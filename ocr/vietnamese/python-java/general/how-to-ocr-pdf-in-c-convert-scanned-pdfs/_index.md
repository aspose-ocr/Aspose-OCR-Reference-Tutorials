# Cách OCR PDF trong C# – Chuyển Đổi PDF Quét

Bạn đã bao giờ cần **how to OCR PDF** nhưng lại gặp khó khăn với một tệp quét không hợp tác? Bạn không phải là người duy nhất. Trong nhiều dự án thực tế—như số hoá hoá đơn hoặc lưu trữ các báo cáo lịch sử—bạn sẽ có một đống hình ảnh ẩn trong một PDF, và cách duy nhất để làm chúng có thể tìm kiếm được là chạy OCR trên chúng.  

Tin tốt là gì? Chỉ với vài dòng C# bạn có thể **convert scanned PDF** thành một **create searchable PDF**, thêm một watermark nhẹ nhàng, và thậm chí lấy văn bản thô ra để xử lý tiếp theo. Dưới đây bạn sẽ thấy một ví dụ hoàn chỉnh, sẵn sàng chạy thực hiện đúng như vậy.

## Nội Dung Hướng Dẫn Này

* Cách **how to OCR PDF** bằng lớp `OcrEngine`.  
* Chuyển một PDF quét thành **create searchable PDF**.  
* Thêm watermark vào trang đầu tiên (**add watermark pdf**).  
* Lấy các chuỗi văn bản thuần từ kết quả (**extract text from pdf**).  
* Các bẫy thường gặp—như xử lý tài liệu đa trang hoặc đối phó với ảnh quét độ phân giải thấp.

Không có liên kết tài liệu bên ngoài, không có đoạn mã chưa hoàn thiện. Mọi thứ bạn cần đều có ở đây.

## Yêu Cầu Trước

* .NET 6.0 trở lên (mã cũng biên dịch được với .NET 5).  
* Một tham chiếu tới thư viện OCR cung cấp `OcrEngine` và `ImageFormats` (ví dụ: `Aspose.OCR` hoặc một gói tương tự).  
* Một file PDF đầu vào có tên `input.pdf` đặt trong thư mục bạn quản lý.  
* Kiến thức cơ bản về ứng dụng console C#.

Nếu bạn đã có những thứ này, tuyệt—hãy bắt đầu.

## Bước 1: Thiết Lập Dự Án và Nhập Các Phụ Thuộc

Tạo một dự án console mới và thêm gói OCR qua NuGet:

```bash
dotnet new console -n OcrPdfDemo
cd OcrPdfDemo
dotnet add package Aspose.OCR
```

Bây giờ mở `Program.cs`. Ở đầu file, nhập các namespace bạn sẽ cần:

```csharp
using System;
using Aspose.OCR;          // OcrEngine lives here
using Aspose.OCR.Image;   // ImageFormats enum
```

*​Tại sao điều này quan trọng*: Nhập đúng các namespace cho phép trình biên dịch tìm thấy `OcrEngine`. Bỏ qua bước này sẽ gây lỗi `CS0246` và dừng quá trình biên dịch.

## Bước 2: Khởi Tạo Engine OCR

Engine là trái tim của quá trình. Bạn tạo một lần và tái sử dụng cho mọi trang.

```csharp
// Step 2: Initialize the OCR engine
OcrEngine engine = new OcrEngine();
```

*Mẹo chuyên nghiệp*: Nếu bạn dự định xử lý nhiều PDF liên tiếp, hãy cân nhắc tái sử dụng cùng một thể hiện `engine` để tránh việc kiểm tra giấy phép lặp lại.

## Bước 3: Tải PDF Nguồn

Cho engine biết file nào sẽ được xử lý. Phương thức `SetImageFromFile` chấp nhận bất kỳ nguồn ảnh‑hoặc‑PDF nào.

```csharp
// Step 3: Load the scanned PDF you want to OCR
engine.SetImageFromFile(@"YOUR_DIRECTORY\input.pdf");
```

> **Tại sao bước này quan trọng** – Engine cần một biểu diễn bitmap của mỗi trang. Khi bạn cung cấp một PDF, nó sẽ raster hoá từng trang dựa trên DPI mặc định (thường là 300). Nếu PDF nguồn đã là dạng văn bản, engine sẽ chỉ truyền qua mà không thay đổi, giúp bạn tiết kiệm thời gian.

## Bước 4: Thực Hiện Nhận Diện OCR

Bây giờ công việc nặng nề diễn ra. Phương thức `Recognize` quét mọi trang, trích xuất glyph và tạo lớp có thể tìm kiếm.

```csharp
// Step 4: Perform OCR on the loaded document
engine.Recognize();
```

*Câu hỏi thường gặp*: *Nếu PDF có 50 trang thì sao?*  
`Recognize` mặc định xử lý toàn bộ tài liệu. Đối với môi trường hạn chế bộ nhớ, bạn có thể đặt `engine.PageIndex` và `engine.PageCount` để làm việc theo từng trang.

## Bước 5: Lưu PDF Có Thể Tìm Kiếm (Watermark trên Trang 1)

Chúng ta sẽ lưu kết quả dưới dạng PDF. Trang đầu tiên sẽ tự động nhận watermark vì thư viện OCR thêm lớp phủ “Generated by Aspose.OCR” theo mặc định. Nếu bạn cần watermark tùy chỉnh, có thể sửa thuộc tính `engine.Watermark` trước khi lưu.

```csharp
// Step 5: Save the OCR‑processed PDF with a watermark on page 1
engine.Save(@"YOUR_DIRECTORY\output.pdf", ImageFormats.Pdf);
```

Sau lệnh này bạn sẽ có một **create searchable PDF** cho phép bạn chọn, sao chép và tìm kiếm văn bản như bất kỳ PDF gốc nào.

## Bước 6: Trích Xuất Văn Bản Thuần (Tùy Chọn nhưng Hữu Ích)

Nếu bạn cũng muốn **extract text from pdf** để lập chỉ mục hoặc phân tích sâu hơn, engine cung cấp cho bạn truy cập tới chuỗi thô.

```csharp
// Step 6: Pull the recognized text into a string
string extractedText = engine.Text;
Console.WriteLine("=== Extracted Text Start ===");
Console.WriteLine(extractedText);
Console.WriteLine("=== Extracted Text End ===");
```

Thuộc tính `engine.Text` nối văn bản của tất cả các trang, giữ nguyên ngắt dòng. Bạn có thể tách bằng `\f` (form feed) nếu cần độ chi tiết theo từng trang.

## Ví Dụ Hoàn Chỉnh Hoạt Động

Dưới đây là chương trình đầy đủ. Sao chép‑dán vào `Program.cs`, thay `YOUR_DIRECTORY` bằng đường dẫn tuyệt đối hoặc tương đối, và chạy `dotnet run`.

```csharp
using System;
using Aspose.OCR;
using Aspose.OCR.Image;

namespace OcrPdfDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // 1️⃣ Initialize the OCR engine
            OcrEngine engine = new OcrEngine();

            // 2️⃣ Load the source PDF to be processed
            // Make sure the file exists; otherwise you'll get a FileNotFoundException.
            string inputPath = @"YOUR_DIRECTORY\input.pdf";
            engine.SetImageFromFile(inputPath);

            // 3️⃣ Perform OCR recognition on the loaded document
            engine.Recognize();

            // 4️⃣ Save the recognized document as a PDF (watermark appears on page 1)
            string outputPath = @"YOUR_DIRECTORY\output.pdf";
            engine.Save(outputPath, ImageFormats.Pdf);

            // 5️⃣ (Optional) Extract plain text for further use
            string extractedText = engine.Text;
            Console.WriteLine("=== Extracted Text Start ===");
            Console.WriteLine(extractedText);
            Console.WriteLine("=== Extracted Text End ===");

            Console.WriteLine($"OCR complete! Searchable PDF saved to: {outputPath}");
        }
    }
}
```

### Kết Quả Mong Đợi

Chạy chương trình sẽ in văn bản đã trích xuất ra console và tạo `output.pdf`. Mở PDF bằng bất kỳ trình xem nào (Adobe Acrobat, Edge, v.v.) và thử tìm kiếm một từ đã xuất hiện trong bản quét gốc—bạn sẽ thấy nó giờ đã có thể tìm kiếm. Trang đầu tiên hiển thị watermark mặc định, xác nhận bước **add watermark pdf** đã thành công.

## Câu Hỏi Thường Gặp & Trường Hợp Đặc Biệt

| Question | Answer |
|----------|--------|
| **Nếu bản quét có độ phân giải thấp?** | Tăng DPI trước khi nhận dạng: `engine.ImageInfo.DpiX = engine.ImageInfo.DpiY = 600;` Điều này cung cấp cho engine OCR nhiều chi tiết hơn để làm việc. |
| **Tôi có thể chọn watermark tùy chỉnh không?** | Có. Đặt `engine.Watermark = "Confidential – Processed on " + DateTime.Now.ToShortDateString();` trước khi `Save`. |
| **Làm sao xử lý PDF có mật khẩu?** | Sử dụng `engine.LoadPassword = "yourPassword";` trước `SetImageFromFile`. |
| **Có cách nào giới hạn OCR chỉ ở các trang cụ thể không?** | Đặt `engine.PageIndex = 2;` và `engine.PageCount = 5;` để chỉ xử lý các trang 3‑7. |
| **Nếu tôi cần giữ nguyên các hình ảnh gốc thì sao?** | Sau khi OCR, bạn có thể hợp nhất PDF gốc với lớp OCR bằng một thư viện PDF (ví dụ: `Aspose.PDF`) để giữ nguyên độ trung thực của hình ảnh. |

## Mẹo & Thực Hành Tốt Nhất

* **Mẹo chuyên nghiệp:** Chạy OCR trên một luồng nền nếu bạn tích hợp vào ứng dụng UI—nếu không giao diện sẽ bị treo.  
* **Cẩn thận với:** PDF có nội dung raster và vector hỗn hợp. Engine chỉ OCR các trang raster; văn bản vector vẫn có thể chọn được tự động.  
* **Điều chỉnh hiệu năng:** Lưu cache dữ liệu ngôn ngữ OCR (`engine.Language = OcrLanguage.English;`) để tránh tải lại cho mỗi tài liệu.

Bằng cách khởi tạo `OcrEngine`, tải file quét, nhận dạng văn bản, lưu một **create searchable pdf**, thêm watermark, và tùy chọn **extract text from pdf**, bạn có thể biến bất kỳ PDF dựa trên hình ảnh nào thành tài sản có thể tìm kiếm và lập chỉ mục.

Bước tiếp theo? Hãy thử nối quy trình này với hệ thống quản lý tài liệu, hoặc thử nghiệm các ngôn ngữ khác (`engine.Language = OcrLanguage.French;`). Bạn cũng có thể khám phá xử lý hàng loạt nhiều file trong một thư mục—chỉ cần lặp lại các bước với một thể hiện `engine` mới mỗi lần.
