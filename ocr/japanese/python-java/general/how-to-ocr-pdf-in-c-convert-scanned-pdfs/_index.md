# C#でPDFをOCRする方法 – スキャンされたPDFを変換

Ever needed to **how to OCR PDF** but were stuck on a scanned file that refused to cooperate? You're not the only one. In many real‑world projects—think invoice digitization or archiving historic reports—you end up with a pile of images tucked inside a PDF, and the only way to make them searchable is to run OCR over them.  

**how to OCR PDF** が必要だったのに、協力しないスキャンファイルで行き詰まったことはありませんか？ あなただけではありません。実務のプロジェクトでは、請求書のデジタル化や歴史的レポートのアーカイブなど、PDF 内に画像が多数埋め込まれた状態になることが多く、検索可能にする唯一の方法は OCR を実行することです。  

The good news? With just a few lines of C# you can **convert scanned PDF** into a **create searchable PDF**, sprinkle a subtle watermark, and even pull the raw text out for further processing. Below you’ll see a complete, ready‑to‑run example that does exactly that.

## 前提条件

* .NET 6.0 以上（コードは .NET 5 でもコンパイル可能）。  
* `OcrEngine` と `ImageFormats` を提供する OCR ライブラリへの参照（例: `Aspose.OCR` または類似パッケージ）。  
* `input.pdf` という名前の入力 PDF を、管理できるフォルダーに配置。  
* C# コンソールアプリの基本的な知識。

これらがすでに揃っているなら、素晴らしいです—さっそく始めましょう。

## ステップ 1: プロジェクトの設定と依存関係のインポート

Create a new console project and add the OCR package via NuGet:

```bash
dotnet new console -n OcrPdfDemo
cd OcrPdfDemo
dotnet add package Aspose.OCR
```

Now open `Program.cs`. At the top, bring in the namespaces you’ll need:

```csharp
using System;
using Aspose.OCR;          // OcrEngine lives here
using Aspose.OCR.Image;   // ImageFormats enum
```

*Why this matters*: Importing the correct namespaces lets the compiler locate `OcrEngine`. Skipping this step will throw a `CS0246` error and halt your build.

## ステップ 2: OCRエンジンの初期化

The engine is the heart of the process. You instantiate it once and reuse it for every page.

```csharp
// Step 2: Initialize the OCR engine
OcrEngine engine = new OcrEngine();
```

*Pro tip*: If you plan to process many PDFs in a row, consider reusing the same `engine` instance to avoid repeated license checks.

## ステップ 3: ソースPDFの読み込み

Tell the engine which file to work on. The method `SetImageFromFile` accepts any image‑or‑PDF source.

```csharp
// Step 3: Load the scanned PDF you want to OCR
engine.SetImageFromFile(@"YOUR_DIRECTORY\input.pdf");
```

> **Why this step is crucial** – The engine needs a bitmap representation of each page. When you feed it a PDF, it internally rasterizes each page based on the default DPI (usually 300). If the source PDF is already text‑based, the engine will simply pass it through unchanged, saving you time.

## ステップ 4: OCR認識の実行

Now the heavy lifting happens. The `Recognize` method scans every page, extracts glyphs, and builds a searchable layer.

```csharp
// Step 4: Perform OCR on the loaded document
engine.Recognize();
```

*Common question*: *What if the PDF has 50 pages?*  
`Recognize` processes the entire document by default. For memory‑constrained environments you can set `engine.PageIndex` and `engine.PageCount` to work page‑by‑page.

## ステップ 5: 検索可能PDFの保存（1ページ目にウォーターマーク）

We’ll save the result as a PDF. The first page will automatically get a watermark because the OCR library adds a “Generated by Aspose.OCR” overlay by default. If you need a custom watermark, you can modify the `engine.Watermark` property before saving.

```csharp
// Step 5: Save the OCR‑processed PDF with a watermark on page 1
engine.Save(@"YOUR_DIRECTORY\output.pdf", ImageFormats.Pdf);
```

After this call you’ll have a **create searchable PDF** where you can select, copy, and search the text just like any native PDF.

## ステップ 6: プレーンテキストの抽出（オプションだが便利）

If you also want to **extract text from pdf** for indexing or further analysis, the engine gives you access to the raw string.

```csharp
// Step 6: Pull the recognized text into a string
string extractedText = engine.Text;
Console.WriteLine("=== Extracted Text Start ===");
Console.WriteLine(extractedText);
Console.WriteLine("=== Extracted Text End ===");
```

The `engine.Text` property concatenates the text of all pages, preserving line breaks. You can split it by `\f` (form feed) if you need per‑page granularity.

## 完全な動作例

Below is the complete program. Copy‑paste it into `Program.cs`, replace `YOUR_DIRECTORY` with an absolute or relative path, and run `dotnet run`.

```csharp
using System;
using Aspose.OCR;
using Aspose.OCR.Image;

namespace OcrPdfDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // 1️⃣ Initialize the OCR engine
            OcrEngine engine = new OcrEngine();

            // 2️⃣ Load the source PDF to be processed
            // Make sure the file exists; otherwise you'll get a FileNotFoundException.
            string inputPath = @"YOUR_DIRECTORY\input.pdf";
            engine.SetImageFromFile(inputPath);

            // 3️⃣ Perform OCR recognition on the loaded document
            engine.Recognize();

            // 4️⃣ Save the recognized document as a PDF (watermark appears on page 1)
            string outputPath = @"YOUR_DIRECTORY\output.pdf";
            engine.Save(outputPath, ImageFormats.Pdf);

            // 5️⃣ (Optional) Extract plain text for further use
            string extractedText = engine.Text;
            Console.WriteLine("=== Extracted Text Start ===");
            Console.WriteLine(extractedText);
            Console.WriteLine("=== Extracted Text End ===");

            Console.WriteLine($"OCR complete! Searchable PDF saved to: {outputPath}");
        }
    }
}
```

### 期待される出力

Running the program prints the extracted text to the console and creates `output.pdf`. Open the PDF in any viewer (Adobe Acrobat, Edge, etc.) and try searching for a word that appeared in the original scan—you’ll see it’s now searchable. The first page shows the default watermark, confirming the **add watermark pdf** step succeeded.

## よくある質問とエッジケース

| Question | Answer |
|----------|--------|
| **What if the scan is low‑resolution?** | Increase the DPI before recognition: `engine.ImageInfo.DpiX = engine.ImageInfo.DpiY = 600;` This gives the OCR engine more detail to work with. |
| **Can I choose a custom watermark?** | Yes. Set `engine.Watermark = "Confidential – Processed on " + DateTime.Now.ToShortDateString();` before `Save`. |
| **How do I handle password‑protected PDFs?** | Use `engine.LoadPassword = "yourPassword";` before `SetImageFromFile`. |
| **Is there a way to limit OCR to specific pages?** | Set `engine.PageIndex = 2;` and `engine.PageCount = 5;` to process pages 3‑7 only. |
| **What if I need to keep the original images untouched?** | After OCR you can merge the original PDF with the OCR layer using a PDF library (e.g., `Aspose.PDF`) to preserve image fidelity. |

## ヒントとベストプラクティス

* **Pro tip:** Run OCR on a background thread if you’re integrating this into a UI app—otherwise the UI will freeze.  
* **Watch out for:** PDFs with mixed raster and vector content. The engine will only OCR raster pages; vector text remains selectable automatically.  
* **Performance tweak:** Cache the OCR language data (`engine.Language = OcrLanguage.English;`) to avoid re‑loading it for every document.

## 結論

You now have a complete, end‑to‑end solution for **how to OCR PDF** in C#. By initializing the `OcrEngine`, loading a scanned file, recognizing the text, saving a **create searchable pdf**, adding a watermark, and optionally **extracting text from pdf**, you can turn any image‑based PDF into a searchable, indexable asset.

Next steps? Try chaining this workflow with a document‑management system, or experiment with different languages (`engine.Language = OcrLanguage.French;`). You could also explore batch processing multiple files in a folder—just loop over the steps with a fresh `engine` instance each time.
