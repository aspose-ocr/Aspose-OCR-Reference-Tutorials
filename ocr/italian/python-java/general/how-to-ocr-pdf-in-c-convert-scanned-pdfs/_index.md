# Come fare OCR su PDF in C# – Convertire PDF scansionati

Hai mai avuto bisogno di **come fare OCR su PDF** ma ti sei trovato bloccato su un file scansionato che si rifiutava di collaborare? Non sei l'unico. In molti progetti reali—pensa alla digitalizzazione di fatture o all'archiviazione di rapporti storici—ti ritrovi con una pila di immagini inserite in un PDF, e l'unico modo per renderle ricercabili è eseguire l'OCR su di esse.  

La buona notizia? Con poche righe di C# puoi **convertire PDF scansionati** in un **PDF ricercabile**, aggiungere una leggera filigrana e persino estrarre il testo grezzo per ulteriori elaborazioni. Di seguito vedrai un esempio completo, pronto all'uso, che fa esattamente questo.

## Cosa copre questo tutorial

* Come **come fare OCR su PDF** usando la classe `OcrEngine`.  
* Convertire un PDF scansionato in un **PDF ricercabile**.  
* Aggiungere una filigrana nella prima pagina (**add watermark pdf**).  
* Estrarre stringhe di testo semplice dal risultato (**extract text from pdf**).  
* Problemi comuni—come gestire documenti multi‑pagina o scansioni a bassa risoluzione.

## Prerequisiti

* .NET 6.0 o versioni successive (il codice si compila anche con .NET 5).  
* Un riferimento alla libreria OCR che fornisce `OcrEngine` e `ImageFormats` (ad es., `Aspose.OCR` o un pacchetto simile).  
* Un PDF di input chiamato `input.pdf` posizionato in una cartella di tua scelta.  
* Familiarità di base con le app console C#.

Se li hai già, ottimo—passiamo subito.

## Passo 1: Configurare il progetto e importare le dipendenze

Create a new console project and add the OCR package via NuGet:

```bash
dotnet new console -n OcrPdfDemo
cd OcrPdfDemo
dotnet add package Aspose.OCR
```

Now open `Program.cs`. At the top, bring in the namespaces you’ll need:

```csharp
using System;
using Aspose.OCR;          // OcrEngine lives here
using Aspose.OCR.Image;   // ImageFormats enum
```

*Perché è importante*: Importare i namespace corretti permette al compilatore di trovare `OcrEngine`. Saltare questo passo genererà un errore `CS0246` e interromperà la compilazione.

## Passo 2: Inizializzare il motore OCR

The engine is the heart of the process. You instantiate it once and reuse it for every page.

```csharp
// Step 2: Initialize the OCR engine
OcrEngine engine = new OcrEngine();
```

*Consiglio*: Se prevedi di elaborare molti PDF consecutivamente, considera di riutilizzare la stessa istanza `engine` per evitare controlli di licenza ripetuti.

## Passo 3: Caricare il PDF di origine

Tell the engine which file to work on. The method `SetImageFromFile` accepts any image‑or‑PDF source.

```csharp
// Step 3: Load the scanned PDF you want to OCR
engine.SetImageFromFile(@"YOUR_DIRECTORY\input.pdf");
```

> **Perché questo passo è fondamentale** – Il motore ha bisogno di una rappresentazione bitmap di ogni pagina. Quando gli fornisci un PDF, lo rasterizza internamente in base alla DPI predefinita (solitamente 300). Se il PDF di origine è già basato su testo, il motore lo passerà semplicemente attraverso senza modifiche, facendoti risparmiare tempo.

## Passo 4: Eseguire il riconoscimento OCR

Now the heavy lifting happens. The `Recognize` method scans every page, extracts glyphs, and builds a searchable layer.

```csharp
// Step 4: Perform OCR on the loaded document
engine.Recognize();
```

*Domanda comune*: *E se il PDF ha 50 pagine?*  
`Recognize` elabora l'intero documento per impostazione predefinita. Per ambienti con memoria limitata puoi impostare `engine.PageIndex` e `engine.PageCount` per lavorare pagina per pagina.

## Passo 5: Salvare il PDF ricercabile (Filigrana sulla pagina 1)

We’ll save the result as a PDF. The first page will automatically get a watermark because the OCR library adds a “Generated by Aspose.OCR” overlay by default. If you need a custom watermark, you can modify the `engine.Watermark` property before saving.

```csharp
// Step 5: Save the OCR‑processed PDF with a watermark on page 1
engine.Save(@"YOUR_DIRECTORY\output.pdf", ImageFormats.Pdf);
```

Dopo questa chiamata avrai un **PDF ricercabile** dove potrai selezionare, copiare e cercare il testo proprio come in qualsiasi PDF nativo.

## Passo 6: Estrarre testo semplice (Opzionale ma utile)

If you also want to **extract text from pdf** for indexing or further analysis, the engine gives you access to the raw string.

```csharp
// Step 6: Pull the recognized text into a string
string extractedText = engine.Text;
Console.WriteLine("=== Extracted Text Start ===");
Console.WriteLine(extractedText);
Console.WriteLine("=== Extracted Text End ===");
```

La proprietà `engine.Text` concatena il testo di tutte le pagine, preservando le interruzioni di riga. Puoi suddividerlo con `\f` (form feed) se ti serve la granularità per pagina.

## Esempio completo funzionante

Below is the complete program. Copy‑paste it into `Program.cs`, replace `YOUR_DIRECTORY` with an absolute or relative path, and run `dotnet run`.

```csharp
using System;
using Aspose.OCR;
using Aspose.OCR.Image;

namespace OcrPdfDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // 1️⃣ Initialize the OCR engine
            OcrEngine engine = new OcrEngine();

            // 2️⃣ Load the source PDF to be processed
            // Make sure the file exists; otherwise you'll get a FileNotFoundException.
            string inputPath = @"YOUR_DIRECTORY\input.pdf";
            engine.SetImageFromFile(inputPath);

            // 3️⃣ Perform OCR recognition on the loaded document
            engine.Recognize();

            // 4️⃣ Save the recognized document as a PDF (watermark appears on page 1)
            string outputPath = @"YOUR_DIRECTORY\output.pdf";
            engine.Save(outputPath, ImageFormats.Pdf);

            // 5️⃣ (Optional) Extract plain text for further use
            string extractedText = engine.Text;
            Console.WriteLine("=== Extracted Text Start ===");
            Console.WriteLine(extractedText);
            Console.WriteLine("=== Extracted Text End ===");

            Console.WriteLine($"OCR complete! Searchable PDF saved to: {outputPath}");
        }
    }
}
```

### Output previsto

Running the program prints the extracted text to the console and creates `output.pdf`. Open the PDF in any viewer (Adobe Acrobat, Edge, etc.) and try searching for a word that appeared in the original scan—you’ll see it’s now searchable. The first page shows the default watermark, confirming the **add watermark pdf** step succeeded.

## Domande frequenti & casi particolari

| Question | Answer |
|----------|--------|
| **E se la scansione è a bassa risoluzione?** | Aumenta la DPI prima del riconoscimento: `engine.ImageInfo.DpiX = engine.ImageInfo.DpiY = 600;` Questo fornisce al motore OCR più dettagli con cui lavorare. |
| **Posso scegliere una filigrana personalizzata?** | Sì. Imposta `engine.Watermark = "Confidential – Processed on " + DateTime.Now.ToShortDateString();` prima di `Save`. |
| **Come gestire PDF protetti da password?** | Usa `engine.LoadPassword = "yourPassword";` prima di `SetImageFromFile`. |
| **È possibile limitare l'OCR a pagine specifiche?** | Imposta `engine.PageIndex = 2;` e `engine.PageCount = 5;` per elaborare solo le pagine 3‑7. |
| **E se devo mantenere intatte le immagini originali?** | Dopo l'OCR puoi unire il PDF originale con lo strato OCR usando una libreria PDF (ad es., `Aspose.PDF`) per preservare la fedeltà delle immagini. |

## Consigli & migliori pratiche

* **Consiglio:** Esegui l'OCR su un thread in background se integri questo in un'app UI—altrimenti l'interfaccia si bloccherà.  
* **Attenzione a:** PDF con contenuti raster e vettoriali misti. Il motore OCRerà solo le pagine raster; il testo vettoriale rimane selezionabile automaticamente.  
* **Ottimizzazione delle prestazioni:** Metti in cache i dati della lingua OCR (`engine.Language = OcrLanguage.English;`) per evitare di ricaricarli per ogni documento.

## Conclusione

Ora hai una soluzione completa, end‑to‑end, per **come fare OCR su PDF** in C#. Inizializzando `OcrEngine`, caricando un file scansionato, riconoscendo il testo, salvando un **PDF ricercabile**, aggiungendo una filigrana e opzionalmente **estrarre testo da PDF**, puoi trasformare qualsiasi PDF basato su immagini in una risorsa ricercabile e indicizzabile.

Prossimi passi? Prova a concatenare questo flusso di lavoro con un sistema di gestione documentale, o sperimenta con lingue diverse (`engine.Language = OcrLanguage.French;`). Puoi anche esplorare l'elaborazione batch di più file in una cartella—basta iterare sui passaggi con una nuova istanza `engine` ogni volta.

Buon coding, e che i tuoi PDF siano sempre ricercabili!
