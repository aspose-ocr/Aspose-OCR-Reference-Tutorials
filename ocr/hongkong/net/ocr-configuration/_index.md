# OCR 配置

## 簡介

如果您正在尋找 **how to extract OCR** 資料從影像中於 .NET 專案中使用，您來對地方了。Aspose.OCR for .NET 讓 OCR 影像辨識變得簡單、快速且高度可自訂。在本指南中，我們將逐步說明核心設定情境——包括處理壓縮檔、資料夾、語言選擇與基於清單的處理——讓您能自信地從影像中擷取文字。

## 快速解答
- **What does “how to extract OCR” mean?** 它指的是使用 OCR 引擎讀取並將影像中的文字轉換為可編輯的字串。  
- **Which library is recommended for .NET?** Aspose.OCR 提供功能豐富的 API，無需外部相依即可執行 OCR 影像辨識。  
- **Do I need a license?** 免費試用可用於評估；商業使用則需購買正式授權。  
- **Which .NET versions are supported?** .NET Framework 4.5+、.NET Core 3.1+、.NET 5/6/7+。  
- **Can I process multiple languages?** 可以——使用 OCR 語言選擇功能即可處理多語言文件。

## 如何在 .NET 中提取 OCR 內容？
Extracting OCR 意味著將影像（或影像集合）傳遞給 Aspose.OCR，該引擎會分析視覺內容、辨識字元，並回傳識別出的文字。此流程可應用於數位化掃描合約、讀取收據或為 PDF 建立索引等情境。

## 為什麼選擇 Aspose.OCR 進行 OCR 影像辨識？
- **High accuracy** 在各種字型與影像品質下皆具高準確度。  
- **Built‑in language support**——可即時切換語言，使用 OCR 語言選擇功能。  
- **No external installations**；此函式庫在 Windows、Linux 與 macOS 上即開箱即用。  
- **Flexible input sources**——支援壓縮檔、資料夾、串流或記憶體中的清單。

## 前提條件
- .NET 開發環境（Visual Studio 2022 或 VS Code）。  
- 已安裝 Aspose.OCR for .NET NuGet 套件 (`Install-Package Aspose.OCR`)。  
- 包含欲處理影像的範例圖片或壓縮檔。

## 常見問題解答

**問：我可以在商業 .NET 應用程式中使用 Aspose.OCR 嗎？ ** 答：可以，一旦您獲得有效的 Aspose 許可證，該庫即可完全支援商業用途。

**問：OCR 語言選擇功能是否支援從右到左的文字？ ** 答：完全支持。 Aspose.OCR 支援阿拉伯語、希伯來語和其他從右到左的語言。

**問：如何處理包含影像的 PDF 檔案？ ** 答：先擷取影像頁面（例如，使用 Aspose.PDF），然後提供給 Aspose.OCR。

**問：如果壓縮包包含非圖片檔案怎麼辦？ ** 答：API 將跳過不支援的檔案；如有需要，您可以按檔案副檔名進行篩選。

**問：是否可以取得每個辨識單字的置信度分數？ ** 答：可以，`OcrResult` 物件提供置信度值，您可以透過程式設計方式進行檢查。
