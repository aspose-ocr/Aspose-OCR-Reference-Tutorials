# 如何在 C# 中 OCR PDF – 轉換掃描的 PDF

是否曾經需要 **how to OCR PDF**，卻被一個不配合的掃描檔卡住？你並非唯一遇到這種情況的人。在許多實務專案中——例如發票數位化或歷史報告的歸檔——你會得到一堆嵌在 PDF 內的影像，而讓它們可搜尋的唯一方法就是對它們執行 OCR。  

好消息是？只要幾行 C# 程式碼，你就能 **convert scanned PDF** 成為 **create searchable PDF**，加上一個淡淡的浮水印，甚至提取原始文字以供後續處理。以下你會看到一個完整、可直接執行的範例，正好做到這些。

## 本教學涵蓋內容

* 使用 `OcrEngine` 類別 **how to OCR PDF** 的方法。  
* 將掃描的 PDF 轉換為 **create searchable PDF**。  
* 在第一頁加入浮水印（**add watermark pdf**）。  
* 從結果中提取純文字字串（**extract text from pdf**）。  
* 常見陷阱——例如處理多頁文件或低解析度掃描。

沒有外部文件連結，沒有未完成的程式碼片段。你需要的一切都在這裡。

## 先決條件

* .NET 6.0 或更新版本（此程式碼亦可在 .NET 5 上編譯）。  
* 對提供 `OcrEngine` 與 `ImageFormats` 的 OCR 函式庫的參考（例如 `Aspose.OCR` 或類似套件）。  
* 一個名為 `input.pdf` 的輸入 PDF，放置於你可控制的資料夾中。  
* 具備基本的 C# 主控台應用程式知識。

如果你已具備上述條件，太好了——讓我們直接開始。

## 步驟 1：設定專案並匯入相依性

建立一個新的主控台專案，並透過 NuGet 加入 OCR 套件：

```bash
dotnet new console -n OcrPdfDemo
cd OcrPdfDemo
dotnet add package Aspose.OCR
```

現在開啟 `Program.cs`。在檔案頂部，引入你需要的命名空間：

```csharp
using System;
using Aspose.OCR;          // OcrEngine lives here
using Aspose.OCR.Image;   // ImageFormats enum
```

*為什麼這很重要*：匯入正確的命名空間讓編譯器能找到 `OcrEngine`。若省略此步驟，會拋出 `CS0246` 錯誤，導致建置中斷。

## 步驟 2：初始化 OCR 引擎

引擎是此流程的核心。你只需實例化一次，之後在每一頁都重複使用它。

```csharp
// Step 2: Initialize the OCR engine
OcrEngine engine = new OcrEngine();
```

*專業提示*：如果你打算連續處理多個 PDF，請考慮重複使用同一個 `engine` 實例，以避免重複的授權檢查。

## 步驟 3：載入來源 PDF

告訴引擎要處理哪個檔案。`SetImageFromFile` 方法接受任何影像或 PDF 來源。

```csharp
// Step 3: Load the scanned PDF you want to OCR
engine.SetImageFromFile(@"YOUR_DIRECTORY\input.pdf");
```

> **為什麼此步驟關鍵** – 引擎需要每一頁的點陣圖表示。當你提供 PDF 時，它會根據預設 DPI（通常為 300）在內部將每頁光柵化。若來源 PDF 已是文字型別，引擎會直接原樣通過，為你節省時間。

## 步驟 4：執行 OCR 辨識

現在開始進行繁重的工作。`Recognize` 方法會掃描每一頁，提取字形，並建立可搜尋的圖層。

```csharp
// Step 4: Perform OCR on the loaded document
engine.Recognize();
```

*常見問題*：*如果 PDF 有 50 頁怎麼辦？*  
`Recognize` 預設會處理整個文件。若在記憶體受限的環境中，你可以設定 `engine.PageIndex` 與 `engine.PageCount` 以逐頁處理。

## 步驟 5：儲存可搜尋的 PDF（第 1 頁加浮水印）

我們將結果儲存為 PDF。第一頁會自動加上浮水印，因為 OCR 函式庫預設會加入「Generated by Aspose.OCR」的覆蓋層。若需要自訂浮水印，可在儲存前修改 `engine.Watermark` 屬性。

```csharp
// Step 5: Save the OCR‑processed PDF with a watermark on page 1
engine.Save(@"YOUR_DIRECTORY\output.pdf", ImageFormats.Pdf);
```

呼叫此方法後，你將得到一個 **create searchable PDF**，可像一般原生 PDF 一樣選取、複製與搜尋文字。

## 步驟 6：提取純文字（可選但很實用）

如果你也想 **extract text from pdf** 以作索引或進一步分析，引擎會提供原始字串的存取。

```csharp
// Step 6: Pull the recognized text into a string
string extractedText = engine.Text;
Console.WriteLine("=== Extracted Text Start ===");
Console.WriteLine(extractedText);
Console.WriteLine("=== Extracted Text End ===");
```

`engine.Text` 屬性會將所有頁面的文字串接起來，保留換行。若需要逐頁的細分，可依 `\f`（換頁符）分割。

## 完整範例程式

以下是完整程式。將其複製貼上至 `Program.cs`，將 `YOUR_DIRECTORY` 替換為絕對或相對路徑，然後執行 `dotnet run`。

```csharp
using System;
using Aspose.OCR;
using Aspose.OCR.Image;

namespace OcrPdfDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // 1️⃣ Initialize the OCR engine
            OcrEngine engine = new OcrEngine();

            // 2️⃣ Load the source PDF to be processed
            // Make sure the file exists; otherwise you'll get a FileNotFoundException.
            string inputPath = @"YOUR_DIRECTORY\input.pdf";
            engine.SetImageFromFile(inputPath);

            // 3️⃣ Perform OCR recognition on the loaded document
            engine.Recognize();

            // 4️⃣ Save the recognized document as a PDF (watermark appears on page 1)
            string outputPath = @"YOUR_DIRECTORY\output.pdf";
            engine.Save(outputPath, ImageFormats.Pdf);

            // 5️⃣ (Optional) Extract plain text for further use
            string extractedText = engine.Text;
            Console.WriteLine("=== Extracted Text Start ===");
            Console.WriteLine(extractedText);
            Console.WriteLine("=== Extracted Text End ===");

            Console.WriteLine($"OCR complete! Searchable PDF saved to: {outputPath}");
        }
    }
}
```

### 預期輸出

執行程式會將提取的文字印到主控台，並產生 `output.pdf`。在任何檢視器（Adobe Acrobat、Edge 等）開啟 PDF，搜尋原始掃描中出現的字詞，你會發現現在已可搜尋。第一頁顯示預設浮水印，證實 **add watermark pdf** 步驟成功。

## 常見問題與特殊情況

| Question | Answer |
|----------|--------|
| **如果掃描解析度低怎麼辦？** | 在辨識前提升 DPI：`engine.ImageInfo.DpiX = engine.ImageInfo.DpiY = 600;` 這會讓 OCR 引擎取得更多細節。 |
| **我可以自訂浮水印嗎？** | 可以。於 `Save` 前設定 `engine.Watermark = "Confidential – Processed on " + DateTime.Now.ToShortDateString();`。 |
| **如何處理受密碼保護的 PDF？** | 在 `SetImageFromFile` 前使用 `engine.LoadPassword = "yourPassword";`。 |
| **有沒有辦法只對特定頁面執行 OCR？** | 設定 `engine.PageIndex = 2;` 與 `engine.PageCount = 5;`，即可僅處理第 3‑7 頁。 |
| **如果需要保持原始影像不變該怎麼辦？** | OCR 完成後，可使用 PDF 函式庫（例如 `Aspose.PDF`）將原始 PDF 與 OCR 圖層合併，以保留影像品質。 |

## 技巧與最佳實踐

* **專業提示**：若將此功能整合至 UI 應用程式，請在背景執行緒上執行 OCR，否則 UI 會卡住。  
* **注意**：含有光柵與向量混合內容的 PDF。引擎僅會對光柵頁面執行 OCR，向量文字會自動保持可選取。  
* **效能調整**：快取 OCR 語言資料（`engine.Language = OcrLanguage.English;`），避免每次處理文件時重新載入。

## 結論

現在你已擁有一套完整、端到端的 **how to OCR PDF** C# 解決方案。透過初始化 `OcrEngine`、載入掃描檔、辨識文字、儲存 **create searchable pdf**、加入浮水印，並可選擇性 **extract text from pdf**，即可將任何影像型 PDF 轉換為可搜尋、可索引的資產。

接下來的步驟？可嘗試將此工作流程與文件管理系統串接，或試驗不同語言（`engine.Language = OcrLanguage.French;`）。亦可探索在資料夾中批次處理多個檔案——只需在每次迴圈中使用新的 `engine` 實例。

祝程式開發順利，願你的 PDF 永遠可搜尋！
