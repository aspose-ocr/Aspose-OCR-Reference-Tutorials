# 如何在 C# 中 OCR PDF – 转换扫描的 PDF

是否曾经需要 **how to OCR PDF**，但被一份不配合的扫描文件卡住？你并不是唯一遇到这种情况的人。在许多真实项目中——比如发票数字化或归档历史报告——你会得到一堆嵌入 PDF 的图像，而让它们可搜索的唯一方法是对其进行 OCR。  

好消息是？只需几行 C# 代码，你就可以将 **convert scanned PDF** 转换为 **create searchable PDF**，添加细微的水印，甚至提取原始文本以便进一步处理。下面你将看到一个完整、可直接运行的示例，正是实现了这些功能。

## 本教程涵盖内容

* 如何使用 `OcrEngine` 类进行 **how to OCR PDF**。  
* 将扫描的 PDF 转换为 **create searchable PDF**。  
* 在首页添加水印（**add watermark pdf**）。  
* 从结果中提取纯文本字符串（**extract text from pdf**）。  
* 常见陷阱——例如处理多页文档或低分辨率扫描。  

没有外部文档链接，也没有未完成的代码片段。你需要的一切都在这里。

## 前提条件

* .NET 6.0 或更高版本（代码在 .NET 5 上也能编译）。  
* 对提供 `OcrEngine` 和 `ImageFormats` 的 OCR 库的引用（例如 `Aspose.OCR` 或类似的包）。  
* 一个名为 `input.pdf` 的输入 PDF，放置在你可控制的文件夹中。  
* 对 C# 控制台应用有基本了解。  

如果你已经具备这些，太好了——我们开始吧。

## 步骤 1：设置项目并导入依赖

通过 NuGet 创建一个新的控制台项目并添加 OCR 包：

```bash
dotnet new console -n OcrPdfDemo
cd OcrPdfDemo
dotnet add package Aspose.OCR
```

打开 `Program.cs`。在顶部，引入你需要的命名空间：

```csharp
using System;
using Aspose.OCR;          // OcrEngine lives here
using Aspose.OCR.Image;   // ImageFormats enum
```

*为什么这很重要*：导入正确的命名空间让编译器能够找到 `OcrEngine`。跳过此步骤会导致 `CS0246` 错误并中止构建。

## 步骤 2：初始化 OCR 引擎

引擎是整个过程的核心。你只需实例化一次，并在每页中重复使用它。

```csharp
// Step 2: Initialize the OCR engine
OcrEngine engine = new OcrEngine();
```

*专业提示*：如果计划连续处理多个 PDF，考虑复用同一个 `engine` 实例，以避免重复的许可证检查。

## 步骤 3：加载源 PDF

告诉引擎要处理哪个文件。方法 `SetImageFromFile` 接受任意图像或 PDF 源。

```csharp
// Step 3: Load the scanned PDF you want to OCR
engine.SetImageFromFile(@"YOUR_DIRECTORY\input.pdf");
```

> **为何此步骤至关重要** – 引擎需要每页的位图表示。当你提供 PDF 时，它会基于默认 DPI（通常为 300）在内部对每页进行光栅化。如果源 PDF 已经是基于文本的，引擎会直接原样通过，节省时间。

## 步骤 4：运行 OCR 识别

现在开始繁重的工作。`Recognize` 方法会扫描每页，提取字形，并构建可搜索层。

```csharp
// Step 4: Perform OCR on the loaded document
engine.Recognize();
```

*常见问题*：*如果 PDF 有 50 页怎么办？*  
`Recognize` 默认会处理整个文档。对于内存受限的环境，你可以设置 `engine.PageIndex` 和 `engine.PageCount` 来逐页处理。

## 步骤 5：保存可搜索的 PDF（第 1 页添加水印）

我们将把结果保存为 PDF。默认情况下，OCR 库会在第一页自动添加 “Generated by Aspose.OCR” 覆盖层，从而生成水印。如果需要自定义水印，可以在保存前修改 `engine.Watermark` 属性。

```csharp
// Step 5: Save the OCR‑processed PDF with a watermark on page 1
engine.Save(@"YOUR_DIRECTORY\output.pdf", ImageFormats.Pdf);
```

调用此方法后，你将得到一个 **create searchable PDF**，可以像原生 PDF 一样选择、复制和搜索文本。

## 步骤 6：提取纯文本（可选但实用）

如果你还想 **extract text from pdf** 用于索引或进一步分析，引擎会提供原始字符串的访问。

```csharp
// Step 6: Pull the recognized text into a string
string extractedText = engine.Text;
Console.WriteLine("=== Extracted Text Start ===");
Console.WriteLine(extractedText);
Console.WriteLine("=== Extracted Text End ===");
```

`engine.Text` 属性会将所有页的文本连接在一起，保留换行。如果需要按页粒度，可以使用 `\f`（换页符）进行拆分。

## 完整工作示例

下面是完整的程序。复制粘贴到 `Program.cs`，将 `YOUR_DIRECTORY` 替换为绝对或相对路径，然后运行 `dotnet run`。

```csharp
using System;
using Aspose.OCR;
using Aspose.OCR.Image;

namespace OcrPdfDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // 1️⃣ Initialize the OCR engine
            OcrEngine engine = new OcrEngine();

            // 2️⃣ Load the source PDF to be processed
            // Make sure the file exists; otherwise you'll get a FileNotFoundException.
            string inputPath = @"YOUR_DIRECTORY\input.pdf";
            engine.SetImageFromFile(inputPath);

            // 3️⃣ Perform OCR recognition on the loaded document
            engine.Recognize();

            // 4️⃣ Save the recognized document as a PDF (watermark appears on page 1)
            string outputPath = @"YOUR_DIRECTORY\output.pdf";
            engine.Save(outputPath, ImageFormats.Pdf);

            // 5️⃣ (Optional) Extract plain text for further use
            string extractedText = engine.Text;
            Console.WriteLine("=== Extracted Text Start ===");
            Console.WriteLine(extractedText);
            Console.WriteLine("=== Extracted Text End ===");

            Console.WriteLine($"OCR complete! Searchable PDF saved to: {outputPath}");
        }
    }
}
```

### 预期输出

运行程序会在控制台打印提取的文本，并生成 `output.pdf`。在任意查看器（Adobe Acrobat、Edge 等）中打开 PDF，尝试搜索原始扫描中出现的词语——你会发现它现在是可搜索的。第一页显示默认水印，确认 **add watermark pdf** 步骤已成功。

## 常见问题与边缘情况

| Question | Answer |
|----------|--------|
| **如果扫描分辨率低怎么办？** | 在识别前提高 DPI：`engine.ImageInfo.DpiX = engine.ImageInfo.DpiY = 600;` 这会为 OCR 引擎提供更多细节。 |
| **我可以选择自定义水印吗？** | 可以。在 `Save` 之前设置 `engine.Watermark = "Confidential – Processed on " + DateTime.Now.ToShortDateString();`。 |
| **如何处理受密码保护的 PDF？** | 在 `SetImageFromFile` 之前使用 `engine.LoadPassword = "yourPassword";`。 |
| **有没有办法仅对特定页面进行 OCR？** | 设置 `engine.PageIndex = 2;` 和 `engine.PageCount = 5;` 可仅处理第 3‑7 页。 |
| **如果我需要保持原始图像不被修改怎么办？** | OCR 完成后，你可以使用 PDF 库（例如 `Aspose.PDF`）将原始 PDF 与 OCR 层合并，以保持图像保真度。 |

## 提示与最佳实践

* **专业提示**：如果将其集成到 UI 应用中，请在后台线程运行 OCR——否则 UI 会卡死。  
* **注意**：包含光栅和矢量混合内容的 PDF。引擎只会对光栅页面进行 OCR；矢量文本会自动保持可选择。  
* **性能调优**：缓存 OCR 语言数据（`engine.Language = OcrLanguage.English;`），避免对每个文档重新加载。  

## 结论

现在你已经拥有一个完整的端到端解决方案，能够在 C# 中 **how to OCR PDF**。通过初始化 `OcrEngine`、加载扫描文件、识别文本、保存 **create searchable pdf**、添加水印，以及可选的 **extract text from pdf**，你可以将任何基于图像的 PDF 转换为可搜索、可索引的资产。

下一步？尝试将此工作流与文档管理系统串联，或尝试不同语言（`engine.Language = OcrLanguage.French;`）。你也可以探索对文件夹中的多个文件进行批处理——只需在每次循环中使用新的 `engine` 实例即可。
