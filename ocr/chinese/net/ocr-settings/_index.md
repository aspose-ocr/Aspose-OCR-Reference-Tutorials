## 如何使用 Aspose.OCR 提取文本图像 – 常用设置概览  

以下是您在各教程中会遇到的设置快速参考：  

| 设置 | 目的 | 何时使用 |
|------|------|----------|
| **Save Result as Document** | 将 OCR 输出导出为 PDF/DOCX/TXT | 需要可复用文档时 |
| **Threads Count** | 控制并行处理 | 大批量或对性能要求高的应用 |
| **Threshold Value** | 调整图像二值化 | 低对比度或噪声较多的图像 |
| **Allowed Characters** | 白名单特定符号 | 行业特定数据（如序列号） |
| **Ignored Characters** | 黑名单不需要的符号 | 去除标点等噪声 |
| **Language Packs** | 启用多语言识别 | 包含非拉丁文字的文档 |

## 常见问题  

**问：我可以在 .NET Core 项目中使用 Aspose.OCR 吗？**  
答：可以，Aspose.OCR for .NET 完全支持 .NET Core、.NET 5+ 和 .NET 6+。  

**问：如何提升低分辨率图像的 OCR 准确率？**  
答：提高阈值，启用语言特定的预处理，并考虑指定允许的字符以减少误报。  

**问：能直接从 PDF 提取文本吗？**  
答：Aspose.OCR 主要针对图像文件，但您可以先使用 Aspose.PDF 将 PDF 页面转换为图像，然后再进行 OCR。  

**问：生产环境需要什么许可证？**  
答：部署时需要商业版 Aspose.OCR 许可证；提供免费试用供评估。  

**问：处理的图像大小是否有限制？**  
答：库可处理高达数百兆字节的图像，但对非常大的文件可能需要调整线程数和内存设置。
