# 使用 Aspose.OCR .NET 从图像中提取文本

## 介绍

在本综合指南中，您将了解如何使用 Aspose.OCR for .NET **从图像中提取文本**。无论您需要从单行、整段文字，还是连续的图像流中提取文本，本教程都将通过清晰的逐步说明带您完成每种场景。让我们深入了解，看看如何在 .NET 应用程序中无缝集成 OCR 功能。

## 快速答案
- **Aspose.OCR 的作用是什么？** 它能够高精度地从图像文件、PDF 和流中提取文本。  
- **我可以从单行提取文本吗？** 可以——使用“Getting Rectangles for Lines”教程。  
- **支持段落提取吗？** 当然；请参阅“Getting Rectangles for Paragraphs”指南。  
- **生产环境需要许可证吗？** 生产使用需要商业许可证；提供免费试用版。  
- **兼容哪些 .NET 版本？** .NET Framework 4.5 及以上、.NET Core 3.1 及以上、.NET 5/6/7。

## 如何使用 Aspose.OCR 提取图像中的文本？

Aspose.OCR 提供灵活的 API，允许您定位特定文本区域、处理图像流或执行全尺度 OCR。针对不同场景选择合适的方法，可提升性能和准确性，同时保持代码简洁。

## 在 OCR 图像识别中获取行的矩形

在许多应用中，您只需要单行出现的文本——例如扫描收据或提取标签。教程 [Getting Rectangles for Lines](./get-rectangles-for-lines/) 说明了如何隔离每行的边界框，并仅对该区域执行 OCR。

## 在 OCR 图像识别中获取段落的矩形

在处理文档时，提取整段文字往往比逐行处理更有用。指南 **[Getting Rectangles for Paragraphs](./get-rectangles-for-paragraphs/)** 带您了解如何检测段落边界，并将这些区域传递给 OCR 引擎以实现准确的文本提取。

## 在 OCR 图像识别中从流中识别图像

直接从流中处理图像（例如通过 Web API 上传文件）可避免保存临时文件的开销。教程 **[Recognize Image from Stream](./recognize-image-from-stream/)** 演示了如何将 `Stream` 对象传递给 Aspose.OCR 并即时获取提取的文本。

## 在 OCR 图像识别中无文本区域检测的图像识别

有时您希望在整个图像上运行 OCR，而无需先定位文本区域。教程 **[Aspose.OCR for .NET](./recognize-image-without-text-area-detection/)** 说明了如何执行全图扫描，这对于快速检查或布局不可预测的情况非常实用。

## 在 OCR 图像识别中识别行

如果您需要专注于特定行——例如验证序列号或条码标签——**[Recognize Line](./recognize-line/)** 提供了一个简明的步骤，帮助您仅隔离并读取该行。

## 在 OCR 图像识别中对图像执行 OCR

本节介绍 **对图像执行 OCR** 的核心操作。通过本教程的步骤，您将学习如何配置 OCR 引擎、选择合适的语言，并从任何受支持的图像格式中获取干净、可搜索的文本。

总之，Aspose.OCR for .NET 提供了丰富的教程，帮助开发者掌握 OCR 图像识别的各个方面——从行、段落到全图流。浏览下方链接，立即开始构建强大的文本提取功能。

## 常见问题

**Q: 我可以在 Web 应用程序中使用 Aspose.OCR 吗？**  
A: 可以，API 支持 ASP.NET、ASP.NET Core 以及其他基于 .NET 的 Web 框架。

**Q: 如何提升低分辨率图像的识别准确率？**  
A: 在将图像传递给 OCR 引擎之前进行预处理（提升对比度、二值化或放大）。

**Q: 能否从加密的 PDF 中提取文本？**  
A: 当在加载选项中提供正确的密码时，Aspose.OCR 可以打开受密码保护的 PDF。

**Q: 开箱即支持哪些语言？**  
A: 超过 100 种语言，包括英语、西班牙语、中文、阿拉伯语等。

**Q: 每个 .NET 平台都需要单独的许可证吗？**  
A: 不需要，单一的 Aspose.OCR 许可证覆盖所有受支持的 .NET 平台。
