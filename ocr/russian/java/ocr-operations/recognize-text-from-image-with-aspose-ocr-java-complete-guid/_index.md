# Распознавание текста с изображения с помощью Aspose OCR Java – Полное руководство

Когда‑нибудь вам нужно было **распознать текст с изображения**, но вы не знали, какую библиотеку выбрать? Вы не одиноки. Во многих реальных проектах — например, сканирование чеков, оцифровка форм или извлечение подписей со скриншотов — получение чистого, исходного текста из растрового изображения является ежедневной задачей.  

В этом руководстве мы пройдём через практический **пример Aspose OCR Java**, который покажет, как **загрузить изображение для OCR**, запустить движок и даже **отключить корректор орфографии**, когда нужны неизменённые символы. К концу вы получите готовую к запуску программу, извлекающую текст из изображения без нежелательных правок.

## Что вы получите

- Чёткое представление о рабочем процессе **Aspose OCR** для Java.  
- Точный код, необходимый для **распознавания текста с изображения** и **извлечения текста из изображения** в его оригинальном виде.  
- Советы, когда стоит отключать встроенный корректор орфографии и как сделать это безопасно.  
- Быстрая проверка, которую вы можете выполнить, чтобы убедиться, что вывод соответствует вашим ожиданиям.

### Предварительные требования (минимум)

- Java 8 или новее, установленная на вашем компьютере.  
- Maven или Gradle для управления зависимостями (мы покажем фрагмент Maven).  
- JAR‑файл `Aspose.OCR` (можно скачать бесплатную trial‑версию с сайта Aspose).  
- Файл изображения (PNG, JPG, BMP и т.д.), содержащий текст, который вы хотите прочитать. Для демонстрации мы будем использовать `mixed-lang.png`.

> **Pro tip:** Если вы планируете обрабатывать множество изображений, рассмотрите возможность загрузки их как потоков, чтобы избежать утечек файловых дескрипторов.

## Шаг 1 – Настройка проекта и добавление зависимости Aspose OCR

Прежде чем мы сможем вызвать любые методы OCR, библиотека должна находиться в classpath. Если вы используете Maven, добавьте следующее в ваш `pom.xml`:

```xml
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-ocr</artifactId>
    <version>23.9</version> <!-- Use the latest stable version -->
</dependency>
```

Если вы предпочитаете Gradle, эквивалент выглядит так:

```gradle
implementation 'com.aspose:aspose-ocr:23.9'
```

После того как зависимость будет разрешена, вы можете импортировать два необходимых класса:

```java
import com.aspose.ocr.OcrEngine;
import com.aspose.ocr.Image;
```

> **Why this matters:** Добавление JAR‑файла через систему сборки гарантирует, что во всех окружениях будет использована правильная версия, что избавляет от ошибок «class not found» в дальнейшем.

## Шаг 2 – Создание OCR‑движка и отключение корректора орфографии

Aspose OCR поставляется со встроенным корректором орфографии, который пытается угадать, что вы имели в виду. Это полезно для чистых документов, но если вы сканируете многоязычные вывески или фрагменты кода, вы получите «исправления», которые вам не нужны. Вот как их отключить:

```java
// Initialize the OCR engine
OcrEngine ocrEngine = new OcrEngine();

// Disable the automatic spell correction
ocrEngine.getSpellCorrector().setEnabled(false);
```

**What’s happening under the hood?** Объект `SpellCorrector` выполняет проход на основе словаря после декодирования сырых глифов. Вызвав `setEnabled(false)`, мы говорим движку пропустить этот проход, сохранив точную последовательность обнаруженных символов.

## Шаг 3 – Загрузка изображения для OCR

Теперь мы действительно **загружаем изображение для OCR**. Метод `Image.load` из Aspose принимает путь к файлу, `InputStream` или даже массив байтов. Для простоты используем путь к файлу:

```java
// Replace the path with the location of your image file
Image inputImage = Image.load("YOUR_DIRECTORY/mixed-lang.png");
```

> **Edge case:** Если изображение больше 5 МБ, рассмотрите возможность предварительного масштабирования. Большие изображения увеличивают потребление памяти и могут замедлять работу движка распознавания.

## Шаг 4 – Распознавание текста и получение сырого вывода

С готовым движком и изображением в памяти распознавание сводится к одной строке:

```java
// Perform OCR and store the raw text
String rawText = ocrEngine.recognize(inputImage);
```

Метод `recognize` возвращает `String`, содержащий **извлечённый текст из изображения** точно так, как его увидел движок — без проверки орфографии и постобработки.

## Шаг 5 – Вывод результата (без проверки орфографии)

Наконец, выведем сырой OCR‑вывод в консоль, чтобы вы могли его проверить:

```java
System.out.println("Raw OCR (no spell‑check):\n" + rawText);
```

При запуске программы вы должны увидеть что‑то вроде:

```
Raw OCR (no spell‑check):
Hello, 世界!
12345
```

Если изображение содержало несколько языков или специальные символы, они останутся неизменными, потому что мы отключили корректор орфографии.

## Полный, готовый к запуску пример

Объединив все части, представляем **полный пример Aspose OCR Java**, который можно скопировать в файл `SpellCorrectionDemo.java`:

```java
import com.aspose.ocr.OcrEngine;
import com.aspose.ocr.Image;

public class SpellCorrectionDemo {
    public static void main(String[] args) throws Exception {

        // Step 1: Create an OCR engine instance
        OcrEngine ocrEngine = new OcrEngine();

        // Step 2: Turn off the built‑in spell corrector
        ocrEngine.getSpellCorrector().setEnabled(false);

        // Step 3: Load the image you want to recognize
        Image inputImage = Image.load("YOUR_DIRECTORY/mixed-lang.png");

        // Step 4: Run OCR on the image and obtain the raw text
        String rawText = ocrEngine.recognize(inputImage);

        // Step 5: Display the OCR result without spell‑checking
        System.out.println("Raw OCR (no spell‑check):\n" + rawText);
    }
}
```

### Как запустить

1. Сохраните файл как `SpellCorrectionDemo.java`.  
2. Скомпилируйте: `javac -cp "path/to/aspose-ocr-23.9.jar" SpellCorrectionDemo.java`  
3. Запустите: `java -cp ".;path/to/aspose-ocr-23.9.jar" SpellCorrectionDemo`

Замените `path/to` на фактический путь к JAR‑файлу Aspose на вашей системе.

## Часто задаваемые вопросы и подводные камни

### Что делать, если изображение в другом формате (например, PDF)?

Aspose OCR также может напрямую читать страницы PDF, но сначала нужно преобразовать страницу PDF в изображение или использовать перегрузку `OcrEngine.recognizePdf`. Это отдельный урок, но принцип **распознавания текста с изображения** остаётся тем же.

### Влияет ли отключение корректора орфографии на производительность?

Немного. Пропуск словарного прохода экономит несколько миллисекунд на страницу, что складывается при обработке тысяч файлов. Компромисс — отсутствие автоматических исправлений опечаток, поэтому решайте исходя из качества ваших данных.

### Можно ли получить результаты, специфичные для языка?

Да. Движок автоматически определяет скрипт, но вы можете принудительно задать язык, вызвав, например, `ocrEngine.setLanguage(OcrEngine.Language.English)`. Это полезно, когда известно, что изображение содержит только один язык и требуется повысить точность.

### Как работать с многостраничными TIFF?

Обрабатывайте каждую страницу как отдельный объект `Image`: `Image.load("file.tif", pageIndex)`. Пройдитесь по страницам в цикле, распознайте каждую и объедините результаты.

## Pro Tips для реальных проектов

- **Пакетная обработка:** Оберните логику OCR в метод, принимающий `InputStream`. Это позволяет передавать изображения из S3, Azure Blob или любого другого хранилища без обращения к файловой системе.  
- **Управление памятью:** Вызовите `ocrEngine.dispose()` после завершения, чтобы освободить нативные ресурсы.  
- **Логирование:** Сохраняйте необработанный вывод в файл журнала для аудита — особенно важно, когда отключён корректор орфографии.  
- **Тестирование:** Напишите unit‑тест, который подаёт известное изображение и проверяет ожидаемую необработанную строку. Это гарантирует, что будущие обновления библиотеки не изменят поведение без вашего ведома.
