# Создание поискового PDF в Java – пошаговое руководство

Задумывались ли вы когда‑нибудь, как **создать поисковый PDF** файлы из стопки отсканированных изображений? Вы не одиноки — многие разработчики сталкиваются с этой проблемой, когда им нужны документы с возможностью текстового поиска для архивирования или соответствия требованиям. Хорошая новость в том, что с несколькими строками кода на Java и Aspose OCR вы можете превратить любой отсканированный PDF в полностью поисковый PDF за секунды.

В этом руководстве мы пройдем весь процесс: от настройки библиотеки Aspose OCR до настройки DPI и языковых параметров, и, наконец, вызова метода конвертации. К концу вы будете знать, как **как конвертировать pdf** файлы программно, поймете нюансы **java pdf ocr**, и сможете ответить на вопрос «**как сделать поисковый pdf**?» для своих проектов.

* Почему DPI и язык важны, когда вы **java pdf ocr** документ.  
* Советы по работе с многоязычными PDF и большими файлами.  

> **Prerequisites:** Java 17 или новее, Maven или Gradle, и лицензия Aspose OCR для Java (бесплатная пробная версия подходит для тестирования). Другие сторонние библиотеки не требуются.

## Обзор создания поискового PDF

Ядро решения находится в классе `PdfOcrProcessor`, предоставляемом Aspose. Он читает каждую страницу отсканированного PDF, запускает OCR и записывает распознанный текст обратно в PDF в виде невидимого текстового слоя. Этот слой делает файл поисковым, сохраняя оригинальный вид изображения.

Ниже приведена полная, готовая к запуску Java‑программа. Смело скопируйте её в свою IDE и нажмите **Run**.

```java
import com.aspose.ocr.*;
import com.aspose.ocr.pdf.*;

public class PdfToSearchablePdf {
    public static void main(String[] args) throws Exception {

        // Step 1: Define the source scanned PDF and the target searchable PDF paths
        String inputPdfPath = "YOUR_DIRECTORY/input.pdf";
        String outputPdfPath = "YOUR_DIRECTORY/searchable_output.pdf";

        // Step 2: Create an instance of the PDF OCR processor
        PdfOcrProcessor pdfProcessor = new PdfOcrProcessor();

        // Step 3: (Optional) Configure OCR settings – DPI and language
        pdfProcessor.getConfiguration().setDpi(300);               // higher DPI can improve accuracy
        pdfProcessor.getConfiguration().setLanguage(Language.ENGLISH);

        // Step 4: Convert the scanned PDF into a searchable PDF
        pdfProcessor.convertToSearchablePdf(inputPdfPath, outputPdfPath);

        // Step 5: Inform the user where the result was saved
        System.out.println("Searchable PDF created at: " + outputPdfPath);
    }
}
```

Running the program prints something like:

```
Searchable PDF created at: YOUR_DIRECTORY/searchable_output.pdf
```

Откройте полученный файл в Adobe Reader, нажмите **Ctrl + F**, и вы увидите, что текст, введённый в поле поиска, теперь совпадает с содержимым отсканированных страниц. Это момент, когда вы понимаете, что успешно **создали поисковый PDF**.

## Шаг 1: Настройка Aspose OCR для Java

Прежде чем вы сможете вызвать `PdfOcrProcessor`, вам нужны JAR‑файлы Aspose OCR в вашем classpath.

**Пользователи Maven** добавьте следующую зависимость в `pom.xml`:

```xml
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-ocr</artifactId>
    <version>23.10</version> <!-- check for the latest version -->
</dependency>
```

**Пользователи Gradle** добавьте эту строку в `build.gradle`:

```gradle
implementation 'com.aspose:aspose-ocr:23.10'
```

Если вы предпочитаете ручную загрузку, скачайте JAR с портала Aspose и поместите его в `libs/`. Не забудьте указать IDE путь к JAR, иначе возникнут ошибки компиляции.

> **Pro tip:** Используйте последнюю версию Aspose OCR, чтобы воспользоваться улучшениями производительности и новыми языковыми пакетами.  

## Шаг 2: Настройка параметров OCR (необязательно, но рекомендуется)

Конфигурация OCR по умолчанию работает, но настройка DPI и языка может существенно улучшить результат, когда вы **конвертировать отсканированный PDF**, содержащий мелкий шрифт или текст не на английском.

```java
pdfProcessor.getConfiguration().setDpi(300); // 300 DPI is a sweet spot
pdfProcessor.getConfiguration().setLanguage(Language.ENGLISH);
```

* **DPI** – Более высокий DPI дает движку OCR больше пикселей для анализа, что обычно приводит к более высокой точности. Однако это также увеличивает использование памяти, поэтому 300 DPI является практичным компромиссом для большинства документов.  
* **Language** – Установка правильного языка снижает количество ложных срабатываний. Aspose поддерживает более 60 языков; просто замените `Language.ENGLISH` на `Language.FRENCH`, `Language.SPANISH` и т.д., если необходимо.  

Если вам нужно **как сделать поисковый PDF** на нескольких языках, вы можете вызвать `setLanguage` несколько раз или использовать `Language.MULTI` (если библиотека поддерживает это).

## Шаг 3: Конвертация отсканированного PDF в поисковый PDF

Теперь происходит магия. Метод `convertToSearchablePdf` выполняет всю тяжелую работу:

```java
pdfProcessor.convertToSearchablePdf(inputPdfPath, outputPdfPath);
```

Внутри Aspose читает изображение каждой страницы, запускает OCR и добавляет скрытый текстовый слой. Исходное изображение остаётся нетронутым, что означает сохранение визуального оформления исходного PDF.

**Edge case:** Если ваш исходный PDF защищён паролем, сначала нужно его разблокировать с помощью `PdfDocument`, прежде чем передавать путь процессору OCR. Библиотека предоставляет `pdfDocument.decrypt("password")` для этой цели.

## Шаг 4: Проверка результата

После конвертации откройте полученный файл в любом PDF‑просмотрщике, поддерживающем поиск текста (Adobe Acrobat Reader, Foxit и др.) и попробуйте найти слово, которое, как вы знаете, присутствует на отсканированном изображении. Если поиск находит слово, вы успешно **создали поисковый PDF**.

Вы также можете программно проверить наличие текстового слоя с помощью Aspose PDF:

```java
PdfDocument doc = new PdfDocument(outputPdfPath);
boolean hasText = doc.getPages().get_Item(1).getExtractedText().length() > 0;
System.out.println("Text layer detected: " + hasText);
```

Если `hasText` выводит `true`, слой OCR присутствует.

## Следующие шаги: выход за пределы базового

Теперь, когда вы знаете **как конвертировать pdf** с помощью Aspose OCR, вы можете исследовать:

* **Тонкая настройка производительности** – экспериментируйте с более низким DPI для ускорения обработки, когда точность не критична.
