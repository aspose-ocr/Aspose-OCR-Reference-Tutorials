# Извлечение текста из изображений с помощью Aspose.OCR – разрешённые символы

## Введение

Извлечение текста из изображений – распространённая задача в современных приложениях: будь то обработка счетов, сканирование чеков или оцифровка печатных документов. **Aspose.OCR for Java** упрощает эту задачу, обеспечивая высокую точность распознавания и гибкие параметры настройки, такие как указание разрешённых символов. В этом руководстве мы пройдём полный **aspose ocr java tutorial**, показывающий, как настроить библиотеку, выполнить OCR и ограничить набор символов под ваши нужды.

## Что такое «извлечение текста из изображений»?
Извлечение текста из изображений — это процесс преобразования визуального текста (например, печатного или рукописного) в машинно‑читаемые строки. Это позволяет выполнять дальнейшие задачи, такие как поиск, индексация или анализ данных.

## Пошаговое руководство

### Шаг 1: Укажите каталог документов

Определите папку, в которой будут храниться результаты OCR‑обработки. Этот путь позже используется для поиска файла изображения.

```java
String dataDir = "Your Document Directory";
```

### Шаг 2: Укажите путь к изображению

Укажите API путь к изображению, которое необходимо проанализировать.

```java
String imagePath = dataDir + "0001460985.Jpeg";
```

### Шаг 3: Создайте экземпляр Aspose.OCR

Создайте объект OCR‑движка, передав строку лицензии. Лицензия может быть временной или постоянной.

```java
AsposeOCR api = new AsposeOCR("YourLicenseKey");
```

### Шаг 4: Выполните распознавание OCR

Вызовите метод `RecognizeLine`, чтобы извлечь строку текста из изображения. Результат — обычная строка, которую можно дальше обрабатывать или сохранять.

```java
try {
    String result = api.RecognizeLine(imagePath);
    System.out.println("File: " + imagePath);
    System.out.println("Result line: " + result);
} catch (IOException e) {
    e.printStackTrace();
}
```

> **Полезный совет:** Если нужно ограничить вывод определённым набором символов (например, только цифры), используйте метод `setAllowedCharacters` у экземпляра `AsposeOCR` перед вызовом `RecognizeLine`. Это заставит движок игнорировать любые символы, не входящие в заданный набор.

## Распространённые проблемы и решения

| Проблема | Причина | Решение |
|----------|----------|----------|
| **Отсутствует вывод или пустая строка** | Неправильный путь к изображению или неподдерживаемый формат | Проверьте `imagePath` и используйте поддерживаемый формат (JPEG, PNG, BMP) |
| **Ошибки распознавания** | Низкое разрешение изображения или шумный фон | Предобработайте изображение (увеличьте контраст, бинаризуйте) перед OCR |
| **Лицензия не применена** | Отсутствует или неверный ключ лицензии | Убедитесь, что строка лицензии корректна и передана в конструктор `AsposeOCR` |
