# Как исправить наклон изображения для OCR – Руководство по предобработке на Java

Вы когда‑нибудь задумывались **как исправить наклон изображения** файлов, полученных со сканера под странным углом? Вы не одиноки — многие разработчики сталкиваются с этой проблемой, когда пытаются извлечь текст из документов, насыщенных изображениями. Хорошая новость? С несколькими строками кода на Java и Aspose OCR вы можете выпрямить, удалить шум и получить чистый текст без усилий.

В этом руководстве мы пройдем весь процесс: загрузим шумный, повернутый скан, применим фильтр выравнивания, удалим визуальный шум и, наконец, **извлечем текст из изображения**. К концу вы будете знать, как **предобработать изображение для OCR**, **очистить отсканированные изображения** и **повысить точность OCR** для любого проекта, требующего надежного извлечения текста.

## Что понадобится

- Java 17 (или любой современный JDK) — код использует стандартные возможности языка.
- Библиотека Aspose OCR for Java (бесплатная пробная версия подходит для экспериментов).
- Пример изображения, которое одновременно шумное и повернуто (например, `noisy-rotated.png`).
- Ваш любимый IDE (IntelliJ IDEA, Eclipse, VS Code…) — любой, способный компилировать Java.

Никаких дополнительных фреймворков, без необходимости в Maven/Gradle; единственные импортируемые классы — те, что показаны ниже.

## Как исправить наклон изображения с помощью Aspose OCR

Первое, что нужно решить, — наклон изображения. Если строки текста не горизонтальны, движок OCR будет неправильно распознавать символы. `DeskewFilter` выполняет основную работу.

```java
import com.aspose.ocr.OcrEngine;
import com.aspose.ocr.Image;
import com.aspose.ocr.preprocess.DeskewFilter;
import com.aspose.ocr.preprocess.DenoiseFilter;

public class PreprocessDemo {
    public static void main(String[] args) throws Exception {
        // Step 1: Create the OCR engine
        OcrEngine ocrEngine = new OcrEngine();

        // Step 2: Load the raw scanned image
        Image scannedImage = Image.load("YOUR_DIRECTORY/noisy-rotated.png");

        // Step 3: Correct the image orientation (deskew)
        DeskewFilter deskewFilter = new DeskewFilter();
        scannedImage = deskewFilter.apply(scannedImage);

        // Step 4: Reduce visual noise (denoise)
        DenoiseFilter denoiseFilter = new DenoiseFilter();
        scannedImage = denoiseFilter.apply(scannedImage);

        // Step 5: Perform OCR on the cleaned image
        String recognizedText = ocrEngine.recognize(scannedImage);

        // Step 6: Output the extracted text
        System.out.println(recognizedText);
    }
}
```

> **Почему это важно:** `DeskewFilter` анализирует геометрию изображения, оценивает угол поворота и вращает bitmap обратно к горизонтальному уровню. Без этого шага большинство OCR‑движков воспринимают наклонные буквы как совершенно другие глифы, сводя ваши усилия по **повышению точности OCR** к провалу.

> **Совет:** Если вы работаете с документами, которые могут быть перевёрнуты, включите флаг `setAutoRotate` в `DeskewFilter` (доступен в более новых версиях Aspose). Он автоматически поворачивает изображение на 180° при необходимости.

## Предобработка изображения для OCR — Удаление шума и очистка

После того как изображение выровнено, следующая преграда — визуальный шум: пятна, артефакты сжатия или слабые фоновые узоры, которые сбивают с толку OCR‑движок. `DenoiseFilter` применяет тонкий алгоритм сглаживания, сохраняющий границы (буквы) и устраняющий зернистость.

```java
// Step 4 (continued): Reduce visual noise (denoise)
DenoiseFilter denoiseFilter = new DenoiseFilter();
scannedImage = denoiseFilter.apply(scannedImage);
```

### Когда следует менять настройки удаления шума

- **Очень тёмные сканы:** Увеличьте силу фильтра (`denoiseFilter.setStrength(2)`) для удаления теней фона.
- **Шрифты мелкого кегля:** Уменьшите силу, чтобы не размыть крошечные засечки.
- **Цветные документы:** Сначала преобразуйте в градации серого (`scannedImage = scannedImage.toGrayscale();`) — OCR‑движок лучше работает с одно‑канальными изображениями.

Эти настройки являются частью лучших практик **очистки отсканированных изображений**; более чистый bitmap напрямую повышает коэффициенты уверенности OCR‑движка.

## Извлечение текста из изображения — Запуск OCR‑движка

Теперь, когда изображение выровнено и очищено, пришло время **извлечь текст из изображения**. Класс `OcrEngine` обрабатывает всё за кулисами: сегментацию, классификацию символов и моделирование языка.

```java
// Step 5: Perform OCR on the cleaned image
String recognizedText = ocrEngine.recognize(scannedImage);
System.out.println(recognizedText);
```

#### Ожидаемый вывод

Если ваш исходный файл содержит строку “**Invoice # 12345**”, консоль должна вывести что‑то вроде:

```
Invoice # 12345
Date: 2026-03-18
Total: $1,250.00
```

Если вывод выглядит искажённым, проверьте предыдущие шаги — особенно выравнивание. Даже наклон в 1 градус может испортить цифры и символы.

## Распространённые ошибки и советы по **повышению точности OCR**

| Проблема | Почему это ухудшает точность | Быстрое решение |
|----------|------------------------------|-----------------|
| **Остаточный поворот** | OCR ожидает горизонтальные базовые линии. | Проверьте с помощью `deskewFilter.getAngle()` и запишите значение. |
| **Чрезмерное удаление шума** | Размывает тонкие штрихи, превращая “i” в “l”. | Используйте `setStrength(0.5)` для деликатных шрифтов. |
| **Неправильный формат изображения** | Сжатие JPEG добавляет артефакты. | Предпочтительно использовать PNG или TIFF для без потерь. |
| **Неправильный язык** | По умолчанию движок использует английский; другие алфавиты требуют явной настройки. | `ocrEngine.setLanguage(OcrEngine.Language.Spanish);` |
| **Низкое DPI (≤150)** | Недостаточно пикселей для надёжной сегментации. | Пересэмплировать до 300 DPI перед обработкой (`scannedImage = scannedImage.resample(300);`). |

### Бонус: пакетная обработка

Если у вас есть папка со сканами, оберните демонстрацию в цикл:

```java
File folder = new File("YOUR_DIRECTORY");
for (File file : folder.listFiles((dir, name) -> name.endsWith(".png"))) {
    Image img = Image.load(file.getAbsolutePath());
    img = new DeskewFilter().apply(img);
    img = new DenoiseFilter().apply(img);
    String text = ocrEngine.recognize(img);
    System.out.println("=== " + file.getName() + " ===");
    System.out.println(text);
}
```

Этот шаблон позволяет **предобрабатывать изображения для OCR** в масштабе, поддерживая чистоту кода и предсказуемую производительность.
