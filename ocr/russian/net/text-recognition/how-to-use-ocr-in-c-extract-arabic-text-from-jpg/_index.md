# Как использовать OCR в C# – извлечь арабский текст из JPG

Когда‑нибудь задумывались **как использовать OCR**, имея картинку с арабским текстом на жёстком диске? Вы не одиноки. Многие разработчики сталкиваются с тем же: есть JPEG, нужны слова внутри, а писать нейронную сеть с нуля не хочется.  

Хорошие новости? Пара строк кода на C# позволяют **распознавать текст из изображений**, извлекать каждый арабский символ и получать чистую строку, которую можно сохранять, искать или отображать. В этом руководстве мы пройдём весь процесс — установку библиотеки, настройку языковой модели, запуск сканирования и вывод результата. К концу вы сможете **преобразовать JPG в текст** за считанные секунды.

## Что вы узнаете

- Установить NuGet‑пакет Aspose.OCR для .NET.  
- Инициализировать OCR‑движок в режиме CPU (идеально для небольших задач).  
- Автоматически загрузить модель арабского языка.  
- Выполнить OCR над JPEG‑изображением и получить распознанный текст.  
- Обработать типичные проблемы, такие как большие файлы или отсутствие языковых данных.

Предыдущий опыт работы с OCR не требуется; достаточно базовых знаний C# и Visual Studio. Готовы? Поехали.

## Установите Aspose OCR для .NET

Прежде чем любой код начнёт работать, нужна библиотека Aspose.OCR. Она поставляется в виде NuGet‑пакета, поэтому установка — одна команда:

```bash
dotnet add package Aspose.OCR
```

Или, если предпочитаете UI Visual Studio, откройте **Tools → NuGet Package Manager → Manage NuGet Packages for Solution**, найдите **Aspose.OCR** и нажмите **Install**. Пакет подтянет всё необходимое, включая файлы языковых данных, которые движок загрузит при первом использовании.

> **Pro tip:** Держите ваш проект нацеленным на .NET 6 или новее; старые фреймворки тоже работают, но вы упустите улучшения производительности.

## Инициализируйте OCR‑движок

Теперь, когда библиотека на месте, можно создать экземпляр `OcrEngine`. Для большинства небольших задач режим CPU по умолчанию более чем достаточен — он использует процессор хоста и избегает дополнительной настройки, требуемой для ускорения на GPU.

```csharp
using Aspose.OCR;
using Aspose.OCR.Settings;

// Initialise the OCR engine (CPU mode is sufficient for small jobs)
var ocrEngine = new OcrEngine();
```

Зачем создавать новый движок каждый раз? Объект `OcrEngine` хранит конфигурацию, такую как язык, DPI и формат вывода. Ограничивая его одной операцией, вы гарантируете чистое состояние и избегаете случайных конфликтов между разными языковыми моделями.

## Загрузите модель арабского языка

Aspose поставляет языковые пакеты по запросу. При первом присвоении `Language.Arabic` библиотека скачает примерно 30 МБ данных в кэш‑папку на вашем компьютере. Эта одноразовая загрузка делает последующие запуски молниеносными.

```csharp
// Choose the language model to load.
// The first assignment triggers an automatic download of the language data (~30 MB).
ocrEngine.Settings.Language = Language.Arabic;
```

Если понадобится поддержка дополнительных скриптов (например, английского или хинди), просто измените значение enum — дополнительный код не нужен. Движок будет кэшировать каждый язык отдельно.

## Выполните OCR над JPG‑изображением

С готовым движком и загруженной арабской моделью укажите путь к изображению, которое хотите обработать. Путь может быть абсолютным или относительным; просто убедитесь, что файл существует и доступен для чтения.

```csharp
// Run OCR on the input image.
var recognitionResult = ocrEngine.Recognize(@"YOUR_DIRECTORY/arabic_doc.jpg");
```

**Пограничный случай:** Если ваш JPEG огромный (более 5 МБ), имеет смысл сначала уменьшить его. Большие изображения увеличивают потребление памяти и могут замедлить распознавание. Быстрая предварительная ресайз‑операция с помощью `System.Drawing` или `ImageSharp` существенно сократит время обработки.

## Получите и отобразите распознанный текст

Метод `Recognize` возвращает объект `OcrResult`. Его свойство `Text` содержит текстовое представление всего, что удалось прочитать движку.

```csharp
// Retrieve the recognised text and output it.
string extractedText = recognitionResult.Text;
Console.WriteLine(extractedText);
```

При запуске программы вы должны увидеть арабские символы, выведенные в консоль, с сохранением исходного порядка и разрывов строк. Если нужен файл вместо консоли, просто запишите строку с помощью `File.WriteAllText`.

### Полный рабочий пример

Объединив всё вместе, получаем полностью готовое консольное приложение:

```csharp
using System;
using Aspose.OCR;
using Aspose.OCR.Settings;

class Program
{
    static void Main()
    {
        // 1️⃣ Initialise the OCR engine (CPU mode works fine for most cases)
        var ocrEngine = new OcrEngine();

        // 2️⃣ Load the Arabic language pack – triggers a one‑time download (~30 MB)
        ocrEngine.Settings.Language = Language.Arabic;

        // 3️⃣ Specify the image path – replace with your actual file location
        string imagePath = @"YOUR_DIRECTORY/arabic_doc.jpg";

        // 4️⃣ Run OCR and capture the result
        var result = ocrEngine.Recognize(imagePath);

        // 5️⃣ Extract the text and display it
        string extractedText = result.Text;
        Console.WriteLine("=== Extracted Arabic Text ===");
        Console.WriteLine(extractedText);
    }
}
```

**Ожидаемый вывод (пример):**

```
=== Extracted Arabic Text ===
هذا نص تجريبي باللغة العربية
تم استخراج هذا النص بنجاح
```

Если вывод выглядит «крякозябрами», проверьте, что изображение чёткое и язык установлен на `Arabic`. Размытие сканов или повернутые страницы — частые причины неудач OCR.

## Часто задаваемые вопросы и советы

- **Что делать, если изображение содержит и арабский, и английский текст?**  
  Установите `ocrEngine.Settings.Language = Language.Multilingual;`, чтобы Aspose автоматически определял несколько скриптов.

- **Можно ли ускорить обработку с помощью GPU?**  
  Да — замените движок по умолчанию на `new OcrEngine(OcrEngineMode.Gpu)`, если у вас совместимая видеокарта и установлен runtime CUDA.

- **Как обрабатывать отображение справа‑налево в UI?**  
  Полученная строка — Unicode; большинство современных UI‑фреймворков (WinForms, WPF, ASP.NET) автоматически поддерживают RTL‑разметку при установке соответствующего `FlowDirection`.

- **Есть ли ограничение по размеру изображения?**  
  Технически нет, но потребление памяти растёт с разрешением. Для огромных сканов (например, книг) лучше обрабатывать по одной странице.

## Следующие шаги

Теперь, когда вы освоили **как использовать OCR** для арабских JPEG‑файлов, можно расширять решение:

- **Пакетная обработка:** Пройдитесь по папке с изображениями и запишите каждый результат в отдельный файл `.txt`.  
- **Пост‑обработка:** Используйте регулярные выражения для очистки лишних знаков пунктуации или разрывов строк.  
- **Интеграция:** Передайте извлечённые строки в поисковый индекс (например, Azure Cognitive Search) для полнотекстового поиска по сканированным документам.

Если интересуют другие языки, просто замените значение enum `Language`. Хотите извлекать таблицы или рукописные заметки? Aspose.OCR также предлагает функции `LayoutAnalysis`, позволяющие сегментировать блоки перед распознаванием.
