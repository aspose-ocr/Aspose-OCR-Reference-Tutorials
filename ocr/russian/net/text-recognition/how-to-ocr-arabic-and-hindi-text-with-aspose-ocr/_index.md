# как распознать арабский и хинди текст с помощью Aspose OCR

Когда‑нибудь задавались вопросом, **как распознать арабский** текст, который пишется справа налево, одновременно извлекая хинди‑символы из чека? Вы не одиноки. Многие разработчики сталкиваются с той же проблемой, когда им нужно **распознавать арабский текст** и **распознавать хинди текст** в одном рабочем процессе.  

В этом руководстве мы пройдем полный, готовый к запуску пример на C#, который покажет, как **извлекать текст из изображения** файлов, **преобразовывать изображение в текст**, и работать с арабскими и хинди скриптами с помощью Aspose OCR. Никаких расплывчатых ссылок — только код, который можно скопировать и вставить, плюс объяснение каждой строки.

> **Pro tip:** Если вы никогда ранее не использовали Aspose OCR, сначала установите пакет NuGet `Aspose.OCR`. Это однократный клик в Visual Studio, и он загрузит все необходимые нативные бинарные файлы для распознавания на CPU.

## как распознать арабский – настройка окружения

Прежде чем погрузиться в код, убедимся, что среда разработки готова.

1. **Target framework** – .NET 6.0 или новее. Более старые версии всё ещё компилируются, но вы упустите новейшие возможности языка.  
2. **Package** – Выполните `dotnet add package Aspose.OCR` в терминале или используйте UI NuGet Package Manager.  
3. **Images** – Поместите два образца изображений в папку, к которой можно обратиться, например `C:\OCRSamples\arabic_sign.jpg` и `C:\OCRSamples\hindi_receipt.png`. Арабское изображение должно содержать чёткие, контрастные арабские символы; изображение с хинди может быть сканированным чеком или фотографией знака.  

Это всё — без дополнительных файлов конфигурации, без драйверов GPU, просто простой OCR‑движок, работающий на CPU.

## распознавание арабского текста – загрузка и обработка

Сейчас мы действительно **распознаем арабский текст**. Ключ в том, чтобы указать движку, какой язык ожидать; иначе движок по умолчанию будет использовать латинские символы, и вы получите искажённый вывод.

```csharp
using Aspose.OCR;
using Aspose.OCR.Models;

class MultiLangDemo
{
    static void Main()
    {
        // 1️⃣ Create the OCR engine (CPU‑based by default)
        var ocrEngine = new OcrEngine();

        // 2️⃣ Load an image that contains Arabic script
        var arabicImage = OcrImage.FromFile(@"C:\OCRSamples\arabic_sign.jpg");

        // 3️⃣ Recognize the Arabic text – note the Language.Arabic enum
        var arabicResult = ocrEngine.Recognize(arabicImage, Language.Arabic);

        // 4️⃣ Print the result to the console
        System.Console.WriteLine("Arabic: " + arabicResult.Text);
```

**Почему это работает:**  
* `OcrEngine` выполняет всю тяжёлую работу — предобработку, сегментацию и классификацию символов.  
* Передавая `Language.Arabic`, мы активируем движок раскладки справа налево и набор арабских символов. Без этого движок будет рассматривать изображение как левосторонний латинский текст, что приводит к отсутствию диакритических знаков и сломанным словам.  

**Ожидаемый вывод** (ваш реальный текст будет отличаться в зависимости от изображения):

```
Arabic: مرحبا بكم في متجرنا
```

Если вы видите пустые строки, проверьте, что изображение не слишком тёмное и путь к файлу указан правильно.  

## извлечение текста из изображения – обработка скриптов справа налево

Арабский — не единственный скрипт, требующий особой обработки. Языки справа налево (RTL) требуют, чтобы OCR‑движок обратил визуальный порядок после распознавания. Aspose делает это автоматически, когда вы указываете `Language.Arabic`, но это стоит иметь в виду для будущих расширений (например, иврит).

*Совет:* При последующем отображении результата OCR в пользовательском интерфейсе убедитесь, что элемент управления поддерживает рендеринг RTL; иначе текст будет отображаться искажённым.

## преобразование изображения в текст – работа с хинди

Переключаясь, давайте **преобразуем изображение в текст** для хинди‑чека. Процесс аналогичен арабскому, но мы используем `Language.Hindi`.

```csharp
        // 5️⃣ Load the Hindi (Devanagari) image
        var hindiImage = OcrImage.FromFile(@"C:\OCRSamples\hindi_receipt.png");

        // 6️⃣ Recognize Hindi text – Language.Hindi tells the engine to use Devanagari models
        var hindiResult = ocrEngine.Recognize(hindiImage, Language.Hindi);

        // 7️⃣ Output the Hindi OCR result
        System.Console.WriteLine("Hindi: " + hindiResult.Text);
    }
}
```

**Почему мы переиспользуем тот же экземпляр `ocrEngine`:**  
Создание нового движка для каждого языка расходует память и время инициализации. Движок Aspose потокобезопасен для последовательных вызовов, поэтому его повторное использование эффективно и чисто.

**Пример вывода в консоль** (again, depends on your image):

```
Hindi: कुल राशि: ₹ 1,250.00
```

Если хинди‑текст выглядит как искажённые латинские символы, вы, вероятно, пропустили указание языка или разрешение изображения слишком низкое (<300 dpi). Увеличение масштаба изображения или применение простого бинаризационного фильтра может значительно повысить точность.

## распознавание хинди текста – распространённые подводные камни и граничные случаи

Даже при правильном указании языка, несколько мелких проблем могут вас подвести:

| Проблема | Симптом | Решение |
|----------|---------|---------|
| Низкий контраст | Многие символы становятся «?» или пропускаются | Предобработать с помощью `OcrImage.AdjustContrast(1.5)` |
| Искажённое изображение | Текст выглядит повернутым, OCR возвращает пустую строку | Вызвать `ocrEngine.PreprocessImage(arabicImage, ImageProcessingOptions.Rotate)` |
| Смешанные языки | Арабская строка, за которой следуют английские цифры | Выполнить два прохода: сначала с `Language.Arabic`, затем с `Language.English` на том же изображении и объединить результаты |
| Большой размер файла | Медленное распознавание или ошибки нехватки памяти | Уменьшить до максимальной ширины 2000 px с помощью `OcrImage.Resize(2000, 0)` |

Эти советы помогут вам **извлекать текст из изображения** файлов, которые не идеально отсканированы, что часто встречается в реальных проектах.

## Сводим всё вместе — полный рабочий пример

Ниже представлен полный код программы, который можно скопировать непосредственно в новый консольный проект. Нет скрытых зависимостей, нет дополнительной конфигурации — только чистый C#.

```csharp
using Aspose.OCR;
using Aspose.OCR.Models;

class MultiLangDemo
{
    static void Main()
    {
        // Initialize the OCR engine (CPU‑based)
        var ocrEngine = new OcrEngine();

        // -------------------- Arabic --------------------
        var arabicPath = @"C:\OCRSamples\arabic_sign.jpg";
        var arabicImage = OcrImage.FromFile(arabicPath);
        var arabicResult = ocrEngine.Recognize(arabicImage, Language.Arabic);
        System.Console.WriteLine("Arabic: " + arabicResult.Text);

        // -------------------- Hindi --------------------
        var hindiPath = @"C:\OCRSamples\hindi_receipt.png";
        var hindiImage = OcrImage.FromFile(hindiPath);
        var hindiResult = ocrEngine.Recognize(hindiImage, Language.Hindi);
        System.Console.WriteLine("Hindi: " + hindiResult.Text);
    }
}
```

**Запуск программы** выведет как арабские, так и хинди строки в консоль, подтверждая, что вы успешно **распознали арабский текст** и **распознали хинди текст** за один запуск.
