# Как выполнять OCR в C# – Извлекать арабский текст из изображений

Когда‑нибудь задумывались **как выполнять OCR** на арабском счете‑фактуре или отсканированном документе? Вы не одиноки — многие разработчики сталкиваются с проблемой, когда нужно извлечь арабские символы из bitmap. Хорошая новость в том, что с помощью нескольких строк C# вы можете распознавать текст из файлов изображений, преобразовать изображение в текст и в конечном итоге **извлекать арабский текст** для последующей обработки.

В этом руководстве мы пройдем полный, готовый к запуску пример, который покажет, как именно выполнять OCR, почему каждый шаг важен и на что следует обратить внимание при работе с правосторонними скриптами. К концу вы сможете **извлекать текст из изображения** на арабском, урду, курдском или любом другом языке, поддерживаемом OCR‑движком.

## Требования

- .NET 6.0 или новее (код также компилируется с .NET Core)  
- Ссылка на библиотеку OCR, предоставляющую `OcrEngine` (например, `MyOcrSdk.dll`).  
- Файл изображения, содержащий арабский текст, например `invoice_arabic.png`.  
- Базовое знакомство с консольными приложениями C#.

> **Совет:** Если у вас нет под рукой OCR‑SDK, бесплатная community‑edition *MyOcrSdk* подходит для тестирования и поддерживает используемые языки.

---

## Шаг 1 – Настройка проекта и импорт пространства имён OCR

Прежде чем мы сможем **распознавать текст из изображения**, нужен скелет проекта и правильные директивы `using`.

```csharp
using System;
using MyOcrSdk;          // <-- Replace with the actual namespace of your OCR library
using MyOcrSdk.IO;       // For ImageStream helper class

namespace ArabicOcrDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // The rest of the OCR workflow will go here
        }
    }
}
```

*Почему это важно:* Импорт нужных пространств имён дает доступ к `OcrEngine`, `Language` и `ImageStream`. Пропуск этого шага приводит к ошибкам компиляции, которые могут расстроить новичков.

---

## Шаг 2 – Создание экземпляра OCR‑движка (включён основной ключевой запрос)

Теперь мы действительно **выполняем OCR**, создавая экземпляр движка.

```csharp
// Step 2: Create an OCR engine instance
var ocrEngine = new OcrEngine();
```

Объект `OcrEngine` — сердце операции; он хранит конфигурацию, выполняет тяжёлую работу и возвращает объект результата, содержащий извлечённую строку. Можно сравнить его с «мозгом», который **преобразует изображение в текст**.

---

## Шаг 3 – Выбор языка распознавания

Арабский, урду, курдский… все используют одну и ту же семейство скриптов, поэтому нужно указать движку, какой язык ожидать. Это значительно повышает точность.

```csharp
// Step 3: Choose the language for recognition (Arabic, Urdu, Kurdish, etc.)
ocrEngine.Config.Language = Language.Arabic;   // You can switch to Language.Urdu, etc.
```

*Почему это важно:* OCR‑движки опираются на языковые модели. Выбор правильной модели уменьшает ошибочное распознавание похожих символов, особенно для правосторонних скриптов.

---

## Шаг 4 – Загрузка изображения с текстом

Нужен bitmap, который движок сможет проанализировать. Вспомогательный метод `ImageStream.FromFile` абстрагирует детали ввода‑вывода файлов.

```csharp
// Step 4: Load the image that contains the text to be recognized
ocrEngine.Image = ImageStream.FromFile(@"YOUR_DIRECTORY/invoice_arabic.png");
```

Убедитесь, что путь указывает на **валидное изображение**. Если файл отсутствует или повреждён, движок бросит исключение, и вы не сможете успешно **извлекать текст из изображения**.

---

## Шаг 5 – Запуск процесса OCR и получение результата

Наконец, вызываем `Recognize()` и выводим извлечённую строку.

```csharp
// Step 5: Perform the OCR operation
var ocrResult = ocrEngine.Recognize();

// Step 6: Display the extracted text
Console.WriteLine("=== Extracted Arabic Text ===");
Console.WriteLine(ocrResult.Text);
```

Свойство `ocrResult.Text` содержит текстовое представление всего, что движок смог прочитать. В большинстве случаев вы увидите смесь арабских символов и цифр — идеально для дальнейшей обработки, такой как вставка в базу данных или перевод.

### Ожидаемый вывод

```
=== Extracted Arabic Text ===
فاتورة رقم: 12345
التاريخ: 2026/03/15
المبلغ: 2500.00 ريال
...
```

Если видите «мусорные» символы, проверьте настройку языка и убедитесь, что изображение высокого разрешения (300 dpi и выше — оптимально).

---

## Полный рабочий пример

Ниже представлена **полная, автономная программа**, которую можно скопировать в новый консольный проект и сразу запустить.

```csharp
// Full OCR demo – extracts Arabic text from an image
using System;
using MyOcrSdk;
using MyOcrSdk.IO;

namespace ArabicOcrDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // 1️⃣ Create the OCR engine
            var ocrEngine = new OcrEngine();

            // 2️⃣ Set language – Arabic (change to Urdu or Kurdish if needed)
            ocrEngine.Config.Language = Language.Arabic;

            // 3️⃣ Load the source image
            // Replace YOUR_DIRECTORY with the folder that holds your PNG/JPG
            string imagePath = @"YOUR_DIRECTORY/invoice_arabic.png";
            ocrEngine.Image = ImageStream.FromFile(imagePath);

            try
            {
                // 4️⃣ Run OCR
                var result = ocrEngine.Recognize();

                // 5️⃣ Output the text
                Console.WriteLine("=== Extracted Arabic Text ===");
                Console.WriteLine(result.Text);
            }
            catch (Exception ex)
            {
                // Graceful error handling – useful when you need to **extract text from image** in batch jobs
                Console.Error.WriteLine($"OCR failed: {ex.Message}");
            }
        }
    }
}
```

> **Примечание:** Если ваш OCR‑SDK требует лицензирования, инициализируйте лицензию до шага 1 (например, `OcrEngine.SetLicense("YOUR_LICENSE_KEY");`). Эта строка опущена для краткости.

## Обработка распространённых граничных случаев

| Ситуация | Почему происходит | Быстрое решение |
|-----------|-------------------|-----------------|
| **Размытое или низкокачественное изображение** | Точность OCR падает ниже 70 % | Сканируйте с 300 dpi или увеличьте изображение с помощью бикубического алгоритма перед передачей в движок. |
| **Смешанные языки (арабский + английский)** | Движок может остановиться после первого языкового блока | Включите режим нескольких языков: `ocrEngine.Config.Language = Language.Arabic | Language.English;` |
| **Проблемы отображения справа‑налево** | Консоль выводит символы слева‑направо, из‑за чего арабский выглядит перевёрнутым | Установите `Console.OutputEncoding = System.Text.Encoding.UTF8;` и используйте терминал, поддерживающий RTL‑скрипты. |
| **Большие PDF, разбитые на множество страниц** | Потребление памяти резко возрастает | Обрабатывайте по одной странице: загружайте каждую страницу как отдельное изображение и повторяйте цикл OCR. |
| **Специальные символы (валюта, даты)** | Некоторые модели OCR считают их шумом | Пост‑обработайте `ocrResult.Text` с помощью regex для нормализации известных шаблонов. |

## Расширение решения – от OCR к извлечению данных

Теперь, когда вы **знаете, как выполнять OCR**, вам может быть интересно: *Что можно сделать с извлечённым арабским текстом?* Вот несколько идей, которые естественно следуют:

1. **Разбор счетов** — используйте регулярные выражения для извлечения номеров счетов, дат и сумм.  
2. **Отправка в API перевода** — передайте арабскую строку в Azure Translator или Google Cloud Translate.  
3. **Сохранение в базе данных** — вставьте очищенный текст в таблицу SQL для отчётности.  
4. **Запуск рабочих процессов** — комбинируйте с очередью сообщений (например, RabbitMQ) для начала последующей обработки.

Во всех этих сценариях используется одна и та же базовая операция: **преобразовать изображение в текст**, а затем работать с результатом.
