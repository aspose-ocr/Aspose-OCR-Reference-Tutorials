# Извлечение текста из изображения в C# – Руководство по офлайн OCR

Когда‑то вам нужно было **извлекать текст из изображения**, но идея отправлять файлы через интернет вас отталкивала? Вы не одиноки. Во многих регулируемых отраслях данные не могут покидать территорию компании, поэтому решение офлайн OCR становится необходимым. В этом руководстве показано, как именно извлекать текст из изображения в C# с помощью Aspose OCR в офлайн‑режиме — без сетевых вызовов, только локальная обработка.

Мы пройдём через загрузку изображения в C#‑коде, настройку языковой модели и, наконец, получение распознанного текста из картинки. К концу вы получите готовое консольное приложение, которое извлекает текст из изображения, не обращаясь к облаку. Без лишних деталей, только практичное решение «от начала до конца», которое можно сразу внедрять в свои проекты.

## Что вам понадобится

- **.NET 6 или новее** (код работает также с .NET Core и .NET Framework)
- **Aspose.OCR for .NET** пакет NuGet (версия 23.6 или новее)
- Пример изображения (PNG, JPG или TIFF), содержащего чёткий, разборчивый текст
- Visual Studio, Rider или любой предпочитаемый вами редактор C#

И всё—никаких дополнительных сервисов, никаких API‑ключей. Если у вас уже есть среда разработки C#, вы готовы к работе.

## Шаг 1 – Создание OCR‑движка и включение офлайн‑режима  

Первое, что нужно сделать, — создать экземпляр `OcrEngine` и установить флаг `OfflineMode`. Это заставит Aspose OCR использовать только языковые пакеты, поставляемые вместе с библиотекой.

```csharp
using System;
using System.Drawing;
using Aspose.OCR;

class OfflineModeTutorial
{
    static void Main()
    {
        // Initialize the OCR engine and force offline processing
        var ocrEngine = new OcrEngine
        {
            OfflineMode = true   // <-- crucial for on‑premise scenarios
        };
```

**Почему это важно:**  
Когда `OfflineMode` установлен в `true`, движок не будет пытаться загрузить какие‑либо языковые данные или отправлять телеметрию. Это гарантирует соответствие строгим политикам конфиденциальности данных.

## Шаг 2 – Загрузка изображения в стиле C#  

Теперь, когда движок готов, нам нужно передать ему изображение. Загрузка файла изображения в C# — простая задача с помощью `System.Drawing.Image.FromFile`. Просто убедитесь, что путь указывает на реальный файл на диске.

```csharp
        // Step 2: Load the image you want to process
        string imagePath = "YOUR_DIRECTORY/offline_test.png";
        ocrEngine.Image = Image.FromFile(imagePath);
```

> **Pro tip:** Если вы целитесь в .NET Core на Linux, добавьте пакет `System.Drawing.Common` и задайте переменную `LD_LIBRARY_PATH`, указывающую на `libgdiplus`. Иначе вы получите исключение во время выполнения.

**Внимание к граничному случаю:**  
Пустое или повреждённое изображение вызовет `FileNotFoundException` или `ArgumentException`. Оберните код загрузки в блок try‑catch, если ожидаете ненадёжный ввод.

## Шаг 3 – Установка языка OCR перед распознаванием  

Aspose OCR поставляется с несколькими языковыми пакетами, но вы должны указать движку, какие из них использовать. Здесь мы **устанавливаем язык OCR** для текущей сессии.

```csharp
        // Step 3: Specify the language model(s) that are available locally
        ocrEngine.Language = new[] { "English" }; // you can add more, e.g., "French", "Spanish"
```

**Почему следует установить язык:**  
Ограничение движка только теми языками, которые действительно нужны, ускоряет распознавание и уменьшает потребление памяти. Если пропустить этот шаг, движок будет пытаться угадывать, что может быть медленнее и менее точно.

## Шаг 4 – Выполнение операции OCR  

При полной настройке фактическое извлечение текста сводится к единому вызову метода. Метод `Recognize` возвращает распознанную строку.

```csharp
        // Step 4: Perform the OCR operation
        string recognizedText = ocrEngine.Recognize();

        // Step 5: Output the recognized text to the console
        Console.WriteLine(recognizedText);
    }
}
```

**Что вы увидите:**  
Если изображение содержит фразу «Hello World», консоль выведет:

```
Hello World
```

Если на изображении несколько строк, каждая строка будет разделена символом новой строки (`\n`). Вы можете дополнительно пост‑обрабатывать строку — обрезать пробелы, разбивать на слова или передавать в последующий NLP‑конвейер.

## Полный рабочий пример  

Ниже представлен полный код программы, который можно скопировать и вставить в новый консольный проект. Не забудьте заменить `YOUR_DIRECTORY/offline_test.png` на реальный путь к вашему тестовому изображению.

```csharp
using System;
using System.Drawing;
using Aspose.OCR;

class OfflineModeTutorial
{
    static void Main()
    {
        // 1️⃣ Create OCR engine and enable offline mode
        var ocrEngine = new OcrEngine
        {
            OfflineMode = true
        };

        // 2️⃣ Load image file C# style
        string imagePath = "YOUR_DIRECTORY/offline_test.png";
        ocrEngine.Image = Image.FromFile(imagePath);

        // 3️⃣ Set OCR language (English only in this demo)
        ocrEngine.Language = new[] { "English" };

        // 4️⃣ Run OCR and capture the result
        string recognizedText = ocrEngine.Recognize();

        // 5️⃣ Show the output
        Console.WriteLine("=== Recognized Text ===");
        Console.WriteLine(recognizedText);
    }
}
```

Запустите программу (`dotnet run` в терминале или нажмите F5 в Visual Studio) и наблюдайте вывод консоли. Если всё настроено правильно, вы только что **извлекли текст из изображения** без выхода за пределы вашего компьютера.

## Часто задаваемые вопросы и подводные камни  

- **Что делать, если нужен язык, который не включён в поставку?**  
  Aspose OCR предоставляет дополнительные языковые пакеты, которые можно скачать с портала Aspose. Поместите файл `.dat` в ту же папку, где находится ваш исполняемый файл, и добавьте его имя в массив `Language`.

- **Можно ли обрабатывать PDF вместо PNG?**  
  Да. Сначала преобразуйте каждую страницу PDF в изображение (например, с помощью `Aspose.PDF`), а затем передайте полученный bitmap в OCR‑движок. Рабочий процесс остаётся тем же.

- **Является ли движок потокобезопасным?**  
  Один экземпляр `OcrEngine` не следует использовать одновременно в нескольких потоках. Создавайте новый движок для каждого запроса, если вы строите веб‑сервис.

- **Подсказка по производительности:**  
  При пакетной обработке переиспользуйте один и тот же движок, но вызывайте `ocrEngine.Reset()` между изображениями. Это избавит от накладных расходов на повторную инициализацию языковых данных.

## Следующие шаги  

Теперь, когда вы можете **извлекать текст из изображения**, рассмотрите следующие идеи:

1. **Сохранить результаты** — записать распознанный текст в базу данных или файл JSON.  
2. **Комбинировать с ИИ** — передать вывод в Azure Cognitive Services для анализа настроений.  
3. **Пакетный режим** — пройтись по папке с изображениями, собрать результаты и сформировать сводный отчёт.  

Во всех этих расширениях также будет задействована загрузка файлов изображений в C# и, возможно, установка языка OCR для каждой партии, но базовый шаблон остаётся тем же.
