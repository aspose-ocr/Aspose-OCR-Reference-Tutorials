# Извлечение текста из изображения – Полное руководство по офлайн OCR

Когда‑нибудь вам нужно было **извлечь текст из изображения**, но вы беспокоились о задержках сети или ограничениях лицензирования? Вы не одиноки. Многие разработчики сталкиваются с проблемой, когда их приложение должно работать в изолированной среде, но при этом требует надёжного OCR. Хорошие новости? С Aspose OCR вы можете запускать весь конвейер локально, **извлекая текст из изображения** без обращения к интернету.

В этом руководстве мы пройдём практический пример, который показывает **как извлечь текст**, настроить офлайн‑движок, **выполнить распознавание OCR**, и даже **распознать кириллический текст**. К концу у вас будет готовое к запуску консольное приложение C#, которое выводит обнаруженную строку прямо в консоль.

## Что понадобится

- .NET 6.0 SDK (или любая современная версия .NET)  
- Visual Studio 2022 или VS Code – что вам больше нравится  
- NuGet‑пакет Aspose.OCR для .NET  
- Папка, содержащая файлы моделей Aspose OCR (скачайте один раз с портала Aspose)  
- Файл изображения, содержащий английские и кириллические символы (например, `cyrillic_doc.jpg`)

Никаких внешних сервисов, никаких скрытых загрузок во время выполнения – всё находится на вашем диске.

## Шаг 1: Установить Aspose.OCR и подготовить ресурсы

Сначала добавьте NuGet‑пакет Aspose.OCR в ваш проект:

```bash
dotnet add package Aspose.OCR
```

Затем создайте папку с именем `AsposeOCRResources` где‑нибудь на вашем компьютере и скопируйте в неё файлы моделей, скачанные с Aspose. OCR‑движок будет искать языковые пакеты в этом каталоге, поэтому убедитесь, что путь указан правильно.

> **Pro tip:** Держите папку ресурсов рядом с файлом `.csproj`; это упрощает работу с путями во время разработки.

## Шаг 2: Сборка офлайн‑движка OCR

Теперь мы создадим экземпляр движка и укажем ему папку ресурсов. Это ключевой шаг, позволяющий нам **выполнять распознавание OCR** полностью офлайн.

```csharp
using Aspose.OCR;
using Aspose.OCR.Enums;

class OfflineDemo
{
    static void Main()
    {
        // 1️⃣ Create the OCR engine instance
        OcrEngine ocrEngine = new OcrEngine();

        // 2️⃣ Tell the engine where the language models live
        ocrEngine.Settings.ResourcesPath = @"YOUR_DIRECTORY/AsposeOCRResources";

        // 3️⃣ Load the languages you need – English and Cyrillic
        ocrEngine.LoadLanguage(Language.English);
        ocrEngine.LoadLanguage(Language.Cyrillic);
```

Зачем явно загружать языки? Потому что мы указали движку работать офлайн; он не будет обращаться к серверам Aspose за недостающими пакетами. Если вы забудете загрузить язык, любые символы этого скрипта будут игнорироваться.

## Шаг 3: Передать изображение в движок

С готовым движком мы теперь передаём изображение, которое хотим обработать. Вспомогательная функция `ImageStream.FromFile` считывает файл в формат, понятный OCR‑движку.

```csharp
        // 4️⃣ Load the target image (make sure the path points to your file)
        ImageStream image = ImageStream.FromFile(@"YOUR_DIRECTORY/cyrillic_doc.jpg");
```

Если ваше изображение большое, рассмотрите возможность предварительного изменения его размера для повышения скорости и точности. OCR‑движок лучше всего работает при DPI около 300.

## Шаг 4: Выполнить процесс распознавания

Вызов `Recognize` выполняет основную работу. Метод возвращает объект `OcrResult`, содержащий извлечённую строку, оценки уверенности и многое другое.

```csharp
        // 5️⃣ Run the OCR recognition
        OcrResult ocrResult = ocrEngine.Recognize(image);
```

За кулисами Aspose разбирает растровое изображение, запускает нейронные сети, специфичные для языка, и соединяет символы вместе. Поскольку мы загрузили как английский, так и кириллический языки, документы со смешанными скриптами обрабатываются без проблем.

## Шаг 5: Вывести извлечённый текст

Наконец, мы выводим результат. В реальном приложении вы могли бы сохранить его в базе данных или передать другому сервису, но для этой демонстрации мы просто выведем его.

```csharp
        // 6️⃣ Show the extracted text on the console
        Console.WriteLine("=== OCR Output ===");
        Console.WriteLine(ocrResult.Text);
    }
}
```

Запуск программы должен вывести что‑то вроде:

```
=== OCR Output ===
This is an English line.
Это строка на кириллице.
```

Если вы видите искажённые символы, дважды проверьте, что кириллический языковой пакет правильно размещён в папке ресурсов и что изображение не слишком размыто.

## Обработка распространённых проблем

### Отсутствующие языковые пакеты

Если вы получаете исключение с сообщением «Language data not found», движок не смог найти файлы моделей. Проверьте `ResourcesPath` и убедитесь, что в папке находятся файлы `english.dat` и `cyrillic.dat` (или файлы с аналогичными названиями).

### Низкие оценки уверенности

Иногда OCR‑движок возвращает низкую уверенность для некоторых символов, особенно если изображение содержит шум. Вы можете повысить точность, сделав следующее:

- Преобразовать изображение в градации серого перед передачей в движок  
- Применить медианный фильтр для уменьшения пятен  
- Убедиться, что текст выровнен по горизонтали (при необходимости повернуть)

### Большие изображения

Обработка фотографии в 10 МП может быть медленной. Уменьшите её до максимальной ширины 2000 px при сохранении пропорций; движок всё равно захватит большинство символов с точностью.

## Расширение примера

- **Пакетная обработка:** Оберните логику распознавания в цикл, который проходит по всем файлам в каталоге.  
- **Форматы вывода:** `OcrResult` также предоставляет коллекцию `TextLines`, включающую ограничивающие рамки — полезно для подсветки текста в UI‑приложениях.  
- **Дополнительные языки:** Просто вызовите `LoadLanguage` с любым другим поддерживаемым значением перечисления (например, `Language.French`).  

Все эти расширения по‑прежнему следуют принципу **как извлечь текст** — просто загрузите соответствующие языковые пакеты, и движок выполнит остальное.

## Полный перечень исходного кода

Ниже представлен полный готовый к копированию код программы. Замените `YOUR_DIRECTORY` фактическим путём на вашем компьютере.

```csharp
using Aspose.OCR;
using Aspose.OCR.Enums;

class OfflineDemo
{
    static void Main()
    {
        // Step 1: Create an OCR engine instance
        OcrEngine ocrEngine = new OcrEngine();

        // Step 2: Point the engine to the folder that contains OCR model files
        ocrEngine.Settings.ResourcesPath = @"YOUR_DIRECTORY/AsposeOCRResources";

        // Step 3: Load the languages you need (no automatic download will occur)
        ocrEngine.LoadLanguage(Language.English);
        ocrEngine.LoadLanguage(Language.Cyrillic);

        // Step 4: Load the image you want to recognize
        ImageStream image = ImageStream.FromFile(@"YOUR_DIRECTORY/cyrillic_doc.jpg");

        // Step 5: Run the recognition process
        OcrResult ocrResult = ocrEngine.Recognize(image);

        // Step 6: Display the extracted text
        Console.WriteLine("=== OCR Output ===");
        Console.WriteLine(ocrResult.Text);
    }
}
```

Сохраните это как `Program.cs`, запустите `dotnet run` и наблюдайте, как консоль выводит текст, полученный движком из вашего изображения. Всё—вы успешно **извлекли текст из изображения** офлайн.
