# Как использовать OCR в C# – извлечение текста из изображений и файлов DJVU

Когда‑то задавались вопросом **как использовать OCR**, чтобы вытянуть слова со сканированной страницы без ручного ввода? Вы не одиноки. Во многих проектах — будь то оцифровка старых книг или извлечение данных из чеков — получение текста из изображения является ежедневной проблемой. Хорошая новость? С Aspose.OCR вы можете сделать это в несколько строк, даже если источник — файл DJVU.

В этом руководстве мы пройдёмся по всему, что нужно для **извлечения текста из изображений**, **загрузки изображения для OCR** и даже **конвертации DJVU в текст** с помощью C#. К концу вы получите готовое консольное приложение, которое выводит распознанный текст прямо в консоль. Никаких загадок, никаких дополнительных зависимостей, только чистый C# и Aspose.

## Что вы узнаете

- Как установить и подключить библиотеку Aspose.OCR.  
- Точный код, необходимый для **загрузки изображения для OCR**, будь то PNG, JPEG или страница DJVU.  
- Как вызвать движок и получить результат.  
- Советы по работе с большими документами и типичные подводные камни.  
- Полный, готовый к запуску пример, который можно скопировать‑вставить в Visual Studio.

> **Требования:** .NET 6.0 или новее и действующая лицензия Aspose.OCR (или бесплатная пробная версия). Если у вас ещё нет библиотеки, получите её из NuGet с помощью `dotnet add package Aspose.OCR`.

## Как использовать OCR с Aspose в C#

Это основной шаг, где мы действительно отвечаем на вопрос **как использовать OCR** в проекте C#. Приведённый ниже код — полная программа; её можно вставить в новое консольное приложение и нажать F5.

```csharp
using System;
using Aspose.OCR;

namespace AsposeOcrDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // Step 1: Create the OCR engine instance.
            var ocrEngine = new OcrEngine();

            // Step 2: Load the image (or DJVU page) you want to analyze.
            // Replace the path with your own file location.
            ocrEngine.Image = ImageStream.FromFile("YOUR_DIRECTORY/book-page.djvu");

            // Step 3: Run the recognition process.
            OcrResult ocrResult = ocrEngine.Recognize();

            // Step 4: Output the extracted text.
            Console.WriteLine("=== OCR Result ===");
            Console.WriteLine(ocrResult.Text);
        }
    }
}
```

**Почему это работает:**  
- `OcrEngine` — точка входа; он абстрагирует тяжёлую работу по предобработке изображения, определению языка и классификации символов.  
- `ImageStream.FromFile` умеет работать со многими форматами, включая DJVU, что позволяет **конвертировать DJVU в текст** без отдельного шага преобразования.  
- `Recognize()` возвращает объект `OcrResult`, содержащий простой текстовый вывод, оценки уверенности и даже ограничивающие рамки, если они понадобятся позже.

Запуск программы выводит что‑то вроде:

```
=== OCR Result ===
In the beginning God created the heavens and the earth.
```

И всё — ваша первая успешная операция **извлечения текста из DJVU**.

## Загрузка изображения для OCR

Прежде чем движок сможет что‑то прочитать, необходимо **загрузить изображение для OCR** правильно. Aspose поддерживает большинство растровых форматов «из коробки», но несколько советов могут сделать распознавание более плавным:

- **Измените размер больших изображений** до максимального 2000 px по самой длинной стороне. Слишком большие файлы увеличивают потребление памяти и могут замедлять движок.  
- **Преобразуйте цветные изображения в градации серого**, если замечаете шумный фон. Используйте `ocrEngine.Image = ImageStream.FromFile(path).ConvertToGrayscale();`.  
- **Установите DPI вручную** для сканов низкого разрешения (`ocrEngine.Image.DpiX = 300; ocrEngine.Image.DpiY = 300;`).

Эти настройки необязательны, но часто повышают точность при **извлечении текста из изображений**, например сканированных чеков.

## Извлечение текста из изображения – Лучшие практики

Когда вы работаете с типичными форматами изображений (PNG, JPEG, BMP), шаги остаются теми же, но можно дополнительно настроить движок:

```csharp
ocrEngine.Image = ImageStream.FromFile("sample.png");

// Optional: enable language packs for better accuracy.
ocrEngine.Language = Language.English;

// Optional: set recognition mode to auto.
ocrEngine.RecognitionMode = RecognitionMode.Auto;
```

- **Выбор языка** имеет значение. Если вы обрабатываете многоязычные документы, задайте `ocrEngine.Language = Language.Multilingual;`.  
- **RecognitionMode** может быть `Auto`, `Fast` или `Accurate`. `Accurate` даёт более высокое качество ценой скорости — используйте его для архивных задач.

## Конвертация DJVU в текст – Работа с многостраничными файлами

Файлы DJVU часто содержат несколько страниц. Aspose рассматривает каждую страницу как отдельный поток изображения, поэтому вы можете перебрать их в цикле:

```csharp
using Aspose.OCR;
using System.Collections.Generic;

var ocrEngine = new OcrEngine();
List<string> allPagesText = new List<string>();

int pageCount = ImageStream.GetPageCount("book.djvu");
for (int i = 1; i <= pageCount; i++)
{
    ocrEngine.Image = ImageStream.FromFile("book.djvu", i); // Load page i
    OcrResult result = ocrEngine.Recognize();
    allPagesText.Add(result.Text);
}

// Combine all pages into a single string.
string fullText = string.Join("\n--- Page Break ---\n", allPagesText);
Console.WriteLine(fullText);
```

**Зачем цикл?**  
Файлы DJVU по сути представляют собой стек изображений. Перебирая каждую страницу, вы **извлекаете текст из DJVU** последовательно, сохраняя оригинальный порядок документа.

## Распространённые проблемы и профессиональные советы

| Проблема | Почему происходит | Решение |
|----------|-------------------|---------|
| **«Мусорные» символы** | Низкое DPI или сильное сжатие | Увеличьте изображение, задайте `ocrEngine.Image.DpiX/Y = 300` |
| **Медленная обработка** | Использование режима `Accurate` на огромных файлах | Переключитесь на режим `Fast` для массовой обработки |
| **Отсутствие поддержки языка** | По умолчанию поддерживается только английский | Загрузите дополнительные языковые пакеты (`ocrEngine.Language = Language.Spanish;`) |
| **Ошибки «Out‑of‑memory»** | Одновременная загрузка многих высоко‑разрешённых страниц | Обрабатывайте страницы по одной и освобождайте ресурсы (`ocrEngine.Dispose();`) |

Быстрый **профессиональный совет**: всегда вызывайте `ocrEngine.Dispose()` после завершения работы со страницей. Это освобождает нативные ресурсы и предотвращает скрытые утечки памяти, которые могут привести к сбоям длительно работающих сервисов.

## Тестирование вашего OCR‑конвейера

Чтобы убедиться, что всё работает, выполните простые проверки:

1. **Выведите длину** распознанной строки: `Console.WriteLine($"Characters recognized: {ocrResult.Text.Length}");`  
2. **Сравните с известным текстом** с помощью простой библиотеки diff, если у вас есть эталон.  
3. **Запишите оценки уверенности** (`ocrResult.Confidence`), чтобы автоматически выявлять страницы низкого качества.

Если уверенность постоянно ниже 0.7, вернитесь к шагам предобработки изображения, описанным выше.
