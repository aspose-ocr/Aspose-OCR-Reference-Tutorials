# Как выполнять пакетное OCR в C# – Извлечение текста из PNG

Вы когда‑нибудь задумывались **как выполнять пакетное OCR** для кучи скриншотов, не пиша отдельную программу для каждого файла? Вы не одиноки. Во многих проектах у нас оказывается десятки PNG‑файлов, из которых нужно извлечь текст, а делать это по одному — настоящая головная боль.  

Хорошие новости? С Aspose OCR вы можете быстро создать небольшое консольное приложение на C#, которое обрабатывает все эти изображения параллельно, обеспечивая быструю **пакетную извлечение текста** и чистый набор результатов. В этом руководстве мы пройдём полный **c# ocr tutorial**, объясним, почему каждый элемент важен, и покажем, как выглядит вывод.

К концу этой статьи вы сможете:

* Загрузить список PNG‑файлов (или любого поддерживаемого изображения) за один раз.  
* Настроить общий `OcrEngine`, чтобы параметры оставались одинаковыми для всей партии.  
* Запустить очередь распознавания с до четырьмя параллельными рабочими потоками.  
* Получить распознанный текст для каждой страницы и вывести его в консоль.

Никакой магии, просто надёжный код, который вы можете добавить в своё решение уже сегодня.

## Что понадобится

Прежде чем погрузиться в детали, убедитесь, что у вас есть:

* .NET 6 SDK (или любая современная версия .NET).  
* Действующая лицензия Aspose OCR или временный оценочный ключ.  
* Папка, содержащая PNG‑файлы, которые нужно обработать.  
* Visual Studio 2022 или ваш любимый редактор.

И всё — никаких дополнительных пакетов NuGet, кроме `Aspose.OCR` и стандартного `System.Collections.Generic`.

## Как выполнить пакетное OCR – Настройка проекта

Первым делом создайте новый консольный проект и подключите библиотеку Aspose OCR.

```bash
dotnet new console -n BatchOcrDemo
cd BatchOcrDemo
dotnet add package Aspose.OCR
```

После завершения восстановления откройте **Program.cs** (или создайте новый файл) и добавьте обычные директивы `using`:

```csharp
using System;
using System.Collections.Generic;
using Aspose.OCR;
```

Эта простая заготовка даёт нам доступ к `OcrEngine`, `RecognitionQueue` и вспомогательным классам, которые понадобятся позже.

## Извлечение текста из PNG – Подготовка списка изображений

Теперь нам нужно указать программе, **какие PNG** обрабатывать через OCR. Самый простой способ – построить `List<string>`, содержащий абсолютные или относительные пути.

```csharp
// Step 1: Define the image files to be processed
var imagePaths = new List<string>
{
    @"YOUR_DIRECTORY/page1.png",
    @"YOUR_DIRECTORY/page2.png",
    @"YOUR_DIRECTORY/page3.png",
    @"YOUR_DIRECTORY/page4.png"
};
```

Замените `YOUR_DIRECTORY` на реальный путь к папке. Если набор файлов динамический, можно также использовать `Directory.GetFiles(@"YOUR_DIRECTORY", "*.png")` и передать полученный массив в список. Главное, что **извлечение текста из PNG** сводится к передаче правильных имён файлов в очередь.

## C# OCR tutorial – Настройка очереди распознавания

Сердцем пакетной операции является `RecognitionQueue`. Представьте её как конвейер, который передаёт каждое изображение общему `OcrEngine`. Делая движок общим, мы экономим память и гарантируем одинаковые настройки для каждой страницы.

```csharp
// Step 2: Create a recognition queue with shared OCR engine and parallelism
var recognitionQueue = new RecognitionQueue
{
    MaxDegreeOfParallelism = 4,          // run up to 4 images at the same time
    Engine = new OcrEngine()            // shared configuration for all images
};
```

Почему `MaxDegreeOfParallelism` установлен в 4? На типичном ноутбуке с четырьмя ядрами это обеспечивает лучшую пропускную способность без перегрузки ОС. Если вы работаете на сервере с большим числом ядер, увеличьте значение соответственно.

### Совет

Если нужны пользовательские языковые пакеты, настройки DPI или обрезка области интереса, сделайте это **один раз** на общем `Engine` до добавления изображений в очередь. Например:

```csharp
recognitionQueue.Engine.Language = Language.English;
recognitionQueue.Engine.Dpi = 300;
```

Все последующие распознавания автоматически наследуют эти параметры — это суть **how to create OCR** конвейеров, которые остаются согласованными.

## Пакетное извлечение текста – Добавление изображений в очередь и запуск обработки

Когда очередь готова, следующий шаг — добавить каждое изображение в неё. Метод `Enqueue` принимает экземпляр `OcrImage`, который мы создаём из пути к файлу.

```csharp
// Step 3: Enqueue each image for OCR processing
foreach (var path in imagePaths)
    recognitionQueue.Enqueue(OcrImage.FromFile(path));
```

После того как все файлы помещены в очередь, запускаем обработку:

```csharp
// Step 4: Run the queue and collect OCR results
List<OcrResult> ocrResults = recognitionQueue.ExecuteAll();
```

`ExecuteAll` блокирует выполнение, пока не завершится обработка всех изображений, а затем возвращает список, где каждый элемент соответствует порядку ввода. Это гарантирует, что результат страницы 1 находится в индексе 0, страницы 2 — в индексе 1 и так далее — удобно, когда нужно сопоставить результаты с исходными файлами.

## How to create OCR – Вывод результатов

Наконец, выведем распознанный текст в консоль. Здесь вы действительно увидите **пакетное извлечение текста** в действии.

```csharp
// Step 5: Output the recognized text for each page
for (int i = 0; i < ocrResults.Count; i++)
{
    Console.WriteLine($"--- Page {i + 1} ---");
    Console.WriteLine(ocrResults[i].Text);
}
```

При запуске программы (`dotnet run`) вы должны увидеть что‑то вроде:

```
--- Page 1 ---
Welcome to the quarterly report.
--- Page 2 ---
Revenue increased by 12% YoY.
--- Page 3 ---
All figures are provisional.
--- Page 4 ---
Contact finance@example.com for details.
```

Если какое‑то изображение не удалось обработать (например, файл повреждён), соответствующий `OcrResult` будет иметь пустое свойство `Text`, и вы сможете изучить `ocrResults[i].Exception` для диагностики.

## How to create OCR – Советы, особые случаи и лучшие практики

### Обработка больших партий

Обработка сотен PNG может всё ещё потреблять много памяти, если держать все объекты `OcrResult` в живых. В таких случаях сразу записывайте вывод в файл или базу данных, как только появляется каждый результат:

```csharp
recognitionQueue.OnResultReady += (sender, args) =>
{
    File.AppendAllText("OcrOutput.txt",
        $"--- {args.Image.Path} ---{Environment.NewLine}{args.Result.Text}{Environment.NewLine}");
};
```

### Работа с форматами, отличными от PNG

Aspose OCR также поддерживает JPEG, BMP и TIFF «из коробки». Просто измените расширение файлов в списке или используйте поиск по шаблону. Те же шаги **c# ocr tutorial** применимы — никаких изменений коду не требуется.

### Пропуск пустых страниц

Если у вас есть отсканированные PDF, иногда содержащие пустые страницы, вы можете отфильтровать результаты:

```csharp
if (!string.IsNullOrWhiteSpace(result.Text))
    // process non‑empty text
```

### Вопросы лицензирования

Оценочная версия ставит водяной знак на каждую страницу. Для продакшн‑использования убедитесь, что вы внедрили файл лицензии в начале `Main`:

```csharp
License lic = new License();
lic.SetLicense("Aspose.OCR.lic");
```

### Настройка параллелизма

`MaxDegreeOfParallelism` по умолчанию равно `Environment.ProcessorCount`. Если вы замечаете высокую загрузку CPU или давление на память, уменьшите значение. И наоборот, на облачной ВМ с множеством ядер повышайте его, чтобы полностью использовать оборудование.
