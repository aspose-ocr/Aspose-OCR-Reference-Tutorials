# Извлечение текста из изображений с помощью операции OCR в папках

## Введение

Добро пожаловать в мир оптического распознавания символов (OCR) с **Aspose.OCR for .NET**! Если вам нужно **извлекать текст из изображений** массово — например, из всей папки отсканированных документов — этот учебник проведёт вас через практическое решение. Мы рассмотрим всё: от настройки проекта до вывода распознанного текста, чтобы вы могли быстро интегрировать OCR по папкам в свои C#‑приложения.

## Быстрые ответы
- **Что изучает этот учебник?** Как извлекать текст из изображений, хранящихся в папке, с помощью Aspose.OCR.
- **Какой язык и платформа?** C# с .NET (Framework или .NET Core).
- **Ключевое требование?** Библиотека Aspose.OCR for .NET (ссылка для загрузки ниже).
- **Сколько строк кода?** Всего семь лаконичных блоков кода.
- **Можно ли конвертировать изображения в текст?** Да — пример показывает именно это.

## Что такое «извлечение текста из изображений»?
Извлечение текста из изображений означает использование технологии OCR для чтения символов, встроенных в картинки, PDF‑файлы или отсканированные документы, и преобразования их в редактируемые, поисковые строки. Aspose.OCR предоставляет надёжный движок, поддерживающий множество форматов изображений и языков.

## Почему стоит использовать Aspose.OCR для OCR по папкам?
- **Высокая точность** благодаря встроенному определению языка.  
- **Пакетная обработка** через `RecognizeMultipleImages`, идеально подходит для папок.  
- **Простой API**, естественно вписывающийся в C#‑проекты.  
- **Масштабируемость** — работает как на настольных, так и на серверных средах.

## Предварительные требования

- Базовые навыки разработки на C# и .NET.  
- Visual Studio (любая современная версия).  
- Библиотека **Aspose.OCR for .NET** — скачайте её [здесь](https://releases.aspose.com/ocr/net/).  
- Понимание концепций OCR (необязательно, но полезно).

## Подключение пространств имён

Добавьте необходимые директивы `using` в начало вашего C#‑файла, чтобы компилятор знал, где находятся классы OCR.

```csharp
using System;
using System.Collections.Generic;
using System.Drawing;
using System.IO;
using Aspose.OCR;
```

## Пошаговое руководство

### Шаг 1: Установите каталог документов
Определите папку, в которой находятся изображения для обработки.

```csharp
// ExStart:1   
// The path to the documents directory.
string dataDir = "Your Document Directory";
```

> **Полезный совет:** Используйте абсолютный путь или `Path.Combine`, чтобы избежать проблем с разделителями пути на разных ОС.

### Шаг 2: Инициализируйте Aspose.OCR
Создайте экземпляр движка OCR.

```csharp
// Initialize an instance of AsposeOcr
AsposeOcr api = new AsposeOcr();
```

### Шаг 3: Укажите путь к изображениям
Укажите API конкретную подпапку, содержащую ваши файлы изображений.

```csharp
// Image Path
string fullPath = dataDir + "OCR";
```

> **Почему это важно:** Метод `RecognizeMultipleImages` ожидает путь к папке, а не к отдельному файлу.

### Шаг 4: Распознайте изображения
Запустите OCR для каждого изображения в папке. При необходимости можно настроить `RecognitionSettings`, задав подсказки по языку или специфическую предобработку.

```csharp
// Recognize image           
RecognitionResult[] result = api.RecognizeMultipleImages(fullPath, new RecognitionSettings
{
    //default or custom
});
```

### Шаг 5: Выведите результаты
Пройдитесь по массиву `RecognitionResult` и выведите извлечённый текст.

```csharp
// Print result
for (int i = 0; i < result.Length; i++)
{
    Console.WriteLine($"Image: {i}\n Result:\n {result[i].RecognitionText}");
}
```

> **Распространённая ошибка:** Не проверять `result.Length` может привести к `IndexOutOfRangeException`, если папка пуста. Всегда проверяйте содержимое папки заранее.

### Шаг 6: Сообщение о завершении
Сигнализируйте об успешном выполнении.

```csharp
// ExEnd:1
Console.WriteLine("OCROperationWithFolder executed successfully");
```

## Распространённые проблемы и решения

| Проблема | Причина | Решение |
|----------|---------|---------|
| Нет вывода | Неправильный или пустой путь к папке | Убедитесь, что `fullPath` указывает на нужный каталог и содержит поддерживаемые форматы изображений (PNG, JPEG, TIFF). |
| Искажение символов | Неправильные настройки языка | Передайте сконфигурированный `RecognitionSettings` с `Language`, установленным в соответствующий ISO‑код. |
| Замедление при большом количестве изображений | Последовательная обработка в UI‑потоке | Выполняйте OCR в фоновом потоке или используйте асинхронные паттерны для сохранения отзывчивости UI. |
