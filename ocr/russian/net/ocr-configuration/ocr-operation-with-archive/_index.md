# Как выполнять OCR на изображениях в архиве с помощью Aspose.OCR для .NET

## Introduction

В этом всестороннем руководстве вы узнаете, **как выполнять OCR** на архивных файлах изображений с использованием библиотеки Aspose.OCR для .NET. Независимо от того, нужно ли вам **преобразовать изображения в текст** или **извлечь текст из архива**, пошаговое руководство ниже проведет вас через всё — от настройки среды разработки до получения распознанного текста из каждого изображения внутри ZIP‑архива.

## What is OCR and Why Use It on Archives?

Оптическое распознавание символов (OCR) преобразует отсканированные изображения или PDF‑файлы в поисковый, редактируемый текст. Когда изображения упакованы внутри архива (например, ZIP‑файла), их извлечение и распознавание каждого изображения за один проход экономит время и упрощает код. Метод `RecognizeMultipleImages` Aspose.OCR делает этот процесс простым.

## Step 1: Set up Your Document Directory

Начните с инициализации пути к каталогу ваших документов:

```csharp
// ExStart:1
// The path to the documents directory.
string dataDir = "Your Document Directory";
// ExEnd:1
```

> **Совет:** Используйте `Path.Combine` для кросс‑платформенной обработки путей.

## Step 2: Initialize Aspose.OCR

Создайте экземпляр класса Aspose.OCR, чтобы начать операции OCR:

```csharp
// ExStart:3
AsposeOcr api = new AsposeOcr();
// ExEnd:3
```

## Step 3: Specify Image Path

Укажите полный путь к вашему архивному изображению (ZIP‑файл, содержащий картинки, которые нужно прочитать):

```csharp
// ExStart:4
string fullPath = dataDir + "OCR.zip";
// ExEnd:4
```

## Step 4: Recognize Image

Выполните распознавание OCR на указанном архиве, используя настройки по умолчанию или пользовательские. Этот вызов автоматически извлекает каждое изображение из ZIP и запускает OCR:

```csharp
// ExStart:5
RecognitionResult[] result = api.RecognizeMultipleImages(fullPath, new RecognitionSettings
{
   //default or custom settings
});
// ExEnd:5
```

> Вы можете настроить `RecognitionSettings` для повышения точности для определённых языков или качества изображений.

## Step 5: Print Results

Пройдитесь по результатам и выведите распознанный текст для каждого изображения внутри архива:

```csharp
// ExStart:6
for (int i = 0; i < result.Length; i++)
{
	 Console.WriteLine($"Image: {i}\n Result:\n {result[i].RecognitionText}");
}
// ExEnd:6
```

Вывод показывает индекс каждого изображения, за которым следует извлечённая строка, эффективно **преобразуя изображения в текст** и **извлекая текст из архивных** файлов.

## Frequently Asked Questions

**В: Можно ли использовать Aspose.OCR для .NET без лицензии?**  
О: Да, доступна бесплатная пробная версия для оценки, но для продакшн‑развёртываний требуется лицензированная версия.

**В: Поддерживает ли библиотека ZIP‑архивы, защищённые паролем?**  
О: В текущей версии `RecognizeMultipleImages` работает со стандартными ZIP‑файлами. Для зашифрованных архивов сначала извлеките изображения с помощью сторонней библиотеки, затем передайте массив изображений в OCR‑движок.

**В: Как улучшить точность распознавания рукописного текста?**  
О: Включите флаг `RecognitionSettings.EnableHandwritingRecognition` и задайте более высокое значение DPI (например, 300).

**В: Можно ли получить оценки уверенности для каждой распознанной строки?**  
О: Каждый `RecognitionResult` содержит свойство `Confidence`, которое можно записать в журнал или использовать для фильтрации результатов с низкой уверенностью.
