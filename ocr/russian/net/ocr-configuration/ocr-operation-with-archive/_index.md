# Как выполнять OCR на изображениях в архиве с помощью Aspose.OCR для .NET

## Введение

В этом всестороннем руководстве вы узнаете, **как выполнять распознавание символов** в архивных файлах изображений с использованием библиотеки Aspose.OCR для .NET. Независимо от того, нужно ли вам **преобразовать изображение в текст** или **извлекать текст из архива**, приведенное ниже пошаговое руководство поможет вам пройти все через настройки среды разработки до получения распознанного текста из каждого изображения внутри ZIP-архива.

## Быстрые ответы
- **О чем этот учебник?** Выполнение оптического распознавания изображений в архиве (ZIP) с помощью Aspose.OCR для .NET.
- **Какое последнее ключевое слово используется?** *как выполнить ocr*.
- **Нужна ли лицензия?** Бесплатная пробная версия подходит для оценки; Для продакшн требуется коммерческая лицензия.
- **Какие версии .NET используются?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6+.
- **Можно ли настроить параметры измерения?** Да — воспользуйтесь `RecognitionSettings` для настройки точности.

## Что такое OCR и зачем его использовать в архивах?

Символы оптического распознавания (OCR) преобразуют отсканированные изображения или PDF-файлы в поисковый, редактируемый текст. Когда изображения вставляются внутрь архива (например, ZIP‑файла), их извлечение и распознавание каждого изображения за один проход экономят время и физический код. Метод `RecounceeMultipleImages` Aspose.OCR делает этот процесс простым.

- Действительная лицензия Aspose.OCR для использования в продакшене (доступна пробная версия).

## Шаг 1. Настройте каталог документов

Начните с инициализации пути к каталогу ваших документов:

```csharp
// ExStart:1
// The path to the documents directory.
string dataDir = "Your Document Directory";
// ExEnd:1
```

> **Совет:** Используйте `Path.Combine` для кросс‑платформенной обработки путей.

## Шаг 2: Инициализация Aspose.OCR

Создайте экземпляр класса Aspose.OCR, чтобы начать операции OCR:

```csharp
// ExStart:3
AsposeOcr api = new AsposeOcr();
// ExEnd:3
```

## Шаг 3: Указание пути к изображению

Укажите полный путь к вашему архивному изображению (ZIP‑файл, содержащий картинки, которые нужно прочитать):

```csharp
// ExStart:4
string fullPath = dataDir + "OCR.zip";
// ExEnd:4
```

## Шаг 4: Распознавание изображения

Выполните распознавание OCR на указанном архиве, используя настройки по умолчанию или пользовательские. Этот вызов автоматически извлекает каждое изображение из ZIP и запускает OCR:

```csharp
// ExStart:5
RecognitionResult[] result = api.RecognizeMultipleImages(fullPath, new RecognitionSettings
{
   //default or custom settings
});
// ExEnd:5
```

> Вы можете настроить `RecognitionSettings` для повышения точности для определённых языков или качества изображений.

## Шаг 5: Вывод результатов

Пройдитесь по результатам и выведите распознанный текст для каждого изображения внутри архива:

```csharp
// ExStart:6
for (int i = 0; i < result.Length; i++)
{
	 Console.WriteLine($"Image: {i}\n Result:\n {result[i].RecognitionText}");
}
// ExEnd:6
```

Вывод показывает индекс каждого изображения, за которым следует извлечённая строка, эффективно **преобразуя изображения в текст** и **извлекая текст из архивных** файлов.

## Распространенные проблемы и устранение неполадок

| Проблема | Причина | Решение |
|----------|---------|----------|
| Текст не возвращается | Низкое качество изображения | Предобработайте изображение (например, бинаризацию) или замените `RecognitionSettings.Dpi` |
| Исключение при чтении ZIP | Недействительный путь к архиву | Убедитесь, что `fullPath` указывает на действительный файл `.zip` и приложение имеет права чтения |
| Лицензия не применяется | Отсутствует файл в свободном состоянии или он не загружен | Вызовите `License License = new License(); License.SetLicense("Aspose.OCR.lic");` перед созданием экземпляра `AsposeOcr` |

## Часто задаваемые вопросы

**В: Можно ли использовать Aspose.OCR для .NET без лицензии?**
О: Да, доступна бесплатная пробная версия для оценки, но для продакшн‑развёртываний требуется бесплатная версия.

**В: Поддерживает ли библиотеку ZIP‑архивов, защищённых паролем?**
О: В текущей версии `RecounceeMultipleImages` работает со стандартными ZIP‑файлами. Для зашифрованных архивов сначала извлеките изображения с помощью временной библиотеки, затем передайте массив изображений в OCR‑движок.

**В: Как улучшить точность рукописного текста?**
О: Включите флаг RecognitionSettings.EnableHandwritingRecognition и задайте более высокое значение DPI (например, 300).

**В: Можно ли получить оценку достоверности для каждой распознанной строки?**
О: Каждый `RecognitionResult` содержит свойство `Confidence`, которое можно записать в журнал или использовать для фильтрации результатов с низкой интенсивностью.
