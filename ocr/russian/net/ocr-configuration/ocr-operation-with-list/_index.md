# Как пакетно выполнять OCR изображений из списка в Aspose.OCR для .NET

## Введение

Добро пожаловать в наш подробный учебник о **том, как пакетно выполнять OCR** нескольких изображений с помощью Aspose.OCR для .NET. Оптическое распознавание символов (OCR) преобразует отсканированные бумажные документы, PDF‑файлы или изображения в редактируемый, индексируемый текст. В этом руководстве вы узнаете, как **извлекать текст из изображений**, читать текст JPEG и обрабатывать несколько файлов за один вызов — идеально для сценариев, когда нужно **сканировать документ в текст** быстро и надёжно.

## Быстрые ответы
- **Что делает «мульти‑изображение OCR»?** Позволяет распознавать текст из списка файлов изображений одним вызовом API.  
- **Какие форматы поддерживаются?** JPEG, PNG, BMP, TIFF, GIF и многие другие.  
- **Нужна ли лицензия?** Для продакшна требуется временная лицензия; бесплатная пробная версия подходит для оценки.  
- **Можно ли настроить распознавание?** Да — используйте `RecognitionSettings` для изменения языка, разрешения и предобработки.  
- **Сколько изображений можно обработать одновременно?** Практически любое количество; API потоково передаёт каждый файл, поэтому потребление памяти остаётся низким.

## Что такое пакетный OCR и почему это важно?

**Пакетный OCR** (или «как пакетно выполнять OCR») — это возможность передать коллекцию путей к изображениям в Aspose.OCR и получить распознанный текст для каждого изображения за одну операцию. Такой подход уменьшает количество сетевых запросов, экономит время разработки и упрощает интеграцию OCR в автоматизированные конвейеры обработки документов, такие как обработка счетов, архивирование или автоматизация ввода данных.

## Почему стоит использовать Aspose.OCR для пакетной обработки изображений?

- **Высокая точность** на шумных сканах и низкокачественных JPEG.  
- **Встроенное определение языка** для многоязычных документов.  
- **Полная поддержка .NET** — работает с .NET Framework, .NET Core и .NET 5/6+.  
- **Отсутствие внешних зависимостей** — библиотека самостоятельно загружает изображения, выполняет предобработку и извлекает текст.  
- **Опции предобработки OCR‑изображений** позволяют улучшить результаты при плохом качестве сканов.

## Предварительные требования

Прежде чем перейти к коду, убедитесь, что у вас выполнены следующие требования:

1. Библиотека Aspose.OCR для .NET: Убедитесь, что библиотека Aspose.OCR установлена. Скачать её можно со [страницы загрузки Aspose.OCR для .NET](https://releases.aspose.com/ocr/net/).

2. Каталог документов: Создайте каталог, где будут храниться ваши документы и изображения для распознавания OCR.

Теперь, когда всё готово, приступим к пошаговому руководству.

## Импорт пространств имён

В вашем проекте C# добавьте необходимые пространства имён для использования Aspose.OCR для .NET:

```csharp
using System;
using System.Collections.Generic;
using System.Drawing;
using System.IO;
using Aspose.OCR;
```

## Пошаговое руководство

### Шаг 1: Настройте каталог документов

Инициализируйте путь к каталогу документов и создайте экземпляр `AsposeOcr`:

```csharp
// The path to the documents directory.
string dataDir = "Your Document Directory";

// Initialize an instance of AsposeOcr
AsposeOcr api = new AsposeOcr();
```

> **Совет:** Храните файлы изображений в подпапке (например, `dataDir/ocr`), чтобы проект оставался упорядоченным.

### Шаг 2: Укажите пути к изображениям

Определите список файлов изображений, которые нужно обработать. Можно смешивать JPEG, PNG, BMP или любой поддерживаемый формат:

```csharp
List<string> imagePaths = new List<string>
{
    dataDir + "0001460985.Jpeg",
    dataDir + "sample.png"
};
```

> **Почему это важно:** Передача `List<string>` позволяет **пакетно выполнять OCR** без написания собственного цикла — API берёт на себя всю тяжёлую работу.

### Шаг 3: Выполните распознавание OCR изображений

Вызовите `RecognizeMultipleImages` с необязательными `RecognitionSettings`. Здесь вы можете применить **предобработку OCR‑изображений**, например, исправление наклона или удаление шума:

```csharp
RecognitionResult[] result = api.RecognizeMultipleImages(imagePaths, new RecognitionSettings
{
   //default or custom settings
});
```

> **Как извлечь текст с пользовательскими настройками:** Если нужен определённый язык или более высокое DPI, задайте `RecognitionSettings.Language` и `RecognitionSettings.Dpi`.

### Шаг 4: Выведите результаты распознавания

Пройдитесь по результатам и выведите распознанный текст для каждого изображения:

```csharp
for (int i = 0; i < result.Length; i++)
{
	 Console.WriteLine($"Image: {i}\n Result:\n {result[i].RecognitionText}");
}
```

Теперь вы должны увидеть извлечённый текст для каждого файла, напечатанный в консоли, что демонстрирует, как **извлекать текст из изображений** массово.

## Распространённые проблемы и их решения

| Проблема | Причина | Решение |
|----------|---------|----------|
| Текст не возвращается | Слишком низкое качество изображения | Увеличьте DPI или используйте `RecognitionSettings` для включения предобработки |
| Определён неверный язык | Язык по умолчанию — английский | Установите `RecognitionSettings.Language` в нужный код языка |
| Ошибка «Out‑of‑memory» при больших пакетах | Одновременная загрузка многих изображений высокого разрешения | Обрабатывайте изображения небольшими партиями или используйте потоковую передачу через `RecognizeMultipleImages`, которая уже реализует стриминг |
