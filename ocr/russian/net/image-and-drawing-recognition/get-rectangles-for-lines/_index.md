# Получение прямоугольников OCR‑строк для текста на изображении

## Введение

В этом руководстве вы узнаете **как получить прямоугольники OCR‑строк** с помощью Aspose.OCR для .NET. К концу руководства вы сможете **распознавать строки текста на изображении** и **извлекать координаты строк** для каждой обнаруженной строки — это идеально для последующей обработки, такой как анализ макета, извлечение данных или пользовательская отрисовка.

## Быстрые ответы
- **Что означает «получить прямоугольники OCR‑строк»?** Возвращает ограничивающие рамки каждой строки текста, обнаруженной на изображении.  
- **Какой метод API используется?** `AsposeOcr.GetRectangles(..., AreasType.LINES, ...)`.  
- **Нужна ли лицензия?** Бесплатная пробная версия подходит для разработки; для продакшна требуется коммерческая лицензия.  
- **Поддерживаемые форматы изображений?** PNG, JPEG, BMP, TIFF и другие.  
- **Можно ли запускать на .NET Core?** Да, Aspose.OCR полностью поддерживает .NET Core и .NET 5/6.

## Предварительные требования

Прежде чем приступить к руководству, убедитесь, что у вас есть следующие условия:

- Базовые знания C# и разработки под .NET.  
- Интегрированная среда разработки (IDE), например Visual Studio.  
- Библиотека Aspose.OCR для .NET, установленная в проект. Скачать её можно [здесь](https://releases.aspose.com/ocr/net/).  
- Пример изображения с текстом для OCR‑распознавания.

## Импорт пространств имён

Убедитесь, что необходимые пространства имён импортированы в ваш проект. Добавьте следующие строки в начало вашего C#‑файла:

```csharp
using System;
using System.Collections.Generic;
using System.Drawing;
using System.IO;
using Aspose.OCR;
```

Теперь разберём процесс получения прямоугольников для строк в OCR‑распознавании изображения на простые шаги.

## Шаг 1: Настройка каталога документов

```csharp
// ExStart:3
string dataDir = "Your Document Directory";
// ExEnd:3
```

Замените `"Your Document Directory"` реальным путём к папке, где находится ваше тестовое изображение.

## Шаг 2: Инициализация Aspose.OCR

```csharp
// ExStart:4
AsposeOcr api = new AsposeOcr();
// ExEnd:4
```

Создайте экземпляр класса `AsposeOcr`, чтобы получить доступ к функциям OCR.

## Шаг 3: Указание пути к изображению

```csharp
// ExStart:5
string fullPath = dataDir + "sample.png";
// ExEnd:5
```

Определите полный путь к изображению, которое нужно обработать.

## Шаг 4: Распознавание изображения и получение прямоугольников

```csharp
// ExStart:6
List<Rectangle> lines = api.GetRectangles(fullPath, AreasType.LINES, false);
// ExEnd:6
```

Метод `GetRectangles` возвращает список объектов `Rectangle`, каждый из которых представляет координаты обнаруженной строки текста. Это ядро **получения прямоугольников OCR‑строк**.

## Шаг 5: Вывод результата

```csharp
// ExStart:7
Console.WriteLine("Areas coordinates:");
lines.ForEach(a => Console.WriteLine($"x:{a.X} y:{a.Y} width:{a.Width} height:{a.Height}"));
// ExEnd:7
```

Выведите координаты найденных областей в консоль. Вы увидите значения, которые позже можно использовать для **извлечения координат строк** в пользовательской обработке.

## Почему стоит использовать Aspose.OCR для получения прямоугольников строк?

- **Высокая точность** — Продвинутые алгоритмы обнаруживают строки даже в шумных или наклонённых изображениях.  
- **Кросс‑платформенность** — Работает на .NET Framework, .NET Core и .NET 5/6.  
- **Отсутствие внешних зависимостей** — Чистая .NET‑библиотека, без нативных DLL.  
- **Богатый вывод** — Помимо прямоугольников строк, можно получить слова, символы и оценки уверенности.

## Распространённые проблемы и решения

| Проблема | Решение |
|----------|---------|
| **Не возвращаются прямоугольники** | Убедитесь, что на изображении есть чёткий горизонтальный текст и указано `AreasType.LINES`. |
| **Неправильные координаты** | Проверьте DPI изображения; изображения низкого разрешения могут давать неточные границы. |
| **Снижение производительности на больших изображениях** | Перед вызовом `GetRectangles` уменьшите изображение до разумного разрешения. |
| **Исключение лицензии** | Для тестов используйте пробную лицензию; для продакшна примените полную лицензию, чтобы снять ограничения оценки. |

## Часто задаваемые вопросы (FAQ)

**В: Можно ли извлекать отдельные слова вместо целых строк?**  
О: Да, используйте `AreasType.WORDS` с тем же методом `GetRectangles`, чтобы получить ограничивающие рамки на уровне слов.

**В: Поддерживает ли API многостраничные PDF?**  
О: Сначала преобразуйте каждую страницу PDF в изображение, затем вызывайте `GetRectangles` для каждого изображения.

**В: Как обрабатывать повернутый текст?**  
О: Включите опцию авто‑поворота в настройках OCR или предварительно поверните изображение перед обработкой.

**В: Можно ли получить оценки уверенности для каждой строки?**  
О: После получения прямоугольников вызовите `api.RecognizeImage(...).Lines`, чтобы получить объекты строк с включёнными значениями уверенности.

**В: Какие версии .NET совместимы?**  
О: Библиотека работает с .NET Framework 4.5+, .NET Core 3.1+ и .NET 5/6.
