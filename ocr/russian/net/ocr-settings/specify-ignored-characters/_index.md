# Поддержка языков OCR: указание игнорируемых символов при распознавании изображений

## Введение

Поддержка языков OCR является краеугольным камнем современной автоматизации документов, позволяя приложениям считывать текст с изображений на множестве алфавитов и символов. В этом руководстве вы узнаете, как указать **Aspose.OCR for .NET** игнорировать определённые символы во время распознавания — важный приём, когда нужен более чистый вывод или требуется отфильтровать шум, такой как номера страниц или декоративные символы. К концу руководства у вас будет готовый к запуску фрагмент кода, демонстрирующий эту возможность от начала до конца.

## Быстрые ответы
- **Что означает “ignored characters”?** Символы, которые движок OCR пропускает при построении результирующей строки.  
- **Зачем это использовать?** Повышает точность, когда определённые символы не имеют значения для вашей бизнес‑логики.  
- **Какой метод API обрабатывает это?** `RecognitionSettings.IgnoredCharacters`.  
- **Можно ли комбинировать это с языковыми пакетами?** Да — игнорируемые символы работают вместе с любым загруженным языком.  
- **Требуется ли лицензия?** Для использования в продакшене необходима временная или полная лицензия.

## Зачем указывать игнорируемые символы?

Во многих реальных сценариях — например, при обработке счетов, чеков или многоязычных форм — вы можете сталкиваться с повторяющимися символами, которые не являются частью значимых данных (например, дефисы, используемые как разделители, номера страниц или декоративные символы). Указав OCR‑движку пропускать их, вы уменьшаете объём последующей обработки и повышаете общую надёжность последующего анализа.

## Пошаговое руководство

### Шаг 1: Настройте каталог документов

Начните с указания каталога, где хранятся ваши документы. Замените `"Your Document Directory"` на реальный путь к вашим документам.

```csharp
// The path to the documents directory.
string dataDir = "Your Document Directory";
```

### Шаг 2: Инициализируйте Aspose.OCR

Создайте экземпляр OCR‑движка. Этот объект будет обрабатывать все последующие вызовы распознавания.

```csharp
// Initialize an instance of AsposeOcr
AsposeOcr api = new AsposeOcr();
```

### Шаг 3: Распознайте изображение с игнорируемыми символами

Передайте файл изображения вместе с объектом `RecognitionSettings`, в котором перечислены символы, которые вы хотите игнорировать. В этом примере мы игнорируем символы `a`, `b` и `1`.

```csharp
// Recognize image with specified ignored characters
RecognitionResult result = api.RecognizeImage(dataDir + "SpanishOCR.bmp", new RecognitionSettings
{
    IgnoredCharacters = "ab1"
});
```

### Шаг 4: Выведите распознанный текст

Наконец, выведите очищенный текст в консоль или любой другой вывод по вашему выбору.

```csharp
// Display the recognized text
Console.WriteLine(result.RecognitionText);
```

## Распространённые проблемы и советы

- **Неправильный путь:** Убедитесь, что `dataDir` заканчивается разделителем пути (`\` или `/`), соответствующим вашей ОС.  
- **Неподдерживаемый язык:** OCR‑движок должен иметь языковой пакет для исходного изображения; иначе игнорируемые символы не будут применяться корректно.  
- **Ошибки лицензии:** Если вы видите исключение, связанное с лицензией, проверьте, что файл временной лицензии правильно указан в вашем проекте.
