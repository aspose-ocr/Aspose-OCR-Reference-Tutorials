# ocr image to text: Указание разрешённых символов в OCR

## Введение

В постоянно меняющемся ландшафте технологий оптическое распознавание символов (OCR) — или преобразование **ocr image to text** — стал трансформирующим инструментом, позволяющим машинам понимать текст с изображений. Aspose.OCR for .NET выделяется как мощное решение, обеспечивая бесшовную интеграцию для разработчиков, ищущих надёжные возможности OCR в своих .NET‑приложениях.

## Быстрые ответы
- **Что делает “Specify Allowed Characters”?** Огранивает вывод OCR заданным набором символов, например только цифрами.  
- **Какой метод извлекает одну строку текста?** `RecognizeLine` возвращает первую найденную строку.  
- **Можно ли менять настройки распознавания OCR на лету?** Да — используйте `RecognitionSettings` для настройки параметров, таких как `AllowedCharacters`.  
- **Доступна ли пробная версия?** Конечно, скачайте бесплатную пробную версию с сайта Aspose.  
- **Какие версии .NET поддерживаются?** Все современные версии .NET Framework и .NET Core/5/6.

## Предварительные требования

Прежде чем погрузиться в руководство, убедитесь, что у вас есть следующие предварительные требования:

- Рабочие знания разработки на .NET.  
- Библиотека Aspose.OCR for .NET. Вы можете скачать её [здесь](https://releases.aspose.com/ocr/net/).  
- Знакомство с Visual Studio или любой другой предпочтительной средой разработки .NET.

## Импорт пространств имён

В вашем .NET‑проекте импортируйте необходимые пространства имён, чтобы эффективно использовать функциональность Aspose.OCR for .NET:

```csharp
using System;
using System.IO;
using Aspose.OCR;
```

Теперь разберём руководство на серию подробных шагов:

## Шаг 1: Указание разрешённых символов в ocr image to text

Для начала задайте путь к каталогу ваших документов:

```csharp
string dataDir = "Your Document Directory";
```

## Шаг 2: Инициализация Aspose.OCR с разрешёнными символами (распознавание изображения с цифрами)

Создайте экземпляр `AsposeOcr`, указав разрешённые символы. В данном случае разрешаем только цифры (0‑9):

```csharp
AsposeOcr api = new AsposeOcr("0123456789");
```

## Шаг 3: Распознавание изображения

Используйте экземпляр `AsposeOcr` для распознавания текста с изображения:

```csharp
string result = api.RecognizeLine(dataDir + "0001460985.Jpeg");
```

## Шаг 4: Отображение распознанного текста

Выведите распознанный текст в консоль:

```csharp
Console.WriteLine(result);
```

## Шаг 5: Второй случай — распознавание изображения с конкретными настройками OCR

Инициализируйте другой экземпляр `AsposeOcr`, на этот раз с более конкретными настройками, демонстрирующими использование **ocr recognition settings**:

```csharp
AsposeOcr api2 = new AsposeOcr();
RecognitionResult result2 = api2.RecognizeImage(dataDir + "0001460985.Jpeg", 
    new RecognitionSettings { 
        AllowedCharacters = CharactersAllowedType.DIGITS,
        RecognizeSingleLine = true
    });
```

## Шаг 6: Отображение распознанного текста второго случая

Выведите распознанный текст второго случая в консоль:

```csharp
Console.WriteLine(result2.RecognitionText);
```

## Шаг 7: Успешное выполнение

Наконец, подтвердите успешное выполнение руководства **SpecifyAllowedCharacters**:

```csharp
Console.WriteLine("SpecifyAllowedCharacters executed successfully");
```

Следуя этим шагам, вы получили возможность **указать разрешённые символы** в распознавании изображений OCR с помощью Aspose.OCR for .NET, обеспечивая точное преобразование **ocr image to text** для сценариев только с цифрами.

## Почему использовать фильтрацию по разрешённым символам?

- **Более высокая точность:** Ограничение набора символов уменьшает количество ложных распознаваний, особенно на шумных изображениях.  
- **Увеличение производительности:** OCR‑движок пропускает нерелевантные глифы, ускоряя обработку.  
- **Соответствие:** Обеспечивает соблюдение форматов данных (например, номера счетов, серийные коды) непосредственно на этапе OCR.

## Распространённые подводные камни и советы

- **Подводный камень:** Передача пустой строки в качестве разрешённых символов отключает фильтрацию.  
  **Совет:** Всегда передавайте непустую строку или используйте перечисление `CharactersAllowedType`.  
- **Подводный камень:** Использование `RecognizeLine` в многострочных документах может пропустить данные.  
  **Совет:** Переключитесь на `RecognizeImage` с `RecognizeSingleLine = false` для извлечения полной страницы.  
- **Подводный камень:** Ошибка при неправильном объединении пути к каталогу может вызвать `FileNotFoundException`.  
  **Совет:** Используйте `Path.Combine(dataDir, "file.jpg")` для кроссплатформенных путей.
