# Указание разрешённых символов OCR – использование Aspose.OCR для .NET

В этом руководстве вы узнаете, как **specify allowed characters ocr** с помощью Aspose.OCR для .NET, позволяя ограничить вывод OCR только необходимыми символами. Это особенно удобно, когда нужно **recognize digits image** такие как серийные номера, идентификаторы счетов или строки, похожие на штрих‑коды. Мы пройдём через настройку, код и несколько практических сценариев, чтобы вы могли сразу применить эту технику.

## Быстрые ответы
- **What does “specify allowed characters ocr” do?** Он ограничивает OCR предопределённым набором символов, повышая точность для целевых данных.  
- **Which characters can I allow?** Любая комбинация, которая вам нужна — цифры, буквы или пользовательские символы (например, “0123456789”).  
- **Why limit characters?** Сокращает количество ложных распознаваний и ускоряет обработку, когда известен ожидаемый набор символов.  
- **Do I need a license?** Бесплатная пробная версия подходит для разработки; для продакшна требуется коммерческая лицензия.  
- **Which .NET versions are supported?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7.

## Что такое “specify allowed characters ocr”?
Когда OCR сканирует изображение, он пытается сопоставить каждый визуальный паттерн со всем алфавитом возможных символов. С помощью **specify allowed characters ocr** вы говорите движку игнорировать всё, что не входит в ваш белый список, что значительно повышает точность распознавания для ограниченных наборов данных.

## Почему использовать Aspose.OCR для распознавания цифр на изображении?
Aspose.OCR предоставляет чистый, лаконичный API для разработчиков .NET. Встроенный параметр `AllowedCharacters` позволяет сосредоточиться на сценариях только с цифрами без необходимости писать пользовательскую пост‑обработку. Это идеально подходит для:
- Считывания показаний счётчиков, номеров счетов или кодов продукции.  
- Проверки данных, введённых пользователем и полученных со сканированных форм.  
- Ускорения пакетной обработки, когда набор символов известен заранее.

## Как указать разрешённые символы OCR – пошаговое руководство

### Шаг 1: Установите путь к папке с изображениями

Сначала задайте, где хранятся ваши образцы изображений.

```csharp
string dataDir = "Your Document Directory";
```

### Шаг 2: Инициализируйте Aspose.OCR с белым списком только цифр

Создайте экземпляр `AsposeOcr` и передайте символы, которые хотите разрешить — в данном случае все цифры.

```csharp
AsposeOcr api = new AsposeOcr("0123456789");
```

### Шаг 3: Распознайте одну строку, содержащую цифры

Используйте метод `RecognizeLine`, чтобы извлечь текст из изображения, содержащего только числа.

```csharp
string result = api.RecognizeLine(dataDir + "0001460985.Jpeg");
```

### Шаг 4: Выведите распознанные цифры

Выведите результат в консоль, чтобы проверить вывод.

```csharp
Console.WriteLine(result);
```

### Шаг 5: Используйте RecognitionSettings для более тонкой настройки

Если требуется более точный контроль — например, принудительное распознавание одной строки — можно воспользоваться перегрузкой, принимающей `RecognitionSettings`.

```csharp
AsposeOcr api2 = new AsposeOcr();
RecognitionResult result2 = api2.RecognizeImage(dataDir + "0001460985.Jpeg", 
    new RecognitionSettings { 
        AllowedCharacters = CharactersAllowedType.DIGITS,
        RecognizeSingleLine = true
    });
```

### Шаг 6: Отобразите результат второго случая

```csharp
Console.WriteLine(result2.RecognitionText);
```

### Шаг 7: Подтвердите успешное выполнение

```csharp
Console.WriteLine("SpecifyAllowedCharacters executed successfully");
```

Следуя этим шагам, вы научились **specify allowed characters ocr** и эффективно **recognize digits image** с помощью Aspose.OCR для .NET.

## Распространённые ошибки и их устранение

- **Пустой результат:** Убедитесь, что качество изображения достаточное (чёткий контраст, минимум шума).  
- **Неправильные символы:** Проверьте, что строка белого списка точно соответствует ожидаемым символам.  
- **Файл не найден:** Убедитесь, что `dataDir` указывает на правильную папку и имя файла учитывает регистр.
