# Извлечение текста из изображения с прямоугольниками – Руководство по OCR

## Введение

Оптическое распознавание символов (OCR) позволяет вам **extract text from image** файлы, делая их доступными для поиска и редактирования. В этом руководстве мы покажем, как повысить точность OCR, подготовив пользовательские прямоугольники, которые фокусируют движок на нужных зонах. С помощью Aspose.OCR for .NET вы увидите полный рабочий процесс — от настройки проекта до получения распознанных строк — чтобы вы могли внедрить надёжное преобразование изображения в текст в любое .NET приложение.

## Быстрые ответы
- **Что означает “extract text from image”?** Он преобразует визуальные символы на изображении в машинно‑читаемые строки.  
- **Какая библиотека обрабатывает это в .NET?** Aspose.OCR for .NET предоставляет полнофункциональный OCR‑движок.  
- **Нужна ли лицензия для продакшн?** Бесплатная пробная версия подходит для разработки; для развертывания требуется коммерческая лицензия.  
- **Можно ли ограничить OCR конкретными зонами?** Да — определите прямоугольники, чтобы охватить только области, содержащие полезный текст.  
- **Какие версии .NET поддерживаются?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7.

## Что такое “extract text from image” с прямоугольниками?

The `extract text from image` процесс читает символы внутри определённых прямоугольных зон, игнорируя всё остальное. Ограничивая OCR этими зонами, вы получаете более высокую точность, более быструю обработку и меньше усилий на пост‑обработку. Такой подход изолирует нужный вам текст, отбрасывая фоновые графические элементы, декоративные детали и другой визуальный шум, который может сбивать OCR‑движок.

## Почему стоит подготовить прямоугольники перед OCR?

Подготовка прямоугольников позволяет сосредоточить OCR‑движок на наиболее релевантных частях изображения, что улучшает как скорость, так и точность. Сужая область анализа, вы уменьшаете объём данных, которые движок должен просмотреть, что приводит к более быстрым результатам и меньшему количеству ошибок распознавания, вызванных посторонним визуальным шумом.

- **Сфокусироваться на релевантном содержимом:** Пропускать заголовки, нижние колонтитулы или декоративные графики, которые могут сбить движок.  
- **Повысить производительность:** Меньшие области требуют меньше вычислений, сокращая время выполнения до 40 % на больших сканах.  
- **Улучшить точность:** Снижение визуального шума повышает процент распознавания символов с ~85 % до >95 % на шумных документах.

## Почему это важно для реальных проектов

Во многих бизнес‑документах — чеки, счета, удостоверения личности — текст смешивается с логотипами или штрих‑кодами. Рисуя прямоугольники вокруг нужных полей, вы извлекаете только ценные данные, сокращая последующую очистку и повышая надёжность автоматизированных рабочих процессов. Такое целевое извлечение уменьшает усилия пост‑обработки и помогает соблюдать стандарты обращения с данными.

## Как извлечь текст из изображения с использованием нескольких прямоугольников

Загрузите изображение один раз, затем передайте список прямоугольников OCR‑движку. Каждый прямоугольник определяет область интереса, и движок возвращает отдельную строку для каждой области, позволяя обрабатывать поля по отдельности и комбинировать результаты по необходимости.

### Определите прямоугольники

Объекты `Rectangle` описывают координаты X‑Y и размер каждой зоны, которую вы хотите сканировать.  

```csharp
List<Rectangle> rects = new List<Rectangle>()
{
    new Rectangle(138, 352, 2033, 537),
    new Rectangle(147, 890, 2033, 1157),
    new Rectangle(923, 2045, 465, 102),
    new Rectangle(104, 2147, 2076, 819)
};
```

### Выполните распознавание OCR

Метод `RecognizeImage` обрабатывает изображение, используя предоставленный список прямоугольников, и возвращает распознанный текст для каждой области.  

```csharp
// first case
List<string> listResult = api.RecognizeImage(dataDir + "sample.png", rects);

// Display the recognized text
foreach (string s in listResult)
{
    Console.WriteLine(s);
}
```

## Как извлечь текст из изображения с использованием RecognitionSettings (альтернативный подход)

Если вы предпочитаете вызов на основе настроек, вы можете достичь того же результата с помощью `RecognitionSettings`. Этот объект позволяет объединить определения прямоугольников с языком, DPI и другими параметрами OCR, предоставляя чистый вызов API с одним параметром.

### Определите параметры распознавания

`RecognitionSettings` позволяет собрать список прямоугольников и дополнительные параметры (например, язык, DPI) в один объект.  

```csharp
RecognitionResult result = api.RecognizeImage(dataDir + "sample.png", new RecognitionSettings
{
    RecognitionAreas = rects
});
```

### Выведите распознанный текст

Итерируйте возвращённые строки и выводите текст каждой области.  

```csharp
// Display the recognized text
foreach (string s in result.RecognitionAreasText)
{
    Console.WriteLine(s);
}
```

## Распространённые проблемы и советы

- **Неправильные координаты прямоугольника:** Убедитесь, что `X`, `Y`, `Width` и `Height` соответствуют нужной области.  
- **Качество изображения:** Низкое разрешение или сильное сжатие ухудшают результаты OCR; рассмотрите предобработку, например бинаризацию.  
- **Пустые результаты:** Убедитесь, что прямоугольники действительно содержат читаемый текст; иначе движок вернёт пустые строки.

## Устранение неполадок и лучшие практики

| Симптом | Вероятная причина | Решение |
|---------|-------------------|--------|
| Нет вывода или пустые строки | Прямоугольники находятся за пределами изображения | Проверьте размеры изображения и координаты прямоугольников |
| Искажённые символы | Плохой контраст или шум | Примените преобразование в градации серого и пороговую обработку перед OCR |
| Низкая производительность на больших файлах | Слишком много прямоугольников или очень большое изображение | Разделите изображение или уменьшите количество прямоугольников, где это возможно |
