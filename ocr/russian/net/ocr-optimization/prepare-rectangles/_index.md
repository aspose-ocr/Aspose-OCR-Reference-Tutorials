# Подготовка прямоугольников в распознавании изображений OCR

## Введение

Оптическое распознавание символов (OCR) необходимо для преобразования визуального контента в поисковый, редактируемый текст. В этом руководстве вы **извлечёте текст из изображения** путем подготовки пользовательских прямоугольников, которые фокусируют движок OCR на определённых областях. С помощью Aspose.OCR for .NET мы пройдем каждый шаг — от настройки проекта до получения распознанного текста — чтобы вы могли интегрировать мощную функцию преобразования изображения в текст в свои .NET‑приложения.

## Быстрые ответы
- **Что означает «извлечь текст из изображения»?** Это преобразование визуальных символов на картинке в машинно‑читаемые строки.  
- **Какая библиотека помогает в этом в .NET?** Aspose.OCR for .NET.  
- **Нужна ли лицензия для разработки?** Бесплатная пробная версия подходит для тестирования; лицензия требуется для продакшн.  
- **Можно ли нацеливаться на конкретные области?** Да, определяя прямоугольники, ограничивающие область OCR.  
- **Какие версии .NET поддерживаются?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7.

## Что такое «извлечь текст из изображения» с прямоугольниками?
Когда вы определяете прямоугольные зоны на изображении, движок OCR обрабатывает только эти зоны. Это повышает точность, сокращает время обработки и позволяет игнорировать шумные фоны или несущественные части.

## Зачем готовить прямоугольники перед OCR?
- **Сфокусироваться на релевантном контенте:** Пропускать заголовки, нижние колонтитулы или декоративные графики.  
- **Повысить производительность:** Меньшие области означают более быстрое распознавание.  
- **Улучшить точность:** Меньше визуального шума приводит к более чистым результатам.

// The path to the documents directory.

## Как извлечь текст из изображения с использованием нескольких прямоугольников

### Шаг 2: Распознавание изображения с несколькими прямоугольниками

#### 2.1 Определите прямоугольники

Создайте список объектов `Rectangle`, определяющих области, которые вы хотите, чтобы движок OCR сканировал.

```csharp
List<Rectangle> rects = new List<Rectangle>()
{
    new Rectangle(138, 352, 2033, 537),
    new Rectangle(147, 890, 2033, 1157),
    new Rectangle(923, 2045, 465, 102),
    new Rectangle(104, 2147, 2076, 819)
};
```

#### 2.2 Выполнить распознавание OCR

Передайте путь к изображению и список прямоугольников в `RecognizeImage`. Метод возвращает коллекцию строк — каждая запись соответствует одному прямоугольнику.

```csharp
// first case
List<string> listResult = api.RecognizeImage(dataDir + "sample.png", rects);

// Display the recognized text
foreach (string s in listResult)
{
    Console.WriteLine(s);
}
```

### Шаг 3: Распознавание изображения с настройками распознавания (альтернативный подход)

Если вы предпочитаете использовать `RecognitionSettings`, вы можете достичь того же результата с немного другим вызовом API.

#### 3.1 Определите настройки распознавания

```csharp
RecognitionResult result = api.RecognizeImage(dataDir + "sample.png", new RecognitionSettings
{
    RecognitionAreas = rects
});
```

#### 3.2 Отобразить распознанный текст

```csharp
// Display the recognized text
foreach (string s in result.RecognitionAreasText)
{
    Console.WriteLine(s);
}
```

## Распространённые проблемы и советы

- **Неправильные координаты прямоугольника:** Убедитесь, что значения `X`, `Y`, `Width` и `Height` правильно соответствуют нужной области.  
- **Качество изображения:** Изображения с низким разрешением могут давать плохие результаты OCR; рассмотрите предварительную обработку (например, бинаризацию).  
- **Пустые результаты:** Убедитесь, что в прямоугольниках действительно есть текст; иначе движок вернёт пустые строки.
