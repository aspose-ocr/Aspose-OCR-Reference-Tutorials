# Извлечение текста из изображений – Преобразование изображений в PDF C#

## Введение

В этом руководстве вы узнаете, как **извлекать текст из изображений** при **преобразовании изображений в PDF C#** с помощью Aspose.OCR для .NET, а затем сохранить многостраничный результат OCR в виде документа. Независимо от того, нужно ли вам **пакетно преобразовывать изображения в pdf** для архивирования, **преобразовывать отсканированные изображения в pdf** для соответствия требованиям, или просто извлекать поисковый текст из картинок, мы пройдем каждый шаг с понятными объяснениями, практическими советами и рекомендациями лучших практик.

## Быстрые ответы
- **Что покрывает это руководство?** Преобразование нескольких изображений в PDF/Docx/Txt/Xlsx и извлечение текста из изображений с помощью Aspose.OCR в C#.
- **Какие форматы вывода поддерживаются?** Docx, Text, Pdf и Xlsx (вы также можете напрямую выводить PDF).
- **Нужна ли лицензия?** Бесплатная пробная версия подходит для оценки; для продакшн требуется постоянная лицензия.
- **Какие версии .NET совместимы?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7.
- **Можно ли извлекать текст во время преобразования?** Да — используйте результаты OCR, чтобы извлечь текст перед сохранением.

## Что такое «извлечение текста из изображений» и почему сочетать его с преобразованием в PDF?

Извлечение текста из изображений означает использование OCR (Optical Character Recognition) для преобразования визуальных символов в поисковые, редактируемые строки. Когда вы **преобразуете изображения в PDF C#**, вы встраиваете эти строки в PDF, делая документ поисковым и индексируемым — идеально для цифрового архивирования и добычи данных.

## Почему использовать Aspose.OCR для этой задачи?

- **Высокая точность** на десятках языков.  
- **Поддержка многостраничных документов** – обработка пакетов изображений за один вызов (идеально для сценариев **batch image to pdf**).  
- **Прямое сохранение** в популярные офисные форматы без дополнительных шагов преобразования.  
- **Полная интеграция с .NET** – без нативных зависимостей, работает на Windows, Linux и облачных средах выполнения.

## Импорт пространств имён

Начните с добавления необходимых пространств имён в ваш файл C#:

```csharp
using System;
using System.Collections.Generic;
using System.IO;
using System.Linq;
using Aspose.OCR;
```

Эти импорты дают вам доступ к коллекциям, работе с файлами, LINQ и классам Aspose OCR.

## Шаг 1: Установите каталог документов

```csharp
// The path to the documents directory.
string dataDir = "Your Document Directory";
```

Замените `"Your Document Directory"` на абсолютный или относительный путь, где находятся ваши исходные изображения и куда вы хотите сохранять выходные файлы.

## Шаг 2: Инициализировать Aspose.OCR

```csharp
// Initialize an instance of AsposeOcr
AsposeOcr api = new AsposeOcr();
```

Создание объекта `AsposeOcr` предоставляет доступ ко всем операциям OCR, включая рабочий процесс **convert images to PDF C#**.

## Шаг 3: Распознать изображения

```csharp
// Recognize image
List<RecognitionResult> result = api.RecognizeMultipleImages(
    new List<string> { dataDir + "sample.png", dataDir + "sample_bad.png" },
    new RecognitionSettings { }
).ToList();
```

Метод `RecognizeMultipleImages` обрабатывает каждый файл в списке и возвращает коллекцию `RecognitionResult`. Вы можете передать любое количество изображений, что идеально подходит для сценариев **convert scanned images pdf**.

## Шаг 4: Сохранить результаты в предпочтительных форматах

```csharp
// Save the result in your preferred format
AsposeOcr.SaveMultipageDocument(RunExamples.GetDataDir_OCR()+"sample.docx", SaveFormat.Docx, result);
AsposeOcr.SaveMultipageDocument(RunExamples.GetDataDir_OCR() + "sample.txt", SaveFormat.Text, result);
AsposeOcr.SaveMultipageDocument(RunExamples.GetDataDir_OCR() + "sample.pdf", SaveFormat.Pdf, result);
AsposeOcr.SaveMultipageDocument(RunExamples.GetDataDir_OCR() + "sample.xlsx", SaveFormat.Xlsx, result);
```

Выберите формат, который лучше всего подходит для вашего последующего рабочего процесса:

- **Docx** – редактируемый документ Word с поисковым текстом.  
- **Text** – извлечение простого текста для быстрой добычи данных (**extract text from images**).  
- **Pdf** – классический вывод PDF, идеально подходит для архивирования.  
- **Xlsx** – представление в виде таблицы для табличных данных.

## Распространённые случаи использования

- **Цифровое архивирование:** Преобразование отсканированных бумажных контрактов в поисковые PDF.  
- **Автоматизация ввода данных:** Извлечение текста из чеков или счетов и загрузка его в базу данных.  
- **Пакетная обработка:** Обработка тысяч изображений в одной задаче с минимальным кодом — идеально для потребностей **batch image to pdf**.

## Устранение неполадок и советы

- **Большие наборы изображений:** Обрабатывайте изображения небольшими партиями, чтобы избежать всплесков памяти.  
- **Качество изображения:** Убедитесь, что изображения имеют минимум 300 dpi для оптимальной точности OCR.  
- **Ошибки лицензии:** Убедитесь, что ваш файл лицензии загружен корректно перед вызовом методов OCR.  
- **Пустые результаты:** OCR‑движок возвращает пустой `RecognitionResult` для нечитаемых страниц; проверьте `result[i].Text` на null или пустые строки и обработайте соответствующим образом.

## Часто задаваемые вопросы

**В: Можно ли преобразовать изображения в PDF C# без использования OCR?**  
A: Да, вы можете использовать Aspose.PDF или другие библиотеки для чистого преобразования изображений в PDF, но OCR добавляет поисковый текст.

**В: Как извлечь текст из изображений C# после преобразования?**  
A: Список `result`, возвращаемый `RecognizeMultipleImages`, содержит свойства `Text`, которые вы можете записать в файл `.txt` или обработать напрямую.

**В: Можно ли задать пользовательские поля страницы или ориентацию?**  
A: При сохранении в PDF или Docx вы можете изменить макет документа через API Aspose.Words или Aspose.PDF перед вызовом `SaveMultipageDocument`.

**В: Что происходит, если изображение не может быть прочитано?**  
A: OCR‑движок возвращает пустой `RecognitionResult` для этой страницы; вы можете проверить `result[i].Text` на null или пустые строки и обработать соответствующим образом.

**В: Поддерживает ли API развертывание в облаке?**  
A: Да, библиотека работает на любой среде выполнения .NET, включая Azure Functions и AWS Lambda, при условии, что среда соответствует требованиям к версии.
