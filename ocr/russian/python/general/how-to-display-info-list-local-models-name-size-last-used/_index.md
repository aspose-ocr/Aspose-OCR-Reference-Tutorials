# Как вывести информацию – список локальных моделей, имя, размер, последнее использование

Когда‑то задавались вопросом **как вывести информацию** из вашей установки AsposeAI, не копаясь в логах или UI? Вы не одиноки. Во многих конвейерах data‑science первое, что нужно — быстрый взгляд на то, какие модели находятся на вашей машине, как они называются, какого они размера и когда вы их в последний раз использовали.

Именно это мы и рассмотрим: лаконичный, сквозной фрагмент Python, который **выводит список локальных моделей**, затем **показывает имя модели**, **отображает размер модели** и **показывает время последнего использования**. Без внешних библиотек, без скрытой магии — только клиент AsposeAI, который у вас уже есть.

К концу этого руководства вы сможете вставить код в любой скрипт, запустить его и мгновенно получить аккуратную таблицу ваших локально кэшированных моделей. Это идеально для проверки окружения, создания панелей мониторинга или просто удовлетворения любопытства о том, что скрывается на диске.

## Предварительные требования

- Python 3.8 или новее (в примере используются f‑строки, поэтому требуется 3.6+)
- установлен пакет `asposeai` (`pip install asposeai`)
- действующая лицензия AsposeAI или пробный ключ (клиент получит учётные данные из переменных окружения или конфигурационного файла)

Если все пункты уже выполнены, отлично — приступим.

## Шаг 1: Как вывести информацию – инициализация клиента AsposeAI

Прежде чем **вывести список локальных моделей**, нам нужен объект‑клиент, который общается с runtime AsposeAI. Этот шаг — фундамент; без него остальной код вызовет `NameError`.

```python
# Step 1: Create an instance of the AsposeAI client
# The client automatically reads credentials from the environment
aspose_ai = AsposeAI()
```

*Почему это важно*: Инициализация клиента устанавливает сессию с локальным движком вывода, загружая необходимые нативные библиотеки. Она также проверяет, что ваша лицензия активна, предотвращая непонятные ошибки позже, когда вы попытаетесь запросить модели.

> **Pro tip**: Если вы запускаете это на CI‑сервере, задайте переменную `ASPOSEAI_LICENSE` в окружении, чтобы клиент мог стартовать без интерактивных запросов.

## Шаг 2: Список локальных моделей – получение доступных моделей

Теперь, когда клиент готов, мы можем **вывести список локальных моделей**. Метод `list_local()` возвращает коллекцию объектов, каждый из которых содержит свойства `name`, `size_mb` и `last_used`.

```python
# Step 2: Retrieve the list of locally available models
local_models = aspose_ai.list_local()
```

*Что происходит под капотом*: `list_local()` сканирует каталог, где AsposeAI кэширует файлы моделей (`~/.asposeai/models` по умолчанию), и формирует лёгкие метаданные. Это быстро, потому что веса модели не загружаются — читается лишь небольшой JSON‑манифест.

Если когда‑нибудь захотите проверить, кэширована ли конкретная модель, этот вызов — самый быстрый способ получить ответ.

## Шаг 3: Вывод имени модели, размера модели и времени последнего использования

Получив модели, мы наконец **выводим информацию**, проходя по коллекции и печатая каждый атрибут. Здесь мы **выводим имя модели**, **показываем размер модели** и **показываем время последнего использования** в одной аккуратной строке.

```python
# Step 3: Print each model's name, size (in MB), and last‑used timestamp
print("Available local models:")
print("-" * 50)
for model_info in local_models:
    # Using an f‑string to format the output cleanly
    print(f"{model_info.name} – {model_info.size_mb} MB – last used {model_info.last_used}")
```

**Ожидаемый вывод** (ваши timestamps будут другими):

```
Available local models:
--------------------------------------------------
gpt‑tiny‑en – 120 MB – last used 2025-11-02 14:23:11
bert‑base‑fr – 420 MB – last used 2025-10-18 09:07:45
whisper‑large‑audio – 1580 MB – last used 2025-09-30 22:41:02
```

*Почему мы форматируем именно так*: Тире (`–`) разделяет поля для лучшей читаемости, а строка‑заголовок делает вывод в консоль удобным для сканирования. Если позже понадобится машинно‑читаемый формат (CSV, JSON), легко заменить вызов `print` на `writer.writerow` или `json.dump`.

### Обработка граничных случаев

- **Нет кэшированных моделей** – `list_local()` возвращает пустой список. Цикл просто пропустит итерацию, оставив только заголовок. Можно добавить защиту:

  ```python
  if not local_models:
      print("No local models found. Use aspose_ai.download(...) to fetch one.")
  ```

- **Отсутствующие атрибуты** – В редких случаях манифест может быть повреждён. Обращение к `model_info.last_used` может вызвать `AttributeError`. Оберните печать в `try/except`, если такие проблемы ожидаются.

- **Большие каталоги моделей** – Если у вас сотни моделей, рассмотрите постраничный вывод или запись в файл вместо печати в консоль.

## Полный рабочий скрипт

Объединив всё вместе, получаем полностью готовый к запуску скрипт:

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-

"""
How to display info from AsposeAI:
List local models and show each model's name, size, and last‑used timestamp.
"""

from asposeai import AsposeAI

def main():
    # Initialize client
    aspose_ai = AsposeAI()

    # Retrieve local models
    local_models = aspose_ai.list_local()

    # Header
    print("Available local models:")
    print("-" * 50)

    if not local_models:
        print("No local models found. Use aspose_ai.download(...) to fetch one.")
        return

    # Display each model's details
    for model_info in local_models:
        try:
            print(f"{model_info.name} – {model_info.size_mb} MB – last used {model_info.last_used}")
        except AttributeError:
            # Fallback if any attribute is missing
            print(f"{model_info.name} – info incomplete")

if __name__ == "__main__":
    main()
```

Сохраните его как `list_models.py`, сделайте исполняемым (`chmod +x list_models.py`) и запустите:

```bash
./list_models.py
```

Вы увидите аккуратный список, продемонстрированный ранее.

## Заключение

Мы прошли процесс **вывода информации** из AsposeAI, **перечислив локальные модели**, затем **выведя имя модели**, **показав размер модели** и **показав время последнего использования**. Подход лёгкий, требует только стандартного пакета `asposeai` и может быть внедрён в любой конвейер автоматизации или отладочную сессию.

Дальше вы можете:

- Экспортировать вывод в CSV для анализа в таблицах (`csv` module).
- Передавать timestamps в панель мониторинга, чтобы получать оповещения о «застоявшихся» моделях.
- Скомбинировать этот скрипт с `aspose_ai.download()`, чтобы автоматически обновлять модели, которые долго не использовались.

Помните, чёткая видимость вашего инвентаря моделей экономит время, уменьшает накопление ненужных данных и поддерживает ваши AI‑сервисы в стабильной работе. Попробуйте скрипт, подправьте форматирование под свои нужды, и он станет надёжным инструментом в вашем арсенале.
