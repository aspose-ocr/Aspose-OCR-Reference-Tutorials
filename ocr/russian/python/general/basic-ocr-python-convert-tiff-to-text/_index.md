# basic ocr python – Преобразование TIFF в текст с AI‑постобработкой

Ищете способ выполнить **basic ocr python** для отсканированных документов? В этом руководстве мы пройдём процесс преобразования файла TIFF в чистый, индексируемый текст с помощью Aspose OCR и AI‑постобработчика.  

Если вам когда‑нибудь приходилось **преобразовывать TIFF в текст**, а полученный результат был полон опечаток и странных символов, вы не одиноки. Мы также покажем, как **загрузить изображение OCR**, настроить движок для **установки GPU‑слоёв**, и даже **исправить ведущие нули**, часто встречающиеся в номерах счетов.

## Что вы узнаете

- Как инициализировать движок Aspose OCR для печатных документов.  
- Точные шаги для **загрузки изображения OCR** из файла TIFF и получения необработанного текста.  
- Настройка AI‑модели, её автоматическая загрузка и **установка GPU‑слоёв** для ускорения вывода.  
- Добавление встроенной проверки орфографии плюс пользовательской функции, которая **исправляет ведущие нули**.  
- Очистка результата OCR и корректное освобождение ресурсов.  

К концу этого руководства у вас будет один переиспользуемый скрипт Python, который преобразует любой TIFF в отшлифованный, индексируемый текст — без ручного копирования‑вставки.  

### Предварительные требования

- Python 3.8+ установленный на вашем компьютере.  
- Пакет `aspose-ocr` (`pip install aspose-ocr`).  
- По желанию: GPU с минимум 4 ГБ видеопамяти, если хотите **установить GPU‑слои**; иначе код автоматически переключится на CPU.

## basic ocr python – загрузка изображения и распознавание текста

Первое, что нам нужно сделать, — это **загрузить изображение OCR**, чтобы движок мог прочитать пиксели. `OcrEngine` от Aspose поддерживает множество форматов, но здесь мы сосредоточимся на TIFF, поскольку это самый распространённый формат для сканированных счетов.

```python
import aspose.ocr as ocr

# Initialise the OCR engine for printed documents
ocr_engine = ocr.OcrEngine()
ocr_engine.set_recognition_mode(ocr.RecognitionMode.PRINTED)   # use HANDWRITTEN for handwritten docs

# Load the TIFF file – replace with your own path
ocr_engine.load_image("YOUR_DIRECTORY/input.tif")
```

> **Pro tip:** Если вы работаете с многостраничными TIFF, Aspose автоматически обрабатывает каждую страницу последовательно, так что дополнительные циклы не требуются.

После загрузки изображения запустим базовый проход распознавания.

```python
# Perform basic OCR and capture the raw string
raw_text = ocr_engine.recognize()
print("Raw OCR:", raw_text)
```

Вы увидите что‑то вроде:

```
Raw OCR: Inv0ce N0: 0123AB4
Total Amt: $1,200.00
Date: 2024-07-15
```

Обратите внимание на *нули* перед буквами? Это классический артефакт OCR, который мы исправим позже.

## Шаг 2: Преобразование TIFF в текст — подготовка AI‑постобработчика

Необработанный OCR полезен, но большинство производственных конвейеров требуют отшлифованную версию. Aspose предоставляет обёртку `AsposeAI`, которая может загрузить модель с Hugging Face, запустить её на GPU и автоматически применить проверку орфографии.

```python
from aspose.ocr.ai import AsposeAI, AsposeAIModelConfig

# Configure the AI model – we’ll download Qwen2.5‑3B‑Instruct automatically
ai_config = AsposeAIModelConfig(
    allow_auto_download="true",
    directory_model_path="YOUR_DIRECTORY/ocr_ai_models",
    hugging_face_repo_id="Qwen/Qwen2.5-3B-Instruct-GGUF",
    hugging_face_quantization="int8",
    gpu_layers=25          # <-- this is where we **set GPU layers**
)

ocr_ai = AsposeAI(ai_config)
```

### Почему важна `set GPU layers`

Параметр `gpu_layers` сообщает базовой модели GGUF, сколько трансформерных слоёв держать на GPU. Больше слоёв = быстрее вывод, но выше потребление видеопамяти. Если у вас скромный ноутбук, уменьшите значение до `10` или полностью опустите параметр, чтобы работать на CPU.

---

## Шаг 3: Применить проверку орфографии и **исправить ведущие нули**

AI от Aspose поставляется со встроенной проверкой орфографии, которая ловит большинство английских опечаток. Однако специфические исправления — например, замена ведущего `0` на `O` в кодах счетов — требуют пользовательской постобработки.

```python
# Enable the built‑in spell‑check
ocr_ai.set_post_processor("spellcheck")

# Custom function to replace a leading zero before three capital letters
def invoice_fix(txt: str) -> str:
    import re
    # Example: "0ABC" -> "OABC"
    return re.sub(r"\b0([A-Z]{3})\b", r"O\1", txt)

# Register the custom fix – it runs after spellcheck
ocr_ai.set_post_processor(invoice_fix)
```

> **Почему это работает:** Регулярное выражение ищет границу слова (`\b`), ноль, затем ровно три заглавные буквы. Затем ноль заменяется буквой «O». Вы можете расширить шаблон для других особенностей (например, `0[0-9]{2}` для неверно прочитанных чисел).

## Шаг 4: Очистка результата OCR с помощью AI‑постобработчика

Теперь объединяем всё: необработанную строку из **basic ocr python**, проверку орфографии и наше исправление нулей. Метод `run_postprocessor` возвращает очищенную версию, готовую для дальнейших систем.

```python
cleaned_text = ocr_ai.run_postprocessor(raw_text)
print("\nCleaned OCR:", cleaned_text)
```

Типичный вывод после постобработки:

```
Cleaned OCR: Invoice No: O123AB4
Total Amt: $1,200.00
Date: 2024-07-15
```

Вы видите, что ведущий ноль превратился в `O`, а распространённые опечатки исправлены. Текст теперь подходит для индексации в поисковой системе или передачи в конвейер извлечения данных.

## Шаг 5: Освобождение AI‑ресурсов — держите ваш GPU в порядке

Если вы запускаете несколько OCR‑задач в длительно работающем сервисе, рекомендуется освобождать видеопамять модели после завершения.

```python
ocr_ai.free_resources()
```

Пропуск этого шага может привести к ошибкам «out‑of‑memory» при последующих вызовах, особенно если **set GPU layers** установлен на высокое значение.

## Опциональные варианты и граничные случаи

| Ситуация | Что изменить |
|-----------|----------------|
| **Рукописные документы** | Использовать `ocr_engine.set_recognition_mode(ocr.RecognitionMode.HANDWRITTEN)` вместо `PRINTED`. |
| **Отсутствует GPU** | Установить `gpu_layers=0` или опустить аргумент; модель будет работать на CPU (медленнее, но надёжно). |
| **Другой язык** | Заменить ID репозитория Hugging Face на модель для нужного языка, например `microsoft/Florence-2-base`. |
| **Пакетная обработка** | Обернуть шаги в цикл `for file in glob("*.tif"):` и собрать результаты в список или CSV. |
| **Более сложные шаблоны нулей** | Расширить `invoice_fix` дополнительными regex‑ами, например `r"\b0+([A-Z]{2,})\b"` для нескольких ведущих нулей. |
