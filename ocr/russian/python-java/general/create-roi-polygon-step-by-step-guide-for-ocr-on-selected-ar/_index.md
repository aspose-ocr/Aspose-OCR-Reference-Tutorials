# Создание ROI‑многоугольника – Полный учебник по OCR в выбранной области

Когда‑нибудь вам нужно было **создать ROI‑многоугольник**, чтобы запускать OCR только на той части изображения, которая важна? Возможно, вы сканируете чеки и важны только итоги, или у вас есть форма, где нужно распознать только поле подписи. В таких случаях **ocr on selected area** экономит время и повышает точность.  

В этом руководстве мы пройдем всё, что вам понадобится: от определения двух многоугольников, регистрации их в OCR‑движке, до получения распознанного текста одним чистым вызовом. К концу вы получите готовый к запуску скрипт и чёткое понимание, почему фокусировка на областях интереса имеет значение.

## Что вы узнаете

- Как **создать ROI‑многоугольник** с помощью типичной OCR‑библиотеки.  
- Чем отличается определение одной области от нескольких.  
- Как зарегистрировать эти области в движке, чтобы выполнялся `ocr on selected area`.  
- Какой результат ожидать и как устранять распространённые проблемы (например, перекрывающиеся ROI, пустые результаты).

### Предпосылки

- Установлен Python 3.8+.  
- OCR‑библиотека, предоставляющая классы `Polygon`, `Point` и метод `add_region_of_interest` (в примере используется вымышленный модуль `ocr`; замените его на ваш реальный SDK, например обёртки Tesseract‑Python или EasyOCR).  
- Пример изображения (`sample.png`), содержащего текст в координатах, указанных ниже.

> **Полезный совет:** Если вы используете реальную библиотеку, убедитесь, что изображение загружено в той же системе координат (начало в левом верхнем углу, X растёт вправо, Y растёт вниз).  

---  

## Шаг 1: Импортировать OCR‑модуль и загрузить изображение  

Сначала подключим OCR‑пакет и прочитаем изображение, которое нужно обработать.  

```python
import ocr  # Replace with your actual OCR library import
from pathlib import Path

# Load the image – adjust the path as needed
image_path = Path("sample.png")
ocr_engine = ocr.Engine(image_path)
```

*Почему это важно:* Движку нужны данные изображения, прежде чем вы сможете привязать любой **ROI‑многоугольник**. Некоторые библиотеки позволяют передать изображение позже, но ранняя инициализация упрощает рабочий процесс.

## Шаг 2: Определить первый ROI‑многоугольник  

Теперь создаём первую область интереса. Представьте, что это прямоугольник вокруг заголовка таблицы.  

```python
# Step 2: Define the first ROI polygon
first_roi = ocr.Polygon([
    ocr.Point(50, 20),   # top‑left
    ocr.Point(200, 20),  # top‑right
    ocr.Point(200, 80),  # bottom‑right
    ocr.Point(50, 80)    # bottom‑left
])
```

*Объяснение:*  
- `ocr.Point(x, y)` использует пиксельные координаты.  
- Список точек упорядочен по часовой стрелке, что ожидают большинство движков.  
- Можно добавить сколько угодно точек, чтобы получить неправильные формы; прямоугольник — лишь частный случай.

## Шаг 3: Определить дополнительные ROI‑многоугольники (по желанию)  

Не обязательно останавливаться на одной. Ниже второй многоугольник, захватывающий поле подписи в нижней части страницы.  

```python
# Step 3: Define the second ROI polygon
second_roi = ocr.Polygon([
    ocr.Point(300, 150),
    ocr.Point(480, 150),
    ocr.Point(480, 210),
    ocr.Point(300, 210)
])
```

*Зачем добавлять больше?* Выполнение **ocr on selected area** для нескольких зон позволяет извлекать разрозненные данные за один проход, что гораздо быстрее, чем обрезать и обрабатывать каждый фрагмент отдельно.

## Шаг 4: Зарегистрировать ROI в OCR‑движке  

Когда многоугольники готовы, сообщаем движку, какие области проверять.  

```python
# Step 4: Register the ROIs
ocr_engine.add_region_of_interest(first_roi)
ocr_engine.add_region_of_interest(second_roi)
```

*Важно:* Некоторые SDK требуют вызвать метод `clear_regions()` перед добавлением новых областей, если вы переиспользуете движок для другого изображения.

## Шаг 5: Запустить OCR и получить текст  

Наконец, запускаем распознавание. Движок будет искать только внутри только что добавленных многоугольников.  

```python
# Step 5: Perform OCR on the defined regions
recognized_text = ocr_engine.recognize().get_text()
print("=== Recognized Text ===")
print(recognized_text)
```

**Ожидаемый вывод** (при условии, что `sample.png` содержит слова «Invoice Total: $123.45» в первом ROI и «Signature: John Doe» во втором):

```
=== Recognized Text ===
Invoice Total: $123.45
Signature: John Doe
```

Если вывод пустой, дважды проверьте, что координаты действительно пересекаются с текстом и что разрешение изображения соответствует системе координат.

## Пограничные случаи и распространённые подводные камни  

| Ситуация                               | На что обратить внимание                         | Исправление / обход                              |
|----------------------------------------|--------------------------------------------------|-------------------------------------------------|
| **Перекрывающиеся ROI**                | Текст может быть возвращён дважды.              | Делайте полигоны непересекающимися или удаляйте дубли. |
| **ROI за пределами изображения**       | Движок может выдать ошибку или ничего не вернуть.| Ограничьте координаты диапазоном `0 ≤ x < width`, `0 ≤ y < height`. |
| **Очень маленький ROI**                | Точность OCR падает из‑за недостаточного количества пикселей. | Увеличьте полигон на несколько пикселей или предварительно масштабируйте изображение. |
| **Неправильная (непрямоугольная) форма** | Некоторые движки поддерживают только выпуклые полигоны. | Разбейте сложные формы на несколько выпуклых ROI. |
| **Изменение размера изображения**      | Жёстко закодированные координаты становятся недействительными. | Вычисляйте координаты ROI относительно размеров изображения (например, в процентах). |

## Полный рабочий пример  

Ниже полностью готовый скрипт, который можно скопировать‑вставить и запустить (замените `ocr` на вашу реальную библиотеку).  

```python
import ocr                      # <- your OCR library
from pathlib import Path

# -------------------------------------------------
# Configuration
# -------------------------------------------------
IMAGE_PATH = Path("sample.png")

# -------------------------------------------------
# Initialize OCR engine with the image
# -------------------------------------------------
engine = ocr.Engine(IMAGE_PATH)

# -------------------------------------------------
# Define ROI polygons
# -------------------------------------------------
first_roi = ocr.Polygon([
    ocr.Point(50, 20), ocr.Point(200, 20),
    ocr.Point(200, 80), ocr.Point(50, 80)
])

second_roi = ocr.Polygon([
    ocr.Point(300, 150), ocr.Point(480, 150),
    ocr.Point(480, 210), ocr.Point(300, 210)
])

# -------------------------------------------------
# Register ROIs
# -------------------------------------------------
engine.add_region_of_interest(first_roi)
engine.add_region_of_interest(second_roi)

# -------------------------------------------------
# Run OCR on the selected areas
# -------------------------------------------------
result = engine.recognize().get_text()

print("=== Recognized Text ===")
print(result)
```

Запустите его командой `python ocr_roi_example.py`, и вы увидите извлечённые строки из двух определённых зон.

## Следующие шаги  

- **Динамическое создание ROI:** Вместо жёстко заданных координат используйте обработку изображений (например, обнаружение контуров в OpenCV) для автоматического нахождения таблиц или полей.  
- **Постобработка:** Удаляйте пробелы, применяйте регулярные выражения или преобразуйте числа в нужные типы данных.  
- **Пакетная обработка:** Проходите по папке изображений, переиспользуя те же определения ROI, если макет одинаковый.  

Если вам интересно **ocr on selected area** для более сложных документов — подумайте о многостраничных PDF или сканированных формах, где поддерживается регистрация ROI по страницам.
