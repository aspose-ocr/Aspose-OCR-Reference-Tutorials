# Как выполнить OCR ROI в Python – Как установить предобработку

Когда‑нибудь задумывались **как выполнить OCR ROI** на шумном изображении счета без загрузки всей страницы в память? Вы не одиноки. Во многих реальных проектах нам требуется лишь несколько полей — имя клиента, адрес, итоги — поэтому сканировать весь документ избыточно.  

Хорошие новости? С Aspose OCR вы можете указать движку точно **где искать** и даже заставить его предварительно очистить изображение. В этом руководстве мы пройдемся по параметрам **как установить предобработку**, определим области интереса (ROIs) и извлечем чистый текст всего в несколько строк кода на Python.

К концу этого руководства у вас будет готовый к запуску скрипт, который извлекает конкретные блоки из любого счета, чека или формы. Дополнительные инструменты не нужны, только Aspose OCR и немного логики на Python.

---

## Что понадобится

- **Python 3.8+** (код работает на любой современной версии)  
- **Aspose OCR for Python via .NET** – установить с помощью `pip install aspose-ocr`  
- Пример изображения (например, `invoice.png`) в папке, к которой вы можете обратиться  
- Базовое знакомство с функциями Python и объектно‑ориентированным синтаксисом  

Если у вас уже есть всё это, отлично — сразу переходим к коду.

## Шаг 1 – Инициализация OCR‑движка (Как выполнить OCR ROI)

Прежде чем мы сможем сказать движку *где* искать, нам нужен экземпляр процессора OCR. Этот объект хранит всю конфигурацию и выполняет проход распознавания.

```python
import aspose.ocr as aocr
import aspose.storage as storage

# Create the engine – this is the entry point for all OCR work
ocr_engine = aocr.OcrEngine()
```

*Почему это важно*: Класс `OcrEngine` абстрагирует тяжелую работу (обнаружение шрифтов, анализ разметки и т.д.). Создавая один движок, вы избегаете повторной инициализации тяжелых ресурсов для каждого изображения, что сохраняет высокую производительность.

---

## Шаг 2 – Загрузка исходного изображения (Как выполнить OCR ROI)

Aspose Storage упрощает загрузку изображений. Укажите путь к файлу, и вы получите объект `Image`, готовый к обработке.

```python
# Replace YOUR_DIRECTORY with the actual path where invoice.png lives
source_image = storage.Image.load("YOUR_DIRECTORY/invoice.png")
```

*Tip*: Если вы работаете с потоками (например, изображения, загруженные через веб‑API), вы можете передать объект `BytesIO` в `Image.load` вместо пути к файлу.

---

## Шаг 3 – Определение областей интереса (ROIs)

Теперь мы отвечаем на главный вопрос **как выполнить OCR ROI**: сообщаем движку точные прямоугольники, содержащие нужные нам данные. Каждый ROI задаётся координатами левого верхнего угла (`x`, `y`) и размерами (`width`, `height`).

```python
from aspose.ocr import Roi

# Each tuple is (x, y, width, height) in pixels
regions_of_interest = [
    Roi(50, 120, 400, 60),   # Customer name block
    Roi(50, 200, 400, 80)    # Address block
]
```

*Почему использовать ROIs?*  
- **Скорость** – Движок пропускает нерелевантные части изображения.  
- **Точность** – Фокусируясь на небольшом участке, шум в остальных частях не может запутать распознаватель.  
- **Гибкость** – Можно добавить столько ROI, сколько нужно; движок возвращает список результатов в том же порядке.

---

## Шаг 4 – Как установить предобработку для повышения точности OCR

Изображения, полученные со сканеров или телефонов, часто страдают от наклона, низкого контраста или неравномерного освещения. Aspose OCR предоставляет объект `PreprocessingOptions`, позволяющий включать типичные исправления перед запуском распознавания.

```python
from aspose.ocr import PreprocessingOptions

preprocessing_options = PreprocessingOptions()
preprocessing_options.deskew = True          # Auto‑rotate tilted text
preprocessing_options.binarize = True        # Convert to black‑white for clarity
preprocessing_options.contrast = 1.5         # Boost contrast (1.0 = no change)
```

*Как это помогает*:  
- **Устранение наклона** удаляет наклон, который делает формы символов неоднозначными.  
- **Бинаризация** уменьшает цветовой шум, предоставляя OCR‑движку чистое бинарное изображение.  
- **Контраст** усиливает слабые штрихи, что особенно полезно для выцветших чеков.

Вы можете экспериментировать с этими флагами — отключить один и посмотреть, изменится ли вывод. Это и есть суть **как установить предобработку** эффективно.

---

## Шаг 5 – Выполнение OCR только в определённых ROI

С готовыми движком, изображением, ROI и предобработкой пришло время вызвать `recognize`. Метод возвращает объект `OcrResult`, содержащий коллекцию `regions` — каждый элемент соответствует порядку ROI, который мы передали.

```python
ocr_result = ocr_engine.recognize(
    source_image,
    rois=regions_of_interest,
    preprocessing=preprocessing_options
)
```

*Behind the scenes*: Движок обрезает каждый ROI, применяет конвейер предобработки и запускает модель распознавания на очищенном фрагменте. Поскольку мы передали список, результат сохраняет тот же порядок, что упрощает пост‑обработку.

---

## Шаг 6 – Чтение и использование извлечённого текста (Как выполнить OCR ROI)

Наконец, пройдитесь по списку `regions` и выведите — или сохраните — распознанные строки. Объект `Region` раскрывает свойство `text`, содержащее результат в Unicode.

```python
for idx, region in enumerate(ocr_result.regions, start=1):
    print(f"Region {idx} text:")
    print(region.text)
    print("-" * 40)
```

**Ожидаемый вывод (пример)**:

```
Region 1 text:
John Doe
----------------------------------------
Region 2 text:
123 Maple Street
Springfield, IL 62704
----------------------------------------
```

Если текст выглядит искажённым, вернитесь к **как установить предобработку**: возможно, стоит увеличить `contrast` или отключить `binarize` для цветных логотипов.

## Распространённые ошибки и профессиональные советы

| Проблема | Почему происходит | Исправление (Как установить предобработку) |
|----------|-------------------|--------------------------------------------|
| Наклонный текст выглядит как мусор | `deskew` отключён или изображение слишком повернуто | Включите `preprocessing_options.deskew = True` |
| Числа превращаются в точки или случайные знаки | Низкий контраст или агрессивная бинаризация | Уменьшите `contrast` (например, `1.2`) или установите `binarize = False` |
| Координаты ROI смещены на несколько пикселей | Разный DPI или масштаб сканера | Используйте инструмент (например, Paint.NET) для измерения точных позиций пикселей, либо добавьте небольшой отступ (`+5` пикселей) к каждому ROI |
| Пустой результат для области | ROI находится за пределами изображения | Проверьте размеры изображения: `source_image.width`, `source_image.height` |

## Расширение решения (Как выполнить OCR ROI в разных сценариях)

- **Dynamic ROIs**: Если ваши счета имеют переменные макеты, сначала выполните быстрый OCR всей страницы, чтобы найти ключевые слова («Customer:», «Address:»), а затем вычислите ROI «на лету».  
- **Batch Processing**: Оберните вышеописанные шаги в функцию и пройдитесь по папке с изображениями. Не забудьте переиспользовать один и тот же экземпляр `ocr_engine`, чтобы снизить потребление памяти.  
- **Exporting to JSON**: Вместо вывода на экран сформируйте словарь и сохраните его через `json.dumps`; это упростит последующую интеграцию (например, передачу в ERP‑систему).

```python
import json

def extract_invoice_fields(image_path):
    img = storage.Image.load(image_path)
    result = ocr_engine.recognize(
        img,
        rois=regions_of_interest,
        preprocessing=preprocessing_options
    )
    data = {f"field_{i+1}": r.text.strip() for i, r in enumerate(result.regions)}
    return data

print(json.dumps(extract_invoice_fields("invoice.png"), indent=2))
```

Ограничивая движок только нужными прямоугольниками и предварительно очищая изображение, вы получаете более быстрые и чистые результаты — идеально для автоматизации счетов, оцифровки форм или любой ситуации, где важен лишь небольшой фрагмент страницы.
