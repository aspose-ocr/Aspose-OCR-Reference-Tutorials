# Как выполнить OCR – извлечь текст из изображения в Python

Когда‑нибудь задавались вопросом, **как выполнить OCR** на отсканированном счете или скриншоте и получить чистый, индексируемый текст? Вы не одиноки. Во многих проектах узким местом является простая загрузка изображения для OCR, а затем «убеждение» движка понять специфические для домена слова.  

В этом руководстве мы пройдем полный, готовый к запуску пример, который **извлекает текст из файлов‑изображений**, показывает, как **распознавать текст из PNG**‑ресурсов, и даже демонстрирует приёмы **повышения точности OCR** с помощью пользовательских словарей и специальных символов. К концу вы получите автономный скрипт, который можно вставить в любой Python‑проект.

## Что понадобится

- Python 3.8+ (код использует подсказки типов, но работает и в более ранних версиях 3.x)
- Библиотека `ocr`, поставляемая с выбранным OCR‑движком (устанавливается через `pip install ocr‑engine` – замените на реальное название пакета)
- Файл изображения (`input.png`), который нужно обработать
- Необязательно: текстовый файл (`invoice_terms.txt`) с доменно‑специфическими словами, по одному на строку

Никаких тяжёлых внешних зависимостей, никаких облачных API‑ключей — только локальный OCR‑движок.

---

## Шаг 1: Установить и импортировать OCR‑библиотеку

Сначала убедитесь, что пакет OCR установлен. Если вы используете гипотетический пакет `ocr-engine`, выполните:

```bash
pip install ocr-engine
```

Теперь импортируйте необходимые классы:

```python
# Step 1: Import the OCR SDK
import ocr
from ocr import Imaging
```

> **Совет:** Если вы работаете в виртуальном окружении, активируйте его перед установкой, чтобы не «засорять» глобальный Python.

## Шаг 2: Создать экземпляр OCR‑движка

Создание объекта движка — точка входа для любой задачи OCR. Представьте, что это включение сканера в программном обеспечении.

```python
# Step 2: Initialise the OCR engine
ocr_engine = ocr.OcrEngine()
```

Почему это важно: движок хранит конфигурацию, такую как языковые пакеты, режимы распознавания и любые пользовательские ресурсы, которые вы подключите позже. Без него вы не сможете **загрузить изображение для OCR** или запустить конвейер распознавания.

## Шаг 3: Загрузить изображение для распознавания

Здесь мы действительно **загружаем изображение для OCR**. Метод `Imaging.Image.load()` читает файл и преобразует его во внутренний битмап‑формат, ожидаемый движком.

```python
# Step 3: Load the PNG image you want to process
image_path = "YOUR_DIRECTORY/input.png"
ocr_engine.set_image(Imaging.Image.load(image_path))
```

Если у вас JPEG или TIFF, тот же метод работает — просто измените расширение. Движок автоматически определит формат.

> **Особый случай:** Очень большие изображения (более 5 МП) могут вызвать всплеск памяти. При необходимости уменьшайте их с помощью Pillow перед загрузкой.

## Шаг 4: Предоставить пользовательский словарь для повышения точности

Большинство OCR‑движков поставляются с общими языковыми моделями. Для счетов, чеков или юридических документов часто встречаются пропущенные слова. Пользовательский список слов сообщает движку: «эти термины легитимны, считаем их правильными».

```python
# Step 4: Attach a custom dictionary (one word per line)
custom_dict_path = "YOUR_DIRECTORY/invoice_terms.txt"
ocr_engine.set_custom_dictionary(custom_dict_path)
```

Типичные записи могут выглядеть так:

```
VAT
InvoiceNumber
Øre
ß
Ħ
```

Добавление этих слов значительно **повышает точность OCR** — особенно для не‑ASCII символов.

## Шаг 5: Добавить специальные символы, отсутствующие в базовом наборе

Если ваш домен использует символы вроде знака евро (€) или скандинавской Ø, их можно явно добавить:

```python
# Step 5: Register additional characters
ocr_engine.add_custom_characters("ØßĦ")
```

Теперь движок будет рассматривать эти глифы как допустимые во время распознавания, уменьшая вероятность «мусорного» вывода.

## Шаг 6: Запустить процесс OCR и получить текст

Наконец, вызовите распознаватель и извлеките результат в виде простого текста. Метод `recognize()` возвращает сложный объект; для большинства сценариев нам нужен лишь сырой строковый результат.

```python
# Step 6: Execute OCR and print the result
result = ocr_engine.recognize()
print("=== Recognized Text ===")
print(result.get_text())
```

**Ожидаемый вывод** (усечённый для краткости):

```
=== Recognized Text ===
Invoice #12345
Date: 2026-03-01
Total: €1,250.00
VAT: 25%
...
```

Если вывод выглядит искажённым, проверьте, правильно ли загружены ваш пользовательский словарь и специальные символы.

## Часто задаваемые вопросы и подводные камни

### Работает ли это с многостраничными PDF?

Да — просто конвертируйте каждую страницу в PNG (с помощью `pdf2image` или аналогичного инструмента) и передавайте их последовательно одному экземпляру `ocr_engine`. Движок обрабатывает каждое изображение независимо, поэтому вы получите список строк, которые можно объединить.

### Что делать, если изображение повернуто?

Большинство современных OCR‑движков автоматически определяют ориентацию, но при необходимости можно принудительно повернуть изображение:

```python
ocr_engine.set_image(Imaging.Image.load(image_path).rotate(90))
```

### Как работать с языками, отличными от английского?

Смените языковую модель перед загрузкой изображения:

```python
ocr_engine.set_language("de")  # German
```

Убедитесь, что соответствующий языковой пакет установлен.

## Полный скрипт — готов к копированию и вставке

Ниже представлен весь код программы, готовый к запуску после замены путей‑заполнителей.

```python
#!/usr/bin/env python3
"""
How to Run OCR – Complete example that extracts text from a PNG,
uses a custom dictionary, and adds special characters for better accuracy.
"""

# Install the OCR package first:
# pip install ocr-engine

import ocr
from ocr import Imaging

def main():
    # 1️⃣ Initialise the OCR engine
    ocr_engine = ocr.OcrEngine()

    # 2️⃣ Load the image you want to recognize
    image_path = "YOUR_DIRECTORY/input.png"      # <-- change this
    ocr_engine.set_image(Imaging.Image.load(image_path))

    # 3️⃣ Provide a custom dictionary (optional but recommended)
    dict_path = "YOUR_DIRECTORY/invoice_terms.txt"   # one word per line
    ocr_engine.set_custom_dictionary(dict_path)

    # 4️⃣ Add any special characters not covered by the default set
    ocr_engine.add_custom_characters("ØßĦ")   # e.g., currency symbols

    # 5️⃣ Run the OCR process
    result = ocr_engine.recognize()

    # 6️⃣ Output the recognized text
    print("=== Recognized Text ===")
    print(result.get_text())

if __name__ == "__main__":
    main()
```

Запустите его командой:

```bash
python run_ocr.py
```

Вы должны увидеть извлечённый текст, выведенный в консоль. Далее его можно записать в файл, сохранить в базе данных или передать в последующие NLP‑конвейеры.
