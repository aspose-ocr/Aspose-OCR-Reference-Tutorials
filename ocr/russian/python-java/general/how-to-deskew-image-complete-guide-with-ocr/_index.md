# Как исправить наклон изображения – Полное руководство с OCR

Когда‑то задумывались **как исправить наклон изображения**, полученного с дешевого сканера? Вы не одиноки. Криво отсканированная страница выглядит непрофессионально, а главное, наклон может сбить с толку любой OCR‑движок, пытающийся **распознать текст с изображения**.  

В этом руководстве мы пройдем полный **конвейер предобработки изображения**, который исправляет наклон, бинаризует и удаляет шум со скана, а затем передаёт его OCR‑движку, чтобы вы могли **преобразовать отсканированное изображение в текст** без лишних хлопот. Никакой магии, только чистый Python и небольшая библиотека, которая делает всю тяжёлую работу.

## Что понадобится

- Python 3.9+ (код работает на 3.10 и новее)
- Пакет `ocr` (установите через `pip install ocr‑toolkit` – замените на название вашей библиотеки)
- Отсканированный JPEG или PNG с заметным наклоном (например, `skewed_scan.jpg`)
- Небольшая доля любопытства, почему каждый шаг предобработки важен

И всё. Нет тяжёлых зависимостей вроде OpenCV, если только вы не планируете углубляться позже.

## Шаг 1: Загрузка отсканированного изображения

Первое, что нужно сделать — загрузить файл в память. Этот шаг прост, но критичен: если путь указан неверно, всё остальное падает.

```python
# Step 1: Load the scanned image
image_path = "YOUR_DIRECTORY/skewed_scan.jpg"
original_image = ocr.Imaging.Image.load(image_path)
```

> **Зачем?**  
> Загрузка изображения даёт нам объект, с которым может работать `ocr.ImagePreprocessor`. Пропуск этого шага оставит конвейер слепым к реальным пиксельным данным.

## Как исправить наклон изображения и подготовить его к OCR

Теперь, когда изображение загружено, выпрямим его. Метод `deskew()` оценивает угол наклона и вращает картинку обратно в горизонтальное положение.

```python
# Step 2: Create an image preprocessor and correct the orientation
preprocessor = ocr.ImagePreprocessor()
preprocessor.deskew()
```

> **Совет:** Если ваши сканы отклонены лишь слегка (≤ 3°), алгоритм по умолчанию обычно справляется идеально. При экстремальных углах может потребоваться подправить внутренние параметры — см. документацию библиотеки для `max_angle`.

## Бинаризация изображения – Превращение в чёрно‑белое

OCR‑движки любят контрастный ввод. Преобразование картинки в бинарное (чёрно‑белое) представление убирает тонкие оттенки, которые могут запутать распознавание символов.

```python
# Step 3: Convert the image to a binary (black‑and‑white) representation
preprocessor.binarize(threshold=128)
```

> **Что происходит?**  
> Пиксели ярче 128 становятся белыми; всё остальное — чёрным. При необходимости отрегулируйте порог, если ваш скан необычно тёмный или светлый.

## Удаление шума со скана перед OCR

Даже идеально выпрямленное изображение может содержать пятна, пыль или артефакты сжатия. Эти мелкие пятнышки — главная беда любого OCR‑движка.

```python
# Step 4: Reduce noise to improve OCR accuracy
preprocessor.denoise(radius=1)
```

> **Зачем удалять шум?**  
> Шум создаёт ложные контуры, которые OCR может принять за символы. Малый радиус подходит для большинства сканов; увеличьте его для сильно зернистых документов.

## Применение конвейера предобработки изображения

Все параметры заданы — теперь запускаем конвейер на оригинальном изображении.

```python
# Step 5: Apply the preprocessing pipeline to the loaded image
processed_image = preprocessor.apply(original_image)
```

> **Результат:** `processed_image` — очищенный, выпрямленный, контрастный битмап, готовый к извлечению текста.

## Распознавание текста с изображения с помощью OCR‑движка

Пора действительно прочитать текст. Мы инициализируем OCR‑движок, передаём ему наш отполированный файл и запрашиваем сырую строку.

```python
# Step 6: Initialise the OCR engine and provide the processed image
ocr_engine = ocr.OcrEngine()
ocr_engine.set_image(processed_image)

# Step 7: Recognise text and output the result
recognized_text = ocr_engine.recognize().get_text()
print(recognized_text)
```

> **Ожидаемый вывод** (пример):

```
Lorem ipsum dolor sit amet, consectetur adipiscing elit.
Sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.
```

Если видите «мусорные» символы, проверьте шаг выпрямления или поэкспериментируйте с другим значением `threshold`.

## Создание конвейера предобработки изображения — Полный скрипт

Объединив всё вместе, получаем единый исполняемый скрипт, который можно сохранить в файл `.py` и запустить:

```python
import ocr  # Replace with the actual import path of your OCR library

# -------------------------------------------------
# 1️⃣ Load the scanned image
# -------------------------------------------------
image_path = "YOUR_DIRECTORY/skewed_scan.jpg"
original_image = ocr.Imaging.Image.load(image_path)

# -------------------------------------------------
# 2️⃣ Deskew – how to deskew image
# -------------------------------------------------
preprocessor = ocr.ImagePreprocessor()
preprocessor.deskew()

# -------------------------------------------------
# 3️⃣ Binarize – convert to black‑and‑white
# -------------------------------------------------
preprocessor.binarize(threshold=128)

# -------------------------------------------------
# 4️⃣ Denoise – remove noise from scan
# -------------------------------------------------
preprocessor.denoise(radius=1)

# -------------------------------------------------
# 5️⃣ Apply the pipeline
# -------------------------------------------------
processed_image = preprocessor.apply(original_image)

# -------------------------------------------------
# 6️⃣ OCR – recognize text from image
# -------------------------------------------------
ocr_engine = ocr.OcrEngine()
ocr_engine.set_image(processed_image)

# -------------------------------------------------
# 7️⃣ Output – convert scanned image to text
# -------------------------------------------------
recognized_text = ocr_engine.recognize().get_text()
print(recognized_text)
```

> **Подсказка:** Оберните всё в функцию (`def ocr_from_file(path): …`), если планируете обрабатывать множество файлов пакетно.

## Часто задаваемые вопросы и особые случаи

- **Что если изображение уже правильно ориентировано?**  
  Метод `deskew()` обнаружит почти нулевой угол и оставит картинку без изменений, так что её можно безопасно запускать для каждого файла.

- **Мой скан в цвете — оставлять его?**  
  Цвет обычно не добавляет ценности для OCR‑задач. Бинаризация убирает его, ускоряя обработку и снижая расход памяти.

- **Можно ли цепочкой применять несколько шагов предобработки?**  
  Конечно. Класс `ImagePreprocessor` создан для конвейеров; перед вызовом `apply()` можно добавить `sharpen()`, `contrast_enhance()` или собственные фильтры.

- **В выводе OCR появляются лишние переносы строк — как исправить?**  
  Постобработайте строку с помощью `text.replace("\n", " ").strip()` или используйте более продвинутый анализ разметки, например режимы `--psm` в Tesseract.

## Следующие шаги

Теперь, когда вы знаете **как исправить наклон изображения** и имеете надёжный **конвейер предобработки изображения**, вы можете:

- Интегрировать **распознавание текста с изображения** в Flask‑API для загрузки документов «на лету».
- Использовать конвейер для **удаления шума со скана** исторических документов, где важна сохранность.
- Масштабировать процесс до пакетной обработки тысяч страниц и выводить поисковый PDF с помощью `pdfminer` или `PyMuPDF`.

Экспериментируйте с различными порогами, радиусами шумоподавления или даже замените OCR‑бэкенд на Tesseract, если нужна поддержка нескольких языков. Принципы остаются теми же: выпрямить, очистить, затем прочитать.
