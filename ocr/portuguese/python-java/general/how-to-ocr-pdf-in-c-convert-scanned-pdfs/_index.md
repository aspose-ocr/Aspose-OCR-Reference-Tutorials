# Como fazer OCR em PDF em C# – Converter PDFs escaneados

Já precisou **how to OCR PDF** mas ficou preso em um arquivo escaneado que se recusava a cooperar? Você não está sozinho. Em muitos projetos do mundo real — pense em digitalização de faturas ou arquivamento de relatórios históricos — você acaba com uma pilha de imagens dentro de um PDF, e a única maneira de torná‑las pesquisáveis é executar OCR sobre elas.  

A boa notícia? Com apenas algumas linhas de C# você pode **convert scanned PDF** em um **create searchable PDF**, adicionar uma marca d'água sutil e até extrair o texto bruto para processamento adicional. Abaixo você verá um exemplo completo, pronto‑para‑executar, que faz exatamente isso.

## O que este tutorial cobre

* Como **how to OCR PDF** usando a classe `OcrEngine`.  
* Transformar um PDF escaneado em um **create searchable PDF**.  
* Adicionar uma marca d'água na primeira página (**add watermark pdf**).  
* Extrair cadeias de texto simples do resultado (**extract text from pdf**).  
* Armadilhas comuns — como lidar com documentos de várias páginas ou com digitalizações de baixa resolução.

Sem links externos de documentação, sem trechos incompletos. Tudo que você precisa está aqui.

## Pré‑requisitos

* .NET 6.0 ou superior (o código também compila com .NET 5).  
* Uma referência à biblioteca OCR que fornece `OcrEngine` e `ImageFormats` (por exemplo, `Aspose.OCR` ou um pacote similar).  
* Um PDF de entrada chamado `input.pdf` colocado em uma pasta que você controla.  
* Familiaridade básica com aplicativos console em C#.

Se você já tem isso, ótimo — vamos começar.

## Etapa 1: Configurar o projeto e importar dependências

Crie um novo projeto console e adicione o pacote OCR via NuGet:

```bash
dotnet new console -n OcrPdfDemo
cd OcrPdfDemo
dotnet add package Aspose.OCR
```

Agora abra `Program.cs`. No topo, importe os namespaces que você precisará:

```csharp
using System;
using Aspose.OCR;          // OcrEngine lives here
using Aspose.OCR.Image;   // ImageFormats enum
```

*Por que isso importa*: Importar os namespaces corretos permite que o compilador localize `OcrEngine`. Pular esta etapa gerará um erro `CS0246` e interromperá sua compilação.

## Etapa 2: Inicializar o motor OCR

O motor é o coração do processo. Você o instancia uma vez e o reutiliza para cada página.

```csharp
// Step 2: Initialize the OCR engine
OcrEngine engine = new OcrEngine();
```

*Dica profissional*: Se você planeja processar muitos PDFs consecutivamente, considere reutilizar a mesma instância `engine` para evitar verificações de licença repetidas.

## Etapa 3: Carregar o PDF de origem

Informe ao motor qual arquivo usar. O método `SetImageFromFile` aceita qualquer fonte de imagem ou PDF.

```csharp
// Step 3: Load the scanned PDF you want to OCR
engine.SetImageFromFile(@"YOUR_DIRECTORY\input.pdf");
```

> **Por que esta etapa é crucial** – O motor precisa de uma representação bitmap de cada página. Quando você fornece um PDF, ele rasteriza internamente cada página com base no DPI padrão (geralmente 300). Se o PDF de origem já for baseado em texto, o motor simplesmente o passará adiante sem alterações, economizando seu tempo.

## Etapa 4: Executar o reconhecimento OCR

Agora o trabalho pesado acontece. O método `Recognize` escaneia cada página, extrai glifos e constrói uma camada pesquisável.

```csharp
// Step 4: Perform OCR on the loaded document
engine.Recognize();
```

*Pergunta comum*: *E se o PDF tiver 50 páginas?*  
`Recognize` processa o documento inteiro por padrão. Em ambientes com memória limitada, você pode definir `engine.PageIndex` e `engine.PageCount` para trabalhar página por página.

## Etapa 5: Salvar o PDF pesquisável (Marca d'água na página 1)

Vamos salvar o resultado como PDF. A primeira página receberá automaticamente uma marca d'água porque a biblioteca OCR adiciona, por padrão, uma sobreposição “Generated by Aspose.OCR”. Se precisar de uma marca d'água personalizada, você pode modificar a propriedade `engine.Watermark` antes de salvar.

```csharp
// Step 5: Save the OCR‑processed PDF with a watermark on page 1
engine.Save(@"YOUR_DIRECTORY\output.pdf", ImageFormats.Pdf);
```

Após esta chamada você terá um **create searchable PDF** onde pode selecionar, copiar e pesquisar o texto como em qualquer PDF nativo.

## Etapa 6: Extrair texto simples (Opcional, mas útil)

Se você também quiser **extract text from pdf** para indexação ou análise adicional, o motor fornece acesso à string bruta.

```csharp
// Step 6: Pull the recognized text into a string
string extractedText = engine.Text;
Console.WriteLine("=== Extracted Text Start ===");
Console.WriteLine(extractedText);
Console.WriteLine("=== Extracted Text End ===");
```

A propriedade `engine.Text` concatena o texto de todas as páginas, preservando quebras de linha. Você pode dividi‑lo por `\f` (form feed) se precisar de granularidade por página.

## Exemplo completo em funcionamento

Abaixo está o programa completo. Copie‑e cole em `Program.cs`, substitua `YOUR_DIRECTORY` por um caminho absoluto ou relativo e execute `dotnet run`.

```csharp
using System;
using Aspose.OCR;
using Aspose.OCR.Image;

namespace OcrPdfDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // 1️⃣ Initialize the OCR engine
            OcrEngine engine = new OcrEngine();

            // 2️⃣ Load the source PDF to be processed
            // Make sure the file exists; otherwise you'll get a FileNotFoundException.
            string inputPath = @"YOUR_DIRECTORY\input.pdf";
            engine.SetImageFromFile(inputPath);

            // 3️⃣ Perform OCR recognition on the loaded document
            engine.Recognize();

            // 4️⃣ Save the recognized document as a PDF (watermark appears on page 1)
            string outputPath = @"YOUR_DIRECTORY\output.pdf";
            engine.Save(outputPath, ImageFormats.Pdf);

            // 5️⃣ (Optional) Extract plain text for further use
            string extractedText = engine.Text;
            Console.WriteLine("=== Extracted Text Start ===");
            Console.WriteLine(extractedText);
            Console.WriteLine("=== Extracted Text End ===");

            Console.WriteLine($"OCR complete! Searchable PDF saved to: {outputPath}");
        }
    }
}
```

### Saída esperada

Executar o programa imprime o texto extraído no console e cria `output.pdf`. Abra o PDF em qualquer visualizador (Adobe Acrobat, Edge, etc.) e tente buscar uma palavra que apareceu na digitalização original — você verá que agora está pesquisável. A primeira página mostra a marca d'água padrão, confirmando que a etapa **add watermark pdf** foi bem‑sucedida.

## Perguntas Frequentes & Casos de Borda

| Pergunta | Resposta |
|----------|----------|
| **E se a digitalização for de baixa resolução?** | Aumente o DPI antes do reconhecimento: `engine.ImageInfo.DpiX = engine.ImageInfo.DpiY = 600;` Isso fornece ao motor OCR mais detalhes para trabalhar. |
| **Posso escolher uma marca d'água personalizada?** | Sim. Defina `engine.Watermark = "Confidential – Processed on " + DateTime.Now.ToShortDateString();` antes de `Save`. |
| **Como lidar com PDFs protegidos por senha?** | Use `engine.LoadPassword = "yourPassword";` antes de `SetImageFromFile`. |
| **Existe uma forma de limitar o OCR a páginas específicas?** | Defina `engine.PageIndex = 2;` e `engine.PageCount = 5;` para processar apenas as páginas 3‑7. |
| **E se eu precisar manter as imagens originais intactas?** | Após o OCR você pode mesclar o PDF original com a camada OCR usando uma biblioteca PDF (por exemplo, `Aspose.PDF`) para preservar a fidelidade das imagens. |

## Dicas & Melhores Práticas

* **Dica profissional:** Execute o OCR em uma thread em segundo plano se você estiver integrando isso a um aplicativo UI — caso contrário, a UI travará.  
* **Fique atento a:** PDFs com conteúdo misto raster e vetor. O motor só fará OCR nas páginas raster; o texto vetorial permanece selecionável automaticamente.  
* **Ajuste de desempenho:** Cache os dados de idioma OCR (`engine.Language = OcrLanguage.English;`) para evitar recarregá‑los a cada documento.

## Conclusão

Agora você tem uma solução completa, de ponta a ponta, para **how to OCR PDF** em C#. Ao inicializar o `OcrEngine`, carregar um arquivo escaneado, reconhecer o texto, salvar um **create searchable pdf**, adicionar uma marca d'água e, opcionalmente, **extracting text from pdf**, você pode transformar qualquer PDF baseado em imagem em um ativo pesquisável e indexável.

Próximos passos? Experimente encadear este fluxo de trabalho com um sistema de gerenciamento de documentos, ou experimente diferentes idiomas (`engine.Language = OcrLanguage.French;`). Você também pode explorar o processamento em lote de vários arquivos em uma pasta — basta percorrer as etapas com uma nova instância `engine` a cada vez.

Feliz codificação, e que seus PDFs estejam sempre pesquisáveis!
