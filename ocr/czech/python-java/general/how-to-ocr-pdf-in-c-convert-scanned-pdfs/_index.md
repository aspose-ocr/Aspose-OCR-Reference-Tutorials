# Jak OCR PDF v C# – Převod naskenovaných PDF

Už jste někdy potřebovali **how to OCR PDF**, ale uvízli jste u naskenovaného souboru, který nechtěl spolupracovat? Nejste v tom sami. V mnoha reálných projektech—například digitalizace faktur nebo archivace historických zpráv—se vám může dostat hromada obrázků vložených do PDF a jediný způsob, jak je učinit prohledávatelnými, je spustit na nich OCR.  

Dobrá zpráva? S několika řádky C# můžete **convert scanned PDF** na **create searchable PDF**, přidat jemnou vodoznak a dokonce získat surový text pro další zpracování. Níže uvidíte kompletní, připravený příklad, který přesně to dělá.

## Co tento tutoriál pokrývá

* Jak **how to OCR PDF** pomocí třídy `OcrEngine`.  
* Převod naskenovaného PDF na **create searchable PDF**.  
* Přidání vodoznaku na první stránku (**add watermark pdf**).  
* Získání řetězců prostého textu z výsledku (**extract text from pdf**).  
* Běžné úskalí—například zpracování vícestránkových dokumentů nebo práce s nízkým rozlišením skenů.

Žádné externí odkazy na dokumentaci, žádné nedokončené úryvky. Vše, co potřebujete, je zde.

## Požadavky

* .NET 6.0 nebo novější (kód se také kompiluje s .NET 5).  
* Odkaz na OCR knihovnu, která poskytuje `OcrEngine` a `ImageFormats` (např. `Aspose.OCR` nebo podobný balíček).  
* Vstupní PDF pojmenované `input.pdf` umístěné ve složce, kterou ovládáte.  
* Základní znalost C# konzolových aplikací.

Pokud už to máte, skvělé—přeskočíme na praktickou část.

## Krok 1: Nastavení projektu a import závislostí

Vytvořte nový konzolový projekt a přidejte OCR balíček přes NuGet:

```bash
dotnet new console -n OcrPdfDemo
cd OcrPdfDemo
dotnet add package Aspose.OCR
```

Nyní otevřete `Program.cs`. Na začátku přidejte jmenné prostory, které budete potřebovat:

```csharp
using System;
using Aspose.OCR;          // OcrEngine lives here
using Aspose.OCR.Image;   // ImageFormats enum
```

*Proč je to důležité*: Import správných jmenných prostorů umožní kompilátoru najít `OcrEngine`. Vynechání tohoto kroku způsobí chybu `CS0246` a zastaví sestavení.

## Krok 2: Inicializace OCR enginu

Engine je srdcem procesu. Vytvoříte jej jednou a znovu použijete pro každou stránku.

```csharp
// Step 2: Initialize the OCR engine
OcrEngine engine = new OcrEngine();
```

*Tip*: Pokud plánujete zpracovávat mnoho PDF po sobě, zvažte opětovné použití stejné instance `engine`, abyste se vyhnuli opakovaným kontrolám licence.

## Krok 3: Načtení zdrojového PDF

Řekněte enginu, se kterým souborem má pracovat. Metoda `SetImageFromFile` přijímá libovolný zdroj obrázku nebo PDF.

```csharp
// Step 3: Load the scanned PDF you want to OCR
engine.SetImageFromFile(@"YOUR_DIRECTORY\input.pdf");
```

> **Proč je tento krok zásadní** – Engine potřebuje bitmapovou reprezentaci každé stránky. Když mu předáte PDF, interně rasterizuje každou stránku podle výchozího DPI (obvykle 300). Pokud je zdrojové PDF již textové, engine jej prostě předá beze změny, čímž vám ušetří čas.

## Krok 4: Spuštění OCR rozpoznání

Nyní se provádí těžká práce. Metoda `Recognize` prohledá každou stránku, extrahuje glyfy a vytvoří prohledávatelnou vrstvu.

```csharp
// Step 4: Perform OCR on the loaded document
engine.Recognize();
```

*Často kladená otázka*: *Co když PDF má 50 stránek?*  
`Recognize` ve výchozím nastavení zpracuje celý dokument. Pro prostředí s omezenou pamětí můžete nastavit `engine.PageIndex` a `engine.PageCount` a zpracovávat stránku po stránce.

## Krok 5: Uložení prohledávatného PDF (vodoznak na stránce 1)

Uložíme výsledek jako PDF. První stránka automaticky získá vodoznak, protože OCR knihovna ve výchozím nastavení přidá překrytí „Generated by Aspose.OCR“. Pokud potřebujete vlastní vodoznak, můžete před uložením upravit vlastnost `engine.Watermark`.

```csharp
// Step 5: Save the OCR‑processed PDF with a watermark on page 1
engine.Save(@"YOUR_DIRECTORY\output.pdf", ImageFormats.Pdf);
```

Po tomto volání budete mít **create searchable PDF**, ve kterém můžete text vybírat, kopírovat a vyhledávat stejně jako v jakémkoli nativním PDF.

## Krok 6: Extrakce prostého textu (volitelné, ale užitečné)

Pokud také chcete **extract text from pdf** pro indexaci nebo další analýzu, engine vám poskytne přístup k surovému řetězci.

```csharp
// Step 6: Pull the recognized text into a string
string extractedText = engine.Text;
Console.WriteLine("=== Extracted Text Start ===");
Console.WriteLine(extractedText);
Console.WriteLine("=== Extracted Text End ===");
```

Vlastnost `engine.Text` spojuje text všech stránek a zachovává konce řádků. Pokud potřebujete rozdělit podle stránek, můžete použít `\f` (form feed).

## Kompletní funkční příklad

Níže je kompletní program. Zkopírujte jej do `Program.cs`, nahraďte `YOUR_DIRECTORY` absolutní nebo relativní cestou a spusťte `dotnet run`.

```csharp
using System;
using Aspose.OCR;
using Aspose.OCR.Image;

namespace OcrPdfDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // 1️⃣ Initialize the OCR engine
            OcrEngine engine = new OcrEngine();

            // 2️⃣ Load the source PDF to be processed
            // Make sure the file exists; otherwise you'll get a FileNotFoundException.
            string inputPath = @"YOUR_DIRECTORY\input.pdf";
            engine.SetImageFromFile(inputPath);

            // 3️⃣ Perform OCR recognition on the loaded document
            engine.Recognize();

            // 4️⃣ Save the recognized document as a PDF (watermark appears on page 1)
            string outputPath = @"YOUR_DIRECTORY\output.pdf";
            engine.Save(outputPath, ImageFormats.Pdf);

            // 5️⃣ (Optional) Extract plain text for further use
            string extractedText = engine.Text;
            Console.WriteLine("=== Extracted Text Start ===");
            Console.WriteLine(extractedText);
            Console.WriteLine("=== Extracted Text End ===");

            Console.WriteLine($"OCR complete! Searchable PDF saved to: {outputPath}");
        }
    }
}
```

### Očekávaný výstup

Spuštění programu vypíše extrahovaný text do konzole a vytvoří `output.pdf`. Otevřete PDF v libovolném prohlížeči (Adobe Acrobat, Edge atd.) a zkuste vyhledat slovo, které se objevilo v původním skenu—uvidíte, že je nyní prohledávatelné. První stránka zobrazuje výchozí vodoznak, což potvrzuje úspěšnost kroku **add watermark pdf**.

## Často kladené otázky a okrajové případy

| Question | Answer |
|----------|--------|
| **Co když je sken nízkého rozlišení?** | Zvyšte DPI před rozpoznáním: `engine.ImageInfo.DpiX = engine.ImageInfo.DpiY = 600;` To poskytne OCR enginu více detailů. |
| **Mohu zvolit vlastní vodoznak?** | Ano. Nastavte `engine.Watermark = "Confidential – Processed on " + DateTime.Now.ToShortDateString();` před `Save`. |
| **Jak zacházet s PDF chráněnými heslem?** | Použijte `engine.LoadPassword = "yourPassword";` před `SetImageFromFile`. |
| **Existuje způsob, jak omezit OCR na konkrétní stránky?** | Nastavte `engine.PageIndex = 2;` a `engine.PageCount = 5;` pro zpracování stránek 3‑7. |
| **Co když potřebuji zachovat původní obrázky nedotčené?** | Po OCR můžete sloučit původní PDF s OCR vrstvou pomocí PDF knihovny (např. `Aspose.PDF`) pro zachování kvality obrázků. |

## Tipy a osvědčené postupy

* **Tip:** Spusťte OCR na pozadí, pokud tuto funkci integrujete do UI aplikace—jinak se UI zamrzne.  
* **Dejte pozor na:** PDF s kombinovaným rastrovým a vektorovým obsahem. Engine OCR provede pouze rastrové stránky; vektorový text zůstane automaticky vybíratelný.  
* **Optimalizace výkonu:** Uložte do cache data jazyka OCR (`engine.Language = OcrLanguage.English;`), abyste se vyhnuli opakovanému načítání pro každý dokument.

## Závěr

Nyní máte kompletní end‑to‑end řešení pro **how to OCR PDF** v C#. Inicializací `OcrEngine`, načtením naskenovaného souboru, rozpoznáním textu, uložením **create searchable pdf**, přidáním vodoznaku a volitelně **extracting text from pdf** můžete převést jakýkoli PDF založený na obrázcích na prohledávatelný, indexovatelný zdroj.

Další kroky? Zkuste propojit tento workflow s dokumentovým systémem, nebo experimentujte s různými jazyky (`engine.Language = OcrLanguage.French;`). Můžete také prozkoumat hromadné zpracování více souborů ve složce—stačí projít kroky v cyklu s novou instancí `engine` pokaždé.

Šťastné programování a ať jsou vaše PDF vždy prohledávatelná!
