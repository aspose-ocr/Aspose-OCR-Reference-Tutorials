# إنشاء مضلع ROI – دليل كامل لتقنية OCR على المنطقة المختارة

هل احتجت يومًا إلى **إنشاء مضلع ROI** حتى تتمكن من تشغيل OCR على الجزء فقط من الصورة الذي يهم؟ ربما تقوم بمسح الإيصالات ضوئيًا ولا يهم سوى الإجماليات، أو لديك نموذج حيث لا يلزم قراءة حقل التوقيع فقط. في هذه الحالات، **ocr على المنطقة المختارة** يوفر الوقت ويزيد الدقة.  

في هذا الدليل سنستعرض كل ما تحتاجه: من تعريف مضلعين، تسجيلهما مع محرك OCR، إلى استخراج النص المعترف به في استدعاء واحد نظيف. بنهاية القراءة ستحصل على سكريبت جاهز للتنفيذ وفهم قوي لأهمية التركيز على مناطق الاهتمام.

## ما ستتعلمه

- كيفية **إنشاء مضلع ROI** باستخدام مكتبة OCR نمطية.  
- الفرق بين تعريف منطقة واحدة مقابل عدة مناطق.  
- كيفية تسجيل تلك المناطق مع المحرك بحيث يتم تنفيذ `ocr on selected area`.  
- النتيجة المتوقعة وكيفية استكشاف الأخطاء الشائعة (مثل تداخل الـ ROI، النتائج الفارغة).

### المتطلبات المسبقة

- تثبيت Python 3.8+.  
- مكتبة OCR تُظهر الكائنات `Polygon`، `Point`، وطريقة `add_region_of_interest` (المثال يستخدم وحدة `ocr` خيالية؛ استبدلها بـ SDK الفعلي مثل أغلفة Tesseract‑Python أو EasyOCR).  
- ملف صورة تجريبي (`sample.png`) يحتوي على نص داخل الإحداثيات الموضحة أدناه.

> **نصيحة احترافية:** إذا كنت تستخدم مكتبة حقيقية، تأكد من تحميل الصورة في نفس مساحة الإحداثيات (الأصل في أعلى اليسار، X يزداد إلى اليمين، Y يزداد إلى الأسفل).  

---  

## الخطوة 1: استيراد وحدة OCR وتحميل الصورة الخاصة بك  

أولاً، استدعِ حزمة OCR إلى النطاق واقرأ الصورة التي تريد معالجتها.  

```python
import ocr  # Replace with your actual OCR library import
from pathlib import Path

# Load the image – adjust the path as needed
image_path = Path("sample.png")
ocr_engine = ocr.Engine(image_path)
```

*لماذا هذا مهم:* يحتاج المحرك إلى بيانات الصورة قبل أن تتمكن من إرفاق أي **مضلع ROI**. بعض المكتبات تسمح بتمرير الصورة لاحقًا، لكن التهيئة المبكرة تحافظ على تنظيم سير العمل.

## الخطوة 2: تعريف أول مضلع ROI  

الآن نقوم بإنشاء أول منطقة اهتمام. فكر فيها كأنك ترسم مستطيلًا حول عنوان جدول.  

```python
# Step 2: Define the first ROI polygon
first_roi = ocr.Polygon([
    ocr.Point(50, 20),   # top‑left
    ocr.Point(200, 20),  # top‑right
    ocr.Point(200, 80),  # bottom‑right
    ocr.Point(50, 80)    # bottom‑left
])
```

*شرح:*  
- `ocr.Point(x, y)` يستخدم إحداثيات البكسل.  
- قائمة النقاط مرتبة باتجاه عقارب الساعة، وهو ما تتوقعه معظم المحركات.  
- يمكنك إضافة عدد غير محدود من النقاط لإنشاء أشكال غير منتظمة؛ المستطيل هو مجرد حالة خاصة.

## الخطوة 3: تعريف مضلعات ROI إضافية (اختياري)  

ليس عليك التوقف عند واحدة. إليك مضلعًا ثانيًا يلتقط حقل التوقيع الموجود أسفل الصفحة.  

```python
# Step 3: Define the second ROI polygon
second_roi = ocr.Polygon([
    ocr.Point(300, 150),
    ocr.Point(480, 150),
    ocr.Point(480, 210),
    ocr.Point(300, 210)
])
```

*لماذا نضيف المزيد؟* تنفيذ **ocr على المنطقة المختارة** لعدة مناطق يتيح لك استخراج قطع بيانات متباعدة في تمريرة واحدة، وهو أسرع بكثير من قص ومعالجة كل شريحة على حدة.

## الخطوة 4: تسجيل الـ ROI مع محرك OCR  

بعد تجهيز المضلعات، أخبر المحرك أي مناطق يجب فحصها.  

```python
# Step 4: Register the ROIs
ocr_engine.add_region_of_interest(first_roi)
ocr_engine.add_region_of_interest(second_roi)
```

*ملاحظة هامة:* بعض SDKs تتطلب استدعاء طريقة `clear_regions()` قبل إضافة مناطق جديدة إذا كنت تعيد استخدام المحرك لصورة أخرى.

## الخطوة 5: تشغيل OCR واسترجاع النص  

أخيرًا، شغّل عملية التعرف. سيبحث المحرك فقط داخل المضلعات التي أضفناها للتو.  

```python
# Step 5: Perform OCR on the defined regions
recognized_text = ocr_engine.recognize().get_text()
print("=== Recognized Text ===")
print(recognized_text)
```

**الناتج المتوقع** (بافتراض أن `sample.png` يحتوي على الكلمات “Invoice Total: $123.45” في الـ ROI الأول و“Signature: John Doe” في الثاني):

```
=== Recognized Text ===
Invoice Total: $123.45
Signature: John Doe
```

إذا كان الناتج فارغًا، تحقق مرة أخرى من أن الإحداثيات تتقاطع فعليًا مع النص وأن دقة الصورة تتطابق مع نظام الإحداثيات.

## حالات الحافة والمشكلات الشائعة  

| الحالة                                 | ما يجب مراقبته                                 | الحل / طريقة التحايل                           |
|----------------------------------------|------------------------------------------------|-----------------------------------------------|
| **تداخل الـ ROI**                     | قد يتم إرجاع النص مرتين.                        | احرص على أن تكون المضلعات منفصلة أو قم بإزالة التكرار من الناتج. |
| **ROI خارج حدود الصورة**               | قد يرفع المحرك خطأً أو لا يُرجع شيئًا.           | قيد الإحداثيات إلى `0 ≤ x < العرض`, `0 ≤ y < الارتفاع`. |
| **ROI صغير جدًا**                     | تنخفض دقة OCR بسبب قلة البكسلات.               | وسّع المضلع بضع بكسلات أو قم بتكبير الصورة أولًا. |
| **شكل غير مستطيل**                    | بعض المحركات تدعم فقط المضلعات المحدبة.         | قسم الأشكال المعقدة إلى عدة ROI محدبة. |
| **تغيير حجم الصورة**                  | تصبح الإحداثيات الثابتة غير صالحة.              | احسب إحداثيات ROI نسبةً إلى أبعاد الصورة (مثلاً النسب المئوية). |

## مثال كامل يعمل  

فيما يلي السكريبت الكامل الذي يمكنك نسخه ولصقه وتشغيله (استبدل `ocr` بالمكتبة الفعلية الخاصة بك).  

```python
import ocr                      # <- your OCR library
from pathlib import Path

# -------------------------------------------------
# Configuration
# -------------------------------------------------
IMAGE_PATH = Path("sample.png")

# -------------------------------------------------
# Initialize OCR engine with the image
# -------------------------------------------------
engine = ocr.Engine(IMAGE_PATH)

# -------------------------------------------------
# Define ROI polygons
# -------------------------------------------------
first_roi = ocr.Polygon([
    ocr.Point(50, 20), ocr.Point(200, 20),
    ocr.Point(200, 80), ocr.Point(50, 80)
])

second_roi = ocr.Polygon([
    ocr.Point(300, 150), ocr.Point(480, 150),
    ocr.Point(480, 210), ocr.Point(300, 210)
])

# -------------------------------------------------
# Register ROIs
# -------------------------------------------------
engine.add_region_of_interest(first_roi)
engine.add_region_of_interest(second_roi)

# -------------------------------------------------
# Run OCR on the selected areas
# -------------------------------------------------
result = engine.recognize().get_text()

print("=== Recognized Text ===")
print(result)
```

شغّله باستخدام `python ocr_roi_example.py` وسترى السلاسل المستخرجة من المنطقتين المحددتين.

## الخطوات التالية  

- **إنشاء ROI ديناميكي:** بدلاً من ترميز الإحداثيات يدويًا، استخدم معالجة الصور (مثل كشف الحواف بـ OpenCV) لتحديد الجداول أو الحقول تلقائيًا.  
- **ما بعد المعالجة:** إزالة الفراغات، تطبيق تعبيرات regex، أو تحويل الأرقام إلى أنواع بيانات مناسبة.  
- **المعالجة الدفعية:** تكرار عبر مجلد من الصور، وإعادة استخدام تعريفات ROI إذا كان التخطيط ثابتًا.  

إذا كنت مهتمًا بـ **ocr على المنطقة المختارة** لمستندات أكثر تعقيدًا—مثل ملفات PDF متعددة الصفحات أو النماذج الممسوحة—فابحث عن مكتبات تدعم تسجيل ROI على مستوى الصفحات.

## الخلاصة  

لقد **أنشأنا كائنات ROI polygon**، سجلناها مع محرك OCR، ونفّذنا `ocr on selected area` في سكريبت نظيف وقابل لإعادة الاستخدام. بتحديد المحرك على المناطق التي تهمك، تقلل من وقت المعالجة، تحسن الدقة، وتبسط التعامل مع البيانات لاحقًا.  

جرّبه مع صورك الخاصة—عدّل الإحداثيات، أضف المزيد من المضلعات، أو اربط الناتج بقاعدة بيانات. السماء هي الحد عندما تتقن OCR القائم على المناطق.
