# كيفية التعرف الضوئي على النص في ملفات PDF باستخدام C# – تحويل ملفات PDF الممسوحة ضوئياً

هل احتجت يوماً إلى **how to OCR PDF** لكنك علقّت في ملف ممسوح ضوئياً يرفض التعاون؟ لست وحدك. في العديد من المشاريع الواقعية—مثل رقمنة الفواتير أو أرشفة التقارير التاريخية—تنتهي بك الأمور إلى مجموعة من الصور المدمجة داخل ملف PDF، والطريقة الوحيدة لجعلها قابلة للبحث هي تشغيل OCR عليها.  

الخبر السار؟ ببضع أسطر من C# يمكنك **convert scanned PDF** إلى **create searchable PDF**، إضافة علامة مائية خفيفة، وحتى استخراج النص الخام للمعالجة اللاحقة. أدناه ستجد مثالاً كاملاً جاهزاً للتنفيذ يفعل كل ذلك بالضبط.

## ما يغطيه هذا الدرس

* كيفية **how to OCR PDF** باستخدام الفئة `OcrEngine`.  
* تحويل ملف PDF ممسوح ضوئياً إلى **create searchable PDF**.  
* إضافة علامة مائية على الصفحة الأولى (**add watermark pdf**).  
* استخراج سلاسل النص العادي من النتيجة (**extract text from pdf**).  
* الأخطاء الشائعة—مثل التعامل مع المستندات متعددة الصفحات أو التعامل مع المسحات منخفضة الدقة.

لا روابط توثيق خارجية، لا مقتطفات نصف مكتملة. كل ما تحتاجه موجود هنا.

## المتطلبات المسبقة

* .NET 6.0 أو أحدث (الكود يتوافق مع .NET 5 أيضاً).  
* إشارة إلى مكتبة OCR التي توفر `OcrEngine` و `ImageFormats` (مثل `Aspose.OCR` أو حزمة مشابهة).  
* ملف PDF إدخال اسمه `input.pdf` موجود في مجلد تتحكم فيه.  
* إلمام أساسي بتطبيقات C# console.

إذا كان لديك كل ذلك، ممتاز—لنبدأ.

## الخطوة 1: إعداد المشروع واستيراد الاعتمادات

أنشئ مشروع console جديد وأضف حزمة OCR عبر NuGet:

```bash
dotnet new console -n OcrPdfDemo
cd OcrPdfDemo
dotnet add package Aspose.OCR
```

الآن افتح `Program.cs`. في الأعلى، استورد المساحات الاسم التي ستحتاجها:

```csharp
using System;
using Aspose.OCR;          // OcrEngine lives here
using Aspose.OCR.Image;   // ImageFormats enum
```

*لماذا هذا مهم*: استيراد المساحات الاسم الصحيحة يسمح للمُصرّف بالعثور على `OcrEngine`. تخطي هذه الخطوة سيسبب خطأ `CS0246` ويوقف عملية البناء.

## الخطوة 2: تهيئة محرك OCR

المحرك هو قلب العملية. تقوم بإنشائه مرة واحدة وتعيد استخدامه لكل صفحة.

```csharp
// Step 2: Initialize the OCR engine
OcrEngine engine = new OcrEngine();
```

*نصيحة احترافية*: إذا كنت تخطط لمعالجة العديد من ملفات PDF على التوالي، فكر في إعادة استخدام نفس كائن `engine` لتجنب فحص الترخيص المتكرر.

## الخطوة 3: تحميل ملف PDF المصدر

أخبر المحرك أي ملف سيعمل عليه. الطريقة `SetImageFromFile` تقبل أي مصدر صورة‑أو‑PDF.

```csharp
// Step 3: Load the scanned PDF you want to OCR
engine.SetImageFromFile(@"YOUR_DIRECTORY\input.pdf");
```

> **لماذا هذه الخطوة حاسمة** – يحتاج المحرك إلى تمثيل bitmap لكل صفحة. عندما تزوده بملف PDF، يقوم داخلياً بتحويل كل صفحة إلى نقطية بناءً على DPI الافتراضي (عادةً 300). إذا كان ملف PDF المصدر نصياً أصلاً، سيقوم المحرك بتمريره دون تعديل، مما يوفر عليك الوقت.

## الخطوة 4: تشغيل التعرف الضوئي

الآن يحدث الجزء الثقيل. الطريقة `Recognize` تمسح كل صفحة، تستخرج الحروف، وتبني طبقة قابلة للبحث.

```csharp
// Step 4: Perform OCR on the loaded document
engine.Recognize();
```

*سؤال شائع*: *ماذا لو كان PDF يحتوي على 50 صفحة؟*  
`Recognize` تعالج المستند بالكامل بشكل افتراضي. للبيئات ذات الذاكرة المحدودة يمكنك ضبط `engine.PageIndex` و `engine.PageCount` للعمل صفحة‑بصفحة.

## الخطوة 5: حفظ PDF القابل للبحث (علامة مائية على الصفحة 1)

سنحفظ النتيجة كملف PDF. ستحصل الصفحة الأولى تلقائياً على علامة مائية لأن مكتبة OCR تضيف تغطية “Generated by Aspose.OCR” بشكل افتراضي. إذا كنت بحاجة إلى علامة مائية مخصصة، يمكنك تعديل خاصية `engine.Watermark` قبل الحفظ.

```csharp
// Step 5: Save the OCR‑processed PDF with a watermark on page 1
engine.Save(@"YOUR_DIRECTORY\output.pdf", ImageFormats.Pdf);
```

بعد هذا الاستدعاء ستحصل على **create searchable PDF** حيث يمكنك تحديد النص، نسخه، والبحث فيه كما في أي PDF أصلي.

## الخطوة 6: استخراج النص العادي (اختياري لكنه مفيد)

إذا أردت أيضاً **extract text from pdf** للفهرسة أو التحليل الإضافي، يمنحك المحرك إمكانية الوصول إلى السلسلة الخام.

```csharp
// Step 6: Pull the recognized text into a string
string extractedText = engine.Text;
Console.WriteLine("=== Extracted Text Start ===");
Console.WriteLine(extractedText);
Console.WriteLine("=== Extracted Text End ===");
```

خاصية `engine.Text` تجمع نص جميع الصفحات، مع الحفاظ على فواصل الأسطر. يمكنك تقسيمها بواسطة `\f` (form feed) إذا احتجت إلى تفصيل حسب الصفحة.

## مثال كامل يعمل

فيما يلي البرنامج الكامل. انسخه‑الصقه في `Program.cs`، استبدل `YOUR_DIRECTORY` بمسار مطلق أو نسبي، ثم شغّل `dotnet run`.

```csharp
using System;
using Aspose.OCR;
using Aspose.OCR.Image;

namespace OcrPdfDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // 1️⃣ Initialize the OCR engine
            OcrEngine engine = new OcrEngine();

            // 2️⃣ Load the source PDF to be processed
            // Make sure the file exists; otherwise you'll get a FileNotFoundException.
            string inputPath = @"YOUR_DIRECTORY\input.pdf";
            engine.SetImageFromFile(inputPath);

            // 3️⃣ Perform OCR recognition on the loaded document
            engine.Recognize();

            // 4️⃣ Save the recognized document as a PDF (watermark appears on page 1)
            string outputPath = @"YOUR_DIRECTORY\output.pdf";
            engine.Save(outputPath, ImageFormats.Pdf);

            // 5️⃣ (Optional) Extract plain text for further use
            string extractedText = engine.Text;
            Console.WriteLine("=== Extracted Text Start ===");
            Console.WriteLine(extractedText);
            Console.WriteLine("=== Extracted Text End ===");

            Console.WriteLine($"OCR complete! Searchable PDF saved to: {outputPath}");
        }
    }
}
```

### النتيجة المتوقعة

تشغيل البرنامج يطبع النص المستخرج إلى وحدة التحكم وينشئ `output.pdf`. افتح الـ PDF بأي عارض (Adobe Acrobat، Edge، إلخ) وجرب البحث عن كلمة ظهرت في المسح الأصلي—ستلاحظ أنها الآن قابلة للبحث. الصفحة الأولى تظهر العلامة المائية الافتراضية، مما يؤكد نجاح خطوة **add watermark pdf**.

## الأسئلة المتكررة والحالات الخاصة

| السؤال | الإجابة |
|----------|--------|
| **ماذا لو كان المسح منخفض الدقة؟** | زد الـ DPI قبل التعرف: `engine.ImageInfo.DpiX = engine.ImageInfo.DpiY = 600;` هذا يمنح محرك OCR مزيداً من التفاصيل للعمل معها. |
| **هل يمكنني اختيار علامة مائية مخصصة؟** | نعم. اضبط `engine.Watermark = "سري – تم المعالجة في " + DateTime.Now.ToShortDateString();` قبل `Save`. |
| **كيف أتعامل مع ملفات PDF محمية بكلمة مرور؟** | استخدم `engine.LoadPassword = "yourPassword";` قبل `SetImageFromFile`. |
| **هل هناك طريقة لتحديد OCR لصفحات معينة فقط؟** | اضبط `engine.PageIndex = 2;` و `engine.PageCount = 5;` لمعالجة الصفحات 3‑7 فقط. |
| **ماذا لو أردت الحفاظ على الصور الأصلية دون تعديل؟** | بعد OCR يمكنك دمج PDF الأصلي مع طبقة OCR باستخدام مكتبة PDF (مثل `Aspose.PDF`) للحفاظ على جودة الصورة. |

## نصائح وممارسات أفضل

* **نصيحة احترافية:** شغّل OCR على خيط خلفي إذا كنت تدمجه في تطبيق واجهة مستخدم—وإلا سيتجمد الواجهة.  
* **احذر من:** ملفات PDF التي تحتوي على محتوى نقطي ومتجه مختلط. المحرك سيقوم بـ OCR للصفحات النقطية فقط؛ النص المتجه يبقى قابلاً للتحديد تلقائياً.  
* **تحسين الأداء:** خزن بيانات لغة OCR مؤقتاً (`engine.Language = OcrLanguage.English;`) لتجنب إعادة تحميلها لكل مستند.

## الخلاصة

أصبح لديك الآن حل كامل من البداية للنهاية لـ **how to OCR PDF** في C#. من خلال تهيئة `OcrEngine`، تحميل ملف ممسوح، التعرف على النص، حفظ **create searchable pdf**، إضافة علامة مائية، واستخراج **extract text from pdf** إن رغبت، يمكنك تحويل أي PDF مبني على الصور إلى أصل قابل للبحث والفهرسة.

ما الخطوة التالية؟ جرّب ربط هذا التدفق بنظام إدارة مستندات، أو جرب لغات مختلفة (`engine.Language = OcrLanguage.French;`). يمكنك أيضاً استكشاف المعالجة الدفعة لعدة ملفات في مجلد—فقط كرّر الخطوات مع كائن `engine` جديد في كل مرة.

برمجة سعيدة، ولتظل ملفات PDF الخاصة بك قابلة للبحث دائماً!
