# كيفية استخراج المستطيلات للفقرات في التعرف على الصور باستخدام OCR

## المقدمة

مرحبًا بك في دليلنا الشامل حول **كيفية استخراج المستطيلات** للفقرات في التعرف على الصور باستخدام Aspose.OCR لـ .NET. إذا كنت تتطلع إلى تحسين خط أنابيب معالجة المستندات، واستخراج حدود الفقرات، وأتمتة استخراج البيانات، فأنت في المكان الصحيح. سنستعرض كل خطوة — من إعداد البيئة إلى طباعة إحداثيات المستطيلات — حتى تتمكن من البدء في استخدام نتائج OCR فورًا.

## إجابات سريعة
- **ماذا يعني “استخراج المستطيلات”؟** يُعيد صناديق الإحاطة (x, y, العرض, الارتفاع) لمناطق النص المكتشف.  
- **أي طريقة API تُوفر المستطيلات؟** `AsposeOcr.GetRectangles` مع `AreasType.PARAGRAPHS`.  
- **هل أحتاج إلى ترخيص للتطوير؟** النسخة التجريبية المجانية تكفي للاختبار؛ يلزم ترخيص تجاري للإنتاج.  
- **هل يمكنني معالجة عدة صور في آن واحد؟** نعم — كرّر عبر قائمة الصور واستدعِ `GetRectangles` لكل ملف.  
- **ما الصيغ المدعومة؟** PNG، JPEG، TIFF، BMP، والعديد غيرها.

## ما معنى “كيفية استخراج المستطيلات” في OCR؟
في مصطلحات OCR، يعني استخراج المستطيلات تحديد الحدود الهندسية التي تحيط بكل فقرة أو سطر نص داخل الصورة. تتيح لك هذه الإحداثيات تمييز، قص، أو تحليل أقسام محددة من المستند الممسوح ضوئيًا.

## لماذا نحتاج إلى استخراج إحداثيات الفقرات؟
- **معالجة لاحقة دقيقة** – يمكنك تمرير كل مستطيل إلى سير عمل لاحق (مثل الترجمة أو الإخفاء).  
- **تحسين واجهة المستخدم/تجربة المستخدم** – عرض صناديق الإحاطة على الصورة الأصلية لإظهار أماكن النص المكتشف.  
- **أتمتة الدُفعات** – تحديد وعزل الفقرات بسرعة عبر مجموعات كبيرة من المستندات.

## المتطلبات المسبقة

- معرفة أساسية بـ C# وتطوير .NET.  
- بيئة تطوير تحتوي على Aspose.OCR لـ .NET مثبتة – يمكنك تنزيلها [هنا](https://releases.aspose.com/ocr/net/).  
- إلمام بمفاهيم معالجة الصور ولماذا يعتبر OCR ضروريًا لاستخراج النص من الملفات الممسوحة.

## استيراد المساحات الاسمية

في ملف C# الخاص بك، استورد المساحات الاسمية المطلوبة لتصبح فئات OCR متاحة:

```csharp
using System;
using System.Collections.Generic;
using System.Drawing;
using System.IO;
using Aspose.OCR;
```

## الخطوة 1: إعداد دليل المستندات

عرّف المجلد الذي يحتوي على الصور التي تريد تحليلها:

```csharp
string dataDir = "Your Document Directory";
```

## الخطوة 2: تهيئة كائن AsposeOcr

أنشئ كائن `AsposeOcr` – هذا يمنحك الوصول إلى جميع وظائف OCR:

```csharp
AsposeOcr api = new AsposeOcr();
```

## الخطوة 3: تحديد مسار الصورة

حدد ملف الصورة الدقيق الذي ترغب في معالجته:

```csharp
string fullPath = dataDir + "sample.png";
```

## الخطوة 4: التعرف على الصورة والحصول على مستطيلات الفقرات

استدعِ طريقة `GetRectangles`. ضبط `detect_areas` على `true` يخبر المحرك بإرجاع مستطيلات **الفقرات**:

```csharp
List<Rectangle> rectangles = api.GetRectangles(fullPath, AreasType.PARAGRAPHS, true);
```

## الخطوة 5: طباعة النتائج

اعرض الإحداثيات لتتمكن من رؤية **إحداثيات استخراج الفقرات** التي تم اكتشافها:

```csharp
Console.WriteLine("Areas coordinates:");
rectangles.ForEach(a => Console.WriteLine($"x:{a.X} y:{a.Y} width:{a.Width} height:{a.Height}"));
```

## المشكلات الشائعة والحلول

| المشكلة | السبب | الحل |
|-------|--------|-----|
| لم يتم إرجاع أي مستطيلات | جودة الصورة منخفضة جدًا أو `AreasType` غير صحيح | تأكد من وضوح الصورة واستخدم `AreasType.PARAGRAPHS`. |
| الإحداثيات متأخرة بمقدار واحد | عدم تطابق مقياس DPI | قم بتعيين DPI الصحيح عند تحميل الصورة أو استخدم `api.Config.Dpi`. |
| استثناء الترخيص | التشغيل بدون ترخيص صالح في بيئة الإنتاج | تطبيق ترخيص مؤقت أو دائم عبر `api.SetLicense`. |
