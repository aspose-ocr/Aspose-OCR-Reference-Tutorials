# تحسين دقة OCR في C# باستخدام Aspose – دليل خطوة بخطوة

هل تساءلت يومًا كيف **تحسن دقة OCR** عندما تستخرج النص من المستندات الممسوحة ضوئيًا؟ لست وحدك. في العديد من المشاريع الواقعية يواجه محرك OCR ضوضاء، صفحات مائلة، أو أبجديات غير لاتينية، وتظهر النتائج كعبث بدلاً من بيانات مفيدة.  

الخبر السار هو أن مجموعة قليلة من الإعدادات يمكنها تحويل هذا الفوضى إلى نص نظيف وقابل للبحث. في هذا الدرس سنستعرض مثالًا كاملاً قابلاً للتنفيذ **يقرأ النص من PNG**، **يستخرج النص من الصورة**، و**يحمّل الصورة لـ OCR** باستخدام Aspose.OCR لـ .NET. في النهاية ستحصل على أساس قوي للحصول على نتائج موثوقة في كل مرة.

## ما ستتعلمه

- كيفية تثبيت وإضافة مرجع حزمة Aspose.OCR عبر NuGet.  
- لماذا تكوين `RecognitionSettings` هو المفتاح **لتحسين دقة OCR**.  
- الشيفرة الدقيقة التي تحتاجها **لتحميل صورة لـ OCR** من تدفق ملف.  
- كيفية **التعرف على النص من التدفق** ومعالجة اللغة السيرية أو أي لغات أخرى.  
- نصائح لضبط إضافي، مثل تصحيح الميل وإزالة الضوضاء، للحفاظ على الدقة عالية.

لا توجد اختصارات “انظر الوثائق” هنا—فقط حل مستقل يمكنك نسخه ولصقه وتشغيله.

## الخطوة 1: تثبيت Aspose.OCR وإعداد المشروع

أولاً، أضف حزمة Aspose.OCR إلى مشروعك:

```bash
dotnet add package Aspose.OCR
```

لماذا هذا مهم لـ **تحسين دقة OCR**؟ الحزمة تتضمن نماذج لغات مدربة مسبقًا ومجموعة من مرشحات ما قبل المعالجة (تصحيح الميل، إزالة الضوضاء، إلخ) الضرورية للحصول على نتائج نظيفة. تخطي هذه الخطوة يعني أنك ستظل عالقًا بالمحرك الافتراضي الأقل صلابة.

> **نصيحة محترف:** إذا كنت تستهدف لغة معينة، ففكّر في تنزيل حزمة اللغة المقابلة من موقع Aspose؛ يمكنها خفض معدل الخطأ بضع نسب مئوية.

## الخطوة 2: إنشاء محرك OCR وتكوين إعدادات التعرف

الآن سننشئ كائن `OcrEngine` ونخبره أننا نريد **تحسين دقة OCR** عبر تمكين مرشحات ما قبل المعالجة واختيار اللغة الصحيحة.

```csharp
using Aspose.OCR;
using Aspose.OCR.Models;

// Step 2: Initialize the OCR engine with accuracy‑boosting settings
using var ocrEngine = new OcrEngine();

var recognitionSettings = new RecognitionSettings
{
    // Choose the language that matches your image; Cyrillic is used here as an example
    Language = Language.Cyrillic,

    // PreprocessFilters help the engine deal with common image issues
    PreprocessFilters = new[]
    {
        PreprocessFilter.Deskew,   // Straightens tilted text
        PreprocessFilter.Denoise   // Removes background noise
    }
};
```

**لماذا هذه المرشحات؟** تصحيح الميل يضبط زاوية سطور النص، وهو أحد أكبر المسببات لانخفاض درجات OCR. إزالة الضوضاء تقلل البقع التي قد يخطئ المحرك في تفسيرها كحروف. معًا، هذه المرشحات **تحسن دقة OCR** بشكل كبير—غالبًا بنسبة 10‑15 % على المسحات الضوضائية.

## الخطوة 3: تحميل صورة PNG – “قراءة النص من PNG”

بعد ذلك، نحتاج إلى **تحميل صورة لـ OCR**. توفر Aspose الدالة `ImageStream.FromFile`، التي تقرأ الملف إلى تدفق يمكن للمحرك استهلاكه.

```csharp
// Step 3: Load the PNG image you want to process
string imagePath = @"YOUR_DIRECTORY/cyrillic_doc.png";
var imageStream = ImageStream.FromFile(imagePath);
```

إذا كنت تتعامل مع صور مخزنة في قاعدة بيانات أو مستلمة عبر API، يمكنك استبدال `FromFile` بـ `FromBytes` أو `FromStream`—طريقة **التعرف على النص من التدفق** تعمل بنفس الطريقة.

## الخطوة 4: التعرف على النص من التدفق

إليك الاستدعاء الأساسي الذي **يتعرف على النص من التدفق** باستخدام الإعدادات التي عرّفناها سابقًا.

```csharp
// Step 4: Perform OCR and capture the result
string ocrResult = ocrEngine.Recognize(imageStream, recognitionSettings);
```

طريقة `Recognize` تُعيد سلسلة نصية عادية تحتوي على جميع الأحرف المكتشفة. لأننا اخترنا `Language.Cyrillic` وفعلنا تصحيح الميل وإزالة الضوضاء، يصبح المحرك مهيأً لـ **استخراج النص من الصورة** بدقة أعلى.

## الخطوة 5: عرض أو معالجة النص المستخرج

أخيرًا، لنقوم **باستخراج النص من الصورة** وعرضه على وحدة التحكم. في تطبيق حقيقي قد تكتب النتيجة إلى قاعدة بيانات، ملف نصي، أو تُدخلها في فهرس بحث.

```csharp
// Step 5: Output the OCR result
Console.WriteLine("=== OCR Result ===");
Console.WriteLine(ocrResult);
```

## نصائح متقدمة لمزيد من **تحسين دقة OCR**

1. **ضبط دقة الصورة**  
   تعمل محركات OCR بأفضل أداء مع 300 dpi أو أعلى. إذا كانت صورة PNG منخفضة الدقة، قم بزيادة حجمها أولًا باستخدام `ImageProcessor.Resize`.

2. **معالجة مسبقة مخصصة**  
   يسمح Aspose بتركيب عدة مرشحات—أضف `PreprocessFilter.Contrast` إذا كانت الصورة باهتة، أو `PreprocessFilter.Binarize` للمسحات ذات التباين العالي بالأبيض والأسود.

3. **العودة إلى لغة بديلة**  
   إذا كنت تتوقع لغات مختلطة، يمكنك ضبط `Language = Language.AutoDetect`. يكون أبطأ لكنه يمنع الأخطاء عندما يُفرض نموذج لغة غير مناسب.

4. **المعالجة على دفعات**  
   ضع استدعاء OCR داخل حلقة وأعد استخدام نفس كائن `OcrEngine`. هذا يقلل الحمل ويحافظ على الدقة ثابتة عبر الصفحات.

5. **تنظيف ما بعد المعالجة**  
   بعد الاستخراج، نفّذ تعبيرًا نمطيًا بسيطًا لإزالة الأحرف غير المرغوب فيها (`[^\\p{L}\\p{N}\\s]`)—هذا ينظف أي ضوضاء متبقية لم يلتقطها المحرك.

## الخلاصة

لقد استعرضنا مثالًا كاملاً من البداية إلى النهاية يوضح **تحسين دقة OCR** عند قراءة النص من ملفات PNG باستخدام Aspose.OCR. عبر **تحميل صورة لـ OCR**، تكوين `RecognitionSettings`، و**التعرف على النص من التدفق**، يمكنك استخراج **نص من الصورة** بشكل موثوق حتى مع النصوص الصعبة مثل السيرية.

جرّب الشيفرة مع صورك الخاصة، واختبر مرشحات ما قبل المعالجة، وستلاحظ سريعًا كيف يمكن لتعديلات بسيطة أن تُحدث فرقًا كبيرًا في الدقة. هل تحتاج إلى معالجة PDFs أو TIFFs أو مستندات متعددة الصفحات؟ نفس المبادئ تنطبق—فقط قدّم التدفق المناسب إلى `Recognize`.

برمجة سعيدة، ولتكن نتائج OCR دائمًا واضحة كالبلور!
