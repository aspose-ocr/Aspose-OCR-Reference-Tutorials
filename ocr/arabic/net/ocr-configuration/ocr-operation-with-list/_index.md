# كيفية معالجة صور OCR على دفعات باستخدام قائمة في Aspose.OCR لـ .NET

## المقدمة

مرحبًا بكم في دليلنا المتعمق حول **كيفية معالجة OCR على دفعات** لعدة صور باستخدام Aspose.OCR لـ .NET. التعرف الضوئي على الأحرف (OCR) يحول المستندات الورقية الممسوحة ضوئيًا أو ملفات PDF أو ملفات الصور إلى نص قابل للتحرير والبحث. في هذا الدليل ستتعلم كيفية **استخراج النص من الصور**، قراءة نص JPEG، ومعالجة عدة ملفات في استدعاء واحد — وهو مثالي للسيناريوهات التي تحتاج فيها إلى **مسح المستند إلى نص** بسرعة وموثوقية.

## إجابات سريعة
- **ماذا يفعل “OCR للصور المتعددة”؟** يتيح لك التعرف على النص من قائمة ملفات الصور في استدعاء API واحد.  
- **ما الصيغ المدعومة؟** JPEG، PNG، BMP، TIFF، GIF والعديد غيرها.  
- **هل أحتاج إلى ترخيص؟** يلزم ترخيص مؤقت للإنتاج؛ نسخة تجريبية مجانية تكفي للتقييم.  
- **هل يمكنني تخصيص عملية التعرف؟** نعم — استخدم `RecognitionSettings` لضبط اللغة، الدقة، ومعالجة ما قبل التعرف.  
- **كم عدد الصور التي يمكنني معالجتها في آن واحد؟** عمليًا أي عدد؛ الـ API يبث كل ملف، لذا يبقى استهلاك الذاكرة منخفضًا.

## ما هو OCR على دفعات ولماذا هو مهم؟

**OCR على دفعات** (أو “كيفية معالجة OCR على دفعات”) هو القدرة على تمرير مجموعة من مسارات الصور إلى Aspose.OCR والحصول على النص المعترف به لكل صورة في عملية واحدة. يقلل هذا النهج من عدد طلبات الشبكة، ويوفر وقت التطوير، ويسهل دمج OCR في خطوط معالجة المستندات الآلية مثل معالجة الفواتير، الأرشفة، أو أتمتة إدخال البيانات.

## لماذا نستخدم Aspose.OCR لمعالجة الصور على دفعات؟

- **دقة عالية** في المسحات الضوضائية وملفات JPEG منخفضة الدقة.  
- **اكتشاف لغة مدمج** للمستندات متعددة اللغات.  
- **دعم كامل لـ .NET** – يعمل مع .NET Framework، .NET Core، و .NET 5/6+.  
- **بدون تبعيات خارجية** — المكتبة تتعامل داخليًا مع تحميل الصور، المعالجة المسبقة، واستخراج النص.  
- خيارات **معالجة ما قبل OCR للصور** تتيح لك تحسين النتائج للصور ذات الجودة الضعيفة.

## المتطلبات المسبقة

قبل الغوص في الكود، تأكد من توفر المتطلبات التالية:

1. مكتبة Aspose.OCR لـ .NET: تأكد من تثبيت مكتبة Aspose.OCR. يمكنك تنزيلها من [صفحة تنزيل Aspose.OCR لـ .NET](https://releases.aspose.com/ocr/net/).

2. دليل المستندات: أنشئ دليلًا حيث تُخزن مستنداتك وصورك للتعرف باستخدام OCR.

الآن بعد أن لديك الأساسيات، لنبدأ الدليل خطوة بخطوة.

## استيراد المساحات الاسمية

في مشروع C# الخاص بك، أضف المساحات الاسمية الضرورية لاستخدام Aspose.OCR لـ .NET:

```csharp
using System;
using System.Collections.Generic;
using System.Drawing;
using System.IO;
using Aspose.OCR;
```

## دليل خطوة بخطوة

### الخطوة 1: إعداد دليل المستندات الخاص بك

ابدأ بتهيئة المسار إلى دليل المستندات وإنشاء كائن `AsposeOcr`:

```csharp
// The path to the documents directory.
string dataDir = "Your Document Directory";

// Initialize an instance of AsposeOcr
AsposeOcr api = new AsposeOcr();
```

> **نصيحة احترافية:** احفظ ملفات الصور في مجلد فرعي (مثال: `dataDir/ocr`) للحفاظ على تنظيم المشروع.

### الخطوة 2: تحديد مسارات الصور

عرّف قائمة ملفات الصور التي تريد معالجتها. يمكنك خلط JPEG، PNG، BMP، أو أي صيغة مدعومة:

```csharp
List<string> imagePaths = new List<string>
{
    dataDir + "0001460985.Jpeg",
    dataDir + "sample.png"
};
```

> **لماذا هذا مهم:** توفير `List<string>` يتيح لك **معالجة OCR على دفعات** دون الحاجة لكتابة حلقة يدوية — الـ API يقوم بالعمل الشاق.

### الخطوة 3: تنفيذ التعرف على الصور باستخدام OCR

استدعِ `RecognizeMultipleImages` مع `RecognitionSettings` اختياريًا. هنا يمكنك تطبيق **معالجة ما قبل OCR للصور** مثل تصحيح الميل أو تقليل الضوضاء:

```csharp
RecognitionResult[] result = api.RecognizeMultipleImages(imagePaths, new RecognitionSettings
{
   //default or custom settings
});
```

> **كيفية استخراج النص بإعدادات مخصصة:** إذا كنت بحاجة إلى لغة معينة أو DPI أعلى، اضبط `RecognitionSettings.Language` و `RecognitionSettings.Dpi`.

### الخطوة 4: عرض نتائج التعرف

تجول في النتائج واطبع النص المعترف به لكل صورة:

```csharp
for (int i = 0; i < result.Length; i++)
{
	 Console.WriteLine($"Image: {i}\n Result:\n {result[i].RecognitionText}");
}
```

يجب الآن أن ترى النص المستخرج لكل ملف يُطبع على وحدة التحكم، مما يوضح كيفية **استخراج النص من الصور** بالجملة.
