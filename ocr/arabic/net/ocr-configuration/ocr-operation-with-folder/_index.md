# استخراج النص من الصور باستخدام عملية OCR على المجلدات

## المقدمة

مرحبًا بك في عالم التعرف الضوئي على الأحرف (OCR) مع **Aspose.OCR for .NET**! إذا كنت بحاجة إلى **استخراج النص من الصور** بكميات كبيرة—مثل مجلد كامل من المستندات الممسوحة ضوئيًا—ستقودك هذه الدورة التعليمية عبر حل عملي وواقعي. سنغطي كل شيء من إعداد المشروع إلى طباعة النص المُتعرف عليه، حتى تتمكن من دمج OCR القائم على المجلدات بسرعة في تطبيقات C# الخاصة بك. في النهاية، سترى أيضًا كيف يتيح لك هذا النهج **تحويل الصور إلى نص**، **استخراج النص من المستندات الممسوحة**، و**قراءة نص الصورة في C#** ببضع أسطر من الشيفرة فقط.

## إجابات سريعة
- **ما الذي تُعلّمه هذه الدورة؟** كيفية استخراج النص من الصور المخزنة في مجلد باستخدام Aspose.OCR.  
- **ما اللغة والمنصة؟** C# مع .NET (Framework أو .NET Core).  
- **ما المتطلب الأساسي؟** مكتبة Aspose.OCR for .NET (رابط التحميل أدناه).  
- **كم عدد أسطر الشيفرة؟** سبعة كتل شيفرة مختصرة فقط.  
- **هل يمكنني تحويل الصور إلى نص؟** نعم—هذا المثال يوضح ذلك بالضبط.

## ما هو “استخراج النص من الصور”؟
استخراج النص من الصور يعني استخدام تقنية OCR لقراءة الأحرف المدمجة في الصور أو ملفات PDF أو المستندات الممسوحة وتحويلها إلى سلاسل نصية قابلة للتحرير والبحث. توفر Aspose.OCR محركًا قويًا يدعم العديد من صيغ الصور واللغات.

## لماذا نستخدم Aspose.OCR لـ OCR القائم على المجلدات؟
- **دقة عالية** مع اكتشاف اللغة المدمج.  
- **معالجة دفعة** عبر `RecognizeMultipleImages`، مثالية للمجلدات.  
- **واجهة برمجة تطبيقات بسيطة** تتكامل بسهولة مع مشاريع C#.  
- **قابلة للتوسع** – تعمل على كل من بيئات سطح المكتب والخوادم.

## حالات الاستخدام الشائعة
- رقمنة مكتبة من الفواتير أو الإيصالات الممسوحة.  
- تحويل ملفات PNG/JPEG المؤرشفة إلى نص قابل للبحث للفهرسة.  
- أتمتة إدخال البيانات بقراءة النص من صور ملصقات المنتجات.  
- بناء ميزة بحث في المستندات تحتاج إلى **استخراج النص من المستندات الممسوحة** في الوقت الفعلي.

## المتطلبات المسبقة

- إتقان أساسي للغة C# وتطوير .NET.  
- Visual Studio (أي نسخة حديثة).  
- مكتبة **Aspose.OCR for .NET** – حمّلها من [هنا](https://releases.aspose.com/ocr/net/).  
- فهم لمفاهيم OCR (اختياري لكنه مفيد).

## استيراد المساحات الاسمية

أضف توجيهات `using` المطلوبة في أعلى ملف C# حتى يعرف المترجم أين يجد فئات OCR.

```csharp
using System;
using System.Collections.Generic;
using System.Drawing;
using System.IO;
using Aspose.OCR;
```

## دليل خطوة بخطوة

### الخطوة 1: تعيين دليل المستندات
عرّف المجلد الذي يحتوي على الصور التي تريد معالجتها.

```csharp
// ExStart:1   
// The path to the documents directory.
string dataDir = "Your Document Directory";
```

> **نصيحة احترافية:** استخدم مسارًا مطلقًا أو `Path.Combine` لتجنب مشاكل فواصل المسار على أنظمة تشغيل مختلفة.

### الخطوة 2: تهيئة Aspose.OCR
أنشئ مثيلًا لمحرك OCR.

```csharp
// Initialize an instance of AsposeOcr
AsposeOcr api = new AsposeOcr();
```

### الخطوة 3: تحديد مسار الصورة
وجه الواجهة البرمجية إلى المجلد الفرعي المحدد الذي يحتوي على ملفات الصور.

```csharp
// Image Path
string fullPath = dataDir + "OCR";
```

> **لماذا هذا مهم:** طريقة `RecognizeMultipleImages` تتوقع مسار مجلد، وليس ملفًا واحدًا.

### الخطوة 4: التعرف على الصور
نفّذ OCR على كل صورة داخل المجلد. يمكنك تخصيص `RecognitionSettings` إذا كنت بحاجة إلى تلميحات لغة أو معالجة مسبقة معينة.

```csharp
// Recognize image           
RecognitionResult[] result = api.RecognizeMultipleImages(fullPath, new RecognitionSettings
{
    //default or custom
});
```

### الخطوة 5: طباعة النتائج
تجول عبر مصفوفة `RecognitionResult` المُرجعة واطبع النص المستخرج.

```csharp
// Print result
for (int i = 0; i < result.Length; i++)
{
    Console.WriteLine($"Image: {i}\n Result:\n {result[i].RecognitionText}");
}
```

> **مشكلة شائعة:** نسيان فحص `result.Length` قد يسبب استثناء `IndexOutOfRangeException` عندما يكون المجلد فارغًا. تحقق دائمًا من محتوى المجلد أولًا.

### الخطوة 6: رسالة الانتهاء
أظهر رسالة تؤكد نجاح التنفيذ.

```csharp
// ExEnd:1
Console.WriteLine("OCROperationWithFolder executed successfully");
```

## نصائح وأفضل الممارسات

- **حجم الدفعة:** إذا كنت تعالج آلاف الملفات، فكر في تقسيم المجلد إلى دفعات أصغر للحفاظ على استهلاك الذاكرة ضمن نطاق متوقع.  
- **تلميحات اللغة:** توفير رمز اللغة الصحيح في `RecognitionSettings` يحسّن الدقة بشكل كبير، خاصةً للخطوط غير اللاتينية.  
- **المعالجة غير المتزامنة:** غلف استدعاء OCR داخل `Task.Run` أو استخدم async/await للحفاظ على استجابة واجهة المستخدم.  
- **التحقق من الملفات:** قبل استدعاء `RecognizeMultipleImages`، صَفِّ الدليل للامتدادات المدعومة (`.png`, `.jpg`, `.jpeg`, `.tif`, `.tiff`).  

## المشكلات الشائعة والحلول

| المشكلة | السبب | الحل |
|--------|-------|------|
| لا يتم إرجاع أي ناتج | مسار المجلد غير صحيح أو فارغ | تحقق من أن `fullPath` يشير إلى الدليل الصحيح ويحتوي على صيغ صور مدعومة (PNG, JPEG, TIFF). |
| أحرف مشوشة | إعدادات اللغة غير صحيحة | مرّر `RecognitionSettings` مكوَّنًا مع `Language` المحدد برمز ISO المناسب. |
| بطء الأداء مع عدد كبير من الصور | المعالجة تسلسلية على خيط الواجهة | نفّذ OCR على خيط خلفي أو استخدم نمط async للحفاظ على استجابة الواجهة. |
