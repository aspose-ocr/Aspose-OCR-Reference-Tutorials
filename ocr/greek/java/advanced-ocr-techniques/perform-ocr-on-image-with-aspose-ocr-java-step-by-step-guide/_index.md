# Εκτέλεση OCR σε Εικόνα – Πλήρες Java Tutorial

Έχετε χρειαστεί ποτέ να **εκτελέσετε OCR σε εικόνα** αλλά δεν ήξερες από πού να ξεκινήσεις; Δεν είστε μόνοι· πολλοί προγραμματιστές αντιμετωπίζουν το ίδιο εμπόδιο όταν προσπαθούν για πρώτη φορά να εξάγουν κείμενο από δεδομένα εικόνας. Τα καλά νέα είναι ότι με το Aspose OCR for Java μπορείτε να λάβετε αξιόπιστα αποτελέσματα με λίγες μόνο γραμμές κώδικα—και μπορείτε ακόμη να βελτιώσετε την ακρίβεια με ένα προσαρμοσμένο λεξικό και ορθογραφικό έλεγχο.

Σε αυτόν τον οδηγό θα περάσουμε από όλα όσα χρειάζεται να γνωρίζετε: από τη φόρτωση της εικόνας για OCR, στην ενεργοποίηση της διόρθωσης ορθογραφίας, και τέλος στην εκτύπωση του καθαρισμένου κειμένου. Στο τέλος θα μπορείτε να **εκτελέσετε OCR σε εικόνα** σε δικά σας έργα Java, και θα δείτε επίσης πώς να **εξάγετε κείμενο από εικόνα** με τρόπο που λειτουργεί ακόμη και με θορυβώδεις σαρώσεις.

## Τι Θα Χρειαστείτε

Πριν βουτήξουμε, βεβαιωθείτε ότι έχετε τα παρακάτω:

- **Java Development Kit (JDK) 8+** – ο κώδικας χρησιμοποιεί τυπικά Java APIs.
- **Aspose OCR for Java** βιβλιοθήκη (μπορείτε να κατεβάσετε το τελευταίο JAR από τον ιστότοπο της Aspose ή το Maven Central).
- Ένα αρχείο εικόνας (π.χ., `invoice.png`) που θέλετε να επεξεργαστείτε.
- (Προαιρετικά) Ένα αρχείο κειμένου `custom_dict.txt` που περιέχει ειδικές λέξεις του τομέα, μία ανά γραμμή.

Αυτό είναι όλο—χωρίς βαριά πλαίσια, χωρίς εξωτερικές υπηρεσίες. Απλώς καθαρή Java και το Aspose OCR JAR.

## Βήμα 1: Ρύθμιση του Έργου και Εισαγωγή Εξαρτήσεων

Πρώτα, δημιουργήστε ένα νέο Maven (ή Gradle) έργο και προσθέστε την εξάρτηση Aspose OCR. Αν χρησιμοποιείτε Maven, το `pom.xml` σας πρέπει να περιέχει:

```xml
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-ocr</artifactId>
    <version>23.10</version> <!-- Check for the latest version -->
</dependency>
```

Αν προτιμάτε να κατεβάσετε το JAR χειροκίνητα, τοποθετήστε το στον φάκελο `libs` και προσθέστε το στο classpath.

> **Συμβουλή:** Πάντα ελέγχετε τον αριθμό έκδοσης τη στιγμή της συγγραφής· νεότερες εκδόσεις μπορεί να προσθέσουν επιπλέον δυνατότητες όπως πακέτα γλωσσών.

## Βήμα 2: Φόρτωση της Εικόνας για OCR

Το πρώτο πραγματικό βήμα κώδικα είναι να κατευθύνετε τη μηχανή OCR στην εικόνα που θέλετε να αναλύσετε. Το Aspose OCR χρησιμοποιεί έναν wrapper `ImageStream`, ο οποίος μπορεί να διαβάσει από αρχείο, πίνακα byte ή ακόμη και URL.

```java

import java.nio.file.Files;
import java.nio.file.Paths;

public class SpellCorrectDemo {
    public static void main(String[] args) throws Exception {

        // Step 2: Load the image you wish to process
        OcrEngine ocrEngine = new OcrEngine();
        ocrEngine.setImage(ImageStream.fromFile("YOUR_DIRECTORY/invoice.png"));
```

Παρατηρήστε πώς **φορτώνουμε εικόνα για OCR** με μία μόνο κλήση μεθόδου—χωρίς περίπλοκες χειρονομίες `BufferedImage`. Αν η εικόνα σας βρίσκεται στους πόρους, απλώς αντικαταστήστε τη διαδρομή με `getClass().getResourceAsStream(...)`.

## Βήμα 3: Ενεργοποίηση Διόρθωσης Ορθογραφίας (Προαιρετικό αλλά Ισχυρό)

Το Aspose OCR μπορεί αυτόματα να διορθώσει κοινά σφάλματα OCR (π.χ., “l” vs “1”). Η ενεργοποίηση αυτής της δυνατότητας είναι τόσο απλή όσο το άναμμα μιας σημαίας:

```java
        // Step 3: Turn on spell‑checking to improve result quality
        ocrEngine.getEngineOptions().setSpellCorrectionEnabled(true);
```

Γιατί να το κάνετε; Φανταστείτε ότι σαρώσατε ένα τυπωμένο τιμολόγιο όπου η γραμματοσειρά είναι μικρή και ο σαρωτής προσθέτει σπασμένα σημεία. Η διόρθωση ορθογραφίας συχνά θα μετατρέψει το “Inv0ice” πίσω σε “Invoice” χωρίς επιπλέον εργασία από εσάς.

## Βήμα 4: Παροχή Προσαρμοσμένου Λεξικού (Προσαρμόστε τη Μηχανή)

Αν εργάζεστε με ορολογία ειδικού κλάδου—π.χ. ιατρικούς κωδικούς, νομική ορολογία ή SKU προϊόντων—ένα προσαρμοσμένο λεξικό μπορεί να βελτιώσει δραστικά την ακρίβεια. Το λεξικό είναι απλώς ένα αρχείο κειμένου με μία λέξη ανά γραμμή.

```java
        // Step 4: Load a custom dictionary to boost recognition of domain terms
        ocrEngine.getEngineOptions().setCustomDictionary(
                Files.readAllLines(Paths.get("YOUR_DIRECTORY/custom_dict.txt")));
```

Βεβαιωθείτε ότι το αρχείο χρησιμοποιεί κωδικοποίηση UTF‑8· διαφορετικά μπορεί να εμφανιστούν ακατανόητοι χαρακτήρες για μη‑ASCII λέξεις.

## Βήμα 5: Εκτέλεση της Διαδικασίας OCR

Τώρα αφήνουμε τελικά τη μηχανή να κάνει τη μαγεία της. Η κλήση `process()` επιστρέφει ένα αντικείμενο `OcrResult` που περιέχει το αναγνωρισμένο κείμενο, βαθμολογίες εμπιστοσύνης, και ακόμη τη διάταξη αν τη χρειάζεστε αργότερα.

```java
        // Step 5: Execute OCR and capture the result
        OcrResult ocrResult = ocrEngine.process();
```

Αν ποτέ αναρωτηθείτε αν η μηχανή πέταξε εξαίρεση, τυλίξτε αυτήν την κλήση σε μπλοκ try‑catch και ελέγξτε το `ocrResult.getErrorMessage()` για λεπτομέρειες.

## Βήμα 6: Έξοδος του Αναγνωρισμένου (και Διορθωμένου) Κειμένου

Το τελευταίο βήμα είναι η εμφάνιση—ή αποθήκευση—της εξαγόμενης συμβολοσειράς. Για μια γρήγορη επίδειξη θα την εκτυπώσουμε απλώς στην κονσόλα:

```java
        // Step 6: Print the corrected text to the console
        System.out.println(ocrResult.getText());
    }
}
```

Η εκτέλεση του προγράμματος θα πρέπει να εκτυπώσει κάτι όπως:

```
Invoice Number: 12345
Date: 2023‑07‑15
Total Amount: $1,250.00
```

Αν δείτε ακαθόριστους χαρακτήρες, ελέγξτε ξανά το προσαρμοσμένο λεξικό και σκεφτείτε να βελτιώσετε την ποιότητα της εικόνας (π.χ., αυξήστε την αντίθεση πριν τη δώσετε στη μηχανή).

### Πλήρες Παράδειγμα Λειτουργίας

Συνδυάζοντας όλα τα παραπάνω, ιδού η πλήρης, έτοιμη‑για‑εκτέλεση κλάση:

```java
import com.aspose.ocr.*;
import java.nio.file.Files;
import java.nio.file.Paths;

public class SpellCorrectDemo {
    public static void main(String[] args) throws Exception {

        // Step 1: Create an OCR engine instance
        OcrEngine ocrEngine = new OcrEngine();

        // Step 2: Load the image you wish to process
        ocrEngine.setImage(ImageStream.fromFile("YOUR_DIRECTORY/invoice.png"));

        // Step 3: Enable spell‑checking for the OCR result
        ocrEngine.getEngineOptions().setSpellCorrectionEnabled(true);

        // Step 4: Provide a custom dictionary (one word per line)
        ocrEngine.getEngineOptions().setCustomDictionary(
                Files.readAllLines(Paths.get("YOUR_DIRECTORY/custom_dict.txt")));

        // Step 5: Run the OCR process
        OcrResult ocrResult = ocrEngine.process();

        // Step 6: Output the recognized (and corrected) text
        System.out.println(ocrResult.getText());
    }
}
```

Αποθηκεύστε αυτό το αρχείο ως `SpellCorrectDemo.java`, προσαρμόστε τις διαδρομές, μεταγλωττίστε με `javac` και τρέξτε με `java SpellCorrectDemo`. Τώρα θα μπορείτε **να εκτελέσετε OCR σε εικόνα** και να δείτε το εξαγόμενο κείμενο στην οθόνη.

## Συχνές Ερωτήσεις & Ακραίες Περιπτώσεις

### Τι γίνεται αν η εικόνα είναι σε διαφορετική μορφή (PDF, TIFF, κ.λπ.);

Το Aspose OCR μπορεί να χειριστεί πολλές μορφές raster από προεπιλογή. Για PDFs θα χρειαστεί να εξάγετε κάθε σελίδα ως εικόνα πρώτα—το Aspose PDF for Java το κάνει άψογα. Η ίδια κλήση `setImage` λειτουργεί μόλις έχετε ένα `BufferedImage` ή ροή byte.

### Πώς διαχειρίζομαι έγγραφα πολλαπλών σελίδων;

Κάντε βρόχο πάνω από κάθε εικόνα σελίδας, εκτελέστε τα βήματα OCR και συνενώστε τα αποτελέσματα. Θυμηθείτε να επαναρυθμίσετε ή να δημιουργήσετε νέο `OcrEngine` για κάθε σελίδα αν θέλετε ανεξάρτητα συμφραζόμενα ορθογραφικού ελέγχου.

### Μπορώ να περιορίσω τη γλώσσα ή το σύνολο χαρακτήρων;

Ναι. Χρησιμοποιήστε `ocrEngine.getEngineOptions().setLanguage(Language.English);` (ή οποιαδήποτε άλλη υποστηριζόμενη γλώσσα) για να μειώσετε ψευδείς θετικές και να επιταχύνετε την επεξεργασία.

### Τι γίνεται με την απόδοση σε μεγάλα batch;

Το Aspose OCR είναι thread‑safe για λειτουργίες μόνο‑ανάγνωσης. Μπορείτε να δημιουργήσετε μια ομάδα νημάτων και να επεξεργαστείτε εικόνες ταυτόχρονα—απλώς αποφύγετε την κοινή χρήση του ίδιου αντικειμένου `OcrEngine` μεταξύ νημάτων.

## Συμβουλές για Καλύτερη Ακρίβεια

- **Προεπεξεργασία της εικόνας**: αυξήστε την αντίθεση, αφαιρέστε θόρυβο φόντου ή μετατρέψτε σε κλίμακα του γκρι.
- **Χρησιμοποιήστε σάρωση υψηλής ανάλυσης** (300 dpi ή περισσότερο). Χαμηλότερες αναλύσεις συχνά παράγουν ακατάλληλους χαρακτήρες.
- **Κρατήστε το προσαρμοσμένο λεξικό συμπαγές**: πάρα πολλές άσχετες λέξεις μπορούν να μπερδέψουν τον ορθογραφικό ελεγκτή.
- **Επικυρώστε την έξοδο**: αν γνωρίζετε την αναμενόμενη μορφή (π.χ., ημερομηνίες, αριθμούς), τρέξτε ένα regex post‑process για να εντοπίσετε ανωμαλίες.
