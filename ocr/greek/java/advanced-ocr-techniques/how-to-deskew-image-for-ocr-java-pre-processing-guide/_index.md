# Πώς να Διορθώσετε την Κλίση Εικόνας για OCR – Οδηγός Προεπεξεργασίας σε Java

Έχετε αναρωτηθεί ποτέ **πώς να διορθώσετε την κλίση εικόνας** που προέρχεται από σαρωτή με παράξενη γωνία; Δεν είστε μόνοι—πολλοί προγραμματιστές αντιμετωπίζουν αυτό το πρόβλημα όταν προσπαθούν να εξάγουν κείμενο από έγγραφα με πολλές εικόνες. Τα καλά νέα; Με λίγες γραμμές Java και Aspose OCR μπορείτε να ευθυγραμμίσετε, να αφαιρέσετε θόρυβο και να εξάγετε καθαρό κείμενο χωρίς κόπο.

Σε αυτό το tutorial θα περάσουμε από όλη τη ροή εργασίας: φόρτωση ενός θορυβώδους, περιστραμμένου σκαναρίσματος, εφαρμογή φίλτρου διόρθωσης κλίσης, αφαίρεση οπτικού «σκότους», και τέλος **εξαγωγή κειμένου από εικόνα**. Στο τέλος θα ξέρετε πώς να **προεπεξεργαστείτε εικόνα για OCR**, **καθαρίσετε σαρωμένη εικόνα** και **βελτιώσετε την ακρίβεια του OCR** για οποιοδήποτε έργο απαιτεί αξιόπιστη εξαγωγή κειμένου.

## Τι Θα Χρειαστείτε

- Java 17 (ή οποιοδήποτε πρόσφατο JDK) – ο κώδικας χρησιμοποιεί τις τυπικές δυνατότητες της γλώσσας.
- Βιβλιοθήκη Aspose OCR for Java (η δωρεάν δοκιμή λειτουργεί άψογα για πειραματισμό).
- Ένα δείγμα εικόνας που είναι ταυτόχρονα θορυβώδης και περιστραμμένη (π.χ. `noisy-rotated.png`).
- Το αγαπημένο σας IDE (IntelliJ IDEA, Eclipse, VS Code…) – οτιδήποτε μπορεί να μεταγλωττίσει Java.

Δεν απαιτούνται επιπλέον frameworks, κανένα Maven/Gradle μαγικό; οι μόνοι import δηλώσεις είναι αυτές που φαίνονται παρακάτω.

## Πώς να Διορθώσετε την Κλίση Εικόνας με Aspose OCR

Το πρώτο βήμα είναι η κλίση της εικόνας. Αν οι γραμμές κειμένου δεν είναι οριζόντιες, η μηχανή OCR θα διαβάσει λανθασμένα χαρακτήρες. Το `DeskewFilter` κάνει το βαριά δουλειά.

```java
import com.aspose.ocr.OcrEngine;
import com.aspose.ocr.Image;
import com.aspose.ocr.preprocess.DeskewFilter;
import com.aspose.ocr.preprocess.DenoiseFilter;

public class PreprocessDemo {
    public static void main(String[] args) throws Exception {
        // Step 1: Create the OCR engine
        OcrEngine ocrEngine = new OcrEngine();

        // Step 2: Load the raw scanned image
        Image scannedImage = Image.load("YOUR_DIRECTORY/noisy-rotated.png");

        // Step 3: Correct the image orientation (deskew)
        DeskewFilter deskewFilter = new DeskewFilter();
        scannedImage = deskewFilter.apply(scannedImage);

        // Step 4: Reduce visual noise (denoise)
        DenoiseFilter denoiseFilter = new DenoiseFilter();
        scannedImage = denoiseFilter.apply(scannedImage);

        // Step 5: Perform OCR on the cleaned image
        String recognizedText = ocrEngine.recognize(scannedImage);

        // Step 6: Output the extracted text
        System.out.println(recognizedText);
    }
}
```

> **Γιατί είναι σημαντικό:** Το `DeskewFilter` αναλύει τη γεωμετρία της εικόνας, εκτιμά τη γωνία περιστροφής και περιστρέφει το bitmap πίσω σε οριζόντια θέση. Χωρίς αυτό το βήμα, οι περισσότερες μηχανές OCR αντιμετωπίζουν τις κεκλιμένες γράμματα ως εντελώς διαφορετικά σύμβολα, μειώνοντας τις προσπάθειές σας για **βελτίωση της ακρίβειας του OCR**.

> **Συμβουλή:** Αν εργάζεστε με έγγραφα που μπορεί να είναι ανάποδα, ενεργοποιήστε τη σημαία `setAutoRotate` στο `DeskewFilter` (διαθέσιμη σε νεότερες εκδόσεις Aspose). Αυτό αυτόματα περιστρέφει 180° όταν χρειάζεται.

## Προεπεξεργασία Εικόνας για OCR – Αφαίρεση Θορύβου και Καθαρισμός

Μόλις η εικόνα είναι ευθεία, το επόμενο εμπόδιο είναι ο οπτικός θόρυβος—εκκενώσεις, τεχνητά σήματα συμπίεσης ή αχνά μοτίβα φόντου που μπερδεύουν τη μηχανή OCR. Το `DenoiseFilter` εφαρμόζει έναν ήπιο αλγόριθμο εξομάλυνσης που διατηρεί τις άκρες (τα γράμματα) ενώ αφαιρεί το «κόκκο».

```java
// Step 4 (continued): Reduce visual noise (denoise)
DenoiseFilter denoiseFilter = new DenoiseFilter();
scannedImage = denoiseFilter.apply(scannedImage);
```

### Πότε να Ρυθμίσετε τις Ρυθμίσεις Αφαίρεσης Θορύβου

- **Πολύ σκοτεινά σκαναρίσματα:** Αυξήστε τη δύναμη του φίλτρου (`denoiseFilter.setStrength(2)`) για να αφαιρέσετε τις σκιές του φόντου.
- **Γραμματοσειρές μικρού μεγέθους:** Μειώστε τη δύναμη ώστε να μην θολώσετε τα λεπτά serif.
- **Έγχρωμα έγγραφα:** Μετατρέψτε πρώτα σε κλίμακα του γκρι (`scannedImage = scannedImage.toGrayscale();`)—η μηχανή OCR λειτουργεί καλύτερα σε εικόνες μονόκαναλ.

Αυτές οι προσαρμογές είναι μέρος των βέλτιστων πρακτικών για **καθαρή σαρωμένη εικόνα**· ένα καθαρό bitmap μεταφράζεται άμεσα σε υψηλότερες βαθμολογίες εμπιστοσύνης από τη μηχανή OCR.

## Εξαγωγή Κειμένου από Εικόνα – Εκτέλεση της Μηχανής OCR

Τώρα που η εικόνα είναι ευθεία και ήσυχη, ήρθε η ώρα να **εξάγετε κείμενο από εικόνα**. Η κλάση `OcrEngine` διαχειρίζεται τα πάντα στο παρασκήνιο: τμηματοποίηση, ταξινόμηση χαρακτήρων και μοντελοποίηση γλώσσας.

```java
// Step 5: Perform OCR on the cleaned image
String recognizedText = ocrEngine.recognize(scannedImage);
System.out.println(recognizedText);
```

#### Αναμενόμενο Αποτέλεσμα

Αν το αρχείο προέλευσης περιέχει τη γραμμή “**Invoice # 12345**”, η κονσόλα θα πρέπει να εκτυπώσει κάτι σαν:

```
Invoice # 12345
Date: 2026-03-18
Total: $1,250.00
```

Αν το αποτέλεσμα φαίνεται χαοτικό, ελέγξτε ξανά τα προηγούμενα βήματα—ιδιαίτερα τη διόρθωση κλίσης. Ακόμη και μια κλίση 1° μπορεί να καταστρέψει αριθμούς και σύμβολα.

## Συνηθισμένα Προβλήματα & Συμβουλές για **Βελτίωση της Ακρίβειας του OCR**

| Πρόβλημα | Γιατί επηρεάζει την ακρίβεια | Γρήγορη λύση |
|----------|----------------------------|--------------|
| **Υπολειπόμενη κλίση** | Το OCR αναμένει οριζόντιες βάσεις. | Επαληθεύστε με `deskewFilter.getAngle()` και καταγράψτε την τιμή. |
| **Υπερβολική αφαίρεση θορύβου** | Θολώνει λεπτές γραμμές, μετατρέποντας το “i” σε “l”. | Χρησιμοποιήστε `setStrength(0.5)` για ευαίσθητες γραμματοσειρές. |
| **Λάθος μορφή εικόνας** | Η συμπίεση JPEG προσθέτει τεχνητά σήματα. | Προτιμήστε PNG ή TIFF για αποθήκευση χωρίς απώλειες. |
| **Λανθασμένη γλώσσα** | Η μηχανή προεπιλογή είναι τα Αγγλικά· άλλα αλφάβητα απαιτούν ρητή ρύθμιση. | `ocrEngine.setLanguage(OcrEngine.Language.Spanish);` |
| **Χαμηλό DPI (≤150)** | Δεν υπάρχουν επαρκή δεδομένα pixel για αξιόπιστη τμηματοποίηση. | Επαναδειγματοληψία σε 300 DPI πριν την επεξεργασία (`scannedImage = scannedImage.resample(300);`). |

### Bonus: Επεξεργασία σε Παρτίδες

Αν έχετε φάκελο με σκαναρίσματα, τυλίξτε το demo σε βρόχο:

```java
File folder = new File("YOUR_DIRECTORY");
for (File file : folder.listFiles((dir, name) -> name.endsWith(".png"))) {
    Image img = Image.load(file.getAbsolutePath());
    img = new DeskewFilter().apply(img);
    img = new DenoiseFilter().apply(img);
    String text = ocrEngine.recognize(img);
    System.out.println("=== " + file.getName() + " ===");
    System.out.println(text);
}
```

Αυτό το μοτίβο σας επιτρέπει να **προεπεξεργαστείτε εικόνα για OCR** σε κλίμακα, διατηρώντας τον κώδικα οργανωμένο και την απόδοση προβλέψιμη.

## Περίληψη & Επόμενα Βήματα

Καλύψαμε **πώς να διορθώσετε την κλίση εικόνας**, **προεπεξεργασία εικόνας για OCR**, και τέλος **εξαγωγή κειμένου από εικόνα** χρησιμοποιώντας το Aspose OCR Java. Με την ευθυγράμμιση του σκαναρίσματος, τον καθαρισμό του θορύβου και την παροχή ενός άψογου bitmap στη μηχανή, θα δείτε αισθητή **βελτίωση της ακρίβειας του OCR** σε όλα τα έργα.

Τι ακολουθεί; Σκεφτείτε τις εξής επεκτάσεις:

- **Ανίχνευση γλώσσας** – αλλάξτε το `ocrEngine.setLanguage` βάσει του ανιχνευμένου αλφαβήτου.
- **Έξοδος PDF** – τροφοδοτήστε το αναγνωρισμένο κείμενο σε γεννήτρια PDF για έγγραφα με δυνατότητα αναζήτησης.
- **Μετα-επεξεργασία μηχανικής μάθησης** – εκτελέστε ορθογραφικό έλεγχο ή προσαρμοσμένα λεξικά στο αποτέλεσμα του OCR.

Δοκιμάστε αυτές τις ιδέες, πειραματιστείτε με διαφορετικές δυνάμεις φίλτρου, και σύντομα θα έχετε μια αξιόπιστη αλυσίδα επεξεργασίας για οποιοδήποτε έργο ψηφιοποίησης εγγράφων.
