# Αφαίρεση Υδατογραφήματος Αξιολόγησης – Πλήρης Οδηγός Άδειας Java

Έχετε αναρωτηθεί ποτέ πώς να **αφαιρέσετε το υδατογράφημα αξιολόγησης** από την έξοδο του Aspose OCR χωρίς να παλεύετε με μια ατελείωτη οθόνη εκκίνησης; Δεν είστε ο μόνος. Σε πολλά έργα Java, το πρώτο πράγμα που εμφανίζεται μετά από μια δοκιμαστική εκτέλεση είναι αυτό το επίμονο υδατογράφημα, και μπορεί να κάνει μια παρουσίαση να φαίνεται άσχημα γρήγορα.  

Τα καλά νέα; Η λύση είναι τόσο απλή όσο η φόρτωση μιας έγκυρης άδειας από τον διακομιστή σας και η επιβεβαίωση ότι είναι ενεργή. Σε αυτόν τον οδηγό θα δείτε **πώς να φορτώσετε άδεια**, **πώς να χρησιμοποιήσετε την άδεια Aspose** σωστά, και ακόμη **πώς να ελέγξετε την εγκυρότητα της άδειας** ώστε το υδατογράφημα να μην εμφανίζεται ξανά.

> **Συμβουλή:** Εάν έχετε ήδη ένα αρχείο άδειας στο δίσκο, μπορείτε να παραλείψετε το βήμα του διακομιστή, αλλά η φόρτωση από έναν κεντρικό διακομιστή αδειών διατηρεί τις κατασκευές σας καθαρές και τα κλειδιά σας ασφαλή.

---

## Προαπαιτούμενα

* Java 17 (ή οποιοδήποτε πρόσφατο JDK) εγκατεστημένο.
* Maven ή Gradle για διαχείριση εξαρτήσεων.
* Άδεια Aspose OCR for Java (θα λάβετε ένα αρχείο `.lic` από την Aspose).
* Πρόσβαση σε διακομιστή αδειών που μπορεί να σερβίρει το αρχείο `.lic` μέσω HTTPS – εδώ έρχεται σε εφαρμογή η **load license from server**.
* Βασική εξοικείωση με IDE Java (IntelliJ IDEA, Eclipse κ.λπ.).

Εάν λείπει κάποιο από αυτά, αποκτήστε το τώρα· το υπόλοιπο του οδηγού υποθέτει ότι είναι διαθέσιμα.

---

## Πώς Φαίνεται η Τελική Λύση

Παρακάτω βρίσκεται το **πλήρες, εκτελέσιμο πρόγραμμα Java** που αφαιρεί το υδατογράφημα αξιολόγησης φορτώνοντας μια άδεια από απομακρυσμένο διακομιστή και εκτυπώνοντας αν η άδεια είναι έγκυρη.

```java
import com.aspose.ocr.*;

public class LicenseDemo {
    public static void main(String[] args) throws Exception {

        // Step 1: Create a License instance – this object will manage the product license
        License license = new License();

        // Step 2: Load the license from your licensing server.
        // Replace the URL and product name with your actual values.
        // This is the part that actually *remove evaluation watermark*.
        license.setLicenseFromServer("https://license.mycompany.com", "Aspose.OCR.Java");

        // Step 3: Verify that the license was applied successfully.
        // If true, no evaluation watermark will appear in OCR results.
        System.out.println("License applied: " + license.isValid());

        // Optional: Use the OCR engine – you’ll see clean output without a watermark.
        OcrEngine engine = new OcrEngine();
        engine.setImage("sample.png");
        engine.process();
        System.out.println("Recognized text: " + engine.getText());
    }
}
```

**Αναμενόμενη έξοδος κονσόλας (όταν η άδεια είναι έγκυρη):**

```
License applied: true
Recognized text: Hello World!
```

Εάν η άδεια δεν μπορεί να ληφθεί ή είναι μη έγκυρη, το `license.isValid()` θα επιστρέψει `false` και η έξοδος του OCR θα περιέχει το υδατογράφημα αξιολόγησης.

## Βήμα‑βήμα Εξήγηση

### Βήμα 1: Προσθήκη Εξάρτησης Aspose OCR

Πρώτα, πείτε στο Maven (ή Gradle) από πού να πάρει τη βιβλιοθήκη Aspose OCR. Σε ένα `pom.xml` αυτό φαίνεται ως εξής:

```xml
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-ocr</artifactId>
    <version>23.9</version> <!-- Use the latest version available -->
</dependency>
```

> **Γιατί είναι σημαντικό:** Χωρίς τη σωστή εξάρτηση οι κλάσεις `License` και `OcrEngine` δεν θα μεταγλωττιστούν, και δεν θα μπορέσετε ποτέ να **αφαιρέσετε το υδατογράφημα αξιολόγησης**.

### Βήμα 2: Αφαίρεση Υδατογραφήματος Αξιολόγησης με Φόρτωση Άδειας

Η καρδιά του οδηγού βρίσκεται εδώ. Δημιουργείτε ένα αντικείμενο `License` και το κατευθύνετε σε ένα απομακρυσμένο endpoint που σερβίρει το αρχείο `.lic`. Αυτή η προσέγγιση είναι πιο ασφαλής από το ενσωμάτωση της άδειας σε έλεγχο πηγής.

```java
License license = new License();
license.setLicenseFromServer("https://license.mycompany.com", "Aspose.OCR.Java");
```

* Το `setLicenseFromServer` επικοινωνεί με το URL, κατεβάζει την άδεια και την καταχωρεί στο runtime της Aspose.
* Το δεύτερο όρισμα είναι το όνομα του προϊόντος όπως έχει καταχωρηθεί στην Aspose· πρέπει να ταιριάζει ακριβώς.

**Κοινά προβλήματα**  
* **Απαιτείται HTTPS:** Η Aspose μπλοκάρει το απλό HTTP για ασφάλεια. Αν δοκιμάσετε `http://` θα έχετε σιωπηλή αποτυχία και το υδατογράφημα θα παραμείνει.
* **Λανθασμένο όνομα προϊόντος:** Η λανθασμένη ορθογραφία του `"Aspose.OCR.Java"` οδηγεί το `license.isValid()` να επιστρέφει `false`.

### Βήμα 3: Έλεγχος Εγκυρότητας Άδειας

Ακόμη και μετά από επιτυχή λήψη, είναι σοφό να επιβεβαιώσετε ότι η άδεια είναι πράγματι έγκυρη. Εδώ η **check license validity** ξεχωρίζει.

```java
boolean valid = license.isValid();
System.out.println("License applied: " + valid);
```

Εάν το `valid` εκτυπώσει `false`, ελέγξτε ξανά το URL του διακομιστή, την αλυσίδα πιστοποιητικών, και ότι το αρχείο άδειας δεν έχει λήξει.

### Βήμα 4: Εκτέλεση OCR Χωρίς Υδατογράφημα

Τώρα που η άδεια είναι ενεργή, οποιαδήποτε λειτουργία OCR θα εκτελείται χωρίς υδατογράφημα.

```java
OcrEngine engine = new OcrEngine();
engine.setImage("sample.png");
engine.process();
System.out.println("Recognized text: " + engine.getText());
```

Μπορείτε να αντικαταστήσετε το `"sample.png"` με οποιαδήποτε εικόνα χρειάζεται να επεξεργαστείτε. Το κύριο συμπέρασμα: μόλις φορτωθεί η άδεια, το Aspose OCR συμπεριφέρεται ακριβώς όπως η επί πληρωμή έκδοση—χωρίς μηνύματα αξιολόγησης, χωρίς κρυφούς περιορισμούς.

### Βήμα 5: (Προαιρετικό) Εναλλακτική Λύση με Τοπικό Αρχείο Άδειας

Εάν ο διακομιστής είναι εκτός λειτουργίας, ίσως θελήσετε να επιστρέψετε σε ένα τοπικό αντίγραφο. Εδώ είναι ένα γρήγορο πρότυπο:

```java
try {
    license.setLicenseFromServer("https://license.mycompany.com", "Aspose.OCR.Java");
} catch (Exception e) {
    // Server unreachable – load from local file instead
    license.setLicense("C:/licenses/Aspose.OCR.Java.lic");
}
```

Αυτή η υβριδική προσέγγιση εξασφαλίζει ότι η εφαρμογή σας δεν θα καταρρεύσει ποτέ λόγω έλλειψης άδειας, και εξακολουθεί να **αφαιρεί το υδατογράφημα αξιολόγησης** υπό κανονικές συνθήκες.

## Συχνές Ερωτήσεις (FAQ)

| Question | Answer |
|----------|--------|
| **Χρειάζεται να επανεκκινήσω το JVM μετά τη φόρτωση της άδειας;** | Όχι. Η άδεια τίθεται σε ισχύ αμέσως για το τρέχον runtime. |
| **Μπορώ να φορτώσω την άδεια περισσότερες από μία φορές;** | Ναι, αλλά δεν είναι απαραίτητο· η πρώτη επιτυχής φόρτωση καταχωρεί το κλειδί παγκοσμίως. |
| **Τι γίνεται αν ο διακομιστής μου χρησιμοποιεί αυτο‑υπογεγραμμένα πιστοποιητικά;** | Είτε εισάγετε το πιστοποιητικό στο trust store του JVM, είτε απενεργοποιείτε την επικύρωση πιστοποιητικών (δεν συνιστάται για παραγωγή). |
| **Είναι το `setLicenseFromServer` thread‑safe;** | Είναι ασφαλές να το καλέσετε μία φορά κατά την εκκίνηση. Εάν το καλέσετε ταυτόχρονα, μπορεί να προκύψουν συνθήκες αγώνα. |
| **Θα επανεμφανιστεί το υδατογράφημα μετά από ανανέωση της άδειας;** | Μόνο αν το νέο αρχείο άδειας δεν ληφθεί σωστά. Πάντα να επαληθεύετε το `license.isValid()` μετά την ανανέωση. |

## Καλές Πρακτικές & Συμβουλές

* **Αποθηκεύστε το URL της άδειας σε αρχείο ρυθμίσεων** (π.χ., `application.properties`) ώστε να μπορείτε να αλλάζετε περιβάλλοντα χωρίς επαναμεταγλώττιση.
* **Καταγράψτε το αποτέλεσμα του `license.isValid()`** κατά την εκκίνηση· μια απλή προειδοποίηση μπορεί να σας εξοικονομήσει ώρες εντοπισμού σφαλμάτων αργότερα.
* **Ποτέ μην κάνετε commit το ακατέργαστο αρχείο `.lic`** σε δημόσιο αποθετήριο. Η χρήση διακομιστή κρατά το κλειδί εκτός ελέγχου πηγής.
* **Διατηρήστε τις βιβλιοθήκες Aspose ενημερωμένες** – νεότερες εκδόσεις μπορεί να εισάγουν πρόσθετα χαρακτηριστικά επικύρωσης που κάνουν το βήμα **check license validity** ακόμη πιο αξιόπιστο.
* **Δοκιμάστε τη διαδρομή αποτυχίας**: κατευθύνετε σκόπιμα σε μη έγκυρο URL και βεβαιωθείτε ότι η εφαρμογή σας υποχωρεί ομαλά (ίσως εμφανίζοντας ένα φιλικό προς τον χρήστη μήνυμα).
