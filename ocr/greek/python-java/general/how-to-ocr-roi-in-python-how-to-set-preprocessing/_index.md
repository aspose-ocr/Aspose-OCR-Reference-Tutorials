# Πώς να κάνετε OCR ROI σε Python – Πώς να ορίσετε προεπεξεργασία

Αναρωτηθήκατε ποτέ **πώς να κάνετε OCR ROI** σε μια θορυβώδη εικόνα τιμολογίου χωρίς να φορτώνετε ολόκληρη τη σελίδα στη μνήμη; Δεν είστε οι μόνοι. Σε πολλά πραγματικά έργα χρειάζονται μόνο μερικά πεδία — όνομα πελάτη, διεύθυνση, σύνολα — οπότε η σάρωση ολόκληρου του εγγράφου είναι υπερβολική.  

Τα καλά νέα; Με το Aspose OCR μπορείτε να πείτε στη μηχανή ακριβώς **πού να κοιτάξει** και ακόμη και να της ζητήσετε να καθαρίσει πρώτα την εικόνα. Σε αυτό το tutorial θα περάσουμε βήμα‑βήμα από **το πώς να ορίσετε προεπεξεργασία**, να ορίσουμε Regions of Interest (ROIs) και να εξάγουμε καθαρό κείμενο με λίγες μόνο γραμμές Python.

Στο τέλος αυτού του οδηγού θα έχετε ένα έτοιμο‑για‑εκτέλεση script που εξάγει συγκεκριμένα τμήματα από οποιοδήποτε τιμολόγιο, απόδειξη ή φόρμα. Δεν χρειάζονται επιπλέον εργαλεία, μόνο το Aspose OCR και λίγη λογική Python.

- Βασική εξοικείωση με συναρτήσεις Python και αντικειμενοστραφή σύνταξη  

Αν έχετε ήδη όλα αυτά, τέλεια — ας περάσουμε κατευθείαν στον κώδικα.

## Βήμα 1 – Αρχικοποίηση του OCR Engine (Πώς να κάνετε OCR ROI)

Πριν μπορέσουμε να πούμε στη μηχανή *πού* να κοιτάξει, χρειαζόμαστε μια παρουσία του OCR processor. Αυτό το αντικείμενο κρατά όλες τις ρυθμίσεις και εκτελεί τη φάση αναγνώρισης.

```python
import aspose.ocr as aocr
import aspose.storage as storage

# Create the engine – this is the entry point for all OCR work
ocr_engine = aocr.OcrEngine()
```

*Γιατί είναι σημαντικό*: Η κλάση `OcrEngine` αφαιρεί το βάρος της (ανίχνευση γραμματοσειρών, ανάλυση διάταξης κ.λπ.). Δημιουργώντας μία μόνο μηχανή αποφεύγετε την επανεκκίνηση βαρέων πόρων για κάθε εικόνα, κάτι που διατηρεί την απόδοση γρήγορη.

---

## Βήμα 2 – Φόρτωση της Πηγαίας Εικόνας (Πώς να κάνετε OCR ROI)

Το Aspose Storage κάνει τη φόρτωση εικόνων απλή. Δείξτε το στο μονοπάτι αρχείου και θα λάβετε ένα αντικείμενο `Image` έτοιμο για επεξεργασία.

```python
# Replace YOUR_DIRECTORY with the actual path where invoice.png lives
source_image = storage.Image.load("YOUR_DIRECTORY/invoice.png")
```

*Συμβουλή*: Αν εργάζεστε με streams (π.χ., εικόνες που ανεβαίνουν μέσω web API), μπορείτε να περάσετε ένα αντικείμενο `BytesIO` στη `Image.load` αντί για μονοπάτι αρχείου.

---

## Βήμα 3 – Ορισμός των Περιοχών Ενδιαφέροντος (ROIs)

Τώρα απαντάμε στην κεντρική ερώτηση **πώς να κάνετε OCR ROI**: λέμε στη μηχανή τα ακριβή ορθογώνια που περιέχουν τα δεδομένα που μας ενδιαφέρουν. Κάθε ROI ορίζεται από την πάνω‑αριστερή γωνία του (`x`, `y`) και το μέγεθός του (`width`, `height`).

```python
from aspose.ocr import Roi

# Each tuple is (x, y, width, height) in pixels
regions_of_interest = [
    Roi(50, 120, 400, 60),   # Customer name block
    Roi(50, 200, 400, 80)    # Address block
]
```

*Γιατί να χρησιμοποιήσετε ROIs;*  
- **Ταχύτητα** – Η μηχανή παραλείπει άσχετα τμήματα της εικόνας.  
- **Ακρίβεια** – Εστιάζοντας σε μικρή περιοχή, ο θόρυβος αλλού δεν μπορεί να μπερδέσει τον αναγνώστη.  
- **Ευελιξία** – Μπορείτε να προσθέσετε όσες ROIs χρειάζεστε· η μηχανή επιστρέφει μια λίστα αποτελεσμάτων με την ίδια σειρά.

---

## Βήμα 4 – Πώς να Ορίσετε Προεπεξεργασία για Καλύτερη Ακρίβεια OCR

Οι εικόνες που προέρχονται απευθείας από σαρωτές ή τηλέφωνα συχνά έχουν παραμόρφωση, χαμηλή αντίθεση ή άνιση φωτισμό. Το Aspose OCR προσφέρει ένα αντικείμενο `PreprocessingOptions` που σας επιτρέπει να ενεργοποιήσετε κοινές διορθώσεις πριν ξεκινήσει η αναγνώριση.

```python
from aspose.ocr import PreprocessingOptions

preprocessing_options = PreprocessingOptions()
preprocessing_options.deskew = True          # Auto‑rotate tilted text
preprocessing_options.binarize = True        # Convert to black‑white for clarity
preprocessing_options.contrast = 1.5         # Boost contrast (1.0 = no change)
```

*Πώς βοηθά*:  
- **Deskew** αφαιρεί την κλίση που κάνει τα σχήματα χαρακτήρων ασαφή.  
- **Binarize** μειώνει τον χρωματικό θόρυβο, δίνοντας στη μηχανή OCR μια καθαρή δυαδική εικόνα.  
- **Contrast** ενισχύει τα αδύναμα στίγματα, κάτι που είναι ιδιαίτερα χρήσιμο για ξεθωριασμένες αποδείξεις.

Μπορείτε να πειραματιστείτε με αυτές τις σημαίες — απενεργοποιήστε μία και δείτε αν το αποτέλεσμα αλλάζει. Αυτή είναι η ουσία του **πώς να ορίσετε προεπεξεργασία** αποτελεσματικά.

---

## Βήμα 5 – Εκτέλεση OCR Μόνο Μέσα στις Ορισμένες ROIs

Με τη μηχανή, την εικόνα, τις ROIs και την προεπεξεργασία έτοιμες, ήρθε η ώρα να καλέσετε το `recognize`. Η μέθοδος επιστρέφει ένα αντικείμενο `OcrResult` που περιέχει μια συλλογή `regions` — κάθε καταχώρηση ταιριάζει με τη σειρά των ROIs που δώσαμε.

```python
ocr_result = ocr_engine.recognize(
    source_image,
    rois=regions_of_interest,
    preprocessing=preprocessing_options
)
```

*Πίσω από τη σκηνή*: Η μηχανή κόβει κάθε ROI, εφαρμόζει τη σειρά προεπεξεργασίας και τρέχει το μοντέλο αναγνώρισης στο καθαρό τμήμα. Επειδή περάσαμε μια λίστα, το αποτέλεσμα διατηρεί την ίδια σειρά, κάνοντας το post‑processing απλό.

---

## Βήμα 6 – Ανάγνωση και Χρήση του Εξαγόμενου Κειμένου (Πώς να κάνετε OCR ROI)

Τέλος, διατρέξτε τη λίστα `regions` και εκτυπώστε — ή αποθηκεύστε — τις αναγνωρισμένες συμβολοσειρές. Το αντικείμενο `Region` εκθέτει μια ιδιότητα `text` που περιέχει το Unicode αποτέλεσμα.

```python
for idx, region in enumerate(ocr_result.regions, start=1):
    print(f"Region {idx} text:")
    print(region.text)
    print("-" * 40)
```

**Αναμενόμενο αποτέλεσμα (παράδειγμα)**:

```
Region 1 text:
John Doe
----------------------------------------
Region 2 text:
123 Maple Street
Springfield, IL 62704
----------------------------------------
```

Αν το κείμενο φαίνεται ακατανόητο, επανεξετάστε το **πώς να ορίσετε προεπεξεργασία**: ίσως αυξήσετε το `contrast` ή απενεργοποιήσετε το `binarize` για χρωματικά λογότυπα.

## Συνηθισμένα Πίνακες και Επαγγελματικές Συμβουλές

| Πρόβλημα | Γιατί Συμβαίνει | Διόρθωση (Πώς να Ορίσετε Προεπεξεργασία) |
|----------|----------------|------------------------------------------|
| Κείμενο με κλίση εμφανίζεται ως ακατανόητο | `deskew` απενεργοποιημένο ή η εικόνα πολύ περιστραμμένη | Ενεργοποιήστε `preprocessing_options.deskew = True` |
| Οι αριθμοί γίνονται κουκκίδες ή τυχαία σημεία | Χαμηλή αντίθεση ή υπερβολική δεικτοποίηση | Μειώστε το `contrast` (π.χ., `1.2`) ή θέστε `binarize = False` |
| Συντεταγμένες ROI εκτός μερικών pixel | Διαφορετικό DPI ή κλιμάκωση σαρωτή | Χρησιμοποιήστε εργαλείο (π.χ., Paint.NET) για ακριβή μέτρηση pixel, ή προσθέστε μικρό περιθώριο (`+5` pixel) σε κάθε ROI |
| Κενό αποτέλεσμα για μια περιοχή | ROI εκτός ορίων εικόνας | Επαληθεύστε τις διαστάσεις εικόνας: `source_image.width`, `source_image.height` |

## Επέκταση της Λύσης (Πώς να κάνετε OCR ROI σε Διαφορετικά Σενάρια)

- **Δυναμικές ROIs**: Αν τα τιμολόγιά σας έχουν μεταβλητές διατάξεις, μπορείτε πρώτα να τρέξετε ένα γρήγορο OCR πλήρους σελίδας για να εντοπίσετε λέξεις‑κλειδιά (“Customer:”, “Address:”) και μετά να υπολογίσετε τις ROIs δυναμικά.  
- **Επεξεργασία κατά Παρτίδες**: Τυλίξτε τα παραπάνω βήματα σε μια συνάρτηση και επαναλάβετε για έναν φάκελο εικόνων. Θυμηθείτε να επαναχρησιμοποιείτε την ίδια παρουσία `ocr_engine` για να κρατήσετε τη χρήση μνήμης χαμηλή.  
- **Εξαγωγή σε JSON**: Αντί για εκτύπωση, δημιουργήστε ένα λεξικό και αποθηκεύστε το με `json.dumps`; αυτό κάνει την ενσωμάτωση σε downstream συστήματα (π.χ., ERP) τριγύρω.

```python
import json

def extract_invoice_fields(image_path):
    img = storage.Image.load(image_path)
    result = ocr_engine.recognize(
        img,
        rois=regions_of_interest,
        preprocessing=preprocessing_options
    )
    data = {f"field_{i+1}": r.text.strip() for i, r in enumerate(result.regions)}
    return data

print(json.dumps(extract_invoice_fields("invoice.png"), indent=2))
```
