# Ανάλυση Διάταξης OCR – Λήψη Ορθογωνίων Γραμμών από Εικόνες

## Εισαγωγή

Σε αυτό το tutorial θα ανακαλύψετε **πώς να λαμβάνετε ορθογώνια γραμμών OCR** με το Aspose.OCR για .NET. Στο τέλος του οδηγού θα μπορείτε να **αναγνωρίζετε γραμμές κειμένου σε μια εικόνα** και να **εξάγετε τις συντεταγμένες των γραμμών** για κάθε ανιχνευόμενη γραμμή — ιδανικό για επεξεργασία downstream όπως **ανάλυση διάταξης OCR**, εξαγωγή δεδομένων ή προσαρμοσμένη απόδοση.

## Γρήγορες Απαντήσεις
- **Τι σημαίνει “λήψη ορθογωνίων γραμμών OCR”;** Επιστρέφει τα πλαίσια περιορισμού (bounding boxes) κάθε γραμμής κειμένου που εντοπίζεται σε μια εικόνα.  
- **Ποια μέθοδος API χρησιμοποιείται;** `AsposeOcr.GetRectangles(..., AreasType.LINES, ...)`.  
- **Χρειάζεται άδεια χρήσης;** Μια δωρεάν δοκιμή λειτουργεί για ανάπτυξη· απαιτείται εμπορική άδεια για παραγωγή.  
- **Υποστηριζόμενες μορφές εικόνας;** PNG, JPEG, BMP, TIFF και άλλες.  
- **Μπορώ να το τρέξω σε .NET Core;** Ναι, το Aspose.OCR υποστηρίζει πλήρως .NET Core και .NET 5/6.

## Προαπαιτούμενα

Πριν ξεκινήσετε το tutorial, βεβαιωθείτε ότι έχετε τα παρακάτω προαπαιτούμενα:

- Βασικές γνώσεις C# και ανάπτυξης .NET.  
- Ένα ολοκληρωμένο περιβάλλον ανάπτυξης (IDE) όπως το Visual Studio.  
- Βιβλιοθήκη Aspose.OCR για .NET εγκατεστημένη. Μπορείτε να τη κατεβάσετε [εδώ](https://releases.aspose.com/ocr/net/).  
- Ένα δείγμα εικόνας που περιέχει κείμενο για αναγνώριση OCR.

## Εισαγωγή Ονομάτων Χώρου

Βεβαιωθείτε ότι έχετε εισάγει τα απαραίτητα namespaces στο έργο σας. Προσθέστε τις παρακάτω γραμμές στην αρχή του αρχείου C#:

```csharp
using System;
using System.Collections.Generic;
using System.Drawing;
using System.IO;
using Aspose.OCR;
```

Τώρα, ας αναλύσουμε τη διαδικασία λήψης ορθογωνίων για γραμμές στην αναγνώριση εικόνας OCR σε εύκολα ακολουθήσιμα βήματα.

## ανάλυση διάταξης ocr – Οδηγός Βήμα‑Βήμα

### Βήμα 1: Ρύθμιση Καταλόγου Εγγράφων

```csharp
// ExStart:3
string dataDir = "Your Document Directory";
// ExEnd:3
```

Αντικαταστήστε το `"Your Document Directory"` με την πραγματική διαδρομή του φακέλου που περιέχει το δείγμα εικόνας.

### Βήμα 2: Αρχικοποίηση Aspose.OCR

```csharp
// ExStart:4
AsposeOcr api = new AsposeOcr();
// ExEnd:4
```

Δημιουργήστε μια παρουσία της κλάσης `AsposeOcr` για πρόσβαση στη λειτουργικότητα OCR.

### Βήμα 3: Καθορισμός Διαδρομής Εικόνας

```csharp
// ExStart:5
string fullPath = dataDir + "sample.png";
// ExEnd:5
```

Ορίστε τη πλήρη διαδρομή της εικόνας στην οποία θέλετε να εκτελέσετε OCR.

### Βήμα 4: Αναγνώριση Εικόνας και Λήψη Ορθογωνίων

```csharp
// ExStart:6
List<Rectangle> lines = api.GetRectangles(fullPath, AreasType.LINES, false);
// ExEnd:6
```

Η μέθοδος `GetRectangles` επιστρέφει μια λίστα αντικειμένων `Rectangle`, το καθένα από τα οποία αντιπροσωπεύει τις συντεταγμένες μιας ανιχνευόμενης γραμμής κειμένου. Αυτό αποτελεί τον πυρήνα της **λήψης ορθογωνίων γραμμών OCR** και επιτρέπει την **ανάλυση διάταξης OCR**.

### Βήμα 5: Εκτύπωση Αποτελέσματος

```csharp
// ExStart:7
Console.WriteLine("Areas coordinates:");
lines.ForEach(a => Console.WriteLine($"x:{a.X} y:{a.Y} width:{a.Width} height:{a.Height}"));
// ExEnd:7
```

Εκτυπώστε τις συντεταγμένες των ανιχνευόμενων περιοχών στην κονσόλα. Θα δείτε τιμές που μπορείτε αργότερα να χρησιμοποιήσετε για **εξαγωγή συντεταγμένων γραμμών** για προσαρμοσμένη επεξεργασία.

## Γιατί να Χρησιμοποιήσετε το Aspose.OCR για Ορθογώνια Γραμμών;

- **Υψηλή ακρίβεια** – Προηγμένοι αλγόριθμοι εντοπίζουν γραμμές ακόμη και σε θορυβώδεις ή λοξές εικόνες.  
- **Διαπλατφορμική** – Λειτουργεί σε .NET Framework, .NET Core και .NET 5/6.  
- **Χωρίς εξωτερικές εξαρτήσεις** – Καθαρή βιβλιοθήκη .NET, χωρίς native DLLs προς διανομή.  
- **Πλούσιο αποτέλεσμα** – Εκτός από τα ορθογώνια γραμμών, μπορείτε επίσης να ανακτήσετε λέξεις, χαρακτήρες και βαθμούς εμπιστοσύνης.

## Συνηθισμένα Προβλήματα και Λύσεις

| Πρόβλημα | Λύση |
|----------|------|
| **Δεν επιστρέφονται ορθογώνια** | Βεβαιωθείτε ότι η εικόνα περιέχει καθαρό, οριζόντιο κείμενο και ότι έχει οριστεί `AreasType.LINES`. |
| **Λανθασμένες συντεταγμένες** | Ελέγξτε το DPI της εικόνας· εικόνες χαμηλής ανάλυσης μπορεί να προκαλέσουν ανακριβή όρια. |
| **Σ bottleneck απόδοσης σε μεγάλες εικόνες** | Αλλάξτε το μέγεθος της εικόνας σε λογική ανάλυση πριν καλέσετε το `GetRectangles`. |
| **Απόρριψη άδειας** | Χρησιμοποιήστε δοκιμαστική άδεια για δοκιμές· εφαρμόστε πλήρη άδεια για παραγωγή ώστε να αποφύγετε περιορισμούς αξιολόγησης. |

## Συχνές Ερωτήσεις

**Ε: Μπορώ να εξάγω μεμονωμένες λέξεις αντί για ολόκληρες γραμμές;**  
Α: Ναι, χρησιμοποιήστε `AreasType.WORDS` με την ίδια μέθοδο `GetRectangles` για να λάβετε πλαίσια περιορισμού σε επίπεδο λέξης.

**Ε: Υποστηρίζει το API PDF με πολλαπλές σελίδες;**  
Α: Μετατρέψτε κάθε σελίδα PDF σε εικόνα πρώτα, έπειτα καλέστε το `GetRectangles` σε κάθε εικόνα.

**Ε: Πώς διαχειρίζομαι κείμενο που είναι περιστραμμένο;**  
Α: Ενεργοποιήστε την επιλογή auto‑rotate στις ρυθμίσεις OCR ή προ‑περιστρέψτε την εικόνα πριν την επεξεργασία.

**Ε: Υπάρχει τρόπος να λάβω βαθμούς εμπιστοσύνης για κάθε γραμμή;**  
Α: Μετά την λήψη των ορθογωνίων, καλέστε `api.RecognizeImage(...).Lines` για πρόσβαση σε αντικείμενα γραμμής που περιλαμβάνουν τιμές εμπιστοσύνης.

**Ε: Ποιες εκδόσεις .NET είναι συμβατές;**  
Α: Η βιβλιοθήκη λειτουργεί με .NET Framework 4.5+, .NET Core 3.1+, και .NET 5/6.
