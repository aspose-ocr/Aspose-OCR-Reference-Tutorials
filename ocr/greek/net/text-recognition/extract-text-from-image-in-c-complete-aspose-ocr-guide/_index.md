# Εξαγωγή Κειμένου από Εικόνα σε C# – Πλήρης Οδηγός Aspose OCR

Κάποτε χρειάστηκε να **εξάγετε κείμενο από εικόνα** αλλά δεν ήξερες ποια βιβλιοθήκη θα το έκανε χωρίς άπειρες ρυθμίσεις; Δεν είσαι μόνος. Σε πολλά έργα λαμβάνουμε μερικά JPG στιγμιότυπα και το επόμενο βήμα είναι να μετατρέψουμε αυτά τα pixel σε αναζητήσιμες συμβολοσειρές.  

Σε αυτό το tutorial θα περάσουμε από ένα πρακτικό παράδειγμα που δείχνει **πώς να αναγνωρίζετε κείμενο** από ένα αρχείο JPG, **πώς να μετατρέψετε JPG σε κείμενο**, και **πώς να φορτώσετε εικόνα για OCR** χρησιμοποιώντας το καθαρό C# API του Aspose OCR. Στο τέλος θα έχετε ένα έτοιμο πρόγραμμα που εκτυπώνει το εξαγόμενο κείμενο στην κονσόλα.

## Τι Θα Μάθετε

- Πώς να εγκαταστήσετε και να αναφέρετε το πακέτο NuGet Aspose OCR.  
- Η ακριβής ακολουθία κλήσεων που απαιτείται για **εξαγωγή κειμένου από εικόνα**.  
- Γιατί η ρύθμιση του κινητήρα σε λειτουργία αξιολόγησης είναι σημαντική για γρήγορες επιδείξεις.  
- Συνηθισμένα προβλήματα (π.χ. μη υποστηριζόμενες μορφές εικόνας) και πώς να τα αποφύγετε.  
- Πώς να επαληθεύσετε ότι το αποτέλεσμα του OCR ταιριάζει με την αρχική εικόνα.

Δεν απαιτείται προηγούμενη εμπειρία με OCR — αρκεί μια βασική γνώση C# και .NET 6 ή νεότερο εγκατεστημένο στο σύστημά σας.

## Προαπαιτούμενα

| Απαίτηση | Λόγος |
|-------------|--------|
| .NET 6 SDK (ή νεότερο) | Παρέχει το runtime για την εφαρμογή κονσόλας C#. |

Αν έχετε ήδη όλα αυτά, τέλεια — ας ξεκινήσουμε.

## Βήμα 1 – Ρυθμίστε τον Aspose OCR Engine για **Εξαγωγή Κειμένου από Εικόνα**

Πρώτα χρειάζεται μια παρουσία του `OcrEngine`. Αυτό το αντικείμενο είναι η καρδιά της βιβλιοθήκης· κρατά τις ρυθμίσεις και εκτελεί τη βαριά δουλειά.

```csharp
using System;
using Aspose.OCR;

class Program
{
    static void Main()
    {
        // Step 1: Create an OCR engine instance
        OcrEngine ocrEngine = new OcrEngine();

        // Enable evaluation mode – no license file needed for this demo
        ocrEngine.SetEvaluationMode();

        // Choose the language (Latin works for most English/Japanese‑Romanized texts)
        ocrEngine.Language = Language.Latin;
```

**Γιατί είναι σημαντικό:**  
Η δημιουργία του κινητήρα είναι φθηνή, αλλά αν ξεχάσετε να καλέσετε το `SetEvaluationMode()` θα προκληθεί εξαίρεση χρόνου εκτέλεσης εκτός αν έχετε αγοράσει άδεια. Η ρύθμιση της γλώσσας περιορίζει το σύνολο χαρακτήρων, βελτιώνοντας την ακρίβεια και την ταχύτητα επεξεργασίας.

## Βήμα 2 – **Φόρτωση Εικόνας για OCR** – **Αναγνώριση Κειμένου από JPG**

Τώρα δείχνουμε στον κινητήρα το αρχείο που θέλουμε να διαβάσουμε. Η βοηθητική μέθοδος `ImageStream.FromFile` αφαιρεί την ανάγκη να ανοίξουμε χειροκίνητα ένα `FileStream`.

```csharp
        // Step 2: Load the image you want to process
        // Replace YOUR_DIRECTORY with the actual folder path
        ocrEngine.Image = ImageStream.FromFile(@"YOUR_DIRECTORY\sample1.jpg");
```

**Συμβουλή για ακραίες περιπτώσεις:**  
Αν η εικόνα σας είναι σε μορφή PNG ή BMP, το `FromFile` λειτουργεί ακόμη, αλλά η ποιότητα του OCR μπορεί να διαφέρει. Για τα καλύτερα αποτελέσματα, χρησιμοποιήστε υψηλής ανάλυσης JPG (300 dpi ή περισσότερο).  

## Βήμα 3 – Εκτελέστε OCR και **Μετατρέψτε JPG σε Κείμενο**

Με την εικόνα φορτωμένη, μια κλήση στο `Recognize()` κάνει τα υπόλοιπα. Η μέθοδος επιστρέφει ένα `RecognitionResult` που περιέχει τη εξαγόμενη συμβολοσειρά και τις βαθμολογίες εμπιστοσύνης.

```csharp
        // Step 3: Run the OCR engine
        RecognitionResult recognitionResult = ocrEngine.Recognize();

        // Step 4: Display the extracted text
        Console.WriteLine("=== OCR Output ===");
        Console.WriteLine(recognitionResult.Text);
    }
}
```

**Τι συμβαίνει στο παρασκήνιο;**  
Το `Recognize()` εκτελεί μια σειρά βημάτων προεπεξεργασίας εικόνας — δυαδικοποίηση, διόρθωση κλίσης, τμηματοποίηση — πριν περάσει τα δεδομένα σε ένα νευρωνικό δίκτυο εκπαιδευμένο σε λατινικούς χαρακτήρες. Η ιδιότητα `Text` που επιστρέφεται είναι ήδη κωδικοποιημένη σε Unicode, ώστε να μπορείτε να τη γράψετε σε αρχείο, βάση δεδομένων ή να τη στείλετε σε άλλη υπηρεσία.

## Αναμενόμενο Αποτέλεσμα

Αν το `sample1.jpg` περιέχει τη φράση “Hello World”, η κονσόλα θα εμφανίσει:

```
=== OCR Output ===
Hello World
```

Αν η εικόνα είναι θολή, μπορεί να δείτε επιπλέον χαρακτήρες ή χαμηλότερη εμπιστοσύνη. Σε αυτήν την περίπτωση, σκεφτείτε να αυξήσετε το DPI της πηγαίας εικόνας ή να εφαρμόσετε φίλτρο ακονίσματος πριν τη φόρτωση.

## Pro Συμβουλές & Συνηθισμένα Πιθανά Σφάλματα

- **Pro tip:** Τυλίξτε την κλήση OCR σε μπλοκ `try…catch` για να διαχειριστείτε κορεσμένα αρχεία με χάρη.  
- **Πιθανό σφάλμα:** Η παράλειψη του ορισμού της γλώσσας μπορεί να κάνει τον κινητήρα να χρησιμοποιήσει ένα γενικό σύνολο, το οποίο μπορεί να αναγνωρίσει λανθασμένα διακριτικά.  
- **Συμβουλή απόδοσης:** Επαναχρησιμοποιήστε την ίδια παρουσία `OcrEngine` για πολλές εικόνες· η δημιουργία νέου κινητήρα κάθε φορά προσθέτει επιπλέον φόρτο.  
- **Τι γίνεται αν χρειαστεί να επεξεργαστώ PDF;** Το Aspose OCR μπορεί να δεχτεί σελίδες PDF ως εικόνες μέσω `ImageStream.FromPdf`, αλλά θα χρειαστείτε και τη βιβλιοθήκη Aspose.PDF.

## Βήμα 4 – Επαλήθευση της Εξαγωγής και Επόμενα Βήματα

Αφού εκτυπώσετε το αποτέλεσμα OCR, πιθανότατα θα θέλετε να το συγκρίνετε με την αρχική εικόνα χειροκίνητα ή μέσω ενός απλού checksum. Εδώ είναι ένας γρήγορος τρόπος για να γράψετε το αποτέλεσμα σε αρχείο κειμένου για μελλοντική ανασκόπηση:

```csharp
        // Optional: Save the OCR result to a .txt file
        System.IO.File.WriteAllText(@"YOUR_DIRECTORY\sample1_ocr.txt", recognitionResult.Text);
        Console.WriteLine("Result saved to sample1_ocr.txt");
```

**Ε: Λειτουργεί αυτό σε Linux;**  
Α: Απόλυτα. Το Aspose OCR είναι δια-πλατφορμικό· απλώς εγκαταστήστε το .NET runtime για Linux και ο ίδιος κώδικας τρέχει αμετάβλητος.

**Ε: Μπορώ να αναγνωρίσω μη‑λατινικά αλφάβητα;**  
Α: Ναι — το Aspose OCR υποστηρίζει κυριλλικά, αραβικά και αρκετά ασιατικά αλφάβητα. Αλλάξτε το `ocrEngine.Language` στην αντίστοιχη τιμή enum.
