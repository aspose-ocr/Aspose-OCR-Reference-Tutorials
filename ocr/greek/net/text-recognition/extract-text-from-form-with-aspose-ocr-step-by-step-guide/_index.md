# Εξαγωγή Κειμένου από Φόρμα με Aspose OCR – Οδηγός Βήμα‑βήμα

Έχετε ποτέ χρειαστεί να **εξάγετε κείμενο από φόρμα** αλλά ολόκληρη η σελίδα ήταν ένα θορυβώδες χάος; Δεν είστε μόνοι—οι προγραμματιστές παλεύουν συνεχώς με PDFs, σαρωμένα τιμολόγια ή χειρόγραφες έρευνες όπου μόνο ένα πεδίο έχει σημασία. Τα καλά νέα; Μπορείτε να πείτε στο Aspose OCR να κοιτάξει μόνο το τμήμα που σας ενδιαφέρει, αγνοώντας τα υπόλοιπα.  

Σε αυτόν τον οδηγό θα σας δείξουμε ακριβώς πώς να **αναγνωρίσετε κείμενο σε περιοχή** μιας σαρωμένης φόρμας, να εξάγετε την απαιτούμενη τιμή και να διατηρήσετε το υπόλοιπο της εικόνας ανέπαφο. Στο τέλος θα έχετε ένα έτοιμο‑για‑εκτέλεση πρόγραμμα C# που διαχειρίζεται το **μέρος OCR της εικόνας** που σας ενδιαφέρει, χωρίς να χρειάζεται εξωτερικές υπηρεσίες.

## Τι Θα Λάβετε Από Αυτό το Μάθημα

- Μια πλήρης, εκτελέσιμη εφαρμογή C# console που εξάγει ένα πεδίο από εικόνα φόρμας.  
- Μια σαφής εξήγηση του γιατί η στόχευση ενός ορθογωνίου είναι πιο γρήγορη και πιο ακριβής.  
- Συμβουλές για τη διαχείριση κενών αποτελεσμάτων, διαφορετικών ρυθμίσεων DPI και φορμών πολλαπλών σελίδων.  
- Μια γρήγορη λίστα ελέγχου ώστε να προσαρμόσετε τον κώδικα στα δικά σας έργα σε λίγα λεπτά.

**Prerequisites** – Θα χρειαστείτε .NET 6 ή νεότερο, Visual Studio 2022 (ή οποιοδήποτε IDE προτιμάτε), και σύνδεση στο διαδίκτυο την πρώτη φορά για να κατεβάσετε το πακέτο NuGet Aspose.OCR. Δεν απαιτούνται άλλες βιβλιοθήκες.

## Εξαγωγή Κειμένου από Φόρμα – Ρύθμιση του Έργου

Πριν βουτήξουμε στον κώδικα, ας βεβαιωθούμε ότι το περιβάλλον είναι έτοιμο. Τα βήματα είναι σκόπιμα απλά, επειδή η πραγματική μαγεία συμβαίνει μόλις δημιουργηθεί η μηχανή OCR.

1. **Δημιουργήστε ένα νέο έργο console**  
   ```bash
   dotnet new console -n FormOcrDemo
   cd FormOcrDemo
   ```

2. **Προσθέστε το Aspose.OCR μέσω NuGet**  
   ```bash
   dotnet add package Aspose.OCR
   ```

3. **Αντιγράψτε τη σαρωμένη φόρμα σας** στον φάκελο του έργου και μετονομάστε την σε `form.png`.  
   (Αν το αρχείο σας είναι PDF, εξάγετε πρώτα τη σελίδα που χρειάζεστε ως PNG—το Aspose OCR λειτουργεί καλύτερα με ραστερ εικόνες.)

Αυτό είναι όλο. Το υπόλοιπο του tutorial υποθέτει ότι αυτά τα τρία βήματα έχουν ολοκληρωθεί.

## Αναγνώριση Κειμένου σε Περιοχή – Ορισμός της Περιοχής Ενδιαφέροντος

Γιατί να ασχοληθείτε με ένα ορθογώνιο; Φανταστείτε ότι έχετε μια σάρωση 2 MB ολόκληρης φορολογικής δήλωσης. Η εκτέλεση OCR σε όλο το αρχείο σπαταλά κύκλους CPU και μπορεί να παράγει ψευδείς θετικές από μη σχετικούς τομείς. Περιορίζοντας το πεδίο στις ακριβείς συντεταγμένες που περιέχουν την τιμή-στόχο, εσείς:

- **Μειώστε το χρόνο επεξεργασίας** έως και 80 % για μεγάλες εικόνες.  
- **Αυξήστε την ακρίβεια** επειδή η μηχανή αγνοεί ενοχλητικά γραφικά.  
- **Απλοποιήστε την επεξεργασία μετά**—γνωρίζετε ακριβώς το κείμενο που αναμένεται.

Το ορθογώνιο ορίζεται από τέσσερις αριθμούς: `x`, `y`, `width` και `height`. Αυτές οι τιμές μετρώνται σε pixel από την πάνω‑αριστερή γωνία της εικόνας. Αν δεν είστε σίγουροι πού βρίσκεται το πεδίο, ανοίξτε την εικόνα στο Paint, περάστε το ποντίκι πάνω στην πάνω‑αριστερή γωνία για να διαβάσετε τις τιμές X/Y, και μετά σύρετε για να μετρήσετε το πλάτος και το ύψος.

## Μέρος OCR της Εικόνας – Φόρτωση της Φόρμας και Ρύθμιση της Μηχανής

Τώρα που η περιοχή είναι καθορισμένη, ας μιλήσουμε για τη μηχανή ίδια. Το `OcrEngine` είναι η καρδιά του Aspose OCR. Ανιχνεύει αυτόματα τη γλώσσα, διορθώνει την κλίση και επιστρέφει μια συμβολοσειρά plain‑text. Μπορείτε επίσης να ρυθμίσετε τις ιδιότητές του—όπως `Resolution` ή `Language`—αν η φόρμα σας χρησιμοποιεί μη λατινικό σύστημα γραφής. Για τις περισσότερες αγγλικές φόρμες, οι προεπιλογές λειτουργούν καλά.

Παρακάτω είναι το **πλήρες, αυτόνομο πρόγραμμα** που συνδυάζει τα πάντα. Μπορείτε να το αντιγράψετε‑και‑επικολλήσετε στο `Program.cs` και να πατήσετε **F5**.

```csharp
using Aspose.OCR;
using System;
using System.Drawing; // for Rectangle

class RegionExample
{
    static void Main()
    {
        // -------------------------------------------------
        // Step 1: Create the OCR engine (using ensures disposal)
        // -------------------------------------------------
        using (OcrEngine ocrEngine = new OcrEngine())
        {
            // -------------------------------------------------
            // Step 2: Load the image that contains the form
            // -------------------------------------------------
            // ImageStream.FromFile reads the file into Aspose's internal format
            // Replace "YOUR_DIRECTORY/form.png" with the actual path if needed
            ocrEngine.Image = ImageStream.FromFile("form.png");

            // -------------------------------------------------
            // Step 3: Define the region that holds the desired field
            // -------------------------------------------------
            // (x, y, width, height) – adjust these numbers for your form
            Rectangle regionOfInterest = new Rectangle(120, 350, 400, 80);

            // -------------------------------------------------
            // Step 4: Recognize text only inside the defined region
            // -------------------------------------------------
            bool success = ocrEngine.Recognize(regionOfInterest);
            string recognizedText = success ? ocrEngine.Text.Trim() : string.Empty;

            // -------------------------------------------------
            // Step 5: Display the extracted field value
            // -------------------------------------------------
            Console.WriteLine("Field value: " + (string.IsNullOrEmpty(recognizedText)
                ? "[No text detected]"
                : recognizedText));
        }

        // Keep the console window open when debugging
        Console.WriteLine("\nPress any key to exit...");
        Console.ReadKey();
    }
}
```

### Αναμενόμενη Έξοδος

Αν το ορθογώνιο περιλαμβάνει σωστά ένα πεδίο που λέει “Invoice # 12345”, η κονσόλα θα εκτυπώσει:

```
Field value: Invoice # 12345
```

Αν η περιοχή είναι κενή ή το OCR αποτύχει, θα δείτε:

```
Field value: [No text detected]
```

## Αναλυτική Εξήγηση Κώδικα Βήμα‑βήμα

Παρακάτω χωρίζουμε το πρόγραμμα σε μικρά τμήματα, εξηγούμε το **γιατί** πίσω από κάθε γραμμή, και επισημαίνουμε σημεία όπου ίσως χρειαστεί να προσαρμόσετε τον κώδικα.

### Βήμα 1: Εγκατάσταση Aspose.OCR μέσω NuGet *(H3)*

```bash
dotnet add package Aspose.OCR
```

*Γιατί;* Το πακέτο NuGet περιλαμβάνει τη μητρική μηχανή OCR, τα αρχεία δεδομένων γλώσσας και ένα ελαφρύ .NET wrapper. Χωρίς αυτό, η κλάση `OcrEngine` δεν υπάρχει.

### Βήμα 2: Αρχικοποίηση OcrEngine *(H3)*

```csharp
using (OcrEngine ocrEngine = new OcrEngine())
```

*Γιατί να χρησιμοποιήσετε `using`;* Το `OcrEngine` διατηρεί μη διαχειριζόμενους πόρους (αρχικά DLL, buffer εικόνας). Η δήλωση `using` εγγυάται την απελευθέρωσή τους, αποτρέποντας διαρροές μνήμης σε υπηρεσίες που τρέχουν για μεγάλο χρονικό διάστημα.

### Βήμα 3: Φόρτωση της Εικόνας Φόρμας *(H3)*

```csharp
ocrEngine.Image = ImageStream.FromFile("form.png");
```

*Γιατί `ImageStream`;* Απομονώνει την ανάγνωση αρχείων και μετατρέπει αυτόματα κοινές μορφές (PNG, JPEG, TIFF) στην εσωτερική αναπαράσταση bitmap που απαιτεί το Aspose OCR.

### Βήμα 4: Καθορισμός της Περιοχής για Εξαγωγή Κειμένου *(H3)*

```csharp
Rectangle regionOfInterest = new Rectangle(120, 350, 400, 80);
```

*Γιατί ένα `Rectangle`;* Η μηχανή OCR δέχεται ένα `System.Drawing.Rectangle`. Οι τέσσερις παράμετροι σας επιτρέπουν να εντοπίσετε ακριβώς το πεδίο, αφαιρώντας το υπόλοιπο της σελίδας.

*Συμβουλή:* Αν χρειάζεται να υποστηρίξετε πολλαπλά πεδία, αποθηκεύστε κάθε ορθογώνιο σε ένα `Dictionary<string, Rectangle>` και επαναλάβετε τη διαδικασία.

### Βήμα 5: Εκτέλεση Αναγνώρισης και Ανάκτηση Αποτελέσματος *(H3)*

```csharp
bool success = ocrEngine.Recognize(regionOfInterest);
string recognizedText = success ? ocrEngine.Text.Trim() : string.Empty;
```

*Γιατί να ελέγξετε το `success`;* Το OCR μπορεί να αποτύχει για διάφορους λόγους—θολή εικόνα, μη υποστηριζόμενη γλώσσα ή κενή περιοχή. Ο έλεγχος του boolean αποτρέπει την εργασία με παλιά δεδομένα.

### Βήμα 6: Διαχείριση Ακραίων Περιπτώσεων και Συνηθισμένων Παγίδων *(H3)*

- **Διαφορετικό DPI:** Αν η σάρωση είναι χαμηλής ανάλυσης (< 150 DPI), αυξήστε το `ocrEngine.Image.DpiX` και `ocrEngine.Image.DpiY` πριν καλέσετε το `Recognize`.  
- **Πολλαπλές Σελίδες:** Για PDF πολλαπλών σελίδων, εξάγετε κάθε σελίδα ως ξεχωριστό PNG και επαναλάβετε τη λογική του ορθογωνίου για κάθε σελίδα.  
- **Μη‑Αγγλικό Κείμενο:** Ορίστε `ocrEngine.Language = Language.Thai;` (ή οποιαδήποτε υποστηριζόμενη γλώσσα) πριν από την αναγνώριση.  
- **Μείωση Θορύβου:** `ocrEngine.Image.Filters.Add(new GaussianBlurFilter(1.2f));` μπορεί να βελτιώσει τα αποτελέσματα σε σάρωση με σπόρια.

## Προχωρώντας Περαιτέρω – Πραγματικές Παραλλαγές

### Εξαγωγή Πολλαπλών Πεδίων από την Ίδια Φόρμα

Αν χρειάζεται να εξάγετε “Name”, “Date” και “Amount” από ένα έγγραφο, δημιουργήστε μια συλλογή:

```csharp
var fields = new Dictionary<string, Rectangle>
{
    { "Name",   new Rectangle(50, 200, 300, 60) },
    { "Date",   new Rectangle(400, 200, 200, 60) },
    { "Amount", new Rectangle(650, 200, 250, 60) }
};

foreach (var kvp in fields)
{
    bool ok = ocrEngine.Recognize(kvp.Value);
    Console.WriteLine($"{kvp.Key}: {(ok ? ocrEngine.Text.Trim() : "[Not found]")}");
}
```

### Ενσωμάτωση με Βάση Δεδομένων

Μετά την εξαγωγή, ίσως θέλετε να αποθηκεύσετε το αποτέλεσμα σε SQL Server:

```csharp
using (var conn = new SqlConnection(connectionString))
{
    conn.Open();
    var cmd = new SqlCommand(
        "INSERT INTO FormResults (FieldName, FieldValue) VALUES (@name, @value)", conn);
    cmd.Parameters.AddWithValue("@name", "InvoiceNumber");
    cmd.Parameters.AddWithValue("@value", recognizedText);
    cmd.ExecuteNonQuery();
}
```

### Αυτοματοποίηση σε Διακομιστή

Αν σκοπεύετε να το εκτελέσετε ως υπηρεσία παρασκηνίου, τυλίξτε τη λογική σε μια μέθοδο `async` και χρησιμοποιήστε `Task.Run` για να διατηρήσετε το UI ανταποκρινόμενο. Θυμηθείτε να ορίσετε `ocrEngine.ParallelProcessing = true;` για επιταχύνσεις πολλαπλών πυρήνων.
