# Πώς να Χρησιμοποιήσετε OCR σε C# – Εξαγωγή Κειμένου από Εικόνες με GPU

Έχετε αναρωτηθεί ποτέ **πώς να χρησιμοποιήσετε OCR** για να εξάγετε κείμενο από ένα σαρωμένο έγγραφο χωρίς κόπο; Δεν είστε οι μόνοι—οι προγραμματιστές ρωτούν συνεχώς, «Πώς μπορώ να εξάγω κείμενο από αρχεία εικόνας αποδοτικά;» Τα καλά νέα είναι ότι με το Aspose.OCR μπορείτε να το κάνετε ακριβώς αυτό, και μπορείτε ακόμη και να **ενεργοποιήσετε την επεξεργασία με GPU** για αξιοσημείωτη αύξηση ταχύτητας σε υποστηριζόμενο υλικό.

Σε αυτό το tutorial θα περάσουμε από ένα πλήρες, ολοκληρωμένο παράδειγμα που σας δείχνει **πώς να χρησιμοποιήσετε OCR**, πώς να **εξάγετε κείμενο από εικόνα**, πώς να **μετατρέψετε τη σάρωση σε κείμενο**, και τι να κάνετε αν το GPU δεν είναι διαθέσιμο. Στο τέλος θα έχετε μια έτοιμη για εκτέλεση εφαρμογή C# console που εκτυπώνει το αναγνωρισμένο κείμενο και σας ενημερώνει αν το GPU χρησιμοποιήθηκε πραγματικά.

## Τι Θα Χρειαστείτε

- .NET 6 SDK ή νεότερο (ο κώδικας λειτουργεί επίσης με .NET Core)  
- Visual Studio 2022 ή οποιονδήποτε επεξεργαστή προτιμάτε  
- Πακέτο Aspose.OCR για .NET (διαθέσιμο μέσω NuGet)  
- Ένα αρχείο εικόνας υψηλής ανάλυσης (π.χ., `highres_scan.tif`) για δοκιμή  

Δεν απαιτείται περίπλοκη ρύθμιση—απλώς μερικές εντολές NuGet και είστε έτοιμοι.

## Βήμα 1: Εγκατάσταση Aspose.OCR και Προετοιμασία του Έργου

Πρώτα απ' όλα. Πρέπει να προσθέσετε τη βιβλιοθήκη OCR στο έργο σας. Ανοίξτε ένα τερματικό στο φάκελο της λύσης και εκτελέστε:

```bash
dotnet new console -n OcrDemo
cd OcrDemo
dotnet add package Aspose.OCR
```

Αυτό δημιουργεί ένα νέο έργο console με όνομα **OcrDemo** και προσθέτει το πακέτο NuGet `Aspose.OCR`. Η βιβλιοθήκη περιέχει την κλάση `OcrEngine` που θα χρησιμοποιήσουμε.

> **Συμβουλή:** Αν βρίσκεστε σε μηχάνημα με αποκλειστικό GPU, βεβαιωθείτε ότι είναι εγκατεστημένος ο πιο πρόσφατος οδηγός γραφικών· διαφορετικά η βιβλιοθήκη θα επιστρέψει αυτόματα σε λειτουργία CPU.

## Βήμα 2: Γράψτε τον Πλήρη Κώδικα OCR

Τώρα ανοίξτε το `Program.cs` και αντικαταστήστε το περιεχόμενό του με το παρακάτω. Κάθε γραμμή είναι σχολιασμένη ώστε να βλέπετε *γιατί* κάνουμε ό,τι κάνουμε.

```csharp
using System;
using Aspose.OCR;
using Aspose.OCR.Enums;

namespace OcrDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // -------------------------------------------------
            // 1️⃣  Create an OCR engine and request GPU processing.
            //    If the GPU isn’t present, Aspose.OCR will silently
            //    switch to CPU mode – no crash, no extra code needed.
            // -------------------------------------------------
            var ocrEngine = new OcrEngine
            {
                ProcessingMode = OcrProcessingMode.Gpu
            };

            // -------------------------------------------------
            // 2️⃣  Define the path to the image you want to process.
            //    Replace the placeholder with the actual file location.
            // -------------------------------------------------
            string imagePath = @"YOUR_DIRECTORY/highres_scan.tif";

            // -------------------------------------------------
            // 3️⃣  Perform the recognition. The method returns an
            //    OcrResult object that contains the extracted text,
            //    confidence scores, and more.
            // -------------------------------------------------
            OcrResult ocrResult = ocrEngine.RecognizeImage(imagePath);

            // -------------------------------------------------
            // 4️⃣  Report whether the GPU was really used.
            //    This is handy for debugging performance issues.
            // -------------------------------------------------
            Console.WriteLine($"GPU used: {ocrEngine.IsGpuEnabled}");

            // -------------------------------------------------
            // 5️⃣  Output the recognized text to the console.
            //    In a real app you might write this to a file,
            //    a database, or feed it into another workflow.
            // -------------------------------------------------
            Console.WriteLine("=== Extracted Text ===");
            Console.WriteLine(ocrResult.Text);
        }
    }
}
```

### Γιατί Λειτουργεί Αυτό

- **ProcessingMode = Gpu** λέει στη μηχανή να δοκιμάσει πρώτα το GPU. Η βιβλιοθήκη αφαιρεί τις χαμηλού επιπέδου κλήσεις CUDA/OpenCL, έτσι δεν χρειάζεται να διαχειρίζεστε τα context της συσκευής μόνοι σας.  
- **IsGpuEnabled** είναι μια boolean που επιβεβαιώνει αν η διαδρομή GPU πέτυχε. Αν δείτε `false`, η μηχανή επέστρεψε αυτόματα σε CPU—δεν υπάρχει λόγος πανικού.  
- **RecognizeImage** κάνει όλη τη βαριά δουλειά: φορτώνει την εικόνα, εκτελεί το μοντέλο OCR και επιστρέφει το αποτέλεσμα ως απλό κείμενο. Δεν χρειάζεται να προεπεξεργαστείτε το bitmap χειροκίνητα εκτός αν έχετε ειδικές απαιτήσεις (π.χ., διόρθωση κλίσης).

## Βήμα 3: Εκτελέστε την Εφαρμογή και Επαληθεύστε το Αποτέλεσμα

Compile and execute:

```bash
dotnet run
```

If everything is set up correctly, you’ll see something like:

```
GPU used: True
=== Extracted Text ===
This is the first line of the scanned document.
Here is the second line, and so on…
```

Αν το GPU δεν είναι διαθέσιμο, η πρώτη γραμμή θα εμφανίσει `GPU used: False`, αλλά το εξαγόμενο κείμενο θα εμφανιστεί κανονικά—ευχαριστώντας την ομαλή εναλλαγή.

> **Συχνή ερώτηση:** *Τι γίνεται αν η εικόνα μου είναι JPEG αντί για TIFF;*  
> Η μέθοδος `RecognizeImage` δέχεται οποιαδήποτε μορφή υποστηρίζεται από το `System.Drawing` του .NET (JPEG, PNG, BMP, κ.λπ.). Απλώς αλλάξτε την επέκταση αρχείου στο `imagePath`.

## Βήμα 4: Προαιρετικό – Ρυθμίστε τις Ρυθμίσεις για Καλύτερη Ακρίβεια

Το Aspose.OCR προσφέρει μερικές ρυθμίσεις που μπορείτε να προσαρμόσετε:

| Ρύθμιση | Τι κάνει | Πότε να τη χρησιμοποιήσετε |
|---------|----------|---------------------------|
| `ocrEngine.Language` | Επιβάλλει μια συγκεκριμένη γλώσσα (π.χ., `OcrLanguage.English`) | Αν γνωρίζετε τη γλώσσα του εγγράφου |
| `ocrEngine.PageSegMode` | Ελέγχει πώς η μηχανή χωρίζει τις σελίδες σε μπλοκ | Για διατάξεις πολλαπλών στηλών |
| `ocrEngine.DetectOrientation` | Αυτόματη περιστροφή κειμένου που δεν είναι ευθεία | Σαρώσεις που μπορεί να είναι ανάποδες |

Μπορείτε να ορίσετε αυτές τις ιδιότητες πριν καλέσετε το `RecognizeImage`. Για παράδειγμα:

```csharp
ocrEngine.Language = OcrLanguage.English;
ocrEngine.DetectOrientation = true;
```

## Περιπτώσεις Ορίων & Επίλυση Προβλημάτων

1. **Out‑of‑Memory on GPU** – Πολύ μεγάλες εικόνες μπορεί να υπερβούν τη μνήμη GPU. Σε αυτήν την περίπτωση, η βιβλιοθήκη θα επιστρέψει αυτόματα σε CPU. Μπορείτε να προσαρμόσετε το μέγεθος της εικόνας για να μειώσετε τη χρήση μνήμης.  
2. **Unsupported Image Format** – Αν η `RecognizeImage` ρίξει *NotSupportedException*, ελέγξτε την επέκταση του αρχείου και βεβαιωθείτε ότι η εικόνα δεν είναι κατεστραμμένη. Η μετατροπή σε PNG συχνά λύνει το πρόβλημα.  
3. **Low Confidence Scores** – Όταν το αποτέλεσμα OCR περιέχει πολλούς ακατανόητους χαρακτήρες, σκεφτείτε προεπεξεργασία (δυαδικοποίηση, αφαίρεση θορύβου) ή μετάβαση σε σάρωση υψηλότερης ανάλυσης.
