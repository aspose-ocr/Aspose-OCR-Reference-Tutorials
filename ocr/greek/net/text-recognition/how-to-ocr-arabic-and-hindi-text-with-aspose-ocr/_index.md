# πώς να κάνετε OCR αραβικού και ινδικού κειμένου με Aspose OCR

Έχετε αναρωτηθεί ποτέ **πώς να κάνετε OCR αραβικού** χαρακτήρων που ρέουν από δεξιά προς αριστερά, ενώ ταυτόχρονα εξάγετε γλυφές Χίντι από μια απόδειξη; Δεν είστε μόνοι. Πολλοί προγραμματιστές αντιμετωπίζουν το ίδιο πρόβλημα όταν χρειάζεται να **αναγνωρίσουν αραβικό κείμενο** και **αναγνωρίσουν ινδικό κείμενο** στην ίδια ροή εργασίας.  

Σε αυτό το tutorial θα περάσουμε βήμα‑βήμα ένα πλήρες, εκτελέσιμο παράδειγμα C# που σας δείχνει πώς να **εξάγετε κείμενο από εικόνα** αρχεία, **μετατρέψετε εικόνα σε κείμενο**, και να διαχειριστείτε και τα δύο σενάρια, Αραβικό και Χίντι, με το Aspose OCR. Καμία ασαφής αναφορά—μόνο ο κώδικας που μπορείτε να αντιγράψετε‑επικολλήσετε, συν τη λογική πίσω από κάθε γραμμή.

> **Pro tip:** Αν δεν έχετε χρησιμοποιήσει ποτέ το Aspose OCR, εγκαταστήστε πρώτα το πακέτο NuGet `Aspose.OCR`. Είναι μια ενέργεια με ένα κλικ στο Visual Studio, και κατεβάζει όλα τα εγγενή δυαδικά αρχεία που χρειάζεστε για αναγνώριση με βάση την CPU.

## πώς να κάνετε OCR αραβικού – Ρύθμιση του Περιβάλλοντος

Πριν βουτήξουμε στον κώδικα, ας βεβαιωθούμε ότι το περιβάλλον ανάπτυξης είναι έτοιμο.

1. **Target framework** – .NET 6.0 ή νεότερο. Οτιδήποτε παλαιότερο θα εξακολουθεί να μεταγλωττίζεται, αλλά θα χάσετε τις πιο πρόσφατες δυνατότητες της γλώσσας.  
2. **Package** – Εκτελέστε `dotnet add package Aspose.OCR` στο τερματικό, ή χρησιμοποιήστε το UI του NuGet Package Manager.  
3. **Images** – Τοποθετήστε δύο δείγμα εικόνων σε έναν φάκελο που μπορείτε να αναφέρετε, π.χ. `C:\OCRSamples\arabic_sign.jpg` και `C:\OCRSamples\hindi_receipt.png`. Η αραβική εικόνα πρέπει να περιέχει καθαρούς, υψηλής αντίθεσης αραβικούς χαρακτήρες· η εικόνα Χίντι μπορεί να είναι μια σαρωμένη απόδειξη ή μια φωτογραφία πινακίδας.  

Αυτό είναι όλο—χωρίς επιπλέον αρχεία ρυθμίσεων, χωρίς οδηγούς GPU, μόνο μια απλή μηχανή OCR βασισμένη στην CPU.

## Αναγνώριση Αραβικού Κειμένου – Φόρτωση και Επεξεργασία

Τώρα θα **αναγνωρίσουμε αραβικό κείμενο**. Το κλειδί είναι να πείτε στη μηχανή ποια γλώσσα να περιμένει· διαφορετικά η μηχανή προεπιλέγει λατινικούς χαρακτήρες και θα λάβετε ακατάστατο αποτέλεσμα.

```csharp
using Aspose.OCR;
using Aspose.OCR.Models;

class MultiLangDemo
{
    static void Main()
    {
        // 1️⃣ Create the OCR engine (CPU‑based by default)
        var ocrEngine = new OcrEngine();

        // 2️⃣ Load an image that contains Arabic script
        var arabicImage = OcrImage.FromFile(@"C:\OCRSamples\arabic_sign.jpg");

        // 3️⃣ Recognize the Arabic text – note the Language.Arabic enum
        var arabicResult = ocrEngine.Recognize(arabicImage, Language.Arabic);

        // 4️⃣ Print the result to the console
        System.Console.WriteLine("Arabic: " + arabicResult.Text);
```

**Γιατί αυτό λειτουργεί:**  
* `OcrEngine` διαχειρίζεται όλη τη βαριά δουλειά—προεπεξεργασία, τμηματοποίηση και ταξινόμηση χαρακτήρων.  
* Με τη μετάδοση του `Language.Arabic`, ενεργοποιούμε τη μηχανή διάταξης από δεξιά προς αριστερά και το αραβικό σύνολο χαρακτήρων. Χωρίς αυτό, η μηχανή θα αντιμετωπίζει την εικόνα ως κείμενο λατινικό από αριστερά προς δεξιά, κάτι που οδηγεί σε ελλιπείς τόνους και σπασμένες λέξεις.  

**Αναμενόμενη έξοδος** (το πραγματικό κείμενο θα διαφέρει ανάλογα με την εικόνα):

```
Arabic: مرحبا بكم في متجرنا
```

Αν δείτε κενές συμβολοσειρές, ελέγξτε ξανά ότι η εικόνα δεν είναι πολύ σκοτεινή και ότι η διαδρομή του αρχείου είναι σωστή.

## εξαγωγή κειμένου από εικόνα – Διαχείριση Σενάρια Δεξιά‑προς‑Αριστερά

Η αραβική δεν είναι η μόνη γραφή που χρειάζεται ειδική διαχείριση. Οι γλώσσες από δεξιά προς αριστερά (RTL) απαιτούν η μηχανή OCR να αντιστρέφει τη οπτική σειρά μετά την αναγνώριση. Το Aspose το κάνει αυτό αυτόματα όταν καθορίζετε `Language.Arabic`, αλλά αξίζει να το σημειώσετε για μελλοντικές επεκτάσεις (π.χ., Εβραϊκά).

*Tip:* Όταν εμφανίζετε αργότερα το αποτέλεσμα OCR σε UI, βεβαιωθείτε ότι ο έλεγχος υποστηρίζει απόδοση RTL· διαφορετικά το κείμενο θα εμφανιστεί ακατάστατο.

## μετατροπή εικόνας σε κείμενο – Εργασία με Χίντι

Αλλάζοντας εστίαση, ας **μετατρέψουμε εικόνα σε κείμενο** για μια απόδειξη Χίντι. Η διαδικασία αντικατοπτρίζει τη ροή του Αραβικού, αλλά χρησιμοποιούμε `Language.Hindi`.

```csharp
        // 5️⃣ Load the Hindi (Devanagari) image
        var hindiImage = OcrImage.FromFile(@"C:\OCRSamples\hindi_receipt.png");

        // 6️⃣ Recognize Hindi text – Language.Hindi tells the engine to use Devanagari models
        var hindiResult = ocrEngine.Recognize(hindiImage, Language.Hindi);

        // 7️⃣ Output the Hindi OCR result
        System.Console.WriteLine("Hindi: " + hindiResult.Text);
    }
}
```

**Γιατί επαναχρησιμοποιούμε το ίδιο στιγμιότυπο `ocrEngine`:**  
Δημιουργώντας μια νέα μηχανή για κάθε γλώσσα θα σπαταλούσε μνήμη και χρόνο εκκίνησης. Η μηχανή του Aspose είναι ασφαλής ως προς νήματα για διαδοχικές κλήσεις, έτσι η επαναχρησιμοποίηση είναι τόσο αποδοτική όσο και καθαρή.

**Δείγμα εξόδου κονσόλας** (πάλι, εξαρτάται από την εικόνα σας):

```
Hindi: कुल राशि: ₹ 1,250.00
```

Αν το κείμενο Χίντι εμφανίζεται ως ακατάστατοι λατινικοί χαρακτήρες, πιθανώς παραλείψατε το enum γλώσσας ή η ανάλυση της εικόνας είναι πολύ χαμηλή (<300 dpi). Η αύξηση της ανάλυσης της εικόνας ή η εφαρμογή ενός απλού φίλτρου δυαδικοποίησης μπορεί να βελτιώσει δραματικά την ακρίβεια.

## αναγνώριση κειμένου Χίντι – Συνηθισμένα προβλήματα και περιπτώσεις άκρων

Ακόμη και με τη σωστή σημαία γλώσσας, μερικά μικρά προβλήματα μπορούν να σας εμποδίσουν:

| Πρόβλημα | Σύμπτωμα | Διόρθωση |
|----------|----------|----------|
| Χαμηλή αντίθεση | Πολλοί χαρακτήρες γίνονται “?” ή παραλείπονται | Προεπεξεργασία με `OcrImage.AdjustContrast(1.5)` |
| Καμπυλωμένη εικόνα | Το κείμενο εμφανίζεται περιστραμμένο, το OCR επιστρέφει κενή συμβολοσειρά | Κλήση `ocrEngine.PreprocessImage(arabicImage, ImageProcessingOptions.Rotate)` |
| Μικτές γλώσσες | Γραμμή αραβικού ακολουθείται από αριθμούς Αγγλικών | Εκτέλεση δύο περάσεων: πρώτα με `Language.Arabic`, μετά με `Language.English` στην ίδια εικόνα και συνένωση αποτελεσμάτων |
| Μεγάλο μέγεθος αρχείου | Αργή αναγνώριση ή σφάλματα έλλειψης μνήμης | Μείωση μεγέθους σε μέγιστο πλάτος 2000 px με `OcrImage.Resize(2000, 0)` |

Αυτές οι συμβουλές σας βοηθούν να **εξάγετε κείμενο από εικόνα** αρχεία που δεν είναι τέλεια σαρωμένα, κάτι που είναι συχνό σε πραγματικά έργα.

## Συνδυάζοντας τα όλα – Πλήρες λειτουργικό παράδειγμα

Παρακάτω είναι το πλήρες πρόγραμμα που μπορείτε να αντιγράψετε απευθείας σε ένα νέο έργο κονσόλας. Χωρίς κρυφές εξαρτήσεις, χωρίς επιπλέον ρυθμίσεις—απλώς καθαρό C#.

```csharp
using Aspose.OCR;
using Aspose.OCR.Models;

class MultiLangDemo
{
    static void Main()
    {
        // Initialize the OCR engine (CPU‑based)
        var ocrEngine = new OcrEngine();

        // -------------------- Arabic --------------------
        var arabicPath = @"C:\OCRSamples\arabic_sign.jpg";
        var arabicImage = OcrImage.FromFile(arabicPath);
        var arabicResult = ocrEngine.Recognize(arabicImage, Language.Arabic);
        System.Console.WriteLine("Arabic: " + arabicResult.Text);

        // -------------------- Hindi --------------------
        var hindiPath = @"C:\OCRSamples\hindi_receipt.png";
        var hindiImage = OcrImage.FromFile(hindiPath);
        var hindiResult = ocrEngine.Recognize(hindiImage, Language.Hindi);
        System.Console.WriteLine("Hindi: " + hindiResult.Text);
    }
}
```

**Η εκτέλεση του προγράμματος** θα εκτυπώσει τόσο αραβικές όσο και χίντι συμβολοσειρές στην κονσόλα, επιβεβαιώνοντας ότι έχετε επιτυχώς **αναγνωρίσει αραβικό κείμενο** και **αναγνωρίσει χίντι κείμενο** σε μία εκτέλεση.

## Συμπέρασμα

Τώρα έχετε μια ισχυρή, ολοκληρωμένη απάντηση στην ερώτηση **πώς να κάνετε OCR αραβικού** ενώ διαχειρίζεστε και το Χίντι. Δημιουργώντας ένα μόνο `OcrEngine`, φορτώνοντας κάθε εικόνα και περνώντας το κατάλληλο enum `Language`, μπορείτε να **εξάγετε κείμενο από εικόνα**, **μετατρέψετε εικόνα σε κείμενο**, και **να αναγνωρίσετε αραβικό κείμενο** καθώς και **να αναγνωρίσετε χίντι κείμενο** χωρίς επιπλέον βιβλιοθήκες.  

Από εδώ μπορείτε να εξερευνήσετε:

* **Batch processing** – επανάληψη σε φάκελο εικόνων και αποθήκευση αποτελεσμάτων σε βάση δεδομένων.  
* **Post‑processing** – χρήση κανονικών εκφράσεων για καθαρισμό συμβόλων νομισμάτων σε αποδείξεις Χίντι.  
* **Hybrid language detection** – τροφοδοσία του ακατέργαστου bitmap σε μοντέλο αναγνώρισης γλώσσας πριν την επιλογή του enum.
