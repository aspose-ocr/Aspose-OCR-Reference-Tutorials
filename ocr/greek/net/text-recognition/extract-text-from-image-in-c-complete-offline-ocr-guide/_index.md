# Εξαγωγή Κειμένου από Εικόνα – Πλήρης Οδηγός Offline OCR

Έχετε ποτέ χρειαστεί να **εξάγετε κείμενο από εικόνα** αλλά ανησυχείτε για την καθυστέρηση του δικτύου ή περιορισμούς άδειας; Δεν είστε μόνοι. Πολλοί προγραμματιστές συναντούν εμπόδια όταν η εφαρμογή τους πρέπει να λειτουργεί σε περιβάλλον sandbox, αλλά εξακολουθεί να χρειάζεται αξιόπιστο OCR. Τα καλά νέα; Με το Aspose OCR μπορείτε να εκτελέσετε ολόκληρη τη διαδικασία τοπικά, **εξάγοντας κείμενο από εικόνα** χωρίς ποτέ να χρειαστεί να συνδεθείτε στο διαδίκτυο.

Σε αυτό το tutorial θα περάσουμε από ένα πρακτικό παράδειγμα που δείχνει **πώς να εξάγετε κείμενο**, να ρυθμίσετε μια offline μηχανή, **να εκτελέσετε αναγνώριση OCR**, και ακόμη **να αναγνωρίσετε κυριλλικό κείμενο**. Στο τέλος θα έχετε μια έτοιμη για εκτέλεση εφαρμογή C# console που εκτυπώνει το ανιχνευμένο κείμενο απευθείας στην κονσόλα.

## Τι Θα Χρειαστείτε

- .NET 6.0 SDK (ή οποιαδήποτε πρόσφατη έκδοση .NET)  
- Visual Studio 2022 ή VS Code – ό,τι προτιμάτε  
- Πακέτο NuGet Aspose.OCR για .NET  
- Ένας φάκελος που περιέχει τα αρχεία μοντέλων Aspose OCR (κατεβάστε μία φορά από το portal του Aspose)  
- Ένα αρχείο εικόνας που περιλαμβάνει αγγλικούς και κυριλλικούς χαρακτήρες (π.χ., `cyrillic_doc.jpg`)

Καμία εξωτερική υπηρεσία, καμία κρυφή λήψη κατά την εκτέλεση – όλα βρίσκονται στον δίσκο σας.

## Βήμα 1: Εγκατάσταση Aspose.OCR και Προετοιμασία Πόρων

Πρώτα, προσθέστε το πακέτο NuGet Aspose.OCR στο έργο σας:

```bash
dotnet add package Aspose.OCR
```

Στη συνέχεια, δημιουργήστε έναν φάκελο με όνομα `AsposeOCRResources` κάπου στον υπολογιστή σας και αντιγράψτε τα αρχεία μοντέλων που κατεβάσατε από το Aspose σε αυτόν. Η μηχανή OCR θα ψάξει για πακέτα γλώσσας σε αυτόν τον κατάλογο, οπότε βεβαιωθείτε ότι η διαδρομή είναι σωστή.

> **Συμβουλή:** Κρατήστε το φάκελο πόρων δίπλα στο αρχείο `.csproj`· απλοποιεί τη διαχείριση διαδρομών κατά την ανάπτυξη.

## Βήμα 2: Δημιουργία της Offline Μηχανής OCR

Τώρα θα δημιουργήσουμε την μηχανή και θα την κατευθύνουμε στον φάκελο πόρων. Αυτό είναι το κρίσιμο βήμα που μας επιτρέπει να **εκτελέσουμε αναγνώριση OCR** εντελώς offline.

```csharp
using Aspose.OCR;
using Aspose.OCR.Enums;

class OfflineDemo
{
    static void Main()
    {
        // 1️⃣ Create the OCR engine instance
        OcrEngine ocrEngine = new OcrEngine();

        // 2️⃣ Tell the engine where the language models live
        ocrEngine.Settings.ResourcesPath = @"YOUR_DIRECTORY/AsposeOCRResources";

        // 3️⃣ Load the languages you need – English and Cyrillic
        ocrEngine.LoadLanguage(Language.English);
        ocrEngine.LoadLanguage(Language.Cyrillic);
```

Γιατί να φορτώνουμε τις γλώσσες ρητά; Επειδή είπαμε στη μηχανή να παραμείνει offline· δεν θα επικοινωνήσει με τους διακομιστές του Aspose για να κατεβάσει τα ελλιπή πακέτα. Αν ξεχάσετε να φορτώσετε μια γλώσσα, οποιοσδήποτε χαρακτήρας από αυτή τη γραφή θα αγνοηθεί.

## Βήμα 3: Παροχή της Εικόνας στη Μηχανή

Με τη μηχανή έτοιμη, τώρα παρέχουμε την εικόνα που θέλουμε να επεξεργαστούμε. Η βοηθητική μέθοδος `ImageStream.FromFile` διαβάζει το αρχείο σε μορφή που καταλαβαίνει η μηχανή OCR.

```csharp
        // 4️⃣ Load the target image (make sure the path points to your file)
        ImageStream image = ImageStream.FromFile(@"YOUR_DIRECTORY/cyrillic_doc.jpg");
```

Αν η εικόνα σας είναι μεγάλη, σκεφτείτε να την αλλάξετε μέγεθος εκ των προτέρων για να βελτιώσετε την ταχύτητα και την ακρίβεια. Η μηχανή OCR λειτουργεί καλύτερα με DPI περίπου 300.

## Βήμα 4: Εκτέλεση της Διαδικασίας Αναγνώρισης

Καλώντας το `Recognize` εκτελείται η βαριά δουλειά. Η μέθοδος επιστρέφει ένα αντικείμενο `OcrResult` που περιέχει το εξαγόμενο κείμενο, βαθμολογίες εμπιστοσύνης και άλλα.

```csharp
        // 5️⃣ Run the OCR recognition
        OcrResult ocrResult = ocrEngine.Recognize(image);
```

Πίσω από τη σκηνή, το Aspose αναλύει το bitmap, εκτελεί νευρωνικά δίκτυα ειδικά για κάθε γλώσσα και ενώνει τους χαρακτήρες. Επειδή φορτώσαμε τόσο τα αγγλικά όσο και τα κυριλλικά, τα έγγραφα με μεικτές γραφές διαχειρίζονται αβίαστα.

## Βήμα 5: Εμφάνιση του Εξαγόμενου Κειμένου

Τέλος, εμφανίζουμε το αποτέλεσμα. Σε μια πραγματική εφαρμογή μπορεί να το αποθηκεύσετε σε βάση δεδομένων ή να το στείλετε σε άλλη υπηρεσία, αλλά για αυτή τη demo θα το εκτυπώσουμε απλώς.

```csharp
        // 6️⃣ Show the extracted text on the console
        Console.WriteLine("=== OCR Output ===");
        Console.WriteLine(ocrResult.Text);
    }
}
```

Η εκτέλεση του προγράμματος θα πρέπει να σας δώσει κάτι όπως:

```
=== OCR Output ===
This is an English line.
Это строка на кириллице.
```

Αν δείτε ακατανόητους χαρακτήρες, ελέγξτε ξανά ότι το πακέτο κυριλλικής γλώσσας είναι σωστά τοποθετημένο στον φάκελο πόρων και ότι η εικόνα δεν είναι πολύ θολή.

## Διαχείριση Συνηθισμένων Προβλημάτων

### Έλλειψη Πακέτων Γλώσσας

Αν λάβετε μια εξαίρεση που λέει «Language data not found», η μηχανή δεν μπόρεσε να βρει τα αρχεία μοντέλου. Επαληθεύστε το `ResourcesPath` και βεβαιωθείτε ότι ο φάκελος περιέχει τα `english.dat` και `cyrillic.dat` (ή αρχεία με παρόμοια ονόματα).

### Χαμηλές Βαθμολογίες Εμπιστοσύνης

Κατά διαστήματα η μηχανή OCR θα επιστρέψει χαμηλή εμπιστοσύνη για ορισμένους χαρακτήρες, ειδικά αν η εικόνα έχει θόρυβο. Μπορείτε να βελτιώσετε την ακρίβεια κάνοντας:

- Μετατροπή της εικόνας σε αποχρώσεις του γκρι πριν την δώσετε στη μηχανή  
- Εφαρμογή φίλτρου median για μείωση των σπινθηρισμάτων  
- Διασφάλιση ότι το κείμενο είναι οριζόντια ευθυγραμμισμένο (περιστροφή αν χρειάζεται)

### Μεγάλες Εικόνες

Η επεξεργασία μιας φωτογραφίας 10 MP μπορεί να είναι αργή. Μειώστε το μέγεθος σε μέγιστο πλάτος 2000 px διατηρώντας την αναλογία διαστάσεων· η μηχανή θα εξακολουθήσει να καταγράφει τους περισσότερους χαρακτήρες με ακρίβεια.

## Επέκταση του Παραδείγματος

- **Batch processing:** Τυλίξτε τη λογική αναγνώρισης σε βρόχο που επαναλαμβάνει όλα τα αρχεία σε έναν κατάλογο.  
- **Output formats:** Το `OcrResult` παρέχει επίσης μια συλλογή `TextLines` που περιλαμβάνει πλαίσια περιορισμού—χρήσιμο για επισήμανση κειμένου σε εφαρμογές UI.  
- **Additional languages:** Απλώς καλέστε το `LoadLanguage` με οποιαδήποτε άλλη υποστηριζόμενη τιμή enum (π.χ., `Language.French`).  

Όλες αυτές οι επεκτάσεις εξακολουθούν να ακολουθούν την αρχή **πώς να εξάγετε κείμενο**—απλώς φορτώστε τα κατάλληλα πακέτα γλώσσας και αφήστε τη μηχανή να κάνει το υπόλοιπο.

## Πλήρης Επανάληψη Κώδικα

Παρακάτω βρίσκεται το πλήρες πρόγραμμα, έτοιμο για αντιγραφή‑επικόλληση. Αντικαταστήστε το `YOUR_DIRECTORY` με την πραγματική διαδρομή στον υπολογιστή σας.

```csharp
using Aspose.OCR;
using Aspose.OCR.Enums;

class OfflineDemo
{
    static void Main()
    {
        // Step 1: Create an OCR engine instance
        OcrEngine ocrEngine = new OcrEngine();

        // Step 2: Point the engine to the folder that contains OCR model files
        ocrEngine.Settings.ResourcesPath = @"YOUR_DIRECTORY/AsposeOCRResources";

        // Step 3: Load the languages you need (no automatic download will occur)
        ocrEngine.LoadLanguage(Language.English);
        ocrEngine.LoadLanguage(Language.Cyrillic);

        // Step 4: Load the image you want to recognize
        ImageStream image = ImageStream.FromFile(@"YOUR_DIRECTORY/cyrillic_doc.jpg");

        // Step 5: Run the recognition process
        OcrResult ocrResult = ocrEngine.Recognize(image);

        // Step 6: Display the extracted text
        Console.WriteLine("=== OCR Output ===");
        Console.WriteLine(ocrResult.Text);
    }
}
```

Αποθηκεύστε το ως `Program.cs`, εκτελέστε `dotnet run` και παρακολουθήστε την κονσόλα να εκτυπώνει το κείμενο που η μηχανή αντλήθηκε από την εικόνα σας. Αυτό είναι—έχετε εξαγάγει επιτυχώς **κείμενο από εικόνα** offline.
