# Πώς να Εκτελέσετε OCR σε C# – Πλήρες Προγραμματιστικό Εγχειρίδιο

Έχετε αναρωτηθεί ποτέ **πώς να εκτελέσετε OCR** σε μια φωτογραφία που μόλις τραβήξατε με το τηλέφωνό σας; Ίσως χρειάζεστε το κείμενο από μια πινακίδα δρόμου σε ένα JPEG για μια εφαρμογή πλοήγησης, ή έχετε μια σειρά σαρωμένων συμβάσεων και θέλετε να τις μετατρέψετε σε αναζητήσιμο κείμενο. Συνοπτικά, θέλετε να *εξάγετε κείμενο από εικόνα* χωρίς να στέλνετε τίποτα στο σύννεφο.

Το καλό νέο είναι ότι μπορείτε να κάνετε όλα αυτά τοπικά με το Aspose.OCR για .NET. Σε αυτό το tutorial θα περάσουμε από τη φόρτωση μιας εικόνας από αρχείο, την αναγνώριση κειμένου από JPG, και τελικά **να τρέξετε OCR σε εικόνα** εντελώς εκτός σύνδεσης. Στο τέλος θα έχετε ένα έτοιμο κομμάτι κώδικα που εκτυπώνει το αναγνωρισμένο αραβικό κείμενο στην κονσόλα.

> **Τι θα πάρετε:** ένα αυτόνομο, εκτελέσιμο πρόγραμμα C#, εξηγήσεις για το γιατί κάθε γραμμή είναι σημαντική, και συμβουλές για την αντιμετώπιση κοινών περιπτώσεων όπως ελλιπείς πόροι ή μη υποστηριζόμενες γλώσσες.

## Προαπαιτούμενα

Πριν βουτήξουμε, βεβαιωθείτε ότι έχετε:

| Απαίτηση | Λόγος |
|----------|-------|
| .NET 6.0 ή νεότερο (ή .NET Framework 4.7+) | Το Aspose.OCR στοχεύει στο .NET Standard 2.0, οπότε οποιοδήποτε σύγχρονο runtime λειτουργεί. |
| Visual Studio 2022 (ή VS Code με επέκταση C#) | Ένα IDE κάνει εύκολη τη διαχείριση των πακέτων NuGet και την εκτέλεση της κονσόλας. |
| Πακέτο NuGet Aspose.OCR (`Aspose.OCR`) | Αυτή είναι η βιβλιοθήκη που πραγματικά εκτελεί την εργασία OCR. |
| Ένας φάκελος που περιέχει τους offline πόρους OCR (λήψη από το Aspose) | Οι offline πόροι αποφεύγουν οποιεσδήποτε κλήσεις HTTP κατά την αναγνώριση. |
| Ένα αρχείο εικόνας (π.χ., `arabic_sign.jpg`) | Θα χρησιμοποιήσουμε ένα JPEG που περιέχει αραβικό κείμενο, αλλά λειτουργεί με οποιαδήποτε γλώσσα. |

Αν λείπει κάτι από τα παραπάνω, κατεβάστε το τώρα — δεν υπάρχει νόημα να ξεκινήσετε ένα tutorial μόνο και να συναντήσετε έλλειψη εξαρτήσεων στη μέση.

## Βήμα 1: Εγκατάσταση Aspose.OCR και Προετοιμασία Πόρων

Πρώτα, προσθέστε το πακέτο Aspose.OCR στο έργο σας:

```bash
dotnet add package Aspose.OCR
```

Αφού εγκατασταθεί το πακέτο, κατεβάστε το **offline OCR resource bundle** από την ιστοσελίδα του Aspose. Αποσυμπιέστε το σε φάκελο στον υπολογιστή σας, για παράδειγμα:

```
C:\OCRResources\
```

> **Γιατί είναι σημαντικό:** Η φόρτωση των πόρων μία φορά κατά την εκκίνηση εξαλείφει την καθυστέρηση δικτύου και κρατά τη λύση σας φιλική προς το GDPR, επειδή τίποτα δεν φεύγει από το μηχάνημα.

## Βήμα 2: Δημιουργία του OCR Engine και Σύνδεση με τον Φάκελο Πόρων

Τώρα θα δημιουργήσουμε ένα αντικείμενο της κλάσης `Engine` και θα του πούμε πού βρίσκονται οι πόροι. Αυτό είναι η καρδιά του **πώς να εκτελέσετε OCR** τοπικά.

```csharp
using Aspose.OCR;
using Aspose.OCR.Settings;

// Step 2: Initialize the OCR engine with the offline resource folder
Engine ocrEngine = new Engine
{
    // Replace with the path you used in the previous step
    ResourceFolder = @"C:\OCRResources"
};

// Load the resources into memory – this prevents any hidden HTTP calls
ocrEngine.LoadResources();
```

> **Pro tip:** Τυλίξτε την κλήση `LoadResources` σε block `try‑catch` αν υπάρχει πιθανότητα το μονοπάτι του φακέλου να είναι λανθασμένο. Η εξαίρεση θα σας πει ακριβώς ποιο αρχείο λείπει.

## Βήμα 3: Φόρτωση της Εικόνας από Αρχείο

Στη συνέχεια, πρέπει να **φορτώσετε εικόνα από αρχείο** ώστε η μηχανή να την αναλύσει. Το Aspose.OCR δουλεύει με το δικό του wrapper `ImageStream`.

```csharp
// Step 3: Load the JPEG you want to recognize
ImageStream image = ImageStream.FromFile(@"C:\OCRResources\arabic_sign.jpg");
```

Αν η εικόνα σας βρίσκεται κάπου αλλού, απλώς αλλάξτε τη διαδρομή. Η κλάση `ImageStream` αφαιρεί την ανάγκη να ασχοληθείτε με το υποκείμενο bitmap, οπότε δεν χρειάζεται να ανησυχείτε για τη συμβατότητα με GDI+.

## Βήμα 4: Αναγνώριση Κειμένου από JPG Χρησιμοποιώντας Ρυθμίσεις Γλώσσας

Τώρα έρχεται ο πυρήνας του **πώς να εκτελέσετε OCR** — η πραγματική αναγνώριση χαρακτήρων. Θα ζητήσουμε αναγνώριση αραβικών, αλλά μπορείτε να αντικαταστήσετε το `Language.Arabic` με οποιαδήποτε άλλη υποστηριζόμενη γλώσσα.

```csharp
// Step 4: Run OCR specifying the desired language (Arabic in this example)
OcrResult ocrResult = ocrEngine.Recognize(
    image,
    new OcrOptions { Language = Language.Arabic }
);
```

> **Γιατί να καθορίσετε γλώσσα;** Η μηχανή OCR χρησιμοποιεί λεξικά και μοντέλα χαρακτήρων ειδικά για κάθε γλώσσα. Η σωστή γλώσσα βελτιώνει δραματικά την ακρίβεια, ειδικά για γραφές με σύνθετα σχήματα όπως η αραβική.

## Βήμα 5: Εμφάνιση του Εξαγόμενου Κειμένου

Τέλος, ας **εξάγουμε κείμενο από εικόνα** και ας το εκτυπώσουμε. Αυτός είναι ο πιο απλός τρόπος να επαληθεύσετε ότι το OCR πέτυχε.

```csharp
// Step 5: Output the recognized text to the console
Console.WriteLine("=== OCR Result ===");
Console.WriteLine(ocrResult.Text);
```

Όταν τρέξετε το πρόγραμμα, θα δείτε τη φράση στα αραβικά που υπήρχε στην πινακίδα να εμφανίζεται στην κονσόλα. Αν η έξοδος φαίνεται «σπασμένη», ελέγξτε ξανά ότι η σωστή γλώσσα έχει επιλεγεί και ότι ο φάκελος πόρων περιέχει τα αρχεία δεδομένων για τα αραβικά.

## Πλήρες Παράδειγμα Λειτουργίας

Παρακάτω είναι το ολοκληρωμένο, έτοιμο‑για‑μεταγλώττιση πρόγραμμα που ενώνει όλα τα βήματα. Αντιγράψτε‑και‑επικολλήστε το σε ένα νέο έργο κονσόλας (`dotnet new console`) και πατήστε **F5**.

```csharp
using System;
using Aspose.OCR;
using Aspose.OCR.Settings;

namespace AsposeOcrDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // -------------------------------------------------
            // Step 1: Configure the OCR engine with offline resources
            // -------------------------------------------------
            Engine ocrEngine = new Engine
            {
                ResourceFolder = @"C:\OCRResources" // <-- change to your folder
            };
            ocrEngine.LoadResources(); // Loads language packs, fonts, etc.

            // -------------------------------------------------
            // Step 2: Load the image you want to process
            // -------------------------------------------------
            ImageStream image = ImageStream.FromFile(@"C:\OCRResources\arabic_sign.jpg");

            // -------------------------------------------------
            // Step 3: Recognize text – specify language to improve accuracy
            // -------------------------------------------------
            OcrResult ocrResult = ocrEngine.Recognize(
                image,
                new OcrOptions { Language = Language.Arabic } // Change as needed
            );

            // -------------------------------------------------
            // Step 4: Output the result
            // -------------------------------------------------
            Console.WriteLine("=== OCR Result ===");
            Console.WriteLine(ocrResult.Text);
        }
    }
}
```

**Αναμενόμενη έξοδος (παράδειγμα):**

```
=== OCR Result ===
مطار القاهره الدولي
```

Αν αντικαταστήσετε την εικόνα με μια αγγλική πινακίδα και ορίσετε `Language.English`, ο ίδιος κώδικας θα εμφανίσει το αγγλικό κείμενο. Αυτό δείχνει πόσο ευέλικτο μπορεί να είναι το **run OCR on image**.

## Εξαγωγή Κειμένου από Εικόνα – Διαχείριση Συνηθισμένων Σεναρίων

### 1. Πολλές Σελίδες ή Πολυ‑πλαισίου Εικόνες

Ορισμένες μορφές εικόνας (όπως TIFF) μπορούν να περιέχουν πολλές σελίδες. Για **εξαγωγή κειμένου από εικόνα** σε τέτοιες περιπτώσεις, επαναλάβετε τη διαδικασία για κάθε πλαίσιο:

```csharp
for (int i = 0; i < image.FramesCount; i++)
{
    image.CurrentFrame = i;
    OcrResult pageResult = ocrEngine.Recognize(image, new OcrOptions { Language = Language.English });
    Console.WriteLine($"Page {i + 1}: {pageResult.Text}");
}
```

### 2. Εικόνες Χαμηλής Ανάλυσης

Η ακρίβεια του OCR πέφτει δραματικά κάτω από 70 dpi. Αν αντιμετωπίσετε ασαφή αποτελέσματα, σκεφτείτε να αυξήσετε την ανάλυση της εικόνας πρώτα:

```csharp
using System.Drawing;
using System.Drawing.Drawing2D;

// Load as Bitmap, resize, then wrap back into ImageStream
Bitmap bmp = new Bitmap(@"C:\OCRResources\lowres.jpg");
Bitmap highRes = new Bitmap(bmp, new Size(bmp.Width * 2, bmp.Height * 2));
ImageStream highResStream = ImageStream.FromBitmap(highRes);
```

### 3. Έλλειψη Πακέτου Γλώσσας

Αν λάβετε εξαίρεση όπως *“Language data not found”*, ελέγξτε ξανά ότι τα αντίστοιχα αρχεία `.dat` υπάρχουν στο `ResourceFolder`. Το Aspose παρέχει ξεχωριστό zip για κάθε γλώσσα.

## Run OCR on Image – Συμβουλές Απόδοσης

- **Cache το Engine:** Η δημιουργία νέου `Engine` για κάθε εικόνα προσθέτει επιπλέον κόστος. Κρατήστε ένα μόνο αντικείμενο ζωντανό για επεξεργασία παρτίδας.
- **Παράλληλη Εκτέλεση με Ασφάλεια:** Το `Engine` είναι thread‑safe για λειτουργίες μόνο‑ανάγνωσης μετά το `LoadResources`. Μπορείτε να δημιουργήσετε πολλαπλά tasks που κάθε ένα καλεί `Recognize` σε διαφορετικές εικόνες.
- **Αποδέσμευση Όταν Τελειώσετε:** Παρόλο που το `Engine` υλοποιεί `IDisposable`, ο GC του .NET θα καθαρίσει τελικά. Η ρητή κλήση `ocrEngine.Dispose()` μέσα σε `using` block είναι καλή πρακτική.

```csharp
using (Engine ocrEngine = new Engine { ResourceFolder = @"C:\OCRResources" })
{
    ocrEngine.LoadResources();
    // run recognitions here
}
```

## Recognize Text from JPG – Περιπτώσεις που Πρέπει να Προσέξετε

| Κατάσταση | Τι να Ελέγξετε | Διόρθωση |
|-----------|----------------|----------|
| **Κατεστραμμένο JPEG** | `ImageStream.FromFile` ρίχνει `FileNotFoundException` ή `ArgumentException`. | Επαληθεύστε την ακεραιότητα του αρχείου, ίσως ξανα‑αποθηκεύστε την εικόνα με πρόγραμμα επεξεργασίας. |
| **Μη υποστηριζόμενη γλώσσα** | Το enum `Language` δεν περιέχει τη γλώσσα-στόχο σας. | Αναβαθμίστε το Aspose.OCR στην πιο πρόσφατη έκδοση· νέες γλώσσες προστίθενται τακτικά. |
| **Εικόνες μικτής γραφής** (π.χ., Αγγλικά + Αραβικά) | Μία μόνο επιλογή γλώσσας μπορεί να παραλείψει το δευτερεύον σενάριο. | Εκτελέστε OCR δύο φορές με διαφορετικές γλώσσες και συνδυάστε τα αποτελέσματα. |
