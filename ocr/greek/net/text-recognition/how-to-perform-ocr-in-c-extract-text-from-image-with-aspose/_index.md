# Πώς να Εκτελέσετε OCR σε C# – Εξαγωγή Κειμένου από Εικόνα με το Aspose OCR

Έχετε αναρωτηθεί ποτέ **πώς να εκτελέσετε OCR** σε μια σαρωμένη απόδειξη ή σε μια φωτογραφία πινακίδας; Δεν είστε οι μόνοι. Σε πολλά πραγματικά έργα χρειάζεται **να εξάγετε κείμενο από εικόνα**, είτε πρόκειται για απόδειξη, σάρωση διαβατηρίου ή χειρόγραφο σημείωμα. Το καλό νέο; Με το Aspose.OCR μπορείτε να το κάνετε με λίγες γραμμές κώδικα C#, και θα μάθετε ακόμη και πώς να **αναγνωρίζετε κείμενο στα Χίντι** ενώ το κάνετε.

Σε αυτό το tutorial θα περάσουμε από ένα πλήρες, έτοιμο‑για‑εκτέλεση παράδειγμα που **διαβάζει κείμενο από εικόνα**, σας δείχνει πώς να **εξάγετε κείμενο από εικόνα** χρησιμοποιώντας τη μηχανή Aspose OCR, και εξηγεί το “γιατί” πίσω από κάθε βήμα. Χωρίς ασαφείς αναφορές σε εξωτερικά έγγραφα—απλώς μια αυτόνομη λύση που μπορείτε να αντιγράψετε‑επικολλήσετε και να τρέξετε σήμερα.

## Τι Θα Χρειαστείτε

- .NET 6.0 ή νεότερο (ο κώδικας μεταγλωττίζεται επίσης κατά .NET Standard 2.0)
- Visual Studio 2022 (ή οποιοδήποτε IDE προτιμάτε)
- Πακέτο NuGet Aspose.OCR (`Install-Package Aspose.OCR`)
- Ένα αρχείο εικόνας που περιέχει κείμενο στα Χίντι (π.χ., `hindi_invoice.jpg`)
- Ο φάκελος πόρων γλώσσας OCR που παρέχεται με το Aspose (κατεβάστε τον από τον ιστότοπο Aspose)

> **Συμβουλή:** Κρατήστε το φάκελο πόρων OCR δίπλα στο έργο σας για εύκολη διαχείριση διαδρομών.

## Υλοποίηση Βήμα‑Βήμα

Παρακάτω χωρίζουμε τη διαδικασία σε έξι λογικά βήματα. Κάθε βήμα έχει τη δική του επικεφαλίδα H2 (ώστε οι μηχανές αναζήτησης και τα μοντέλα AI να εντοπίζουν γρήγορα την πληροφορία) και έναν σύντομο υπότιτλο H3 που περιλαμβάνει φυσικά δευτερεύουσες λέξεις‑κλειδιά.

### Βήμα 1 – Ορισμός Διαδρομής Πόρων OCR  
**Γιατί είναι σημαντικό:** Το Aspose.OCR βασίζεται σε πακέτα γλώσσας (γραμματοσειρές, λεξικά και αρχεία μοντέλων) που βρίσκονται σε φάκελο που του υποδεικνύετε. Αν η διαδρομή είναι λανθασμένη, η μηχανή πετάει `FileNotFoundException` και δεν θα πάρετε κανένα κείμενο από την εικόνα.

```csharp
using Aspose.OCR;
using Aspose.OCR.Models;

// Tell the OCR engine where to find language resources
OcrEngine.SetResourcesPath(@"C:\MyProject\OcrResources");
```

### Βήμα 2 – Δημιουργία Αντικειμένου OcrEngine  
**Γιατί είναι σημαντικό:** Το `OcrEngine` είναι το βαρέως τύπου αντικείμενο που κρατά το μοντέλο αναγνώρισης στη μνήμη. Η τοποθέτησή του σε μπλοκ `using` εγγυάται σωστή αποδέσμευση, κάτι που είναι ιδιαίτερα σημαντικό όταν επεξεργάζεστε πολλές εικόνες σε παρτίδα.

```csharp
// Instantiate the OCR engine inside a using block
using (var ocrEngine = new OcrEngine())
{
    // Subsequent steps go here...
}
```

### Βήμα 3 – Ρύθμιση Παραμέτρων Αναγνώρισης (Επιλογή Χίντι)  
**Γιατί είναι σημαντικό:** Από προεπιλογή το Aspose προσπαθεί να εντοπίσει αυτόματα τη γλώσσα, αλλά ο ρητός ορισμός του `Language.Hindi` βελτιώνει την ακρίβεια για τα γράμματα Devanagari και επιταχύνει την επεξεργασία.

```csharp
    // Configure the engine to recognize Hindi text
    var recognitionSettings = new RecognitionSettings
    {
        Language = Language.Hindi   // Recognize Hindi characters
    };
```

### Βήμα 4 – Φόρτωση της Εικόνας που Θέλετε να Διαβάσετε  
**Γιατί είναι σημαντικό:** Το `ImageStream.FromFile` αφαιρεί την ανάγκη χειρισμού του bitmap και μεταδίδει τα δεδομένα αποδοτικά. Μπορείτε επίσης να χρησιμοποιήσετε `MemoryStream` αν η εικόνα προέρχεται από αίτημα web.

```csharp
    // Load the target image (replace with your own path)
    var imageStream = ImageStream.FromFile(@"C:\MyProject\Images\hindi_invoice.jpg");
```

### Βήμα 5 – Εκτέλεση της Διαδικασίας OCR  
**Γιατί είναι σημαντικό:** Η μέθοδος `Recognize` εκτελεί το βαρύ έργο—προεπεξεργασία, τμηματοποίηση, ταξινόμηση χαρακτήρων και τελικά συναρμολόγηση κειμένου. Επιστρέφει μια απλή συμβολοσειρά που μπορείτε να αποθηκεύσετε, εμφανίσετε ή να επεξεργαστείτε περαιτέρω.

```csharp
    // Execute OCR and capture the recognized text
    string recognizedText = ocrEngine.Recognize(imageStream, recognitionSettings);
```

### Βήμα 6 – Εμφάνιση του Εξαγόμενου Κειμένου  
**Γιατί είναι σημαντικό:** Για γρήγορο debugging συνήθως θέλετε να γράψετε το αποτέλεσμα στην κονσόλα ή σε αρχείο καταγραφής. Σε παραγωγή μπορεί να το αποθηκεύσετε σε βάση δεδομένων ή να το περάσετε σε επόμενη ροή εργασίας.

```csharp
    // Output the result to the console
    Console.WriteLine("=== OCR Result ===");
    Console.WriteLine(recognizedText);
}
```

## Πλήρες Παράδειγμα Εργασίας

Συνδυάζοντας όλα τα παραπάνω, εδώ είναι το ολοκληρωμένο πρόγραμμα που μπορείτε να ενσωματώσετε σε ένα έργο console app. Φροντίστε να αντικαταστήσετε τις διαδρομές placeholder με τις πραγματικές σας.

```csharp
using System;
using Aspose.OCR;
using Aspose.OCR.Models;

class Program
{
    static void Main()
    {
        // 1️⃣ Set the folder that contains language resources
        OcrEngine.SetResourcesPath(@"C:\MyProject\OcrResources");

        // 2️⃣ Create the OCR engine (auto‑disposed)
        using (var ocrEngine = new OcrEngine())
        {
            // 3️⃣ Tell the engine to look for Hindi characters
            var recognitionSettings = new RecognitionSettings
            {
                Language = Language.Hindi
            };

            // 4️⃣ Load the image you want to read
            var imageStream = ImageStream.FromFile(@"C:\MyProject\Images\hindi_invoice.jpg");

            // 5️⃣ Perform OCR
            string recognizedText = ocrEngine.Recognize(imageStream, recognitionSettings);

            // 6️⃣ Show the extracted text
            Console.WriteLine("=== OCR Result ===");
            Console.WriteLine(recognizedText);
        }

        // Keep console open (useful when running from VS)
        Console.WriteLine("\nPress any key to exit...");
        Console.ReadKey();
    }
}
```

**Αναμενόμενη έξοδος** (συνοπτικά):

```
=== OCR Result ===
Invoice No: 12345
Date: 01/01/2024
Amount: ₹ 15,000
धन्यवाद
```

Αν δείτε ακατανόητο κείμενο αντί για χαρακτήρες Χίντι, ελέγξτε ξανά ότι ο φάκελος πόρων δείχνει στη σωστή έκδοση του πακέτου γλώσσας Χίντι.

## Συνηθισμένα Προβλήματα & Πώς να τα Αντιμετωπίσετε  

| Πρόβλημα | Γιατί Συμβαίνει | Γρήγορη Λύση |
|----------|----------------|--------------|
| **Ακατανόητοι χαρακτήρες** | Λάθος ή ελλιπείς πόροι γλώσσας | Επαληθεύστε ότι το `SetResourcesPath` δείχνει στον φάκελο που περιέχει το `Hindi.cognates` και τα σχετικά αρχεία |
| **Σφάλματα έλλειψης μνήμης** | Φόρτωση πολύ μεγάλης εικόνας χωρίς κλιμάκωση | Χρησιμοποιήστε `ImageStream.FromFile(..., maxWidth: 2000)` για να μειώσετε το μέγεθος κατά τη φόρτωση |
| **Αργή απόδοση** | Λειτουργία αυτόματης ανίχνευσης που σαρώνει πολλές γλώσσες | Ορίστε ρητά `Language = Language.Hindi` (ή οποιαδήποτε άλλη στόχευση) |
| **Καμία έξοδος** | Η εικόνα είναι εντελώς λευκή ή θολή | Προεπεξεργαστείτε την εικόνα (αντίθεση, δυαδικοποίηση) πριν τη δώσετε στο OCR |

## Επέκταση της Λύσης: Άλλες Γλώσσες & Σενάρια  

Αν χρειάζεστε να **διαβάσετε κείμενο από εικόνα** στα Αγγλικά, Ισπανικά ή οποιαδήποτε άλλη γλώσσα, απλώς αλλάξτε το enum `Language`:

```csharp
recognitionSettings.Language = Language.English;   // or Language.Spanish, etc.
```

Μπορείτε επίσης να συνδυάσετε πολλαπλές γλώσσες:

```csharp
recognitionSettings.Language = Language.English | Language.Hindi;
```

Για επεξεργασία παρτίδας, τυλίξτε το μπλοκ `using (var ocrEngine = new OcrEngine())` γύρω από έναν βρόχο `foreach` που διατρέχει έναν φάκελο εικόνων. Η μηχανή θα επαναχρησιμοποιήσει το φορτωμένο μοντέλο, μειώνοντας δραστικά το κόστος εκκίνησης.

## Δοκιμή της Ακρίβειας του OCR  

1. Εκτελέστε το πρόγραμμα με μια γνωστή δοκιμαστική εικόνα (μπορείτε να δημιουργήσετε ένα απλό PNG με κείμενο Χίντι χρησιμοποιώντας οποιονδήποτε επεξεργαστή γραφικών).  
2. Συγκρίνετε την έξοδο της κονσόλας με το αρχικό κείμενο.  
3. Αν το ποσοστό σφάλματος είναι μεγαλύτερο από 5 %, σκεφτείτε να βελτιώσετε την ποιότητα της εικόνας (αυξήστε το DPI στα 300 dpi) ή να εφαρμόσετε βήμα προεπεξεργασίας όπως `imageStream = imageStream.ApplyGaussianBlur(1.5)` (το Aspose παρέχει βασικά φίλτρα).
