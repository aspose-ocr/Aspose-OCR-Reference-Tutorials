# Δημιουργία Αναζητήσιμου PDF από TIF – Πλήρης Οδηγός C#

Έχετε ποτέ χρειαστεί να **δημιουργήσετε αναζητήσιμο PDF** από ένα σαρωμένο τιμολόγιο αλλά δεν ήξερες πώς να διατηρήσεις το κείμενο αναζητήσιμο και τα μεταδεδομένα του αρχείου τακτοποιημένα; Δεν είστε μόνοι. Σε πολλά έργα αυτοματοποίησης το PDF πρέπει να είναι τόσο μηχανικά αναγνώσιμο όσο και σωστά ετικετοποιημένο με πληροφορίες όπως τίτλος, συγγραφέας και όρια εμπιστοσύνης.  

Σε αυτόν τον οδηγό θα περάσουμε βήμα‑βήμα από μια πλήρη λύση που **μετατρέπει μια εικόνα σε PDF**, ενσωματώνει **μεταδεδομένα PDF**, και φιλτράρει αποτελέσματα OCR χαμηλής εμπιστοσύνης — όλα με τη βιβλιοθήκη Aspose.OCR. Στο τέλος θα έχετε ένα έτοιμο κομμάτι κώδικα που μπορείτε να ενσωματώσετε σε οποιοδήποτε έργο .NET, καθώς και μερικές συμβουλές για ειδικές περιπτώσεις.

## Προαπαιτούμενα

- .NET 6.0 ή νεότερο (ο κώδικας λειτουργεί επίσης σε .NET Framework 4.6+)  
- Πακέτο NuGet Aspose.OCR (`Install-Package Aspose.OCR`)  
- Μια εικόνα TIF που θέλετε να μετατρέψετε σε αναζητήσιμο PDF (π.χ., `input.tif`)  
- Βασική εξοικείωση με C# και Visual Studio (ή το αγαπημένο σας IDE)

Δεν απαιτούνται άλλες εξωτερικές υπηρεσίες — όλη η διαδικασία εκτελείται τοπικά.

## Βήμα 1 – Ρύθμιση του OCR Engine (Create Searchable PDF Core)

Το πρώτο που χρειαζόμαστε είναι μια παρουσία `OcrEngine` που ξέρει ποια γλώσσα να αναγνωρίσει. Στις περισσότερες επιχειρηματικές περιπτώσεις η αγγλική είναι αρκετή, αλλά μπορείτε να αντικαταστήσετε το `Language.English` με οποιαδήποτε υποστηριζόμενη γλώσσα.

```csharp
using Aspose.OCR;
using Aspose.OCR.Pdf;   // PDF‑related extensions

// Initialize OCR engine with English language
OcrEngine ocrEngine = new OcrEngine
{
    Language = Language.English
};
```

**Γιατί είναι σημαντικό:** Η μηχανή OCR κάνει το βαριά έργο της μετατροπής των εικονοστοιχείων σε κείμενο Unicode. Η σωστή ρύθμιση της γλώσσας βελτιώνει την ακρίβεια και μειώνει τον αριθμό των λέξεων χαμηλής εμπιστοσύνης που αργότερα φιλτράρονται.

> **Pro tip:** Αν επεξεργάζεστε πολυγλωσσικά έγγραφα, δημιουργήστε πολλαπλές μηχανές ή χρησιμοποιήστε το `Language.Multilingual` ώστε το Aspose να εντοπίζει αυτόματα τη γλώσσα.

## Βήμα 2 – Φόρτωση της Εικόνας TIF (Create PDF from TIF)

Τώρα κατευθύνουμε τη μηχανή στο αρχείο προέλευσης. Η βοηθητική μέθοδος `ImageStream.FromFile` διαβάζει την εικόνα σε ροή που μπορεί να καταναλώσει η μηχανή OCR.

```csharp
// Load the TIF image you want to make searchable
ocrEngine.Image = ImageStream.FromFile(@"YOUR_DIRECTORY/input.tif");
```

Μπορεί να αναρωτιέστε, *«Μπορώ να τροφοδοτήσω JPEG ή PNG αντί αυτού;»* Απόλυτα — το Aspose.OCR υποστηρίζει τις περισσότερες μορφές raster. Απλώς αλλάξτε την επέκταση του αρχείου και είστε έτοιμοι.

## Βήμα 3 – Διαμόρφωση των PDF Options (Add Metadata to PDF)

Πριν τη μετατροπή, ορίζουμε ένα αντικείμενο `PdfOptions`. Εδώ **προσθέτουμε μεταδεδομένα PDF** όπως τίτλο και συγγραφέα, και επίσης λέμε στη μηχανή να απορρίψει λέξεις των οποίων η εμπιστοσύνη είναι κάτω από ένα όριο.

```csharp
PdfOptions searchablePdfOptions = new PdfOptions
{
    Title = "Invoice #12345",
    Author = "MyCompany",
    // Hide words with confidence lower than 0.8 (80%)
    MinConfidence = 0.8f
};
```

**Τι συμβαίνει εδώ;**  
- `Title` και `Author` γίνονται μέρος του λεξικού πληροφοριών του εγγράφου PDF, κάνοντας το αρχείο πιο εύκολο στην ευρετηρίαση από συστήματα διαχείρισης εγγράφων.  
- `MinConfidence` λειτουργεί ως φράγμα προστασίας: το OCR συχνά διαβάζει λανθασμένα αχνά χαρακτήρες· το φιλτράρισμα τους διατηρεί το επίπεδο αναζήτησης καθαρό και βελτιώνει την εξαγωγή κειμένου σε επόμενα βήματα.

Αν χρειάζεστε προσαρμοσμένα μεταδεδομένα (π.χ., `Subject`, `Keywords`), μπορείτε να τα ορίσετε μέσω `PdfOptions.CustomProperties`.

## Βήμα 4 – Μετατροπή της Εικόνας σε Αναζητήσιμο PDF (Convert Image to PDF)

Με τη μηχανή έτοιμη και τις επιλογές ορισμένες, η τελική κλήση εκτελεί τη μετατροπή. Γράφει ένα αναζητήσιμο PDF στο δίσκο, ενσωματώνοντας το επίπεδο κειμένου OCR κάτω από την αρχική εικόνα.

```csharp
ocrEngine.ConvertToSearchablePdf(
    outputPath: @"YOUR_DIRECTORY/output.pdf",
    pdfOptions: searchablePdfOptions);
```

Μετά την εκτέλεση αυτής της γραμμής, το `output.pdf` περιέχει:
- Την αρχική εικόνα TIF ως οπτικό υπόβαθρο  
- Ένα αόρατο επίπεδο κειμένου που μπορούν να διαβάσουν οι μηχανές αναζήτησης και οι λειτουργίες αντιγραφής‑επικόλλησης  
- Τα μεταδεδομένα που ορίσαμε νωρίτερα (τίτλος, συγγραφέας κ.λπ.)

### Αναμενόμενο Αποτέλεσμα

Ανοίξτε το παραγόμενο PDF σε Adobe Reader ή οποιονδήποτε προβολέα PDF και δοκιμάστε να επιλέξετε κείμενο. Θα πρέπει να μπορείτε να αντιγράψετε προτάσεις που ταιριάζουν με το αρχικό σκανάρισμα. Ο διάλογος **Properties** του εγγράφου θα εμφανίσει τον τίτλο “Invoice #12345” και τον συγγραφέα “MyCompany”.

## Βήμα 5 – Επαλήθευση Φιλτραρίσματος Εμπιστοσύνης (Why MinConfidence Helps)

Είναι χρήσιμο να επιβεβαιώσετε ότι οι λέξεις χαμηλής εμπιστοσύνης έχουν πράγματι αφαιρεθεί. Μπορείτε να εξετάσετε το κρυφό κείμενο του PDF με ένα εργαλείο όπως το `pdftotext`:

```bash
pdftotext output.pdf - | grep -i "unlikelyword"
```

Αν η λέξη δεν εμφανίζεται ποτέ, το φίλτρο `MinConfidence` λειτούργησε όπως προβλέπεται. Προσαρμόστε το όριο (π.χ., `0.7f`) αν χρειάζεστε πιο επιθετικό ή πιο χαλαρό φιλτράρισμα.

## Βήμα 6 – Διαχείριση Πολλαπλών Σελίδων (Create Searchable PDF from Multi‑Page TIF)

Πολλά τιμολόγια έρχονται ως πολυσελιδικά TIFF. Το Aspose.OCR αντιμετωπίζει κάθε καρέ ως ξεχωριστή σελίδα αυτόματα. Απλώς δείξτε τη μηχανή στο πολυσελιδικό αρχείο· η ίδια κλήση `ConvertToSearchablePdf` θα δημιουργήσει ένα πολυσελιδικό αναζητήσιμο PDF.

```csharp
ocrEngine.Image = ImageStream.FromFile(@"YOUR_DIRECTORY/multi_page.tif");
ocrEngine.ConvertToSearchablePdf(@"YOUR_DIRECTORY/multi_output.pdf", searchablePdfOptions);
```

**Edge case:** Αν μια σελίδα είναι κενή, το OCR μπορεί ακόμα να δημιουργήσει ένα κενό επίπεδο κειμένου, το οποίο δεν προκαλεί πρόβλημα. Ωστόσο, μπορείτε να παραλείψετε τις κενές σελίδες ελέγχοντας το `ocrEngine.HasText` πριν τη μετατροπή.

## Βήμα 7 – Συσκευασία του Πλήρους Παραδείγματος (All Steps Together)

Παρακάτω υπάρχει ένα ενιαίο, έτοιμο‑για‑εκτέλεση πρόγραμμα που ενώνει όλα τα βήματα. Αντικαταστήστε το `YOUR_DIRECTORY` με τη διαδρομή του φακέλου σας.

```csharp
using System;
using Aspose.OCR;
using Aspose.OCR.Pdf;

class Program
{
    static void Main()
    {
        // 1️⃣ Initialize OCR engine
        OcrEngine ocrEngine = new OcrEngine
        {
            Language = Language.English
        };

        // 2️⃣ Load the TIF image (create pdf from tif)
        string inputPath = @"YOUR_DIRECTORY/input.tif";
        ocrEngine.Image = ImageStream.FromFile(inputPath);

        // 3️⃣ Set PDF metadata and confidence filter
        PdfOptions pdfOptions = new PdfOptions
        {
            Title = "Invoice #12345",
            Author = "MyCompany",
            MinConfidence = 0.8f
        };

        // 4️⃣ Convert to searchable PDF (convert image to pdf)
        string outputPath = @"YOUR_DIRECTORY/output.pdf";
        ocrEngine.ConvertToSearchablePdf(outputPath, pdfOptions);

        Console.WriteLine($"✅ Searchable PDF created at: {outputPath}");
    }
}
```

Τρέξτε το πρόγραμμα (`dotnet run` ή πατήστε **F5** στο Visual Studio) και θα δείτε το μήνυμα επιβεβαίωσης. Το παραγόμενο PDF βρίσκεται δίπλα στην πηγαία εικόνα, έτοιμο για ευρετηρίαση ή αρχειοθέτηση.

## Συχνές Ερωτήσεις & Παγίδες

| Ερώτηση | Απάντηση |
|----------|--------|
| **Μπορώ να προσθέσω προσαρμοσμένη γραμματοσειρά στο επίπεδο αναζήτησης;** | Το επίπεδο OCR χρησιμοποιεί Unicode· η οπτική εμφάνιση καθορίζεται από την υποκείμενη εικόνα, οπότε δεν απαιτείται ενσωμάτωση επιπλέον γραμματοσειράς. |
| **Τι γίνεται αν το TIF μου είναι έγχρωμο;** | Το Aspose.OCR μετατρέπει αυτόματα τις έγχρωμες εικόνες σε γκρι κλίμακα για OCR, αλλά μπορείτε να διατηρήσετε τα αρχικά χρώματα στο PDF ορίζοντας `PdfOptions.PreserveColor = true`. |
| **Είναι η βιβλιοθήκη thread‑safe;** | Κάθε παρουσία `OcrEngine` πρέπει να χρησιμοποιείται από ένα μόνο νήμα. Για παράλληλη επεξεργασία, δημιουργήστε ξεχωριστή μηχανή ανά νήμα. |
| **Πώς κρυπτογραφώ το PDF;** | Χρησιμοποιήστε το `PdfOptions.Security` για να ορίσετε κωδικό πρόσβασης και δικαιώματα μετά τη μετατροπή OCR. |
