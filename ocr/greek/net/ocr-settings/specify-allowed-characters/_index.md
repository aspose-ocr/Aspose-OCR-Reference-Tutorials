# Καθορισμός Επιτρεπόμενων Χαρακτήρων OCR – Χρήση Aspose.OCR για .NET

Σε αυτό το tutorial, θα μάθετε πώς να **specify allowed characters ocr** με Aspose.OCR για .NET, επιτρέποντάς σας να περιορίσετε την έξοδο του OCR μόνο στους χαρακτήρες που χρειάζεστε. Αυτό είναι ιδιαίτερα χρήσιμο όταν πρέπει να **recognize digits image** αρχεία όπως σειριακοί αριθμοί, IDs τιμολογίων ή αλφαριθμητικά τύπου barcode. Θα περάσουμε από τη ρύθμιση, τον κώδικα και μερικά πρακτικά σενάρια ώστε να μπορείτε να εφαρμόσετε τη μέθοδο αμέσως.

## Γρήγορες Απαντήσεις
- **Τι κάνει το “specify allowed characters ocr”;** Περιορίζει το OCR σε ένα προκαθορισμένο σύνολο χαρακτήρων, βελτιώνοντας την ακρίβεια για στοχευμένα δεδομένα.  
- **Ποιοι χαρακτήρες μπορώ να επιτρέψω;** Οποιοσδήποτε συνδυασμός χρειάζεστε — ψηφία, γράμματα ή προσαρμοσμένα σύμβολα (π.χ., “0123456789”).  
- **Γιατί να περιορίσετε τους χαρακτήρες;** Μειώνει τις λανθασμένες αναγνώσεις και επιταχύνει την επεξεργασία όταν το αναμενόμενο σύνολο χαρακτήρων είναι γνωστό.  
- **Χρειάζομαι άδεια;** Μια δωρεάν δοκιμή λειτουργεί για ανάπτυξη· απαιτείται εμπορική άδεια για παραγωγή.  
- **Ποιες εκδόσεις .NET υποστηρίζονται;** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7.

## Τι είναι το “specify allowed characters ocr”;
Όταν το OCR σαρώνει μια εικόνα, προσπαθεί να ταιριάξει κάθε οπτικό μοτίβο με το πλήρες αλφάβητο των πιθανών χαρακτήρων. Με το **specify allowed characters ocr**, λέτε στη μηχανή να αγνοεί ό,τι βρίσκεται εκτός της λευκής λίστας σας, κάτι που βελτιώνει δραστικά την ακρίβεια αναγνώρισης για περιορισμένα σύνολα δεδομένων.

## Γιατί να χρησιμοποιήσετε το Aspose.OCR για την αναγνώριση εικόνων ψηφίων;
Το Aspose.OCR παρέχει ένα καθαρό, ευανάγνωστο API για προγραμματιστές .NET. Η ενσωματωμένη επιλογή `AllowedCharacters` σας επιτρέπει να εστιάσετε σε σενάρια μόνο με ψηφία χωρίς να γράψετε προσαρμοσμένη λογική μετα-επεξεργασίας. Αυτό είναι ιδανικό για:
- Ανάγνωση μετρήσεων μετρητών, αριθμών τιμολογίων ή κωδικών προϊόντων.  
- Επικύρωση δεδομένων που εισάγονται από χρήστες και έχουν καταγραφεί από σαρωμένες φόρμες.  
- Επιτάχυνση επεξεργασίας παρτίδων όπου το σύνολο χαρακτήρων είναι γνωστό εκ των προτέρων.

## Προαπαιτούμενα

Πριν βυθιστείτε στον κώδικα, βεβαιωθείτε ότι έχετε:

- Καλή γνώση ανάπτυξης .NET.

## Πώς να καθορίσετε επιτρεπόμενους χαρακτήρες OCR – Οδηγός βήμα‑βήμα

### Βήμα 1: Ορίστε τη διαδρομή του φακέλου εικόνων σας

Πρώτα, ορίστε πού αποθηκεύονται οι δείγμα εικόνων σας.

```csharp
string dataDir = "Your Document Directory";
```

### Βήμα 2: Αρχικοποιήστε το Aspose.OCR με λευκή λίστα μόνο ψηφίων

Δημιουργήστε μια παρουσία `AsposeOcr` και περάστε τους χαρακτήρες που θέλετε να επιτρέψετε — σε αυτήν την περίπτωση, όλα τα ψηφία.

```csharp
AsposeOcr api = new AsposeOcr("0123456789");
```

### Βήμα 3: Αναγνωρίστε μια μονή γραμμή που περιέχει ψηφία

Χρησιμοποιήστε τη μέθοδο `RecognizeLine` για να εξάγετε το κείμενο από μια εικόνα που περιέχει μόνο αριθμούς.

```csharp
string result = api.RecognizeLine(dataDir + "0001460985.Jpeg");
```

### Βήμα 4: Εξαγωγή των αναγνωρισμένων ψηφίων

Εκτυπώστε το αποτέλεσμα στην κονσόλα ώστε να μπορείτε να επαληθεύσετε την έξοδο.

```csharp
Console.WriteLine(result);
```

### Βήμα 5: Χρησιμοποιήστε το RecognitionSettings για μεγαλύτερο έλεγχο

Αν χρειάζεστε πιο λεπτομερή έλεγχο — όπως την εξαναγκασμένη αναγνώριση μονής γραμμής — μπορείτε να χρησιμοποιήσετε την υπερφόρτωση που δέχεται `RecognitionSettings`.

```csharp
AsposeOcr api2 = new AsposeOcr();
RecognitionResult result2 = api2.RecognizeImage(dataDir + "0001460985.Jpeg", 
    new RecognitionSettings { 
        AllowedCharacters = CharactersAllowedType.DIGITS,
        RecognizeSingleLine = true
    });
```

### Βήμα 6: Εμφάνιση του αποτελέσματος δεύτερης περίπτωσης

```csharp
Console.WriteLine(result2.RecognitionText);
```

### Βήμα 7: Επιβεβαίωση επιτυχούς εκτέλεσης

```csharp
Console.WriteLine("SpecifyAllowedCharacters executed successfully");
```

Ακολουθώντας αυτά τα βήματα, έχετε μάθει πώς να **specify allowed characters ocr** και αποδοτικά να **recognize digits image** περιεχόμενο χρησιμοποιώντας το Aspose.OCR για .NET.

## Συνηθισμένα προβλήματα και αντιμετώπιση

- **Κενό αποτέλεσμα:** Βεβαιωθείτε ότι η ποιότητα της εικόνας είναι επαρκής (καθαρό αντίθεση, ελάχιστος θόρυβος).  
- **Λάθος χαρακτήρες επιστράφηκαν:** Ελέγξτε ξανά ότι η συμβολοσειρά λευκής λίστας ταιριάζει ακριβώς με τους χαρακτήρες που περιμένετε.  
- **Αρχείο δεν βρέθηκε:** Επαληθεύστε ότι το `dataDir` δείχνει στον σωστό φάκελο και ότι το όνομα αρχείου ταιριάζει με την ακριβή κεφαλοποίηση.
