# Προεπεξεργασία OCR Εικόνας σε C# – Πλήρης Οδηγός με Φίλτρα

Έχετε ποτέ δυσκολευτεί να **προεπεξεργαστείτε OCR εικόνας** επειδή οι σάρωσές σας είναι στραβές, σπόρια ή απλώς δύσκολες στην ανάγνωση; Δεν είστε μόνοι. Σε πολλά πραγματικά έργα, μια τρεμουλή φωτογραφία από απόδειξη ή μια θορυβώδης σαρωμένη φόρμα μπορεί να κάνει τη μηχανή OCR να παράγει ακατανόητο κείμενο αντί για χρήσιμα δεδομένα.  

Τα καλά νέα; Μπορείτε να καθαρίσετε αυτές τις εικόνες πριν τις δώσετε στη μηχανή, και το αποτέλεσμα είναι μια δραματικά υψηλότερη ακρίβεια όταν **αναγνωρίζετε εικόνα κειμένου**. Σε αυτό το tutorial θα περάσουμε βήμα-βήμα **πώς να προσθέσετε φίλτρα**, να μειώσετε τον θόρυβο, και τελικά **να εξάγετε απλό κείμενο** με το Aspose.OCR σε C#.

Θα καλύψουμε όλα όσα χρειάζεστε—από την εγκατάσταση της βιβλιοθήκης μέχρι την εκτέλεση του κώδικα και την επαλήθευση του αποτελέσματος—ώστε να μπορείτε να αντιγράψετε‑επικολλήσετε μια λειτουργική λύση αμέσως.

## Τι Θα Χρειαστεί

- **.NET 6+** (ή οποιοδήποτε πρόσφατο .NET runtime· το API λειτουργεί το ίδιο)
- **Aspose.OCR for .NET** NuGet package  
  ```bash
  dotnet add package Aspose.OCR
  ```
- Μια δείγμα εικόνας που υποφέρει από περιστροφή, θόρυβο ή χαμηλή αντίθεση (π.χ., `skewed_noisy.jpg`)
- Ένα IDE ή επεξεργαστή που προτιμάτε (Visual Studio, VS Code, Rider—επιλέξτε το δικό σας)

Αυτό είναι όλο. Δεν χρειάζονται επιπλέον εγγενείς βιβλιοθήκες, ούτε βαριά πλαίσια επεξεργασίας εικόνας. Το Aspose περιλαμβάνει τα φίλτρα που χρειαζόμαστε.

## Βήμα 1 – Δημιουργία του Αντικειμένου OCR Engine  

Το πρώτο που κάνετε είναι να δημιουργήσετε ένα `OcrEngine`. Σκεφτείτε το ως τον εγκέφαλο που θα διαβάσει αργότερα τους χαρακτήρες αφού έχουμε καθαρίσει την εικόνα.

```csharp
using System;
using Aspose.OCR;
using Aspose.OCR.Models;
using Aspose.OCR.Filters;

class FilterExample
{
    static void Main()
    {
        // Initialize the OCR engine – this object will hold our filters and settings
        OcrEngine ocrEngine = new OcrEngine();
```

> **Γιατί είναι σημαντικό:** Η μηχανή διατηρεί μια **pipeline προεπεξεργασίας**. Η προσθήκη φίλτρων αργότερα θα τα εφαρμόζει αυτόματα κάθε φορά που καλείτε το `Recognize`.

## Βήμα 2 – Προσθήκη Φίλτρων για Μείωση Θορύβου Εικόνας και Βελτίωση Αναγνωσιμότητας  

Τώρα **προσθέτουμε φίλτρα**. Κάθε φίλτρο αντιμετωπίζει ένα συγκεκριμένο πρόβλημα:

| Filter | Τι διορθώνει | Τυπικές τιμές |
|--------|--------------|---------------------|
| `DeskewFilter` | Κείμενο με περιστροφή (skew) | `MaxAngle = 12` (degrees) |
| `DenoiseFilter` | Τυχαίες κηλίδες, σκόνη | `Strength = 0.6` (0‑1) |
| `ContrastBoostFilter` | Αμυδρά γράμματα | `Level = 1.3` (1‑2) |
| `BinarizeAdaptiveFilter` | Ασυμμετρικός φωτισμός | defaults work well |

```csharp
        // Step 2: Add preprocessing filters to improve image quality
        //   • Deskew to correct rotation (max 12°)
        //   • Denoise to reduce noise (strength 0.6)
        //   • Contrast boost to enhance visibility (level 1.3)
        //   • Adaptive binarization for better binarization
        ocrEngine.Preprocessing.Add(new DeskewFilter { MaxAngle = 12 });
        ocrEngine.Preprocessing.Add(new DenoiseFilter { Strength = 0.6 });
        ocrEngine.Preprocessing.Add(new ContrastBoostFilter { Level = 1.3 });
        ocrEngine.Preprocessing.Add(new BinarizeAdaptiveFilter());
```

> **Συμβουλή:** Αν οι εικόνες σας είναι ήδη όρθιες, μπορείτε να παραλείψετε το βήμα deskew. Η αφαίρεση περιττών φίλτρων επιταχύνει την επεξεργασία.

## Βήμα 3 – Φόρτωση της Εικόνας που Θέλετε να Επεξεργαστείτε  

Στη συνέχεια, λέμε στη μηχανή ποια εικόνα να καθαρίσει. `ImageStream.FromFile` διαβάζει το αρχείο σε μορφή που καταλαβαίνει η μηχανή OCR.

```csharp
        // Step 3: Load the image that needs OCR processing
        ImageStream image = ImageStream.FromFile(@"YOUR_DIRECTORY/skewed_noisy.jpg");
```

> **Ακραία περίπτωση:** Όταν εργάζεστε με ροές που προέρχονται από web API, αντικαταστήστε το `FromFile` με `FromStream` για να αποφύγετε την πρόσβαση στο σύστημα αρχείων.

## Βήμα 4 – Εκτέλεση του OCR Engine και Αναγνώριση Εικόνας Κειμένου  

Τώρα συμβαίνει η μαγεία. Η μηχανή εφαρμόζει όλα τα φίλτρα που προσθέσαμε, και στη συνέχεια εκτελεί την αναγνώριση χαρακτήρων.

```csharp
        // Step 4: Run the OCR engine on the prepared image
        OcrResult ocrResult = ocrEngine.Recognize(image);
```

> **Γιατί λειτουργεί:** Η pipeline προεπεξεργασίας εξασφαλίζει ότι η εικόνα που δίνεται στον αναγνωριστή είναι όσο το δυνατόν πιο καθαρή, κάτι που αυξάνει άμεσα την ακρίβεια του **recognize text image**.

## Βήμα 5 – Εξαγωγή Απλού Κειμένου και Εμφάνιση του  

Τέλος, εξάγουμε το αποτέλεσμα ως απλό κείμενο και το εκτυπώνουμε στην κονσόλα. Αυτό είναι το μέρος **extract plain text** της ροής εργασίας.

```csharp
        // Step 5: Output the recognized plain text
        Console.WriteLine(ocrResult.PlainText);
    }
}
```

Αν το `skewed_noisy.jpg` περιέχει μια απλή γραμμή τιμολογίου όπως `Total: $123.45`, θα πρέπει να δείτε:

Ακόμα και αν το αρχικό αρχείο ήταν περιστραμμένο 10°, είχε κηλίδες και χαμηλή αντίθεση, τα φίλτρα θα το καθαρίσουν αρκετά ώστε η μηχανή OCR να το διαβάσει σωστά.

## Συχνές Ερωτήσεις & Πιθανά Προβλήματα  

- **Τι γίνεται αν το αποτέλεσμα OCR είναι ακόμα ακατάληπτο;**  
  Δοκιμάστε να αυξήσετε το `DenoiseFilter.Strength` στο `0.8` ή να μειώσετε το `ContrastBoostFilter.Level` στο `1.1`. Η υπερβολική ενίσχυση της αντίθεσης μπορεί μερικές φορές να δημιουργήσει αχνά που μπερδεύουν τον αναγνωριστή.

- **Μπορώ να προσθέσω το δικό μου προσαρμοσμένο φίλτρο;**  
  Ναι. Το Aspose.OCR σας επιτρέπει να υλοποιήσετε το `IFilter` και να το ενσωματώσετε στο `ocrEngine.Preprocessing`. Είναι ένα προχωρημένο θέμα, αλλά χρήσιμο όταν έχετε ανάγκες προεπεξεργασίας ειδικές για το πεδίο σας.

- **Λειτουργεί αυτό με PDF;**  
  Μόνο αν πρώτα μετατρέψετε κάθε σελίδα PDF σε εικόνα (π.χ., χρησιμοποιώντας το Aspose.PDF). Μόλις έχετε ένα bitmap, η ίδια αλυσίδα φίλτρων εφαρμόζεται.

- **Είναι η βιβλιοθήκη thread‑safe;**  
  Το αντικείμενο `OcrEngine` **δεν** είναι thread‑safe. Δημιουργήστε ξεχωριστό engine ανά νήμα ή τυλίξτε τις κλήσεις σε κλείδωμα (lock).

## Επέκταση του Παραδείγματος  

Τώρα που έχετε μια σταθερή βάση, σκεφτείτε τα επόμενα βήματα:

1. **Batch processing** – Επανάληψη σε έναν φάκελο εικόνων, εφαρμογή της ίδιας αλυσίδας φίλτρων, και εγγραφή του κάθε αποτελέσματος σε αρχείο `.txt`.  
2. **Language packs** – Αν χρειάζεται να αναγνωρίσετε Γαλλικά ή Γερμανικά, φορτώστε τα αντίστοιχα δεδομένα γλώσσας μέσω `ocrEngine.Language = "fra"` (ή `"deu"`).  
3. **Region‑of‑interest (ROI)** – Χρησιμοποιήστε `ocrEngine.Recognize(image, new Rectangle(x, y, width, height))` για να εστιάσετε σε συγκεκριμένη περιοχή, κάτι που μπορεί να επιταχύνει μεγάλες σαρώσεις.
