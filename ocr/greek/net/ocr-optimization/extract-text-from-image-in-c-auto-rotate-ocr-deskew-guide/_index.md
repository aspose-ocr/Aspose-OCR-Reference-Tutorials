# Εξαγωγή κειμένου από εικόνα σε C# – Αυτόματη περιστροφή OCR & διόρθωση κλίσης

Έχετε ποτέ χρειαστεί να **εξάγετε κείμενο από εικόνα** αλλά το αρχείο ήρθε με παράξενη γωνία; Είναι ένα συνηθισμένο πρόβλημα—ιδιαίτερα όταν δουλεύετε με σαρωμένα τιμολόγια, φωτογραφημένα αποδείξεις ή λοξά PDF. Τα καλά νέα είναι ότι δεν χρειάζεται να γράψετε έναν προσαρμοσμένο αλγόριθμο περιστροφής. Χρησιμοποιώντας τη ενσωματωμένη λειτουργία *auto rotate OCR* του Aspose OCR, μπορείτε να αφήσετε τη μηχανή να ευθυγραμμίσει την εικόνα και στη συνέχεια να **εξάγετε κείμενο από εικόνα** σε ένα ομαλό βήμα.

Σε αυτό το tutorial θα περάσουμε από ένα πλήρες, έτοιμο‑για‑εκτέλεση πρόγραμμα C# που:

* Αρχικοποιεί τη μηχανή OCR με αυτόματη προσανατολισμό και διόρθωση κλίσης,
* Αναγνωρίζει μια περιστραμμένη ή λοξή εικόνα,
* Επιστρέφει τόσο τη διαγνώσθμενη γωνία περιστροφής όσο και το εξαγόμενο κείμενο.

Δεν χρειάζονται εξωτερικές υπηρεσίες, δεν υπάρχει περίπλοκος μαθηματικός υπολογισμός—μόνο λίγες γραμμές κώδικα και μια σαφής εξήγηση του *πώς να διορθώσετε κλίση σαρωμένης εικόνας* όταν χρειάζεται.

## Τι θα χρειαστείτε

| Προαπαιτούμενο | Γιατί είναι σημαντικό |
|----------------|------------------------|
| .NET 6.0 ή νεότερο (ή .NET Framework 4.6+) | Το Aspose OCR διανέμεται ως πακέτο NuGet που στοχεύει αυτά τα runtime. |
| Visual Studio 2022 (ή οποιοσδήποτε επεξεργαστής C#) | Διευκολύνει την προσθήκη πακέτων NuGet και την εκτέλεση της κονσόλας. |
| Ένα δείγμα εικόνας (`rotated_document.jpg`) | Το αρχείο πρέπει να είναι JPEG, PNG, BMP ή TIFF που δεν είναι τέλεια όρθιο. |
| Πακέτο NuGet Aspose.OCR (`Install-Package Aspose.OCR`) | Παρέχει τις κλάσεις `OcrEngine`, `PreprocessingFilters` και σχετικούς τύπους. |

Αν έχετε τσεκάρει αυτά τα κουτάκια, είστε έτοιμοι να ξεκινήσετε. Διαφορετικά, κατεβάστε το τελευταίο Aspose OCR από το NuGet—είναι εγκατάσταση με ένα κλικ.

## Βήμα 1 – Αρχικοποίηση της μηχανής OCR (Κύρια Λέξη‑Κλειδί σε Δράση)

Το πρώτο που κάνουμε είναι να δημιουργήσουμε ένα αντικείμενο `OcrEngine` και να του πούμε να **auto rotate OCR** και **deskew** οποιαδήποτε εισερχόμενη εικόνα. Αυτές οι δύο σημαίες συνδυάζονται με bitwise OR (`|`) επειδή το `PreprocessingFilters` είναι enum με το χαρακτηριστικό `[Flags]`.

```csharp
using System;
using Aspose.OCR;

class Program
{
    static void Main()
    {
        // Initialise the OCR engine with English language and preprocessing options
        OcrEngine ocrEngine = new OcrEngine
        {
            Language = Language.English,
            // Enable both auto‑rotate and auto‑deskew
            Preprocessing = PreprocessingFilters.AutoRotate | PreprocessingFilters.AutoDeskew
        };
```

> **Γιατί είναι σημαντικό:**  
> *Auto‑rotate OCR* εξετάζει τη βάση κειμένου της εικόνας, υποθέτει τον σωστό προσανατολισμό και την περιστρέφει εσωτερικά πριν από την αναγνώριση. *Auto‑deskew* κάνει το ίδιο για ελαφρά κλίση που συχνά εμφανίζεται σε σαρωμένα έγγραφα. Ενεργοποιώντας και τα δύο, εξασφαλίζετε τα καλύτερα δυνατά αποτελέσματα **εξαγωγής κειμένου από εικόνα** χωρίς χειροκίνητη επεξεργασία εικόνας.

---

## Βήμα 2 – Αναγνώριση της εικόνας και λήψη του αποτελέσματος

Τώρα δίνουμε στη μηχανή τη διαδρομή του αρχείου. Η μέθοδος `RecognizeImage` επιστρέφει ένα αντικείμενο `OcrResult` που περιέχει τη διαγνώσθμενη γωνία περιστροφής, τις βαθμολογίες εμπιστοσύνης και το κείμενο σε απλό κείμενο.

```csharp
        // Recognise the image file and obtain the OCR result
        OcrResult ocrResult = ocrEngine.RecognizeImage("YOUR_DIRECTORY/rotated_document.jpg");
```

> **Συμβουλή:** Αν δουλεύετε με ροή (π.χ., ένα ανεβασμένο αρχείο), χρησιμοποιήστε `ocrEngine.RecognizeImage(Stream)` αντί αυτού. Οι ίδιες σημαίες προεπεξεργασίας εφαρμόζονται, οπότε εξακολουθείτε να επωφελείστε από το **auto rotate OCR**.

---

## Βήμα 3 – Εμφάνιση της γωνίας περιστροφής και του εξαγόμενου κειμένου

Τέλος, εκτυπώνουμε δύο πληροφορίες στην κονσόλα: τη γωνία που η μηχανή θεωρεί ότι η εικόνα έπρεπε να περιστραφεί και το πραγματικό κείμενο που εξήχθη.

```csharp
        // Show the detected rotation and the recognised text
        Console.WriteLine($"Detected rotation: {ocrResult.RotationAngle}°");
        Console.WriteLine("----- Extracted Text -----");
        Console.WriteLine(ocrResult.Text);
    }
}
```

**Αναμενόμενο αποτέλεσμα** (οι αριθμοί σας θα διαφέρουν ανάλογα με την είσοδο εικόνας):

```
Detected rotation: 12.3°
----- Extracted Text -----
Invoice #12345
Date: 2024‑03‑15
Total: $1,250.00
Thank you for your business!
```

Αν η εικόνα ήταν ήδη όρθια, η γωνία περιστροφής θα είναι `0°`, και θα λάβετε καθαρό κείμενο χάρη στον αλγόριθμο *auto‑deskew*.

## Βήμα 4 – Κατανόηση του πώς να διορθώσετε κλίση σαρωμένης εικόνας (Δευτερεύουσα Λέξη‑Κλειδί σε Βάθος)

Μπορεί να αναρωτιέστε *πώς να διορθώσετε κλίση σαρωμένης εικόνας* όταν η μηχανή OCR αναφέρει μη‑μηδενική περιστροφή. Στο παρασκήνιο, το Aspose OCR εφαρμόζει έναν **Hough transform** για να εντοπίσει τον κυρίαρχο προσανατολισμό των γραμμών κειμένου. Στη συνέχεια περιστρέφει το bitmap με το αντίστροφο της γωνίας, ευθυγραμμίζοντας ουσιαστικά το κείμενο.

**Πότε είναι σημαντικό;**  
* Σαρωτές που τροφοδοτούν το χαρτί με μικρή γωνία (συνηθισμένο σε γραφεία).  
* Φωτογραφίες από smartphone που τραβιούνται με το χέρι—σπάνια το ορίζοντα είναι τέλειος.  

**Διαχείριση ακραίων περιπτώσεων:**  
Αν το `RotationAngle` του αποτελέσματος είναι ασυνήθιστα μεγάλο (π.χ., > 45°), η μηχανή μπορεί να έχει παρερμηνεύσει την εικόνα (ίσως είναι λογότυπο ή μη‑κειμενικό γραφικό). Σε αυτήν την περίπτωση μπορείτε:

1. Να περιστρέψετε χειροκίνητα την εικόνα χρησιμοποιώντας `System.Drawing` πριν τη δώσετε στη μηχανή, ή  
2. Να απενεργοποιήσετε το `AutoRotate` και να κρατήσετε μόνο το `AutoDeskew` αν γνωρίζετε ότι το έγγραφο είναι ήδη όρθιο.

```csharp
// Example: manually rotate 90° clockwise before OCR
using (var img = Aspose.OCR.Image.Load("rotated_document.jpg"))
{
    img.Rotate(90);
    OcrResult result = ocrEngine.RecognizeImage(img);
    Console.WriteLine(result.Text);
}
```

## Βήμα 5 – Συνηθισμένα προβλήματα & Συμβουλές (E‑E‑A‑T σε Δράση)

| Πρόβλημα | Γιατί συμβαίνει | Διόρθωση |
|----------|------------------|----------|
| **Θολές ή χαμηλής ανάλυσης εικόνες** | Η ακρίβεια του OCR πέφτει δραματικά· η ανίχνευση περιστροφής μπορεί να αποτύχει. | Χρησιμοποιήστε σαρώσεις τουλάχιστον 300 dpi· εφαρμόστε φίλτρο sharpening πριν το OCR. |
| **Μικτές γλώσσες** | Η μηχανή προεπιλέγει τα Αγγλικά· οι ξένοι χαρακτήρες γίνονται ακατάληπτοι. | Ορίστε `Language = Language.English | Language.Spanish` (ή κατάλληλο συνδυασμό). |
| **Μεγάλα αρχεία (> 10 MB)** | Η πίεση μνήμης μπορεί να προκαλέσει `OutOfMemoryException`. | Μειώστε την ανάλυση της εικόνας πρώτα, ή επεξεργαστείτε σε τμήματα χρησιμοποιώντας `OcrEngine.RecognizeRegion`. |
| **Λανθασμένη διαδρομή αρχείου** | `FileNotFoundException` σταματά το πρόγραμμα. | Χρησιμοποιήστε `Path.Combine(Environment.CurrentDirectory, "rotated_document.jpg")` για μεγαλύτερη ανθεκτικότητα. |

> **Συμβουλή επαγγελματία:** Πάντα καταγράφετε το `ocrResult.RotationAngle` και το `ocrResult.Confidence` (αν είναι διαθέσιμο). Αυτά τα μετρικά σας βοηθούν να αποφασίσετε αν αξίζει μια επανάληψη με διαφορετικές ρυθμίσεις προεπεξεργασίας.

## Βήμα 6 – Επέκταση του παραδείγματος (Τι ακολουθεί;)

Τώρα που μπορείτε να **εξάγετε κείμενο από εικόνα** με αυτόματη περιστροφή και διόρθωση κλίσης, σκεφτείτε τα επόμενα βήματα:

* **Επεξεργασία σε παρτίδες** – Επανάληψη πάνω σε φάκελο εικόνων, αποθήκευση κάθε αποτελέσματος σε βάση δεδομένων, και σήμανση όσων έχουν `RotationAngle > 5°` για χειροκίνητη ανασκόπηση.  
* **Μετατροπή σε PDF** – Συνδυάστε το εξαγόμενο κείμενο με την αρχική εικόνα σε ένα αναζητήσιμο PDF χρησιμοποιώντας Aspose.PDF.  
* **Ανίχνευση γλώσσας** – Χρησιμοποιήστε τη σημαία `AutoDetectLanguage` του Aspose.OCR για να αφήσετε τη μηχανή να επιλέξει αυτόματα τη βέλτιστη γλώσσα.  

Κάθε ένα από αυτά βασίζεται στην ίδια βασική αρχή: αφήστε τη βιβλιοθήκη να διαχειριστεί τον προσανατολισμό, και εσείς εστιάστε στη λογική της επιχείρησής σας.

## Πλήρες λειτουργικό παράδειγμα (Έτοιμο για αντιγραφή‑επικόλληση)

```csharp
using System;
using Aspose.OCR;

class Program
{
    static void Main()
    {
        // 1️⃣ Initialise the OCR engine with auto‑rotate and auto‑deskew
        OcrEngine ocrEngine = new OcrEngine
        {
            Language = Language.English,
            Preprocessing = PreprocessingFilters.AutoRotate | PreprocessingFilters.AutoDeskew
        };

        // 2️⃣ Recognise the image – replace the path with your own file
        string imagePath = "YOUR_DIRECTORY/rotated_document.jpg";
        OcrResult ocrResult = ocrEngine.RecognizeImage(imagePath);

        // 3️⃣ Output the rotation angle and the recognised text
        Console.WriteLine($"Detected rotation: {ocrResult.RotationAngle}°");
        Console.WriteLine("----- Extracted Text -----");
        Console.WriteLine(ocrResult.Text);
    }
}
```

Αποθηκεύστε το αρχείο ως `Program.cs`, τρέξτε `dotnet add package Aspose.OCR`, έπειτα `dotnet run`. Θα πρέπει να δείτε τη γωνία και το καθαρό κείμενο να εκτυπώνονται στην κονσόλα.
