# Πώς να διορθώσετε την κλίση μιας εικόνας και να ενισχύσετε την αντίθεση για καλύτερη ακρίβεια OCR

Έχετε αναρωτηθεί ποτέ **πώς να διορθώσετε την κλίση μιας εικόνας** που προέρχεται από σαρωτή ή smartphone;  
Δεν είστε μόνοι—οι περισσότεροι προγραμματιστές αντιμετωπίζουν αυτό το πρόβλημα όταν η πηγή είναι ελαφρώς κεκλιμένη ή θορυβώδης, και το αποτέλεσμα του OCR καταλήγει σε ακατάστατο μπερδεμένο κείμενο.  

Το καλό νέο είναι ότι με λίγες γραμμές C# μπορείτε να ευθυγραμμίσετε την εικόνα, να καθαρίσετε το φόντο και ακόμη να αυξήσετε την αντίθεση ώστε η μηχανή να διαβάζει το κείμενο σαν επαγγελματίας. Σε αυτόν τον οδηγό θα σας δείξουμε επίσης πώς να **εξάγετε κείμενο από εικόνα** και **αναγνωρίσετε περιεχόμενο εικόνας κειμένου** με το Aspose.OCR, διατηρώντας πάντα στο επίκεντρο το **βελτιώστε την ακρίβεια OCR**.

## Τι θα χρειαστείτε

- **.NET 6.0** ή νεότερο (ο κώδικας μεταγλωττίζεται και σε .NET Framework 4.7+)  
- **Aspose.OCR** πακέτο NuGet (έκδοση 23.12 ή νεότερη) – εγκαταστήστε μέσω `dotnet add package Aspose.OCR`  
- Ένα δείγμα εικόνας που είναι τόσο περιστραμμένη όσο και θορυβώδης (π.χ., `noisy_rotated.jpg`)  
- Visual Studio, VS Code ή οποιοδήποτε IDE προτιμάτε  

Αυτό είναι όλο—χωρίς επιπλέον εγγενείς βιβλιοθήκες, χωρίς βαριές συνδέσεις OpenCV. Απλώς καθαρός διαχειριζόμενος κώδικας.

---

## Βήμα 1: Ρύθμιση του έργου και εισαγωγή namespaces

Πρώτα, δημιουργήστε μια νέα εφαρμογή console και φέρετε τα απαιτούμενα namespaces στο πεδίο ορατότητας. Αυτό το βήμα είναι η βάση για όλα τα επόμενα.

```csharp
// Program.cs
using System;
using Aspose.OCR;
using Aspose.OCR.Filters;

namespace OcrDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // We'll fill this in later
        }
    }
}
```

**Γιατί είναι σημαντικό:**  
`Aspose.OCR` σας παρέχει την κλάση `OcrEngine`, ενώ το `Aspose.OCR.Filters` προσφέρει χρήσιμα φίλτρα προεπεξεργασίας όπως `DeskewFilter` και `ContrastBoostFilter`. Η εισαγωγή τους εκ των προτέρων διατηρεί τον κώδικα τακτοποιημένο και ενημερώνει τον μεταγλωττιστή για το τι προτίθεσθε να χρησιμοποιήσετε.

---

## Βήμα 2: Αρχικοποίηση του OCR Engine και προσθήκη φίλτρου Deskew

Τώρα πραγματικά **πώς να διορθώσετε την κλίση μιας εικόνας**. Το `DeskewFilter` ανιχνεύει αυτόματα τη γωνία περιστροφής (μέχρι το μέγιστο που ορίζετε) και περιστρέφει το bitmap πίσω στην οριζόντια θέση.

```csharp
// Inside Main()
var ocrEngine = new OcrEngine();

// Deskew: correct up to 15 degrees of rotation
ocrEngine.Filters.Add(new DeskewFilter { MaxAngle = 15 });
```

**Τι συμβαίνει στο παρασκήνιο;**  
Το φίλτρο σαρώει την εικόνα για τη μεγαλύτερη οριζόντια γραμμή κειμένου, εκτιμά την κλίση και εφαρμόζει αντίστροφη περιστροφή. Περιορίζοντας το `MaxAngle` στα 15°, αποφεύγουμε την υπερδιόρθωση εικόνων που είναι ήδη ευθείες.

> **Συμβουλή επαγγελματία:** Αν οι πηγαίες εικόνες μπορεί να είναι ανάποδες, αυξήστε το `MaxAngle` στα 180°—το φίλτρο θα βρει ακόμη τη σωστή προσανατολισμό.

---

## Βήμα 3: Μείωση θορύβου με φίλτρο Denoise

Ένα θορυβώδες scan μπορεί να παραπλανήσει ακόμη και το πιο έξυπνο OCR engine. Η προσθήκη ενός `DenoiseFilter` εξομαλύνει τα στίγματα χωρίς να διαγράψει τις λεπτές λεπτομέρειες.

```csharp
// Denoise: strength ranges from 0 (off) to 1 (max)
ocrEngine.Filters.Add(new DenoiseFilter { Strength = 0.7 });
```

**Γιατί το χρειάζεστε:**  
Ο θόρυβος δημιουργεί ψευδείς άκρες που ο αλγόριθμος OCR ερμηνεύει ως χαρακτήρες. Ένα `strength` 0.7 είναι ένα καλό σημείο για τα περισσότερα σαρωμένα έγγραφα· μπορείτε να το προσαρμόσετε για πολύ καθαρά ή πολύ βρώμικα inputs.

---

## Βήμα 4: Ενίσχυση της αντίθεσης – “Πώς να ενισχύσετε την αντίθεση” σε δράση

Εδώ απαντάμε στη δευτερεύουσα λέξη-κλειδί **πώς να ενισχύσετε την αντίθεση**. Το `ContrastBoostFilter` ενισχύει τη διαφορά μεταξύ του σκούρου κειμένου και του φωτεινού φόντου, κάνοντας τα γράμματα να ξεχωρίζουν.

```csharp
// Contrast boost: level > 1 increases contrast
ocrEngine.Filters.Add(new ContrastBoostFilter { Level = 1.3 });
```

**Η λογική:**  
Υψηλότερη αντίθεση μειώνει την πιθανότητα να παραλειφθούν αδύναμα στίγματα. Ένα επίπεδο `1.3` λειτουργεί καλά για τυπικά έγγραφα μαύρο‑σε‑λευκό· για έγχρωμες φωτογραφίες μπορεί να χρειαστεί `1.5` ή περισσότερο.

---

## Βήμα 5: Εκτέλεση OCR και εξαγωγή του κειμένου

Με την εικόνα προεπεξεργασμένη, τελικά **εξάγουμε κείμενο από εικόνα** χρησιμοποιώντας τη μέθοδο `Recognize`. Η μέθοδος επιστρέφει ένα αντικείμενο `OcrResult` που περιέχει το ακατέργαστο κείμενο και τις βαθμολογίες εμπιστοσύνης.

```csharp
// Perform OCR on the prepared image
var ocrResult = ocrEngine.Recognize("YOUR_DIRECTORY/noisy_rotated.jpg");

// Output the recognized text
Console.WriteLine("=== OCR Output ===");
Console.WriteLine(ocrResult.Text);
```

**Τι λαμβάνετε:**  
`ocrResult.Text` περιέχει την απλή κειμενική αναπαράσταση όλων όσων μπόρεσε να διαβάσει η μηχανή. Αν χρειάζεστε εμπιστοσύνη σε επίπεδο λέξης, εξερευνήστε το `ocrResult.Regions`—κάθε περιοχή περιλαμβάνει μια ιδιότητα `Confidence`.

---

## Πλήρες λειτουργικό παράδειγμα (Έτοιμο για αντιγραφή‑επικόλληση)

Παρακάτω είναι το πλήρες πρόγραμμα που μπορείτε να τοποθετήσετε στο `Program.cs`. Βεβαιωθείτε ότι η διαδρομή της εικόνας δείχνει σε πραγματικό αρχείο στο σύστημά σας.

```csharp
using System;
using Aspose.OCR;
using Aspose.OCR.Filters;

namespace OcrDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // 1️⃣ Initialize OCR engine
            var ocrEngine = new OcrEngine();

            // 2️⃣ Deskew – correct up to 15° rotation
            ocrEngine.Filters.Add(new DeskewFilter { MaxAngle = 15 });

            // 3️⃣ Denoise – smooth out speckles (strength 0.7)
            ocrEngine.Filters.Add(new DenoiseFilter { Strength = 0.7 });

            // 4️⃣ Contrast boost – make text pop (level 1.3)
            ocrEngine.Filters.Add(new ContrastBoostFilter { Level = 1.3 });

            // 5️⃣ Recognize the image
            var imagePath = @"YOUR_DIRECTORY\noisy_rotated.jpg";
            var ocrResult = ocrEngine.Recognize(imagePath);

            // 6️⃣ Show the result
            Console.WriteLine("=== OCR Output ===");
            Console.WriteLine(ocrResult.Text);
        }
    }
}
```

**Αναμενόμενη έξοδος (παράδειγμα):**

```
=== OCR Output ===
Invoice #12345
Date: 2024-09-30
Total Amount: $1,250.00
Thank you for your business!
```

Αν το αποτέλεσμα φαίνεται ακατάληπτο, ελέγξτε ξανά την ποιότητα της εικόνας, προσαρμόστε το `Strength` ή το `Level`, ή αυξήστε το `MaxAngle` για πιο έντονη διόρθωση κλίσης.

## Συχνές ερωτήσεις & ειδικές περιπτώσεις

### Τι γίνεται αν η εικόνα είναι ανάποδα;

Ορίστε `MaxAngle = 180` στο `DeskewFilter`. Το φίλτρο θα εντοπίσει την περιστροφή 180° και θα το αναστρέψει σωστά.

### Το έγγραφό μου είναι έγχρωμο (π.χ., μια σαρωμένη φόρμα με μπλε επισημάνσεις).

Δοκιμάστε να προσθέσετε ένα `ColorFilter` πριν την ενίσχυση της αντίθεσης, ή μετατρέψτε την εικόνα σε γκρι κλίμακα χειροκίνητα:

```csharp
ocrEngine.Filters.Add(new GrayscaleFilter());
```

### Χρειάζομαι να επεξεργαστώ πολλά αρχεία σε batch.

Τυλίξτε τη λογική OCR σε έναν βρόχο `foreach` που διατρέχει έναν φάκελο:

```csharp
foreach (var file in Directory.GetFiles(@"C:\Scans", "*.jpg"))
{
    var result = ocrEngine.Recognize(file);
    File.WriteAllText(Path.ChangeExtension(file, ".txt"), result.Text);
}
```

### Πώς μπορώ να γνωρίζω την εμπιστοσύνη του OCR;

Εξετάστε το `ocrResult.Regions`:

```csharp
foreach (var region in ocrResult.Regions)
{
    Console.WriteLine($"Text: {region.Text} – Confidence: {region.Confidence:P2}");
}
```

Υψηλότερες τιμές εμπιστοσύνης (κοντά στο 100%) συνήθως σημαίνουν ότι τα βήματα προεπεξεργασίας ήταν επιτυχείς.

## Συμβουλές για μέγιστη ακρίβεια OCR

| Tip | Why it Helps |
|-----|--------------|
| **Χρησιμοποιήστε μορφές εικόνας χωρίς απώλειες** (PNG, TIFF) | Η συμπίεση JPEG μπορεί να θολώσει τις άκρες, βλάπτοντας την αναγνώριση. |
| **Διατηρήστε το DPI στο 300+** | Περισσότερα pixel ανά χαρακτήρα δίνουν στη μηχανή περισσότερα δεδομένα για επεξεργασία. |
| **Κόψτε τα άσχετα περιθώρια** | Μειώνει τον θόρυβο και επιταχύνει την επεξεργασία. |
| **Εφαρμόστε δυαδικό κατώφλι** (μαύρο/λευκό) μετά την ενίσχυση της αντίθεσης για καθαρά έγγραφα κειμένου | Απλοποιεί την εικόνα σε δύο χρώματα, κάτι που αγαπούν οι περισσότερες μηχανές OCR. |
| **Δοκιμάστε πρώτα με μικρό δείγμα** | Σας επιτρέπει να ρυθμίσετε ακριβώς το `Strength` και το `Level` πριν την κλιμάκωση. |
