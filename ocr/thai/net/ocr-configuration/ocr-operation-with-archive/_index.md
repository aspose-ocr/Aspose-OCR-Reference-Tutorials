# วิธีทำ OCR บนรูปภาพในไฟล์เก็บข้อมูลด้วย Aspose.OCR สำหรับ .NET

## การแนะนำ

ในบทเรียนเชิงลึกนี้คุณจะได้สัมผัส **วิธีทำ OCR** ​​บนไฟล์รูปภาพที่เป็นลักษณะของรูปแบบ เก็บถาวรด้วยไลบรารี Aspose.OCR .NET .NET .NET ต้องการ ** แปลงรูปภาพเป็นข้อความ** หรือ **ดึงข้อความจากไฟล์เก็บถาวร** คู่มือขั้นตอนต่อไปนี้จะพาคุณผ่านทุกขั้นตอนที่ต้องใช้ความพยายามในการปรับปรุงการดึงข้อความสำหรับที่เก็บข้อมูลในแต่ละครั้งภายในไฟล์ ZIP

## คำตอบด่วน
- **บทช่วยสอนครอบคลุมอะไรบ้าง** การใช้ OCR กับอิมเมจที่เก็บถาวร (ZIP) ด้วย Aspose.OCR สำหรับ .NET
- **คำหลักใดถูกกำหนดเป้าหมาย** *วิธีดำเนินการ ocr*
- **ฉันต้องมีใบอนุญาตหรือไม่** ทดลองใช้งานฟรีเพื่อประเมินผลได้ ต้องมีใบอนุญาตเชิงพาณิชย์สำหรับการผลิต
- **รองรับ .NET เวอร์ชันใดบ้าง** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6+
- **ฉันสามารถปรับแต่งการตั้งค่าการจดจำได้หรือไม่** ได้—ใช้ `การตั้งค่าการจดจำ` เพื่อปรับแต่งความแม่นยำ

## OCR คืออะไร และเหตุใดจึงใช้กับเอกสารสำคัญ?

Optical Character Recognition (OCR) แปลงภาพสแกนหรือ PDF ให้เป็นข้อความค้นหาและแก้ไขได้ก่อนรูปภาพถูกบรรจุอยู่ในไฟล์เก็บถาวร (เช่น ไฟล์ ZIP) การดึงและเก็บแต่ละรูปภาพในเวลาเดียวกันจะช่วยเพิ่มประสิทธิภาพของโค้ด Aspose.OCR มีเมธอด `RecognizeMultipleImages` และเฟิร์มแวร์นี้เป็นเรื่องง่าย

## ข้อกำหนดเบื้องต้น

- Visual Studio 2019 หรือใหม่กว่า (หรือ IDE รองรับ .NET เวลส์)
- .NET Framework4.5+ หรือ .NETCore3.1+ ติดตั้งแล้ว
- เข้าถึงไลบรารี Aspose.OCR สำหรับ .NET (ลิงก์ดาวน์โหลดด้านล่าง)
- ไลเซนส์ Aspose.OCR ที่ถูกต้องในผลิตภัณฑ์ (มีการทดลองทดลอง)

## ขั้นตอนที่ 1: ตั้งค่าไดเร็กทอรีเอกสารของคุณ

เริ่มต้นโดยกำหนดพาธไปยังโฟลเดอร์เอกสารของคุณ:

```csharp
// ExStart:1
// The path to the documents directory.
string dataDir = "Your Document Directory";
// ExEnd:1
```

> **เคล็ดลับ:** ใช้ `Path.Combine` เพื่อจัดการพาธแบบข้ามแพลตฟอร์ม

## ขั้นตอนที่ 2: เริ่มต้นใช้งาน Aspose.OCR

สร้างอินสแตนซ์ของคลาส Aspose.OCR เพื่อเริ่มต้นการทำ OCR:

```csharp
// ExStart:3
AsposeOcr api = new AsposeOcr();
// ExEnd:3
```

## ขั้นตอนที่ 3: ระบุเส้นทางของภาพ

กำหนดพาธเต็มไปยังไฟล์ archive (ไฟล์ ZIP ที่บรรจุรูปภาพที่ต้องการอ่าน):

```csharp
// ExStart:4
string fullPath = dataDir + "OCR.zip";
// ExEnd:4
```

## ขั้นตอนที่ 4: จดจำภาพ

เรียกใช้การจดจำ OCR บน archive ที่ระบุโดยใช้การตั้งค่าเริ่มต้นหรือการตั้งค่าที่กำหนดเอง คำสั่งนี้จะดึงรูปภาพแต่ละไฟล์จาก ZIP และทำ OCR ให้โดยอัตโนมัติ:

```csharp
// ExStart:5
RecognitionResult[] result = api.RecognizeMultipleImages(fullPath, new RecognitionSettings
{
   //default or custom settings
});
// ExEnd:5
```

> คุณสามารถปรับ `RecognitionSettings` เพื่อเพิ่มความแม่นยำสำหรับภาษาหรือคุณภาพของภาพที่เฉพาะเจาะจงได้

## ขั้นตอนที่ 5: พิมพ์ผลลัพธ์

วนลูปผลลัพธ์และพิมพ์ข้อความที่จดจำได้สำหรับแต่ละรูปภาพภายใน archive:

```csharp
// ExStart:6
for (int i = 0; i < result.Length; i++)
{
	 Console.WriteLine($"Image: {i}\n Result:\n {result[i].RecognitionText}");
}
// ExEnd:6
```

ผลลัพธ์จะแสดงดัชนีของรูปภาพแต่ละภาพตามด้วยสตริงที่ดึงออกมา ทำให้ **แปลงรูปภาพเป็นข้อความ** และ **ดึงข้อความจากไฟล์ archive** ได้อย่างมีประสิทธิภาพ

## คำถามที่พบบ่อย

**ถาม: การฟัง Aspose.OCR สำหรับ .NET อาจมีไลเซนส์หรือไม่?**
ตอบ: ลองทดลองทดลองได้ฟรีสำหรับค่าส่วนใหญ่ แต่ต้องใช้เซนส์ที่สามารถพบได้ในผลิตภัณฑ์

**ถาม: ไลบรารีรองรับไฟล์ ZIP และการป้องกันด้วยรหัสผ่านหรือไม่**
ตอบ: ปัจจุบัน `RecognizeMultipleImages` รองรับไฟล์ ZIP มาตรฐานเท่านั้นสำหรับไฟล์เก็บถาวรที่เก็บข้อมูลที่ให้คุณดึงรูปภาพออกมาก่อนไลบรารีด้วยและจากนั้นจึงส่งต่อไปให้กับเครื่องมือ OCR

**ถาม: ไม่เชื่อสำหรับข้อความลายมือได้อย่างไร?**
ตอบ: สามารถใช้ฟล `RecognitionSettings.EnableHandwritingRecognition` และตั้งค่า DPI ของ (เช่น 300)

**ถาม: ร้องรับคะแนนความทรงจำที่ไม่มีวันลืม?**
ตอบ: ประสิทธิภาพ `RecognitionResult` คุณลักษณะ `Confidence` เป็นผลสำเร็จบันทึกประโยชน์ที่กรองผลลัพธ์ที่ความรู้สึกต่ำได้
