# วิธีดึงข้อความจากไฟล์ ZIP โดยใช้ Aspose.OCR สำหรับ .NET

## บทนำ

ในบทเรียนเชิงลึกนี้คุณจะได้เรียนรู้ **วิธีดึงข้อความจาก zip** โดยใช้ OCR กับแต่ละภาพภายในไฟล์ zip ไม่ว่าคุณต้องการ **แปลงภาพเป็นข้อความ**, **อ่านภาพจาก zip**, หรือสร้างคลังเอกสารที่ค้นหาได้ คู่มือขั้นตอนต่อไปนี้จะพาคุณผ่านทุกอย่างตั้งแต่การติดตั้ง Aspose.OCR สำหรับ .NET จนถึงการพิมพ์ข้อความที่จดจำได้สำหรับทุกภาพในไฟล์ ZIP

## คำตอบเร็ว
- **บทเรียนนี้ครอบคลุมอะไร?** การดึงข้อความจากไฟล์ ZIP โดยใช้ Aspose.OCR สำหรับ .NET.  
- **คำหลักหลักที่มุ่งเป้า?** *extract text from zip*.  
- **ต้องมีใบอนุญาตหรือไม่?** สามารถใช้รุ่นทดลองฟรีเพื่อประเมินผลได้; ต้องมีใบอนุญาตเชิงพาณิชย์สำหรับการใช้งานจริง.  
- **รองรับเวอร์ชัน .NET ใดบ้าง?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6+.  
- **สามารถปรับแต่งการตั้งค่าการจดจำได้หรือไม่?** ใช่ — ใช้ `RecognitionSettings` เพื่อปรับความแม่นยำสำหรับภาษาต่าง ๆ หรือคุณภาพของภาพ.

## OCR คืออะไรและทำไมต้องใช้กับไฟล์ ZIP?

Optical Character Recognition (OCR) แปลงภาพสแกนหรือ PDF ให้เป็นข้อความที่ค้นหาและแก้ไขได้ เมื่อภาพเหล่านั้นถูกบรรจุอยู่ในไฟล์ ZIP การดึงและจดจำแต่ละภาพในขั้นตอนเดียวช่วยประหยัดเวลาและลดความซับซ้อนของโค้ด Aspose.OCR มีเมธอด `RecognizeMultipleImages` ที่ทำให้กระบวนการนี้ง่ายขึ้น ทำให้คุณ **อ่านภาพจาก zip** แล้วได้ข้อความทันที

## ข้อกำหนดเบื้องต้น

- Visual Studio 2019 หรือใหม่กว่า (หรือ IDE ที่รองรับ .NET ใดก็ได้).  
- .NET Framework 4.5 + หรือ .NET Core 3.1 + ติดตั้งแล้ว.  
- เข้าถึงไลบรารี Aspose.OCR สำหรับ .NET (ลิงก์ดาวน์โหลดด้านล่าง).  
- ใบอนุญาต Aspose.OCR ที่ใช้ได้สำหรับการผลิต (มีรุ่นทดลอง).

## ขั้นตอนที่ 1: ตั้งค่าไดเรกทอรีเอกสารของคุณ

เริ่มต้นโดยกำหนดเส้นทางไปยังไดเรกทอรีเอกสารของคุณ โฟลเดอร์นี้จะบรรจุไฟล์ ZIP ที่ต้องการประมวลผล:

```csharp
// ExStart:1
// The path to the documents directory.
string dataDir = "Your Document Directory";
// ExEnd:1
```

> **เคล็ดลับ:** ใช้ `Path.Combine` เพื่อจัดการเส้นทางแบบข้ามแพลตฟอร์ม

## ขั้นตอนที่ 2: เริ่มต้น Aspose.OCR

สร้างอินสแตนซ์ของคลาส Aspose.OCR เพื่อเริ่มกระบวนการ OCR:

```csharp
// ExStart:3
AsposeOcr api = new AsposeOcr();
// ExEnd:3
```

## ขั้นตอนที่ 3: ระบุเส้นทางไฟล์ ZIP

กำหนดเส้นทางเต็มของไฟล์อาร์ไคฟ์ (ไฟล์ ZIP ที่บรรจุภาพที่ต้องการอ่าน):

```csharp
// ExStart:4
string fullPath = dataDir + "OCR.zip";
// ExEnd:4
```

## ขั้นตอนที่ 4: จดจำภาพภายในไฟล์ ZIP

เรียกใช้การจดจำ OCR บนไฟล์อาร์ไคฟ์ที่ระบุโดยใช้การตั้งค่าเริ่มต้นหรือการตั้งค่าที่กำหนดเอง คำสั่งนี้จะดึงภาพแต่ละภาพจาก ZIP แล้วทำ OCR ให้โดยอัตโนมัติ:

```csharp
// ExStart:5
RecognitionResult[] result = api.RecognizeMultipleImages(fullPath, new RecognitionSettings
{
   //default or custom settings
});
// ExEnd:5
```

> คุณสามารถปรับ `RecognitionSettings` เพื่อเพิ่มความแม่นยำสำหรับภาษาที่เฉพาะเจาะจง, DPI, หรือเปิดการจดจำลายมือ

## ขั้นตอนที่ 5: พิมพ์ข้อความที่ดึงออกมา

วนลูปผลลัพธ์และพิมพ์ข้อความที่จดจำได้สำหรับแต่ละภาพภายในไฟล์อาร์ไคฟ์ นี่คือขั้นตอนที่คุณ **ดึงข้อความจาก zip** จริง ๆ:

```csharp
// ExStart:6
for (int i = 0; i < result.Length; i++)
{
	 Console.WriteLine($"Image: {i}\n Result:\n {result[i].RecognitionText}");
}
// ExEnd:6
```

ผลลัพธ์จะแสดงดัชนีของภาพแต่ละภาพตามด้วยสตริงที่ดึงออกมา ทำให้ **แปลงภาพเป็นข้อความ** และ **ดึงข้อความจากไฟล์อาร์ไคฟ์** ได้ในขั้นตอนเดียว

## ทำไมวิธีนี้ถึงสำคัญ

- **การประมวลผลเป็นชุด:** จัดการภาพจำนวนใดก็ได้ภายใน ZIP โดยไม่ต้องแยกไฟล์ด้วยตนเอง.  
- **ประสิทธิภาพ:** ลดภาระ I/O โดยอ่านโดยตรงจากไฟล์อาร์ไคฟ์.  
- **ความสามารถขยาย:** ทำงานกับไฟล์ ZIP ขนาดใหญ่และสามารถผสานกับรูปแบบ async เพื่อรองรับการทำงานแบบ throughput สูง.

## คำถามที่พบบ่อย

**ถาม: สามารถใช้ Aspose.OCR สำหรับ .NET ได้โดยไม่ต้องมีใบอนุญาตหรือไม่?**  
ตอบ: ใช่ มีรุ่นทดลองฟรีสำหรับการประเมินผล แต่ต้องใช้เวอร์ชันที่มีใบอนุญาตสำหรับการใช้งานในสภาพแวดล้อมการผลิต

**ถาม: ไลบรารีรองรับไฟล์ ZIP ที่มีการป้องกันด้วยรหัสผ่านหรือไม่?**  
ตอบ: ปัจจุบัน `RecognizeMultipleImages` ทำงานกับไฟล์ ZIP มาตรฐานเท่านั้น สำหรับไฟล์ที่เข้ารหัสไว้ ให้ดึงภาพออกมาก่อนด้วยไลบรารีของบุคคลที่สาม แล้วจึงส่งอาร์เรย์ภาพให้กับเครื่องมือ OCR

**ถาม: จะเพิ่มความแม่นยำสำหรับข้อความลายมือได้อย่างไร?**  
ตอบ: เปิดฟลัก `RecognitionSettings.EnableHandwritingRecognition` และตั้งค่า DPI ที่สูงขึ้น (เช่น 300)

**ถาม: มีวิธีรับคะแนนความเชื่อมั่นสำหรับแต่ละบรรทัดที่จดจำได้หรือไม่?**  
ตอบ: แต่ละ `RecognitionResult` มีคุณสมบัติ `Confidence` ที่คุณสามารถบันทึกหรือใช้กรองผลลัพธ์ที่ความเชื่อมั่นต่ำได้
