# ปรับปรุงความแม่นยำของ OCR ด้วยการตรวจสอบการสะกดในภาพ

เมื่อคุณทำงานกับ Optical Character Recognition (OCR) เป้าหมายสูงสุดคือ **ปรับปรุงความแม่นยำของ OCR** เพื่อให้ข้อความที่สกัดออกมาตรงกับภาพต้นฉบับอย่างสมบูรณ์ คำที่สะกดผิดเป็นแหล่งที่มาของข้อผิดพลาดทั่วไป โดยเฉพาะเมื่อภาพต้นฉบับมีสัญญาณรบกวนหรือใช้ฟอนต์แปลกใหม่ Aspose.OCR for .NET มีความสามารถในการตรวจสอบการสะกดในตัวที่ไม่เพียงแต่แก้ไขข้อผิดพลาดเหล่านั้น แต่ยังให้คุณขยายเครื่องมือด้วยพจนานุกรมที่กำหนดเองได้ ในบทเรียนนี้คุณจะได้เรียนรู้วิธีใช้การตรวจสอบการสะกดเพื่อเพิ่มผลลัพธ์ของ OCR ดูผลลัพธ์ก่อนและหลังการแก้ไข และค้นพบวิธีปรับกระบวนการแก้ไขให้ตรงกับความต้องการของภาษาของคุณ

## คำตอบสั้น
- **การตรวจสอบการสะกดทำอะไรกับ OCR?** มันจะตรวจจับคำที่สะกดผิดในผลลัพธ์ OCR โดยอัตโนมัติและแทนที่ด้วยทางเลือกที่เป็นไปได้ที่สุด  
- **ไลบรารีใดให้ฟีเจอร์นี้?** Aspose.OCR for .NET มี API การตรวจสอบการสะกดที่พร้อมใช้งาน  
- **ต้องการการเชื่อมต่ออินเทอร์เน็ตหรือไม่?** ไม่จำเป็น เครื่องตรวจสอบการสะกดทำงานแบบออฟไลน์ทั้งหมด  
- **ฉันสามารถเพิ่มคำศัพท์ของตนเองได้หรือไม่?** ได้ คุณสามารถจัดหาพจนานุกรมผู้ใช้แบบกำหนดเองเพื่อจัดการกับคำเฉพาะโดเมน  
- **วิธีนี้ช่วยให้ฉันจดจำข้อความจากภาพได้อย่างไร?** ด้วยการแก้ไขข้อผิดพลาดที่ OCR สร้างขึ้น ข้อความสุดท้ายจะสะอาดและพร้อมสำหรับการประมวลผลต่อไป

## การตรวจสอบการสะกดใน OCR คืออะไร?
การตรวจสอบการสะกดจะตรวจสอบข้อความดิบที่ได้จากเครื่อง OCR ระบุโทเคนที่ไม่ตรงกับคำที่รู้จักในพจนานุกรมของภาษาที่เลือก และเสนอหรือใช้การแก้ไข การทำขั้นตอนนี้เป็นสิ่งสำคัญสำหรับ **ปรับปรุงความแม่นยำของ OCR** โดยเฉพาะเมื่อประมวลผลเอกสารสแกน ใบเสร็จ หรือแบบฟอร์มที่ OCR อาจตีความอักขระผิดพลาด

## ทำไมต้องใช้การสนับสนุนภาษาของ Aspose OCR?
Aspose.OCR มาพร้อมกับชุดภาษาที่ครอบคลุมและอนุญาตให้คุณเชื่อมต่อพจนานุกรมเพิ่มเติม การใช้ **aspose ocr language support** หมายความว่าคุณสามารถจัดการกับเอกสารหลายภาษาโดยไม่ต้องเขียนพาร์เซอร์เอง และคุณจะได้เข้าถึงกฎเฉพาะภาษาที่ช่วยเพิ่มคุณภาพการจดจำต่อไป

## เมื่อใดที่การปรับปรุงความแม่นยำของ OCR มีความสำคัญที่สุด?
- **เอกสารทางกฎหมายและการปฏิบัติตาม** ที่คำผิดเพียงคำเดียวอาจเปลี่ยนความหมายได้  
- **สายงานการสกัดข้อมูล** ที่ผลลัพธ์ OCR จะถูกส่งต่อไปยังการวิเคราะห์หรือโมเดล AI  
- **แอปพลิเคชันที่ผู้ใช้เห็น** เช่น ตัวสแกนบนมือถือที่ต้องคืนข้อความที่อ่านได้ทันที  

## ข้อกำหนดเบื้องต้น

ก่อนที่เราจะลงลึกในเวทมนตร์การตรวจสอบการสะกด โปรดตรวจสอบว่าคุณมีสิ่งต่อไปนี้พร้อมใช้งาน:

- Aspose.OCR for .NET Library: ดาวน์โหลดและติดตั้งไลบรารี Aspose.OCR จาก [release page](https://releases.aspose.com/ocr/net/)  
- Document Directory: ตรวจสอบว่าคุณมีโฟลเดอร์ที่กำหนดไว้สำหรับเอกสารของคุณ แทนที่ `"Your Document Directory"` ในโค้ดตัวอย่างด้วยเส้นทางจริงของคุณ  

## นำเข้า Namespaces

เริ่มต้นด้วยการนำเข้า namespaces ที่จำเป็นในโครงการ .NET ของคุณ:

```csharp
using System;
using Aspose.OCR.SpellChecker;
using System.Collections.Generic;
```

## ขั้นตอนที่ 1: เริ่มต้น Aspose.OCR

สร้างอินสแตนซ์ของ Aspose.OCR เพื่อเริ่มกระบวนการ OCR

```csharp
// The path to the documents directory.
string dataDir = "Your Document Directory";

// Initialize an instance of AsposeOcr
AsposeOcr api = new AsposeOcr();
```

## ขั้นตอนที่ 2: จดจำภาพ

ต่อไปให้จดจำข้อความในภาพโดยใช้ Aspose.OCR ตัวอย่างโค้ดต่อไปนี้แสดงกระบวนการนี้:

```csharp
// Recognize image
RecognitionResult result = api.RecognizeImage(dataDir + "sample_bad.png", new RecognitionSettings(Language.Eng));
```

## ขั้นตอนที่ 3: ก่อนการแก้ไข

ดึงผลลัพธ์ OCR ก่อนการแก้ไขเพื่อเปรียบเทียบกับเวอร์ชันที่แก้ไขแล้ว

```csharp
// Get result
Console.WriteLine("BEFORE CORRECTION:\n" + result.RecognitionText);
```

## ขั้นตอนที่ 4: หลังการแก้ไข

ใช้การตรวจสอบการสะกดเพื่อรับผลลัพธ์ที่แก้ไขแล้ว โค้ดต่อไปนี้แสดงขั้นตอนนี้:

```csharp
// Get corrected result
string correctedResult = result.GetSpellCheckCorrectedText(SpellCheckLanguage.Eng);
Console.WriteLine("AFTER CORRECTION:\n" + correctedResult);
```

## ขั้นตอนที่ 5: คำที่สะกดผิดและข้อเสนอแนะ

รับรายการคำที่สะกดผิดพร้อมข้อเสนอแนะการแก้ไขโดยใช้โค้ดต่อไปนี้:

```csharp
// Get list of misspelled words with suggestions
List<SpellCheckError> errorsList = result.GetSpellCheckErrorList(SpellCheckLanguage.Eng);
foreach (var word in errorsList)
{
    Console.Write("Word:" + word.Word);
    Console.Write(" StartPosition:" + word.StartPosition);
    Console.WriteLine(" Length:" + word.Length);
    Console.WriteLine("SuggestedWords:");
    foreach (var suggest in word.SuggestedWords)
    {
        Console.Write(suggest.Word + " ");
    }
    Console.WriteLine();
}
```

## ขั้นตอนที่ 6: แก้ไขข้อความของผู้ใช้

แก้ไขข้อความที่ผู้ใช้ระบุโดยใช้ไลบรารี Aspose.OCR:

```csharp
// Correct user text
Console.WriteLine("recogniition -> " + api.CorrectSpelling("recogniition"));
```

## ขั้นตอนที่ 7: การแก้ไขด้วยพจนานุกรมผู้ใช้

เพิ่มประสิทธิภาพการแก้ไขโดยรวมพจนานุกรมผู้ใช้แบบกำหนดเองเข้าไป:

```csharp
// Get corrected result with user dictionary
string correctedResultUserDict = result.GetSpellCheckCorrectedText(SpellCheckLanguage.Eng, dataDir+"dictionary.txt");
Console.WriteLine("AFTER CORRECTION WITH USER DICTIONARY:\n" + correctedResultUserDict);
```

## ปัญหาทั่วไปและวิธีแก้

| ปัญหา | สาเหตุ | วิธีแก้ |
|-------|--------|----------|
| ไม่ได้รับข้อเสนอแนะ | แพ็คภาษาไม่ได้โหลดหรือข้อความสั้นเกินไป | ตรวจสอบให้แน่ใจว่า `RecognitionSettings(Language.Eng)` ตรงกับภาษาของภาพต้นฉบับและผลลัพธ์ OCR มีอักขระเพียงพอ |
| พจนานุกรมกำหนดเองไม่ถูกนำไปใช้ | เส้นทางหรือรูปแบบไฟล์ไม่ถูกต้อง | ยืนยันว่า `dictionary.txt` มีอยู่ในตำแหน่งที่ระบุและใช้หนึ่งคำต่อบรรทัด |
| ตัวตรวจสอบการสะกดทำให้เอกสารขนาดใหญ่ช้าลง | การประมวลผลแต่ละคำแยกกันเพิ่มภาระ | ประมวลผลหน้าเป็นชุดหรือเพิ่มการจัดสรรหน่วยความจำหากรันบน .NET Core |
