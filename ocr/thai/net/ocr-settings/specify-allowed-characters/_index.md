# ระบุอักขระที่อนุญาต OCR – ใช้ Aspose.OCR สำหรับ .NET

ในบทเรียนนี้ คุณจะได้เรียนรู้วิธี **ระบุอักขระที่อนุญาต OCR** ด้วย Aspose.OCR สำหรับ .NET เพื่อจำกัดผลลัพธ์ของ OCR ให้มีเฉพาะอักขระที่คุณต้องการ ซึ่งเป็นประโยชน์อย่างยิ่งเมื่อคุณต้อง **จดจำภาพตัวเลข** เช่น หมายเลขซีเรียล, รหัสใบแจ้งหนี้ หรือสตริงที่คล้ายบาร์โค้ด เราจะเดินผ่านการตั้งค่า, โค้ด, และสถานการณ์ใช้งานจริงสองกรณี เพื่อให้คุณสามารถนำเทคนิคนี้ไปใช้ได้ทันที

## คำตอบสั้น
- **“ระบุอักขระที่อนุญาต OCR” ทำอะไร?** จำกัด OCR ให้รับเฉพาะชุดอักขระที่กำหนดไว้ล่วงหน้า เพิ่มความแม่นยำสำหรับข้อมูลที่ต้องการ  
- **ฉันสามารถอนุญาตอักขระอะไรได้บ้าง?** ผสมผสานใดก็ได้ที่ต้องการ—ตัวเลข, ตัวอักษร, หรือสัญลักษณ์พิเศษ (เช่น “0123456789”)  
- **ทำไมต้องจำกัดอักขระ?** ลดการจดจำผิดและเร่งความเร็วการประมวลผลเมื่อชุดอักขระที่คาดหวังเป็นที่ทราบ  
- **ต้องมีลิขสิทธิ์หรือไม่?** สามารถใช้รุ่นทดลองฟรีสำหรับการพัฒนา; ต้องมีลิขสิทธิ์เชิงพาณิชย์สำหรับการใช้งานจริง  
- **รองรับเวอร์ชัน .NET ใดบ้าง?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7  

## “ระบุอักขระที่อนุญาต OCR” คืออะไร?
เมื่อ OCR สแกนภาพ มันจะพยายามจับคู่รูปแบบภาพทุกอย่างกับอักขระทั้งหมดที่เป็นไปได้โดยทั่ว ๆ ไป ด้วยการ **ระบุอักขระที่อนุญาต OCR** คุณบอกให้เอนจินละเลยทุกอย่างที่อยู่นอกรายการที่คุณกำหนดไว้ ซึ่งจะทำให้ความแม่นยำของการจดจำเพิ่มขึ้นอย่างมากสำหรับชุดข้อมูลที่มีขอบเขตจำกัด

## ทำไมต้องใช้ Aspose.OCR เพื่อจดจำภาพตัวเลข?
Aspose.OCR มี API ที่สะอาดและเป็นธรรมชาติสำหรับนักพัฒนา .NET ตัวเลือก `AllowedCharacters` ในตัวช่วยให้คุณมุ่งเน้นที่กรณีที่ต้องการตัวเลขเท่านั้นโดยไม่ต้องเขียนโค้ดประมวลผลหลังจากจดจำเอง นี่เหมาะอย่างยิ่งสำหรับ:
- การอ่านค่ามิเตอร์, หมายเลขใบแจ้งหนี้, หรือรหัสสินค้า  
- การตรวจสอบข้อมูลที่ผู้ใช้กรอกจากแบบฟอร์มที่สแกน  
- การเร่งกระบวนการประมวลผลแบบกลุ่มเมื่อชุดอักขระทราบล่วงหน้า  

## สิ่งที่ต้องมีก่อนเริ่ม

ก่อนจะลงมือเขียนโค้ด โปรดตรวจสอบว่าคุณมี:

- ความรู้พื้นฐานการพัฒนา .NET  
- ไลบรารี **Aspose.OCR for .NET** คุณสามารถดาวน์โหลดได้ [ที่นี่](https://releases.aspose.com/ocr/net/)  
- Visual Studio (หรือ IDE .NET ที่คุณชื่นชอบ)  

## นำเข้า Namespaces

ในโปรเจกต์ .NET ของคุณ ให้นำเข้า namespaces ที่จำเป็นเพื่อใช้ฟังก์ชันของ Aspose.OCR:

```csharp
using System;
using System.IO;
using Aspose.OCR;
```

ต่อไปเราจะแบ่งบทเรียนออกเป็นขั้นตอนที่ครบถ้วน:

## วิธีระบุอักขระที่อนุญาต OCR – คู่มือขั้นตอนโดยละเอียด

### ขั้นตอนที่ 1: ตั้งค่าเส้นทางไปยังโฟลเดอร์รูปภาพของคุณ

กำหนดตำแหน่งที่เก็บรูปภาพตัวอย่างของคุณก่อน

```csharp
string dataDir = "Your Document Directory";
```

### ขั้นตอนที่ 2: เริ่มต้น Aspose.OCR ด้วย whitelist เฉพาะตัวเลข

สร้างอินสแตนซ์ `AsposeOcr` และส่งอักขระที่ต้องการอนุญาตให้—ในกรณีนี้คือเลขทั้งหมด

```csharp
AsposeOcr api = new AsposeOcr("0123456789");
```

### ขั้นตอนที่ 3: จดจำบรรทัดเดียวที่มีตัวเลข

ใช้เมธอด `RecognizeLine` เพื่อดึงข้อความจากภาพที่มีเฉพาะตัวเลข

```csharp
string result = api.RecognizeLine(dataDir + "0001460985.Jpeg");
```

### ขั้นตอนที่ 4: แสดงผลตัวเลขที่จดจำได้

พิมพ์ผลลัพธ์ลงคอนโซลเพื่อให้คุณตรวจสอบได้

```csharp
Console.WriteLine(result);
```

### ขั้นตอนที่ 5: ใช้ RecognitionSettings เพื่อควบคุมเพิ่มเติม

หากต้องการการควบคุมที่ละเอียดกว่า—เช่นบังคับให้จดจำแบบบรรทัดเดียว—คุณสามารถใช้ overload ที่รับ `RecognitionSettings`

```csharp
AsposeOcr api2 = new AsposeOcr();
RecognitionResult result2 = api2.RecognizeImage(dataDir + "0001460985.Jpeg", 
    new RecognitionSettings { 
        AllowedCharacters = CharactersAllowedType.DIGITS,
        RecognizeSingleLine = true
    });
```

### ขั้นตอนที่ 6: แสดงผลกรณีที่สอง

```csharp
Console.WriteLine(result2.RecognitionText);
```

### ขั้นตอนที่ 7: ยืนยันการทำงานสำเร็จ

```csharp
Console.WriteLine("SpecifyAllowedCharacters executed successfully");
```

โดยทำตามขั้นตอนเหล่านี้ คุณได้เรียนรู้วิธี **ระบุอักขระที่อนุญาต OCR** และจดจำเนื้อหา **ภาพตัวเลข** อย่างมีประสิทธิภาพด้วย Aspose.OCR สำหรับ .NET

## ข้อผิดพลาดทั่วไปและการแก้ไขปัญหา

- **ผลลัพธ์เป็นค่าว่าง:** ตรวจสอบคุณภาพของภาพ (คอนทราสต์ชัด, สัญญาณรบกวนน้อย)  
- **อักขระที่ได้ไม่ตรง:** ตรวจสอบให้แน่ใจว่า string whitelist ตรงกับอักขระที่คุณต้องการอย่างแม่นยำ  
- **ไม่พบไฟล์:** ยืนยันว่า `dataDir` ชี้ไปยังโฟลเดอร์ที่ถูกต้องและชื่อไฟล์ตรงตามตัวพิมพ์ใหญ่‑เล็ก
