# สร้าง ROI Polygon – คู่มือเต็มสำหรับ OCR บนพื้นที่ที่เลือก

เคยต้อง **สร้าง ROI polygon** เพื่อให้คุณสามารถทำ OCR ได้เฉพาะส่วนของภาพที่สำคัญหรือไม่? บางครั้งคุณอาจสแกนใบเสร็จและต้องการเพียงยอดรวม, หรือมีแบบฟอร์มที่ต้องอ่านเฉพาะช่องลายเซ็นเท่านั้น. ในกรณีเหล่านี้, **ocr on selected area** จะช่วยประหยัดเวลาและเพิ่มความแม่นยำ.  

ในคู่มือนี้เราจะพาคุณผ่านทุกขั้นตอนที่ต้องทำ: ตั้งแต่การกำหนดสอง polygon, ลงทะเบียนกับเครื่องมือ OCR, จนถึงการดึงข้อความที่จดจำได้ออกมาในหนึ่งคำสั่งเรียกเดียว. เมื่อเสร็จสิ้นคุณจะมีสคริปต์พร้อมรันและเข้าใจเหตุผลว่าทำไมการโฟกัสที่พื้นที่สนใจจึงสำคัญ.

## สิ่งที่คุณจะได้เรียนรู้

- วิธี **สร้าง ROI polygon** ด้วยไลบรารี OCR ที่ทั่วไป.
- ความแตกต่างระหว่างการกำหนดพื้นที่เดียวกับหลายพื้นที่.
- วิธีลงทะเบียนพื้นที่เหล่านั้นกับเอนจินเพื่อให้ทำ `ocr on selected area`.
- ผลลัพธ์ที่คาดหวังและวิธีแก้ปัญหาข้อผิดพลาดทั่วไป (เช่น ROI ซ้อนกัน, ผลลัพธ์ว่าง).

### ข้อกำหนดเบื้องต้น

- ติดตั้ง Python 3.8+.
- ไลบรารี OCR ที่มีเมธอด `Polygon`, `Point`, และ `add_region_of_interest` (ตัวอย่างใช้โมดูล `ocr` สมมติ; แทนที่ด้วย SDK ของคุณจริง เช่น Tesseract‑Python wrappers หรือ EasyOCR).
- ไฟล์ภาพตัวอย่าง (`sample.png`) ที่มีข้อความตามพิกัดที่แสดงด้านล่าง.

> **เคล็ดลับ:** หากคุณใช้ไลบรารีจริง, ตรวจสอบให้แน่ใจว่าภาพถูกโหลดในระบบพิกัดเดียวกัน (จุดกำเนิดที่มุมซ้ายบน, X เพิ่มไปทางขวา, Y เพิ่มลงล่าง).  

---  

## ขั้นตอนที่ 1: นำเข้าโมดูล OCR และโหลดภาพของคุณ  

แรกสุดให้เรียกใช้แพคเกจ OCR และอ่านภาพที่ต้องการประมวลผล.  

```python
import ocr  # Replace with your actual OCR library import
from pathlib import Path

# Load the image – adjust the path as needed
image_path = Path("sample.png")
ocr_engine = ocr.Engine(image_path)
```

*เหตุผลที่สำคัญ:* เอนจินต้องการข้อมูลภาพก่อนที่คุณจะสามารถแนบ **ROI polygon** ใด ๆ ได้. ไลบรารีบางตัวอาจให้คุณส่งภาพภายหลัง, แต่การเริ่มต้นล่วงหน้าจะทำให้ workflow ดูเป็นระเบียบมากขึ้น.

## ขั้นตอนที่ 2: กำหนด ROI Polygon แรก  

ตอนนี้เราจะสร้างพื้นที่สนใจแรก. คิดว่าเป็นการวาดสี่เหลี่ยมรอบหัวตาราง.  

```python
# Step 2: Define the first ROI polygon
first_roi = ocr.Polygon([
    ocr.Point(50, 20),   # top‑left
    ocr.Point(200, 20),  # top‑right
    ocr.Point(200, 80),  # bottom‑right
    ocr.Point(50, 80)    # bottom‑left
])
```

*คำอธิบาย:*  
- `ocr.Point(x, y)` ใช้พิกัดพิกเซล.  
- รายการจุดจะเรียงตามเข็มนาฬิกา, ซึ่งส่วนใหญ่ของเอนจินคาดหวัง.  
- คุณสามารถเพิ่มจุดได้ตามต้องการเพื่อสร้างรูปทรงที่ไม่เป็นสี่เหลี่ยม; สี่เหลี่ยมเป็นกรณีพิเศษเท่านั้น.

## ขั้นตอนที่ 3: กำหนด ROI Polygon เพิ่มเติม (ไม่บังคับ)  

คุณไม่จำเป็นต้องหยุดที่หนึ่งเท่านั้น. ต่อไปนี้คือตัวอย่าง polygon ที่จับช่องลายเซ็นด้านล่างของหน้า.  

```python
# Step 3: Define the second ROI polygon
second_roi = ocr.Polygon([
    ocr.Point(300, 150),
    ocr.Point(480, 150),
    ocr.Point(480, 210),
    ocr.Point(300, 210)
])
```

*ทำไมต้องเพิ่มอีก?* การทำ **ocr on selected area** สำหรับหลายโซนช่วยให้คุณดึงข้อมูลที่แตกต่างกันในหนึ่งรอบ, ซึ่งเร็วกว่าแครอปและประมวลผลแต่ละส่วนแยกกัน.

## ขั้นตอนที่ 4: ลงทะเบียน ROI กับเอนจิน OCR  

เมื่อ polygon พร้อม, บอกเอนจินว่าต้องตรวจสอบพื้นที่ไหน.  

```python
# Step 4: Register the ROIs
ocr_engine.add_region_of_interest(first_roi)
ocr_engine.add_region_of_interest(second_roi)
```

*หมายเหตุสำคัญ:* SDK บางตัวต้องเรียกเมธอด `clear_regions()` ก่อนเพิ่มใหม่ หากคุณใช้เอนจินเดียวกันประมวลผลหลายภาพ.

## ขั้นตอนที่ 5: รัน OCR และดึงข้อความออกมา  

สุดท้ายให้เรียกการจดจำ. เอนจินจะมองเฉพาะภายใน polygon ที่เราเพิ่มเท่านั้น.  

```python
# Step 5: Perform OCR on the defined regions
recognized_text = ocr_engine.recognize().get_text()
print("=== Recognized Text ===")
print(recognized_text)
```

**ผลลัพธ์ที่คาดหวัง** (สมมติว่า `sample.png` มีคำว่า “Invoice Total: $123.45” ใน ROI แรกและ “Signature: John Doe” ใน ROI ที่สอง):

```
=== Recognized Text ===
Invoice Total: $123.45
Signature: John Doe
```

หากผลลัพธ์เป็นค่าว่าง, ตรวจสอบว่าพิกัดจริง ๆ ตัดกับข้อความและความละเอียดของภาพตรงกับระบบพิกัดหรือไม่.

## กรณีขอบและข้อผิดพลาดทั่วไป  

| สถานการณ์                              | สิ่งที่ต้องระวัง                                 | วิธีแก้ / ทางเลือก                              |
|----------------------------------------|--------------------------------------------------|-----------------------------------------------|
| **ROI ซ้อนกัน**                        | ข้อความอาจถูกคืนค่าซ้ำสองครั้ง.                | ทำให้ polygon ไม่ทับกันหรือทำการ deduplicate ผลลัพธ์. |
| **ROI อยู่นอกขอบภาพ**                | เอนจินอาจโยงข้อผิดพลาดหรือคืนค่าไม่มีอะไร.   | จำกัดพิกัดให้อยู่ในช่วง `0 ≤ x < width`, `0 ≤ y < height`. |
| **ROI ขนาดเล็กมาก**                    | ความแม่นยำของ OCR ลดลงเนื่องจากพิกเซลไม่พอ.   | ขยาย polygon เพิ่มหลายพิกเซลหรืออัปสเกลภาพก่อน. |
| **รูปทรงไม่เป็นสี่เหลี่ยม**            | บางเอนจินรองรับเฉพาะ convex polygon เท่านั้น. | แยกรูปทรงซับซ้อนเป็นหลาย convex ROI. |
| **ขนาดภาพเปลี่ยน**                    | พิกัดที่กำหนดแบบคงที่จะใช้ไม่ได้.              | คำนวณพิกัด ROI ตามขนาดภาพ (เช่น เปอร์เซ็นต์). |

## ตัวอย่างทำงานเต็มรูปแบบ  

ด้านล่างเป็นสคริปต์ทั้งหมดที่คุณสามารถคัดลอก‑วางและรันได้ (แทนที่ `ocr` ด้วยไลบรารีของคุณ).  

```python
import ocr                      # <- your OCR library
from pathlib import Path

# -------------------------------------------------
# Configuration
# -------------------------------------------------
IMAGE_PATH = Path("sample.png")

# -------------------------------------------------
# Initialize OCR engine with the image
# -------------------------------------------------
engine = ocr.Engine(IMAGE_PATH)

# -------------------------------------------------
# Define ROI polygons
# -------------------------------------------------
first_roi = ocr.Polygon([
    ocr.Point(50, 20), ocr.Point(200, 20),
    ocr.Point(200, 80), ocr.Point(50, 80)
])

second_roi = ocr.Polygon([
    ocr.Point(300, 150), ocr.Point(480, 150),
    ocr.Point(480, 210), ocr.Point(300, 210)
])

# -------------------------------------------------
# Register ROIs
# -------------------------------------------------
engine.add_region_of_interest(first_roi)
engine.add_region_of_interest(second_roi)

# -------------------------------------------------
# Run OCR on the selected areas
# -------------------------------------------------
result = engine.recognize().get_text()

print("=== Recognized Text ===")
print(result)
```

รันด้วยคำสั่ง `python ocr_roi_example.py` แล้วคุณจะเห็นข้อความที่ดึงจากสองโซนที่กำหนด.

## ขั้นตอนต่อไป  

- **การสร้าง ROI แบบไดนามิก:** แทนการกำหนดพิกัดคงที่, ใช้การประมวลผลภาพ (เช่น OpenCV contour detection) เพื่อค้นหาโต๊ะหรือฟิลด์โดยอัตโนมัติ.  
- **การประมวลผลหลังจาก OCR:** ตัด whitespace, ใช้ regex, หรือแปลงตัวเลขเป็นชนิดข้อมูลที่เหมาะสม.  
- **การประมวลผลเป็นชุด:** วนลูปผ่านโฟลเดอร์ของภาพ, ใช้ ROI เดิมซ้ำหากเลเอาต์สอดคล้องกัน.  

หากคุณสนใจ **ocr on selected area** สำหรับเอกสารที่ซับซ้อนกว่า—เช่น PDF หลายหน้า หรือแบบฟอร์มสแกน—ลองดูไลบรารีที่รองรับการลงทะเบียน ROI ตามหน้า.
