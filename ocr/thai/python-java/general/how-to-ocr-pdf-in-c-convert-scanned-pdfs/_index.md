# วิธีทำ OCR PDF ใน C# – แปลง PDF ที่สแกน

เคยต้องการ **how to OCR PDF** แต่ติดอยู่กับไฟล์สแกนที่ไม่ยอมทำงาน? คุณไม่ได้เป็นคนเดียว ในหลายโครงการจริง—เช่นการแปลงใบแจ้งหนี้เป็นดิจิทัลหรือการเก็บบันทึกประวัติ—คุณจะเจอกับกองรูปภาพที่ฝังอยู่ใน PDF และวิธีเดียวที่จะทำให้สามารถค้นหาได้คือการทำ OCR กับมัน.  

ข่าวดีคือ? ด้วยไม่กี่บรรทัดของ C# คุณสามารถ **convert scanned PDF** เป็น **create searchable PDF** เพิ่มลายน้ำแบบเบา ๆ และแม้กระทั่งดึงข้อความดิบออกมาสำหรับการประมวลผลต่อไป ด้านล่างคุณจะเห็นตัวอย่างที่สมบูรณ์พร้อมรันที่ทำสิ่งนั้นได้อย่างแม่นยำ.

## สิ่งที่บทเรียนนี้ครอบคลุม

* วิธี **how to OCR PDF** ด้วยคลาส `OcrEngine`.  
* แปลง PDF ที่สแกนเป็น **create searchable PDF**.  
* เพิ่มลายน้ำบนหน้าแรก (**add watermark pdf**).  
* ดึงสตริงข้อความธรรมดาจากผลลัพธ์ (**extract text from pdf**).  
* ข้อผิดพลาดทั่วไป—เช่นการจัดการเอกสารหลายหน้า หรือการจัดการสแกนความละเอียดต่ำ.

ไม่มีลิงก์เอกสารภายนอก ไม่มีโค้ดที่ทำครึ่งหนึ่ง ทุกอย่างที่คุณต้องการอยู่ที่นี่.

## ข้อกำหนดเบื้องต้น

* .NET 6.0 หรือใหม่กว่า (โค้ดนี้ยังคอมไพล์ได้กับ .NET 5 ด้วย).  
* อ้างอิงไปยังไลบรารี OCR ที่ให้ `OcrEngine` และ `ImageFormats` (เช่น `Aspose.OCR` หรือแพ็คเกจที่คล้ายกัน).  
* ไฟล์ PDF อินพุตชื่อ `input.pdf` ที่วางในโฟลเดอร์ที่คุณควบคุม.  
* ความคุ้นเคยพื้นฐานกับแอปคอนโซล C#.

ถ้าคุณมีแล้ว เยี่ยม—มาเริ่มกันเลย.

## ขั้นตอนที่ 1: ตั้งค่าโปรเจกต์และนำเข้าขึ้นต่อ

สร้างโปรเจกต์คอนโซลใหม่และเพิ่มแพคเกจ OCR ผ่าน NuGet:

```bash
dotnet new console -n OcrPdfDemo
cd OcrPdfDemo
dotnet add package Aspose.OCR
```

จากนั้นเปิดไฟล์ `Program.cs` ที่ด้านบน นำเข้า namespace ที่คุณต้องการใช้:

```csharp
using System;
using Aspose.OCR;          // OcrEngine lives here
using Aspose.OCR.Image;   // ImageFormats enum
```

*ทำไมเรื่องนี้สำคัญ*: การนำเข้า namespace ที่ถูกต้องทำให้คอมไพเลอร์หาตำแหน่ง `OcrEngine` ได้ การข้ามขั้นตอนนี้จะทำให้เกิดข้อผิดพลาด `CS0246` และหยุดการสร้าง.

## ขั้นตอนที่ 2: เริ่มต้น OCR Engine

Engine คือหัวใจของกระบวนการ คุณสร้างอินสแตนซ์หนึ่งครั้งและใช้ซ้ำสำหรับทุกหน้า.

```csharp
// Step 2: Initialize the OCR engine
OcrEngine engine = new OcrEngine();
```

*เคล็ดลับ*: หากคุณวางแผนจะประมวลผล PDF หลายไฟล์ต่อเนื่อง ควรใช้อินสแตนซ์ `engine` เดียวกันเพื่อหลีกเลี่ยงการตรวจสอบไลเซนส์ซ้ำ.

## ขั้นตอนที่ 3: โหลด PDF ต้นฉบับ

บอก engine ว่าไฟล์ใดที่จะทำงาน วิธี `SetImageFromFile` ยอมรับแหล่งที่มาของรูปภาพหรือ PDF ใดก็ได้.

```csharp
// Step 3: Load the scanned PDF you want to OCR
engine.SetImageFromFile(@"YOUR_DIRECTORY\input.pdf");
```

> **ทำไมขั้นตอนนี้สำคัญ** – Engine ต้องการการแสดงผลแบบ bitmap ของแต่ละหน้า เมื่อคุณป้อน PDF ให้มัน Engine จะทำการแรสเตอร์แต่ละหน้าโดยอิง DPI เริ่มต้น (โดยทั่วไป 300) หาก PDF ต้นฉบับเป็นข้อความอยู่แล้ว Engine จะผ่านไปโดยไม่เปลี่ยนแปลง ช่วยประหยัดเวลา.

## ขั้นตอนที่ 4: รันการจดจำ OCR

ตอนนี้งานหนักเริ่มทำงาน วิธี `Recognize` จะสแกนทุกหน้า ดึง glyphs และสร้างเลเยอร์ที่ค้นหาได้.

```csharp
// Step 4: Perform OCR on the loaded document
engine.Recognize();
```

*คำถามทั่วไป*: *ถ้า PDF มี 50 หน้า?*  
`Recognize` จะประมวลผลเอกสารทั้งหมดโดยค่าเริ่มต้น สำหรับสภาพแวดล้อมที่มีหน่วยความจำจำกัด คุณสามารถตั้งค่า `engine.PageIndex` และ `engine.PageCount` เพื่อทำงานทีละหน้า.

## ขั้นตอนที่ 5: บันทึก PDF ที่ค้นหาได้ (ลายน้ำบนหน้า 1)

เราจะบันทึกผลลัพธ์เป็น PDF หน้าแรกจะได้รับลายน้ำโดยอัตโนมัติ เนื่องจากไลบรารี OCR เพิ่ม overlay “Generated by Aspose.OCR” เป็นค่าเริ่มต้น หากต้องการลายน้ำแบบกำหนดเอง สามารถแก้ไข property `engine.Watermark` ก่อนบันทึก.

```csharp
// Step 5: Save the OCR‑processed PDF with a watermark on page 1
engine.Save(@"YOUR_DIRECTORY\output.pdf", ImageFormats.Pdf);
```

หลังจากเรียกนี้คุณจะได้ **create searchable PDF** ที่คุณสามารถเลือก คัดลอก และค้นหาข้อความได้เช่นเดียวกับ PDF ธรรมดา.

## ขั้นตอนที่ 6: ดึงข้อความธรรมดา (ไม่บังคับแต่มีประโยชน์)

หากคุณต้องการ **extract text from pdf** เพื่อทำดัชนีหรือวิเคราะห์ต่อไป Engine จะให้คุณเข้าถึงสตริงดิบ.

```csharp
// Step 6: Pull the recognized text into a string
string extractedText = engine.Text;
Console.WriteLine("=== Extracted Text Start ===");
Console.WriteLine(extractedText);
Console.WriteLine("=== Extracted Text End ===");
```

property `engine.Text` จะต่อข้อความของทุกหน้าเข้าด้วยกันโดยคงการขึ้นบรรทัดใหม่ คุณสามารถแยกด้วย `\f` (form feed) หากต้องการระดับหน้าต่างๆ.

## ตัวอย่างทำงานเต็มรูปแบบ

ด้านล่างเป็นโปรแกรมเต็มรูปแบบ คัดลอกและวางลงใน `Program.cs` แทนที่ `YOUR_DIRECTORY` ด้วยพาธแบบเต็มหรือสัมพันธ์ แล้วรัน `dotnet run`.

```csharp
using System;
using Aspose.OCR;
using Aspose.OCR.Image;

namespace OcrPdfDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // 1️⃣ Initialize the OCR engine
            OcrEngine engine = new OcrEngine();

            // 2️⃣ Load the source PDF to be processed
            // Make sure the file exists; otherwise you'll get a FileNotFoundException.
            string inputPath = @"YOUR_DIRECTORY\input.pdf";
            engine.SetImageFromFile(inputPath);

            // 3️⃣ Perform OCR recognition on the loaded document
            engine.Recognize();

            // 4️⃣ Save the recognized document as a PDF (watermark appears on page 1)
            string outputPath = @"YOUR_DIRECTORY\output.pdf";
            engine.Save(outputPath, ImageFormats.Pdf);

            // 5️⃣ (Optional) Extract plain text for further use
            string extractedText = engine.Text;
            Console.WriteLine("=== Extracted Text Start ===");
            Console.WriteLine(extractedText);
            Console.WriteLine("=== Extracted Text End ===");

            Console.WriteLine($"OCR complete! Searchable PDF saved to: {outputPath}");
        }
    }
}
```

### ผลลัพธ์ที่คาดหวัง

การรันโปรแกรมจะพิมพ์ข้อความที่ดึงออกมาที่คอนโซลและสร้างไฟล์ `output.pdf` เปิด PDF ด้วยโปรแกรมดูใดก็ได้ (Adobe Acrobat, Edge ฯลฯ) แล้วลองค้นหาคำที่ปรากฏในสแกนต้นฉบับ คุณจะเห็นว่าตอนนี้สามารถค้นหาได้ หน้าแรกจะแสดงลายน้ำค่าเริ่มต้น ยืนยันว่าขั้นตอน **add watermark pdf** สำเร็จ.

## คำถามที่พบบ่อยและกรณีขอบ

| Question | Answer |
|----------|--------|
| **ถ้าการสแกนมีความละเอียดต่ำ?** | เพิ่ม DPI ก่อนทำการจดจำ: `engine.ImageInfo.DpiX = engine.ImageInfo.DpiY = 600;` ซึ่งจะให้รายละเอียดมากขึ้นกับ OCR engine. |
| **ฉันสามารถเลือกลายน้ำแบบกำหนดเองได้ไหม?** | ได้. ตั้งค่า `engine.Watermark = "Confidential – Processed on " + DateTime.Now.ToShortDateString();` ก่อน `Save`. |
| **ฉันจะจัดการกับ PDF ที่มีรหัสผ่านอย่างไร?** | ใช้ `engine.LoadPassword = "yourPassword";` ก่อน `SetImageFromFile`. |
| **มีวิธีจำกัด OCR ให้ทำงานเฉพาะหน้าที่กำหนดหรือไม่?** | ตั้งค่า `engine.PageIndex = 2;` และ `engine.PageCount = 5;` เพื่อประมวลผลเฉพาะหน้าที่ 3‑7 เท่านั้น. |
| **ถ้าฉันต้องการเก็บภาพต้นฉบับไว้โดยไม่เปลี่ยนแปลงจะทำอย่างไร?** | หลังจาก OCR คุณสามารถรวม PDF ต้นฉบับกับเลเยอร์ OCR ด้วยไลบรารี PDF (เช่น `Aspose.PDF`) เพื่อรักษาความคมชัดของภาพ. |

## เคล็ดลับและแนวทางปฏิบัติที่ดีที่สุด

* **เคล็ดลับ:** รัน OCR บนเธรดพื้นหลังหากคุณนำไปใช้ในแอป UI—ไม่เช่นนั้น UI จะค้าง.  
* **ระวัง:** PDF ที่มีเนื้อหาร่วมระหว่าง raster และ vector Engine จะ OCR เฉพาะหน้าที่เป็น raster; ข้อความ vector จะยังคงเลือกได้โดยอัตโนมัติ.  
* **ปรับประสิทธิภาพ:** แคชข้อมูลภาษาของ OCR (`engine.Language = OcrLanguage.English;`) เพื่อหลีกเลี่ยงการโหลดซ้ำสำหรับแต่ละเอกสาร.

## สรุป

ตอนนี้คุณมีโซลูชันครบวงจรสำหรับ **how to OCR PDF** ใน C# โดยการเริ่มต้น `OcrEngine` โหลดไฟล์สแกน ทำการจดจำข้อความ บันทึกเป็น **create searchable pdf** เพิ่มลายน้ำ และถ้าต้องการ **extract text from pdf** คุณสามารถแปลง PDF ที่เป็นภาพใดก็ได้ให้เป็นทรัพย์สินที่ค้นหาและทำดัชนีได้.

ขั้นตอนต่อไป? ลองเชื่อมโยงเวิร์กโฟลว์นี้กับระบบจัดการเอกสาร หรือทดลองใช้ภาษาต่าง ๆ (`engine.Language = OcrLanguage.French;`) คุณยังสามารถสำรวจการประมวลผลเป็นชุดหลายไฟล์ในโฟลเดอร์—เพียงวนลูปขั้นตอนด้วยอินสแตนซ์ `engine` ใหม่ทุกครั้ง.

ขอให้เขียนโค้ดอย่างสนุกสนาน และขอให้ PDF ของคุณสามารถค้นหาได้เสมอ!
