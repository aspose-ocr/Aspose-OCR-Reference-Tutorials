# Python में ROI को OCR कैसे करें – प्रीप्रोसेसिंग कैसे सेट करें

क्या आपने कभी शोरयुक्त इनवॉइस इमेज में पूरे पेज को मेमोरी में लोड किए बिना **ROI को OCR कैसे करें** के बारे में सोचा है? आप अकेले नहीं हैं। कई वास्तविक‑दुनिया प्रोजेक्ट्स में हमें केवल कुछ फ़ील्ड्स की ज़रूरत होती है—ग्राहक का नाम, पता, कुल—इसलिए पूरे दस्तावेज़ को स्कैन करना ज़रूरत से अधिक है।  

अच्छी खबर? Aspose OCR के साथ आप इंजन को बिल्कुल **कहाँ देखना है** बता सकते हैं और यहाँ तक कि पहले इमेज को साफ़ करने को भी कह सकते हैं। इस ट्यूटोरियल में हम **प्रीप्रोसेसिंग कैसे सेट करें** विकल्पों को समझेंगे, Regions of Interest (ROIs) को परिभाषित करेंगे, और केवल कुछ पंक्तियों के Python कोड से साफ़ टेक्स्ट निकालेंगे।  

इस गाइड के अंत तक आपके पास एक तैयार‑चलाने‑योग्य स्क्रिप्ट होगी जो किसी भी इनवॉइस, रसीद, या फ़ॉर्म से विशिष्ट ब्लॉक्स निकालती है। कोई अतिरिक्त टूल्स नहीं चाहिए, सिर्फ Aspose OCR और थोड़ा Python लॉजिक।

## आपको क्या चाहिए

- **Python 3.8+** (कोड किसी भी हालिया संस्करण पर काम करता है)  
- **Aspose OCR for Python via .NET** – `pip install aspose-ocr` के साथ इंस्टॉल करें  
- एक सैंपल इमेज (जैसे `invoice.png`) को किसी फ़ोल्डर में रखें जिसे आप रेफ़र कर सकें  
- Python फ़ंक्शन्स और ऑब्जेक्ट‑ओरिएंटेड सिंटैक्स की बुनियादी समझ  

यदि आपके पास ये सब है, तो बढ़िया—आइए सीधे कोड में कूदते हैं।

## चरण 1 – OCR इंजन को इनिशियलाइज़ करें (ROI को OCR कैसे करें)

इंजन को *कहाँ* देखना है बताने से पहले, हमें OCR प्रोसेसर की एक इंस्टेंस चाहिए। यह ऑब्जेक्ट सभी कॉन्फ़िगरेशन रखता है और पहचान प्रक्रिया चलाता है।

```python
import aspose.ocr as aocr
import aspose.storage as storage

# Create the engine – this is the entry point for all OCR work
ocr_engine = aocr.OcrEngine()
```

*क्यों महत्वपूर्ण है*: `OcrEngine` क्लास भारी काम (फ़ॉन्ट डिटेक्शन, लेआउट एनालिसिस, आदि) को एब्स्ट्रैक्ट करती है। एक ही इंजन बनाकर आप प्रत्येक इमेज के लिए भारी रिसोर्सेज को फिर से इनिशियलाइज़ करने से बचते हैं, जिससे प्रदर्शन तेज़ रहता है।

---

## चरण 2 – अपना स्रोत इमेज लोड करें (ROI को OCR कैसे करें)

Aspose Storage इमेज लोड करना आसान बनाता है। इसे फ़ाइल पाथ पर पॉइंट करें, और आपको प्रोसेसिंग के लिए तैयार `Image` ऑब्जेक्ट मिल जाता है।

```python
# Replace YOUR_DIRECTORY with the actual path where invoice.png lives
source_image = storage.Image.load("YOUR_DIRECTORY/invoice.png")
```

*टिप*: यदि आप स्ट्रीम्स के साथ काम कर रहे हैं (जैसे, वेब API के माध्यम से अपलोड की गई इमेज), तो आप फ़ाइल पाथ की बजाय `Image.load` में `BytesIO` ऑब्जेक्ट पास कर सकते हैं।

## चरण 3 – Regions of Interest (ROIs) को परिभाषित करें

अब हम मुख्य प्रश्न **ROI को OCR कैसे करें** का उत्तर देते हैं: हम इंजन को उन सटीक आयतों के बारे में बताते हैं जिनमें वह डेटा है जो हमें चाहिए। प्रत्येक ROI को उसके टॉप‑लेफ़्ट कोने (`x`, `y`) और आकार (`width`, `height`) से परिभाषित किया जाता है।

```python
from aspose.ocr import Roi

# Each tuple is (x, y, width, height) in pixels
regions_of_interest = [
    Roi(50, 120, 400, 60),   # Customer name block
    Roi(50, 200, 400, 80)    # Address block
]
```

*ROIs क्यों उपयोग करें?*  
- **स्पीड** – इंजन इमेज के अप्रासंगिक भागों को स्किप करता है।  
- **एक्यूरेसी** – छोटे क्षेत्र पर फोकस करने से अन्य जगह का शोर recognizer को भ्रमित नहीं कर पाता।  
- **लचीलापन** – आप जितने भी ROIs चाहें जोड़ सकते हैं; इंजन उसी क्रम में परिणामों की सूची लौटाता है।

---

## चरण 4 – बेहतर OCR एक्यूरेसी के लिए प्रीप्रोसेसिंग कैसे सेट करें

स्कैनर या फोन से सीधे आए इमेज अक्सर स्क्यू, कम कॉन्ट्रास्ट, या असमान लाइटिंग से प्रभावित होते हैं। Aspose OCR एक `PreprocessingOptions` ऑब्जेक्ट प्रदान करता है जो पहचान चलने से पहले सामान्य सुधारों को सक्षम करने देता है।

```python
from aspose.ocr import PreprocessingOptions

preprocessing_options = PreprocessingOptions()
preprocessing_options.deskew = True          # Auto‑rotate tilted text
preprocessing_options.binarize = True        # Convert to black‑white for clarity
preprocessing_options.contrast = 1.5         # Boost contrast (1.0 = no change)
```

*यह कैसे मदद करता है*:  
- **Deskew** वह झुकाव हटाता है जो अक्षर के आकार को अस्पष्ट बनाता है।  
- **Binarize** रंगीन शोर को कम करता है, OCR इंजन को एक साफ़ बाइनरी इमेज देता है।  
- **Contrast** हल्की स्ट्रोक्स को बढ़ाता है, जो फीके रसीदों के लिए विशेष रूप से उपयोगी है।

आप इन फ़्लैग्स के साथ प्रयोग कर सकते हैं—एक को बंद करें और देखें कि आउटपुट बदलता है या नहीं। यही है **प्रीप्रोसेसिंग कैसे सेट करें** का प्रभावी तरीका।

---

## चरण 5 – केवल परिभाषित ROIs के भीतर OCR करें

इंजन, इमेज, ROIs, और प्रीप्रोसेसिंग तैयार होने के बाद, `recognize` को कॉल करने का समय है। यह मेथड एक `OcrResult` ऑब्जेक्ट लौटाता है जिसमें `regions` कलेक्शन होता है—प्रत्येक एंट्री हमारे द्वारा प्रदान किए गए ROI क्रम से मेल खाती है।

```python
ocr_result = ocr_engine.recognize(
    source_image,
    rois=regions_of_interest,
    preprocessing=preprocessing_options
)
```

*पर्दे के पीछे*: इंजन प्रत्येक ROI को क्रॉप करता है, प्रीप्रोसेसिंग पाइपलाइन लागू करता है, और साफ़ किए गए स्निपेट पर पहचान मॉडल चलाता है। क्योंकि हमने एक लिस्ट पास की थी, परिणाम वही क्रम रखता है, जिससे पोस्ट‑प्रोसेसिंग सरल हो जाता है।

---

## चरण 6 – निकाले गए टेक्स्ट को पढ़ें और उपयोग करें (ROI को OCR कैसे करें)

अंत में, `regions` लिस्ट पर इटरेट करें और पहचाने गए स्ट्रिंग्स को प्रिंट—या स्टोर—करें। `Region` ऑब्जेक्ट एक `text` प्रॉपर्टी एक्सपोज़ करता है जो Unicode परिणाम रखती है।

```python
for idx, region in enumerate(ocr_result.regions, start=1):
    print(f"Region {idx} text:")
    print(region.text)
    print("-" * 40)
```

**अपेक्षित आउटपुट (उदाहरण)**:

```
Region 1 text:
John Doe
----------------------------------------
Region 2 text:
123 Maple Street
Springfield, IL 62704
----------------------------------------
```

यदि टेक्स्ट गड़बड़ दिखता है, तो **प्रीप्रोसेसिंग कैसे सेट करें** को फिर से देखें: शायद `contrast` बढ़ाएँ या रंगीन लोगो के लिए `binarize` को डिसेबल करें।

---

## सामान्य समस्याएँ और प्रो टिप्स

| Issue | Why it Happens | Fix (How to Set Preprocessing) |
|-------|----------------|--------------------------------|
| झुकी हुई टेक्स्ट गड़बड़ दिखती है | `deskew` निष्क्रिय या इमेज बहुत घुमा हुआ | `preprocessing_options.deskew = True` सक्षम करें |
| संख्याएँ बिंदु या बिखरे निशान बन जाती हैं | कम कॉन्ट्रास्ट या अत्यधिक बाइनराइज़ेशन | `contrast` कम करें (उदा., `1.2`) या `binarize = False` सेट करें |
| ROI कॉर्डिनेट्स कुछ पिक्सेल ऑफ़ हैं | विभिन्न DPI या स्कैनर स्केलिंग | किसी टूल (जैसे Paint.NET) से सटीक पिक्सेल पोज़िशन मापें, या प्रत्येक ROI में छोटा मार्जिन (`+5` पिक्सेल) जोड़ें |
| एक क्षेत्र के लिए खाली परिणाम | ROI इमेज की सीमा से बाहर है | इमेज डाइमेंशन जांचें: `source_image.width`, `source_image.height` |

## समाधान का विस्तार (विभिन्न परिदृश्यों में ROI को OCR कैसे करें)

- **Dynamic ROIs**: यदि आपके इनवॉइस का लेआउट बदलता रहता है, तो आप पहले एक तेज़ फुल‑पेज OCR चलाकर कीवर्ड्स (“Customer:”, “Address:”) खोज सकते हैं और फिर ऑन‑द‑फ्लाई ROIs गणना कर सकते हैं।  
- **Batch Processing**: ऊपर के चरणों को एक फ़ंक्शन में रैप करें और इमेज फ़ोल्डर पर लूप चलाएँ। मेमोरी उपयोग कम रखने के लिए वही `ocr_engine` इंस्टेंस पुनः उपयोग करना याद रखें।  
- **Exporting to JSON**: प्रिंट करने के बजाय, एक डिक्शनरी बनाएं और `json.dumps` से डंप करें; यह डाउनस्ट्रीम इंटीग्रेशन (जैसे ERP सिस्टम को फीड करना) को सरल बनाता है।

```python
import json

def extract_invoice_fields(image_path):
    img = storage.Image.load(image_path)
    result = ocr_engine.recognize(
        img,
        rois=regions_of_interest,
        preprocessing=preprocessing_options
    )
    data = {f"field_{i+1}": r.text.strip() for i, r in enumerate(result.regions)}
    return data

print(json.dumps(extract_invoice_fields("invoice.png"), indent=2))
```

## निष्कर्ष

अब आपके पास एक पूर्ण, चलाने योग्य उदाहरण है जो Python में **ROI को OCR कैसे करें** दिखाता है जबकि **प्रीप्रोसेसिंग कैसे सेट करें** को मास्टर करके इष्टतम एक्यूरेसी प्राप्त करता है। इंजन को उन सटीक आयतों तक सीमित करके और पहले इमेज को साफ़ करके, आप तेज़, साफ़ परिणाम प्राप्त करते हैं—इनवॉइस ऑटोमेशन, फ़ॉर्म डिजिटाइज़ेशन, या किसी भी परिदृश्य के लिए परफेक्ट जहाँ पेज का केवल एक हिस्सा महत्वपूर्ण हो।

अगले कदम के लिए तैयार हैं? अलग दस्तावेज़ प्रकार के लिए ROI कॉर्डिनेट्स को समायोजित करने की कोशिश करें, या `sharpen` या `noise_reduction` जैसे अतिरिक्त प्रीप्रोसेसिंग फ़्लैग्स के साथ प्रयोग करें। Aspose OCR की लचीलापन आपको पाइपलाइन को लगभग किसी भी इमेज क्वालिटी के अनुसार अनुकूलित करने देता है।

यदि आप किसी समस्या में फँसते हैं, तो खाली क्षेत्रों के लिए कंसोल आउटपुट देखें और प्रीप्रोसेसिंग सेटिंग्स को फिर से देखें। कोडिंग का आनंद लें, और आपका OCR हमेशा सटीक रहे!
