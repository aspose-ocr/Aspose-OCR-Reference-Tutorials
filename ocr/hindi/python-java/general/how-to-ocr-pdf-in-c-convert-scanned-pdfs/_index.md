# C# में PDF को OCR कैसे करें – स्कैन किए गए PDFs को बदलें

क्या आपको कभी **how to OCR PDF** करने की ज़रूरत पड़ी, लेकिन एक स्कैन किए गए फ़ाइल पर अटक गए जो सहयोग नहीं कर रही थी? आप अकेले नहीं हैं। कई वास्तविक‑दुनिया प्रोजेक्ट्स में—जैसे इनवॉइस डिजिटाइज़ेशन या ऐतिहासिक रिपोर्ट्स का अभिलेख—आपको PDF के अंदर छवियों का ढेर मिल जाता है, और उन्हें सर्चेबल बनाने का एकमात्र तरीका है कि उन पर OCR चलाया जाए।  

अच्छी खबर? सिर्फ कुछ ही पंक्तियों के C# कोड से आप **convert scanned PDF** को **create searchable PDF** में बदल सकते हैं, हल्का वॉटरमार्क जोड़ सकते हैं, और आगे की प्रोसेसिंग के लिए कच्चा टेक्स्ट भी निकाल सकते हैं। नीचे आप एक पूर्ण, तैयार‑चलाने योग्य उदाहरण देखेंगे जो यही करता है।

## इस ट्यूटोरियल में क्या कवर किया गया है

* `OcrEngine` क्लास का उपयोग करके **how to OCR PDF** कैसे करें।  
* स्कैन किए गए PDF को **create searchable PDF** में बदलना।  
* पहले पेज पर वॉटरमार्क जोड़ना (**add watermark pdf**).  
* परिणाम से प्लेन‑टेक्स्ट स्ट्रिंग्स निकालना (**extract text from pdf**).  
* सामान्य जाल—जैसे मल्टी‑पेज दस्तावेज़ों को संभालना या लो‑रिज़ॉल्यूशन स्कैन से निपटना।  

कोई बाहरी दस्तावेज़ लिंक नहीं, कोई आधा‑ख़त्म स्निपेट नहीं। आपको जो चाहिए वह सब यहाँ है।

## आवश्यकताएँ

* .NET 6.0 या बाद का संस्करण (कोड .NET 5 के साथ भी कम्पाइल होता है)।  
* `OcrEngine` और `ImageFormats` प्रदान करने वाली OCR लाइब्रेरी का रेफ़रेंस (जैसे, `Aspose.OCR` या समान पैकेज)।  
* `input.pdf` नामक इनपुट PDF को उस फ़ोल्डर में रखें जिसे आप नियंत्रित करते हैं।  
* C# कंसोल ऐप्स की बुनियादी परिचितता।  

यदि आपके पास ये सब है, तो बढ़िया—आइए शुरू करते हैं।

## चरण 1: प्रोजेक्ट सेट अप करें और डिपेंडेंसीज़ इम्पोर्ट करें

एक नया कंसोल प्रोजेक्ट बनाएं और NuGet के माध्यम से OCR पैकेज जोड़ें:

```bash
dotnet new console -n OcrPdfDemo
cd OcrPdfDemo
dotnet add package Aspose.OCR
```

अब `Program.cs` खोलें। शीर्ष पर, उन नेमस्पेस को इम्पोर्ट करें जिनकी आपको आवश्यकता होगी:

```csharp
using System;
using Aspose.OCR;          // OcrEngine lives here
using Aspose.OCR.Image;   // ImageFormats enum
```

*Why this matters*: सही नेमस्पेस इम्पोर्ट करने से कंपाइलर `OcrEngine` को ढूँढ़ पाता है। इस चरण को छोड़ने पर `CS0246` त्रुटि आएगी और आपका बिल्ड रुक जाएगा।

## चरण 2: OCR इंजन को इनिशियलाइज़ करें

इंजन प्रक्रिया का हृदय है। आप इसे एक बार इंस्टैंसिएट करते हैं और हर पेज के लिए पुनः उपयोग करते हैं।

```csharp
// Step 2: Initialize the OCR engine
OcrEngine engine = new OcrEngine();
```

*Pro tip*: यदि आप कई PDFs को क्रम में प्रोसेस करने की योजना बना रहे हैं, तो बार‑बार लाइसेंस चेक से बचने के लिए वही `engine` इंस्टेंस पुनः उपयोग करने पर विचार करें।

## चरण 3: स्रोत PDF लोड करें

इंजन को बताएं कि किस फ़ाइल पर काम करना है। मेथड `SetImageFromFile` किसी भी इमेज‑या‑PDF स्रोत को स्वीकार करता है।

```csharp
// Step 3: Load the scanned PDF you want to OCR
engine.SetImageFromFile(@"YOUR_DIRECTORY\input.pdf");
```

> **Why this step is crucial** – इंजन को प्रत्येक पेज की बिटमैप प्रतिनिधित्व चाहिए। जब आप इसे PDF देते हैं, तो यह डिफ़ॉल्ट DPI (आमतौर पर 300) के आधार पर प्रत्येक पेज को आंतरिक रूप से रास्टराइज़ करता है। यदि स्रोत PDF पहले से टेक्स्ट‑आधारित है, तो इंजन इसे बिना बदले पास कर देगा, जिससे आपका समय बचता है।

## चरण 4: OCR रिकग्निशन चलाएँ

अब भारी काम होता है। `Recognize` मेथड हर पेज को स्कैन करता है, ग्लिफ़्स निकालता है, और एक सर्चेबल लेयर बनाता है।

```csharp
// Step 4: Perform OCR on the loaded document
engine.Recognize();
```

*Common question*: *यदि PDF में 50 पेज हैं तो क्या होगा?*  
`Recognize` डिफ़ॉल्ट रूप से पूरे दस्तावेज़ को प्रोसेस करता है। मेमोरी‑सीमित वातावरण के लिए आप `engine.PageIndex` और `engine.PageCount` सेट करके पेज‑बाय‑पेज काम कर सकते हैं।

## चरण 5: सर्चेबल PDF सहेजें (पेज 1 पर वॉटरमार्क)

हम परिणाम को PDF के रूप में सहेजेंगे। पहला पेज स्वचालित रूप से वॉटरमार्क प्राप्त करेगा क्योंकि OCR लाइब्रेरी डिफ़ॉल्ट रूप से “Generated by Aspose.OCR” ओवरले जोड़ती है। यदि आपको कस्टम वॉटरमार्क चाहिए, तो सहेजने से पहले `engine.Watermark` प्रॉपर्टी को संशोधित कर सकते हैं।

```csharp
// Step 5: Save the OCR‑processed PDF with a watermark on page 1
engine.Save(@"YOUR_DIRECTORY\output.pdf", ImageFormats.Pdf);
```

इस कॉल के बाद आपके पास एक **create searchable PDF** होगा जहाँ आप टेक्स्ट को चयन, कॉपी और सर्च कर सकते हैं जैसे किसी भी नेटिव PDF में।

## चरण 6: प्लेन टेक्स्ट निकालें (वैकल्पिक लेकिन उपयोगी)

यदि आप इंडेक्सिंग या आगे के विश्लेषण के लिए **extract text from pdf** भी चाहते हैं, तो इंजन आपको कच्ची स्ट्रिंग तक पहुंच देता है।

```csharp
// Step 6: Pull the recognized text into a string
string extractedText = engine.Text;
Console.WriteLine("=== Extracted Text Start ===");
Console.WriteLine(extractedText);
Console.WriteLine("=== Extracted Text End ===");
```

`engine.Text` प्रॉपर्टी सभी पेजों का टेक्स्ट जोड़ती है, लाइन ब्रेक को संरक्षित रखते हुए। यदि आपको पेज‑वार ग्रैन्युलैरिटी चाहिए तो आप इसे `\f` (फॉर्म फ़ीड) से विभाजित कर सकते हैं।

## पूर्ण कार्यशील उदाहरण

नीचे पूरा प्रोग्राम दिया गया है। इसे `Program.cs` में कॉपी‑पेस्ट करें, `YOUR_DIRECTORY` को एक पूर्ण या सापेक्ष पाथ से बदलें, और `dotnet run` चलाएँ।

```csharp
using System;
using Aspose.OCR;
using Aspose.OCR.Image;

namespace OcrPdfDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // 1️⃣ Initialize the OCR engine
            OcrEngine engine = new OcrEngine();

            // 2️⃣ Load the source PDF to be processed
            // Make sure the file exists; otherwise you'll get a FileNotFoundException.
            string inputPath = @"YOUR_DIRECTORY\input.pdf";
            engine.SetImageFromFile(inputPath);

            // 3️⃣ Perform OCR recognition on the loaded document
            engine.Recognize();

            // 4️⃣ Save the recognized document as a PDF (watermark appears on page 1)
            string outputPath = @"YOUR_DIRECTORY\output.pdf";
            engine.Save(outputPath, ImageFormats.Pdf);

            // 5️⃣ (Optional) Extract plain text for further use
            string extractedText = engine.Text;
            Console.WriteLine("=== Extracted Text Start ===");
            Console.WriteLine(extractedText);
            Console.WriteLine("=== Extracted Text End ===");

            Console.WriteLine($"OCR complete! Searchable PDF saved to: {outputPath}");
        }
    }
}
```

### अपेक्षित आउटपुट

प्रोग्राम चलाने से निकाला गया टेक्स्ट कंसोल में प्रिंट होता है और `output.pdf` बनता है। PDF को किसी भी व्यूअर (Adobe Acrobat, Edge, आदि) में खोलें और मूल स्कैन में मौजूद किसी शब्द को खोजने की कोशिश करें—आप देखेंगे कि अब वह सर्चेबल है। पहला पेज डिफ़ॉल्ट वॉटरमार्क दिखाता है, जिससे **add watermark pdf** चरण सफल हुआ यह पुष्टि होती है।

## अक्सर पूछे जाने वाले प्रश्न और किनारे के केस

| Question | Answer |
|----------|--------|
| **यदि स्कैन लो‑रिज़ॉल्यूशन है तो क्या करें?** | पहचान से पहले DPI बढ़ाएँ: `engine.ImageInfo.DpiX = engine.ImageInfo.DpiY = 600;` इससे OCR इंजन को अधिक विवरण मिलता है। |
| **क्या मैं कस्टम वॉटरमार्क चुन सकता हूँ?** | हां। `Save` से पहले `engine.Watermark = "Confidential – Processed on " + DateTime.Now.ToShortDateString();` सेट करें। |
| **पासवर्ड‑प्रोटेक्टेड PDFs को कैसे हैंडल करें?** | `SetImageFromFile` से पहले `engine.LoadPassword = "yourPassword";` उपयोग करें। |
| **क्या OCR को विशिष्ट पेजों तक सीमित करने का तरीका है?** | केवल पेज 3‑7 को प्रोसेस करने के लिए `engine.PageIndex = 2;` और `engine.PageCount = 5;` सेट करें। |
| **यदि मुझे मूल इमेजेज़ को अपरिवर्तित रखना है तो क्या करें?** | OCR के बाद आप PDF लाइब्रेरी (जैसे `Aspose.PDF`) का उपयोग करके मूल PDF को OCR लेयर के साथ मर्ज कर सकते हैं ताकि इमेज की फ़िडेलिटी बनी रहे। |

## टिप्स और बेस्ट प्रैक्टिसेज

* **Pro tip:** यदि आप इसे UI ऐप में इंटीग्रेट कर रहे हैं तो OCR को बैकग्राउंड थ्रेड पर चलाएँ—अन्यथा UI फ्रीज़ हो जाएगा।  
* **Watch out for:** मिश्रित रास्टर और वेक्टर कंटेंट वाले PDFs। इंजन केवल रास्टर पेजों को OCR करेगा; वेक्टर टेक्स्ट स्वचालित रूप से चयन योग्य रहेगा।  
* **Performance tweak:** प्रत्येक दस्तावेज़ के लिए पुनः लोड करने से बचने हेतु OCR भाषा डेटा को कैश करें (`engine.Language = OcrLanguage.English;`)।  

## निष्कर्ष

अब आपके पास C# में **how to OCR PDF** के लिए एक पूर्ण, एंड‑टू‑एंड समाधान है। `OcrEngine` को इनिशियलाइज़ करके, स्कैन किए हुए फ़ाइल को लोड करके, टेक्स्ट को पहचानकर, एक **create searchable pdf** सहेजकर, वॉटरमार्क जोड़कर, और वैकल्पिक रूप से **extracting text from pdf** करके, आप किसी भी इमेज‑आधारित PDF को सर्चेबल, इंडेक्सेबल एसेट में बदल सकते हैं।  

अगले कदम? इस वर्कफ़्लो को डॉक्यूमेंट‑मैनेजमेंट सिस्टम के साथ जोड़ने की कोशिश करें, या विभिन्न भाषाओं (`engine.Language = OcrLanguage.French;`) के साथ प्रयोग करें। आप फ़ोल्डर में कई फ़ाइलों को बैच प्रोसेस करने का भी पता लगा सकते हैं—बस प्रत्येक बार एक नई `engine` इंस्टेंस के साथ चरणों को लूप करें।  

कोडिंग का आनंद लें, और आपके PDFs हमेशा सर्चेबल रहें!
