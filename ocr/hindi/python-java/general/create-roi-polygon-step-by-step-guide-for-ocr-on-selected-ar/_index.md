# ROI Polygon बनाएं – चयनित क्षेत्र पर OCR के लिए पूर्ण ट्यूटोरियल

क्या आपको कभी **create ROI polygon** की आवश्यकता पड़ी है ताकि आप केवल छवि के उस भाग पर OCR चला सकें जो महत्वपूर्ण है? शायद आप रसीदें स्कैन कर रहे हैं और केवल कुल राशि महत्वपूर्ण है, या आपके पास एक फ़ॉर्म है जहाँ केवल हस्ताक्षर फ़ील्ड को पढ़ना आवश्यक है। ऐसे मामलों में, **ocr on selected area** समय बचाता है और सटीकता बढ़ाता है।  

इस गाइड में हम आपको वह सब कुछ दिखाएंगे जिसकी आपको जरूरत है: दो पॉलीगॉन को परिभाषित करने से, उन्हें OCR इंजन के साथ रजिस्टर करने तक, और एक साफ़ कॉल में पहचाने गए टेक्स्ट को निकालने तक। अंत तक आपके पास चलाने के लिए तैयार स्क्रिप्ट होगी और यह समझ भी होगी कि रुचि के क्षेत्रों (regions of interest) पर ध्यान केंद्रित करना क्यों महत्वपूर्ण है।

## आप क्या सीखेंगे

- एक सामान्य OCR लाइब्रेरी का उपयोग करके **create ROI polygon** ऑब्जेक्ट कैसे बनाएं।
- एकल क्षेत्र बनाम कई क्षेत्रों को परिभाषित करने में अंतर।
- इंजन के साथ उन क्षेत्रों को रजिस्टर करने का तरीका ताकि `ocr on selected area` किया जा सके।
- अपेक्षित आउटपुट और सामान्य समस्याओं (जैसे, ओवरलैपिंग ROIs, खाली परिणाम) को कैसे ट्रबलशूट करें।

### आवश्यकताएँ

- Python 3.8+ स्थापित हो।
- `Polygon`, `Point`, और `add_region_of_interest` मेथड्स प्रदान करने वाली OCR लाइब्रेरी (उदाहरण में एक काल्पनिक `ocr` मॉड्यूल का उपयोग किया गया है; इसे अपने वास्तविक SDK से बदलें, जैसे Tesseract‑Python रैपर्स या EasyOCR)।
- एक सैंपल इमेज फ़ाइल (`sample.png`) जिसमें नीचे दिखाए गए निर्देशांक के भीतर टेक्स्ट हो।

> **Pro tip:** यदि आप वास्तविक लाइब्रेरी का उपयोग कर रहे हैं, तो सुनिश्चित करें कि इमेज समान कोऑर्डिनेट स्पेस में लोड हो (उत्पत्ति शीर्ष‑बाएँ पर, X दाएँ बढ़ता है, Y नीचे की ओर)।  

---  

## चरण 1: OCR मॉड्यूल इम्पोर्ट करें और अपनी इमेज लोड करें  

सबसे पहले, OCR पैकेज को स्कोप में लाएँ और वह इमेज पढ़ें जिसे आप प्रोसेस करना चाहते हैं।  

```python
import ocr  # Replace with your actual OCR library import
from pathlib import Path

# Load the image – adjust the path as needed
image_path = Path("sample.png")
ocr_engine = ocr.Engine(image_path)
```

*Why this matters:* इंजन को इमेज डेटा की आवश्यकता होती है इससे पहले कि आप कोई **ROI polygon** संलग्न कर सकें। कुछ लाइब्रेरी आपको बाद में इमेज पास करने की अनुमति भी देती हैं, लेकिन प्रारंभ में इनिशियलाइज़ करने से वर्कफ़्लो साफ़ रहता है।

## चरण 2: पहला ROI Polygon परिभाषित करें  

अब हम पहला region of interest बनाते हैं। इसे एक टेबल हेडर के चारों ओर आयत बनाते हुए सोचें।  

```python
# Step 2: Define the first ROI polygon
first_roi = ocr.Polygon([
    ocr.Point(50, 20),   # top‑left
    ocr.Point(200, 20),  # top‑right
    ocr.Point(200, 80),  # bottom‑right
    ocr.Point(50, 80)    # bottom‑left
])
```

*Explanation:*  
- `ocr.Point(x, y)` पिक्सेल कोऑर्डिनेट्स का उपयोग करता है।  
- बिंदुओं की सूची घड़ी की दिशा में क्रमबद्ध होती है, जैसा कि अधिकांश इंजन अपेक्षा करते हैं।  
- आप अनियमित आकार बनाने के लिए जितने चाहें बिंदु जोड़ सकते हैं; आयत केवल एक विशेष मामला है।

## चरण 3: अतिरिक्त ROI Polygons परिभाषित करें (वैकल्पिक)  

आपको केवल एक पर ही रुकना नहीं है। यहाँ एक दूसरा पॉलीगॉन है जो पेज के नीचे स्थित हस्ताक्षर फ़ील्ड को कैप्चर करता है।  

```python
# Step 3: Define the second ROI polygon
second_roi = ocr.Polygon([
    ocr.Point(300, 150),
    ocr.Point(480, 150),
    ocr.Point(480, 210),
    ocr.Point(300, 210)
])
```

*Why add more?* कई ज़ोन के लिए **ocr on selected area** चलाने से आप एक ही पास में विभिन्न डेटा टुकड़े निकाल सकते हैं, जो प्रत्येक स्लाइस को अलग‑अलग क्रॉप करके प्रोसेस करने से बहुत तेज़ है।

## चरण 4: OCR इंजन के साथ ROIs रजिस्टर करें  

पॉलीगॉन तैयार होने के बाद, इंजन को बताएं कि किन क्षेत्रों की जाँच करनी है।  

```python
# Step 4: Register the ROIs
ocr_engine.add_region_of_interest(first_roi)
ocr_engine.add_region_of_interest(second_roi)
```

*Important note:* कुछ SDKs को नया जोड़ने से पहले `clear_regions()` मेथड को कॉल करना पड़ता है यदि आप इंजन को किसी अन्य इमेज के लिए पुनः उपयोग कर रहे हैं।

## चरण 5: OCR चलाएँ और टेक्स्ट प्राप्त करें  

अंत में, पहचान प्रक्रिया शुरू करें। इंजन केवल उन पॉलीगॉन के अंदर देखेगा जिन्हें हमने अभी जोड़ा है।  

```python
# Step 5: Perform OCR on the defined regions
recognized_text = ocr_engine.recognize().get_text()
print("=== Recognized Text ===")
print(recognized_text)
```

**Expected output** (मान लेते हैं कि `sample.png` में पहले ROI में शब्द “Invoice Total: $123.45” और दूसरे में “Signature: John Doe” हैं):

```
=== Recognized Text ===
Invoice Total: $123.45
Signature: John Doe
```

यदि आउटपुट खाली है, तो दोबारा जांचें कि निर्देशांक वास्तव में टेक्स्ट को काटते हैं और इमेज रेज़ोल्यूशन कोऑर्डिनेट सिस्टम से मेल खाता है।

## किनारे के मामलों और सामान्य समस्याएँ  

| स्थिति                               | क्या देखना है                               | समाधान / वैकल्पिक उपाय                              |
|----------------------------------------|--------------------------------------------------|-----------------------------------------------|
| **Overlapping ROIs**                    | टेक्स्ट दो बार लौट सकता है।                     | पॉलीगॉन को अलग रखें या आउटपुट को डिडुप्लिकेट करें। |
| **ROI outside image bounds**           | इंजन त्रुटि दे सकता है या कुछ नहीं लौटाएगा।    | `0 ≤ x < width`, `0 ≤ y < height` के भीतर निर्देशांक को क्लैंप करें। |
| **Very small ROI**                     | अपर्याप्त पिक्सेल के कारण OCR की सटीकता घटती है। | पॉलीगॉन को कुछ पिक्सेल बढ़ाएँ या पहले इमेज को अपस्केल करें। |
| **Non‑rectangular shape**               | कुछ इंजन केवल कॉन्वेक्स पॉलीगॉन का समर्थन करते हैं।      | जटिल आकार को कई कॉन्वेक्स ROIs में विभाजित करें। |
| **Changing image size**                | हार्ड‑कोडेड निर्देशांक अमान्य हो जाते हैं।           | इमेज के आयामों के सापेक्ष ROI निर्देशांक गणना करें (जैसे, प्रतिशत)। |

## पूर्ण कार्यशील उदाहरण  

नीचे पूर्ण स्क्रिप्ट है जिसे आप कॉपी‑पेस्ट करके चला सकते हैं (`ocr` को अपनी वास्तविक लाइब्रेरी से बदलें)।  

```python
import ocr                      # <- your OCR library
from pathlib import Path

# -------------------------------------------------
# Configuration
# -------------------------------------------------
IMAGE_PATH = Path("sample.png")

# -------------------------------------------------
# Initialize OCR engine with the image
# -------------------------------------------------
engine = ocr.Engine(IMAGE_PATH)

# -------------------------------------------------
# Define ROI polygons
# -------------------------------------------------
first_roi = ocr.Polygon([
    ocr.Point(50, 20), ocr.Point(200, 20),
    ocr.Point(200, 80), ocr.Point(50, 80)
])

second_roi = ocr.Polygon([
    ocr.Point(300, 150), ocr.Point(480, 150),
    ocr.Point(480, 210), ocr.Point(300, 210)
])

# -------------------------------------------------
# Register ROIs
# -------------------------------------------------
engine.add_region_of_interest(first_roi)
engine.add_region_of_interest(second_roi)

# -------------------------------------------------
# Run OCR on the selected areas
# -------------------------------------------------
result = engine.recognize().get_text()

print("=== Recognized Text ===")
print(result)
```

`python ocr_roi_example.py` के साथ चलाएँ और आपको दो परिभाषित ज़ोन से निकाले गए स्ट्रिंग्स दिखेंगे।

## अगले कदम  

- **Dynamic ROI generation:** हार्ड‑कोडेड निर्देशांक के बजाय, इमेज प्रोसेसिंग (जैसे, OpenCV कंटूर डिटेक्शन) का उपयोग करके टेबल या फ़ील्ड को स्वचालित रूप से लोकेट करें।  
- **Post‑processing:** व्हाइटस्पेस हटाएँ, रेगेक्स लागू करें, या संख्याओं को उचित डेटा टाइप में बदलें।  
- **Batch processing:** इमेज की फ़ोल्डर पर लूप चलाएँ, यदि लेआउट समान है तो वही ROI परिभाषाएँ पुनः उपयोग करें।  

यदि आप अधिक जटिल दस्तावेज़ों के लिए **ocr on selected area** के बारे में जिज्ञासु हैं—जैसे मल्टी‑पेज PDFs या स्कैन किए गए फ़ॉर्म—तो उन लाइब्रेरीज़ को देखें जो पेज‑वार ROI रजिस्ट्रेशन का समर्थन करती हैं।

## निष्कर्ष  

हमने अभी **created ROI polygon** ऑब्जेक्ट्स बनाए, उन्हें OCR इंजन के साथ रजिस्टर किया, और `ocr on selected area` को एक साफ़, पुनरुत्पादनीय स्क्रिप्ट में किया। इंजन को उन ज़ोन तक सीमित करके जिनकी आपको ज़रूरत है, आप प्रोसेसिंग समय घटाते हैं, सटीकता बढ़ाते हैं, और डाउनस्ट्रीम डेटा हैंडलिंग को बहुत सरल बनाते हैं।  

इसे अपनी इमेजों के साथ आज़माएँ—निर्देशांक बदलें, अधिक पॉलीगॉन जोड़ें, या आउटपुट को डेटाबेस से जोड़ें। एक बार जब आप region‑based OCR में निपुण हो जाते हैं, तो संभावनाएँ असीमित हैं।
