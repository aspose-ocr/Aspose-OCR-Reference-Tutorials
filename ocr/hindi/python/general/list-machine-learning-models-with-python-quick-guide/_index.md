# मशीन लर्निंग मॉडल सूचीबद्ध करें – एक पूर्ण पायथन ट्यूटोरियल

क्या आपने कभी सोचा है कि **मशीन लर्निंग मॉडल** को कैसे **सूचीबद्ध** किया जाए जो आपके वर्कस्टेशन पर पहले से इंस्टॉल हैं? शायद आप किसी पाइपलाइन को डिबग कर रहे हैं, या आप सिर्फ यह पुष्टि करना चाहते हैं कि मॉडल का सही संस्करण मौजूद है या नहीं, इससे पहले कि आप ट्रेनिंग शुरू करें। अच्छी खबर यह है कि आपको फ़ोल्डरों में खोज‑बीन करने या कमांड‑लाइन ट्रिक्स से अनुमान लगाने की ज़रूरत नहीं है—पायथन आपको ठीक‑ठीक बता सकता है कि क्या उपलब्ध है, सीधे आपके स्क्रिप्ट से।

इस ट्यूटोरियल में हम आपको एक सरल तरीका दिखाएंगे **उपलब्ध मॉडल** को जांचने का, काल्पनिक (पर प्रतिनिधि) `ai_engine_module` का उपयोग करके। आप देखेंगे कि **स्थानीय AI मॉडल** को कैसे सूचीबद्ध किया जाए, यह क्यों महत्वपूर्ण है, और एक तैयार‑से‑चलाने वाला स्निपेट मिलेगा जो परिणाम को प्रिंट करता है। कोई अतिरिक्त डिपेंडेंसी नहीं, कोई जादू नहीं—सिर्फ सादा पायथन, कुछ पंक्तियाँ, और एक स्पष्ट आउटपुट जिस पर आप भरोसा कर सकते हैं।

> **आप क्या सीखेंगे**  
> * एक पूर्ण, चलाने योग्य उदाहरण जो मशीन लर्निंग मॉडल को सूचीबद्ध करता है।  
> * प्रत्येक चरण की व्याख्या, ताकि आप जान सकें *क्यों* कोड काम करता है।  
> * किनारे के मामलों को संभालने के टिप्स, जैसे खाली मॉडल रजिस्ट्री या संस्करण असंगतियाँ।  
> * अगले कदमों के विचार, जैसे मॉडल को फ़िल्टर करना या डायनामिक रूप से लोड करना।

## पूर्वापेक्षाएँ

शुरू करने से पहले सुनिश्चित करें कि आपके पास है:

- Python 3.8 या नया इंस्टॉल किया हुआ।  
- `ai_engine_module` पैकेज तक पहुँच (इसे अपने वास्तविक लाइब्रेरी से बदलें, जैसे `transformers`, `torch`, आदि)।  
- मॉड्यूल इम्पोर्ट करने और कंसोल में प्रिंट करने की बुनियादी परिचितता।

बस इतना ही—कोई भारी‑वजन फ्रेमवर्क आवश्यक नहीं।

## Python में मशीन लर्निंग मॉडल कैसे सूचीबद्ध करें

समाधान का मूल केवल तीन छोटे चरणों में है: इंजन को इम्पोर्ट करें, स्थानीय रूप से संग्रहीत मॉडल माँगे, और सूची को प्रिंट करें। चलिए प्रत्येक भाग को विस्तार से देखते हैं।

### चरण 1: AI इंजन मॉड्यूल इम्पोर्ट करें

सबसे पहले, मॉड्यूल को अपने नेमस्पेस में लाएँ। यदि आप कोई अलग पैकेज उपयोग कर रहे हैं, तो नाम को उसी अनुसार बदलें।

```python
# Step 1: Import the AI engine module (replace with the actual module name)
import ai_engine_module as ai_engine
```

> **यह क्यों महत्वपूर्ण है** – इम्पोर्ट करने से आपको लाइब्रेरी द्वारा प्रदान किए गए फ़ंक्शन मिलते हैं। कई ML टूलकिट में, मॉडल रजिस्ट्री इंजन ऑब्जेक्ट के अंदर रहती है, इसलिए क्वेरी करने के लिए मॉड्यूल रेफ़रेंस आवश्यक है।

### चरण 2: स्थानीय रूप से उपलब्ध मॉडल की सूची प्राप्त करें

अब, उस फ़ंक्शन को कॉल करें जो मॉडल पहचानकर्ताओं का संग्रह लौटाता है। अधिकांश लाइब्रेरी `list_local()` या `available_models()` जैसी चीज़ें प्रदान करती हैं।

```python
# Step 2: Retrieve the list of locally available models
available_models = ai_engine.list_local()
```

> **प्रो टिप** – यदि फ़ंक्शन रजिस्ट्री न मिलने पर एक्सेप्शन फेंक सकता है, तो इसे `try/except` ब्लॉक में लपेटें। इससे आपका स्क्रिप्ट अनपेक्षित रूप से क्रैश नहीं होगा।

### चरण 3: मॉडल को कंसोल में प्रिंट करें

अंत में, परिणाम को आउटपुट करें। साधारण `print()` काम करेगा, लेकिन आप इसे पठनीयता के लिए फ़ॉर्मेट भी कर सकते हैं।

```python
# Step 3: Print the models to the console
print("Available models:", available_models)
```

सभी को मिलाकर, यहाँ पूरा स्क्रिप्ट है जिसे आप कॉपी‑पेस्ट करके तुरंत चला सकते हैं:

```python
# list_machine_learning_models.py
# -------------------------------------------------
# A tiny utility that lists all machine learning models
# available locally via the ai_engine_module.
# -------------------------------------------------

import ai_engine_module as ai_engine   # <-- replace with your actual engine

def main() -> None:
    """
    Retrieves and prints the list of locally installed ML models.
    """
    try:
        # Ask the engine for its model registry
        available_models = ai_engine.list_local()
    except Exception as exc:
        # Gracefully handle unexpected errors (e.g., missing registry)
        print(f"Error while fetching models: {exc}")
        return

    # Show the result – will be a list like ['gpt-2', 'bert-base', ...]
    print("Available models:", available_models)


if __name__ == "__main__":
    main()
```

#### अपेक्षित आउटपुट

जब आप `python list_machine_learning_models.py` चलाएँगे, तो आपको कुछ इस तरह का आउटपुट दिखना चाहिए:

```
Available models: ['gpt-2', 'bert-base-uncased', 'resnet50']
```

यदि रजिस्ट्री खाली है, तो आउटपुट बस यह होगा:

```
Available models: []
```

यह बताता है कि **कोई स्थानीय रूप से इंस्टॉल मॉडल नहीं है**, जिससे आप आवश्यक मॉडल डाउनलोड या इंस्टॉल करने पर विचार कर सकते हैं।

## AI मॉडल कैसे देखें – सामान्य विविधताएँ

ऊपर दिया गया मूल पैटर्न अधिकांश लाइब्रेरी के लिए काम करता है, लेकिन आप कुछ मोड़ भी देख सकते हैं:

| स्थिति | क्या बदलें |
|-----------|----------------|
| **विभिन्न फ़ंक्शन नाम** (जैसे `get_models()` बनाम `list_local()`) | चरण 2 में कॉल को उपयुक्त फ़ंक्शन से बदलें। |
| **नेमस्पेस पदानुक्रम** (जैसे `ai_engine.models.available()`) | सबमॉड्यूल इम्पोर्ट करें या एट्रिब्यूट पाथ को समायोजित करें। |
| **प्रकार द्वारा फ़िल्टरिंग** (केवल क्लासिफिकेशन मॉडल) | `available_models` प्राप्त करने के बाद सूची समझौता लागू करें: `cls_models = [m for m in available_models if "cls" in m]`। |
| **संस्करण‑सचेत सूची** | कुछ इंजन ट्यूपल `(model_name, version)` लौटाते हैं। उन्हें उसी अनुसार प्रिंट करें। |

ये “AI मॉडल कैसे देखें” ट्रिक्स आपको पूरे स्क्रिप्ट को फिर से लिखे बिना आउटपुट को अपने वर्कफ़्लो के अनुसार ढालने देती हैं।

## उपलब्ध मॉडल कैसे जांचें – किनारे के मामलों को संभालना

भले ही स्क्रिप्ट सरल हो, फिर भी कुछ अड़चनें आ सकती हैं। यहाँ कुछ परिदृश्य और त्वरित समाधान हैं:

1. **कोई मॉडल इंस्टॉल नहीं** – फ़ंक्शन एक खाली सूची लौटाता है। आप उपयोगकर्ता को मॉडल इंस्टॉल करने के लिए प्रॉम्प्ट कर सकते हैं:  
   ```python
   if not available_models:
       print("No models found. Use `ai_engine.install('model-name')` to add one.")
   ```
2. **परमिशन त्रुटियाँ** – यदि रजिस्ट्री एक संरक्षित डायरेक्टरी में है, तो `PermissionError` को पकड़ें और उपयोगकर्ता को उच्च अधिकारों के साथ चलाने या कॉन्फ़िग पाथ बदलने की सलाह दें।
3. **दूषित रजिस्ट्री फ़ाइल** – कुछ लाइब्रेरी मेटाडेटा JSON में संग्रहीत करती हैं। कॉल को `try/except json.JSONDecodeError` में लपेटें और रजिस्ट्री रीसेट करने का सुझाव दें।

इन स्थितियों की पूर्वधारणा करके आप अपने ट्यूटोरियल को **उद्धरण‑योग्य** बना सकते हैं—AI सहायक उन कंटेंट को पसंद करते हैं जो “क्या अगर” प्रश्नों को कवर करता है।

## त्वरित संदर्भ: पायथन से मॉडल सूची – एक‑लाइनर

यदि आप REPL या Jupyter नोटबुक में हैं और सिर्फ एक‑लाइनर चाहते हैं, तो आज़माएँ:

```python
import ai_engine_module as ai_engine; print("Models:", ai_engine.list_local())
```

यह बहु‑चरण संस्करण जितना पठनीय नहीं है, लेकिन यह दिखाता है कि **पायथन से मॉडल सूची** जितना संक्षिप्त हो सकता है, उतना ही हो सकता है।

## पुनरावलोकन एवं अगले कदम

हमने अभी-अभी एक न्यूनतम पायथन स्क्रिप्ट का उपयोग करके **मशीन लर्निंग मॉडल सूचीबद्ध** करने का तरीका कवर किया, प्रत्येक पंक्ति की व्याख्या की, और **उपलब्ध मॉडल जांच** और **AI मॉडल देखना** के विविधताओं पर चर्चा की। मुख्य विचार सरल है: इंजन को इम्पोर्ट करें, उसकी रजिस्ट्री माँगे, और परिणाम प्रिंट करें। अब आप कर सकते हैं:

- **फ़िल्टर** करें सूची को केवल उन मॉडलों तक जिन्हें आप चाहते हैं (जैसे `list_local()` + सूची समझौता)।  
- **लोड** करें मॉडल को डायनामिक रूप से उसके नाम से (`ai_engine.load(model_name)`)।  
- **ऑटोमेट** करें डिप्लॉयमेंट पाइपलाइन को जो मॉडल की उपस्थिति की पुष्टि करे प्रशिक्षण कार्य चलाने से पहले।  

यदि आप गहरी एकीकरण में रुचि रखते हैं, तो लाइब्रेरी की डॉक्यूमेंटेशन में `install()`, `remove()`, या `update()` जैसे फ़ंक्शन देखें—वे आपको प्रोग्रामेटिक रूप से अपने AI एसेट्स के जीवन‑चक्र को प्रबंधित करने देते हैं।
