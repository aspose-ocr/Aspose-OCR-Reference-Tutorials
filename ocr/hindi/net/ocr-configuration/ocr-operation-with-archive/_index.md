# Aspose.OCR for .NET के साथ आर्काइव इमेजेज़ पर OCR कैसे करें

## परिचय

इस व्यापक ट्यूटोरियल में आप **OCR कैसे करें** को Aspose.OCR लाइब्रेरी for .NET का उपयोग करके आर्काइव्ड इमेज फ़ाइलों पर सीखेंगे। चाहे आपको **इमेजेज़ को टेक्स्ट में बदलें** हो या **आर्काइव से टेक्स्ट निकालें** फ़ाइलें, नीचे दिया गया चरण‑बद्ध गाइड आपको सब कुछ दिखाएगा—डेवलपमेंट एनवायरनमेंट सेटअप से लेकर ZIP आर्काइव के भीतर प्रत्येक इमेज से पहचाने गए टेक्स्ट को प्राप्त करने तक।

## त्वरित उत्तर
- **यह ट्यूटोरियल क्या कवर करता है?** Aspose.OCR for .NET के साथ आर्काइव (ZIP) इमेजेज़ पर OCR करना।  
- **लक्षित मुख्य कीवर्ड कौन सा है?** *how to perform ocr*.  
- **क्या मुझे लाइसेंस चाहिए?** एक मुफ्त ट्रायल मूल्यांकन के लिए काम करता है; उत्पादन के लिए एक व्यावसायिक लाइसेंस आवश्यक है।  
- **कौन से .NET संस्करण समर्थित हैं?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6+.  
- **क्या मैं रिकग्निशन सेटिंग्स को कस्टमाइज़ कर सकता हूँ?** हाँ—सटीकता को ट्यून करने के लिए `RecognitionSettings` का उपयोग करें।

## OCR क्या है और आर्काइव पर इसका उपयोग क्यों करें?

ऑप्टिकल कैरेक्टर रिकग्निशन (OCR) स्कैन किए गए इमेजेज़ या PDF को सर्चेबल, एडिटेबल टेक्स्ट में बदलता है। जब इमेजेज़ एक आर्काइव (जैसे ZIP फ़ाइल) में बंडल होते हैं, तो एक साथ प्रत्येक चित्र को एक्सट्रैक्ट और पहचानना समय बचाता है और कोड जटिलता को कम करता है। Aspose.OCR की `RecognizeMultipleImages` मेथड इस प्रक्रिया को सरल बनाती है।

## चरण 1: अपना डॉक्यूमेंट डायरेक्टरी सेट करें

सबसे पहले अपने डॉक्यूमेंट डायरेक्टरी का पाथ इनिशियलाइज़ करें:

```csharp
// ExStart:1
// The path to the documents directory.
string dataDir = "Your Document Directory";
// ExEnd:1
```

> **प्रो टिप:** क्रॉस‑प्लेटफ़ॉर्म पाथ हैंडलिंग के लिए `Path.Combine` का उपयोग करें।

## चरण 2: Aspose.OCR को इनिशियलाइज़ करें

OCR ऑपरेशन्स शुरू करने के लिए Aspose.OCR क्लास का एक इंस्टेंस बनाएं:

```csharp
// ExStart:3
AsposeOcr api = new AsposeOcr();
// ExEnd:3
```

## चरण 3: इमेज पाथ निर्दिष्ट करें

अपने आर्काइव इमेज (ZIP फ़ाइल जिसमें पढ़ने के लिए चित्र हैं) का पूरा पाथ परिभाषित करें:

```csharp
// ExStart:4
string fullPath = dataDir + "OCR.zip";
// ExEnd:4
```

## चरण 4: इमेज को पहचानें

डिफ़ॉल्ट या कस्टम सेटिंग्स का उपयोग करके निर्दिष्ट आर्काइव पर OCR पहचान चलाएँ। यह कॉल स्वचालित रूप से ZIP से प्रत्येक इमेज को एक्सट्रैक्ट करती है और उस पर OCR चलाती है:

```csharp
// ExStart:5
RecognitionResult[] result = api.RecognizeMultipleImages(fullPath, new RecognitionSettings
{
   //default or custom settings
});
// ExEnd:5
```

> आप विशिष्ट भाषाओं या इमेज क्वालिटी के लिए सटीकता बढ़ाने हेतु `RecognitionSettings` को ट्यून कर सकते हैं।

## चरण 5: परिणाम प्रिंट करें

परिणामों पर लूप करें और आर्काइव के भीतर प्रत्येक इमेज के लिए पहचाने गए टेक्स्ट को प्रिंट करें:

```csharp
// ExStart:6

{
	 Console.WriteLine($"Image: {i}\n Result:\n {result[i].RecognitionText}");
}
// ExEnd:6
```

आउटपुट प्रत्येक इमेज इंडेक्स के बाद निकाले गए स्ट्रिंग को दिखाता है, प्रभावी रूप से **इमेजेज़ को टेक्स्ट में बदलता** है और **आर्काइव से टेक्स्ट निकालता** है।

## अक्सर पूछे जाने वाले प्रश्न

**प्रश्न: क्या मैं Aspose.OCR for .NET को बिना लाइसेंस के उपयोग कर सकता हूँ?**  
**उत्तर:** हाँ, मूल्यांकन के लिए एक मुफ्त ट्रायल उपलब्ध है, लेकिन उत्पादन डिप्लॉयमेंट के लिए लाइसेंस्ड संस्करण आवश्यक है।

**प्रश्न: क्या लाइब्रेरी पासवर्ड‑प्रोटेक्टेड ZIP आर्काइव्स को सपोर्ट करती है?**  
**उत्तर:** वर्तमान में, `RecognizeMultipleImages` मानक ZIP फ़ाइलों के साथ काम करता है। एन्क्रिप्टेड आर्काइव्स के लिए, पहले थर्ड‑पार्टी लाइब्रेरी का उपयोग करके इमेजेज़ एक्सट्रैक्ट करें, फिर इमेज एरे को OCR इंजन में पास करें।

**प्रश्न: हस्तलेखित टेक्स्ट की सटीकता कैसे बढ़ा सकता हूँ?**  
**उत्तर:** `RecognitionSettings.EnableHandwritingRecognition` फ़्लैग को सक्षम करें और उच्च DPI सेटिंग (जैसे 300) प्रदान करें।

**प्रश्न: प्रत्येक पहचानी गई लाइन के लिए कॉन्फिडेंस स्कोर प्राप्त करने का कोई तरीका है?**  
**उत्तर:** प्रत्येक `RecognitionResult` में एक `Confidence` प्रॉपर्टी होती है जिसे आप लॉग कर सकते हैं या कम कॉन्फिडेंस वाले परिणामों को फ़िल्टर करने के लिए उपयोग कर सकते हैं।
