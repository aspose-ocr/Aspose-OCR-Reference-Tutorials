# OCR भाषा समर्थन: छवि पहचान में अनदेखे अक्षरों को निर्दिष्ट करें

## परिचय

OCR भाषा समर्थन आधुनिक दस्तावेज़ स्वचालन का एक मुख्य आधार है, जो अनुप्रयोगों को कई वर्णमालाओं और प्रतीकों में छवियों से पाठ पढ़ने की अनुमति देता है। इस ट्यूटोरियल में आप सीखेंगे कि **Aspose.OCR for .NET** को पहचान के दौरान विशिष्ट अक्षरों को अनदेखा करने के लिए कैसे कहा जाए—एक आवश्यक ट्रिक जब आपको साफ़ आउटपुट चाहिए या पृष्ठ संख्या या सजावटी प्रतीकों जैसे शोर को फ़िल्टर करना हो। गाइड के अंत तक आपके पास एक तैयार‑चलाने‑योग्य स्निपेट होगा जो इस सुविधा को अंत‑से‑अंत प्रदर्शित करता है।

## त्वरित उत्तर
- **“ignored characters” का क्या अर्थ है?** वह अक्षर जो OCR इंजन परिणाम स्ट्रिंग बनाते समय छोड़ देता है।  
- **इसे क्यों उपयोग करें?** जब कुछ प्रतीक आपके व्यावसायिक तर्क के लिए अप्रासंगिक हों तो सटीकता में सुधार करता है।  
- **कौन सा API मेथड इसे संभालता है?** `RecognitionSettings.IgnoredCharacters`।  
- **क्या मैं इसे भाषा पैक्स के साथ संयोजित कर सकता हूँ?** हाँ—अनदेखे अक्षर किसी भी लोड की गई भाषा के साथ काम करते हैं।  
- **क्या लाइसेंस आवश्यक है?** उत्पादन उपयोग के लिए एक अस्थायी या पूर्ण लाइसेंस आवश्यक है।

## अनदेखे अक्षरों को निर्दिष्ट क्यों करें?

वास्तविक दुनिया के कई परिदृश्यों में—जैसे इनवॉइस, रसीद या बहुभाषी फ़ॉर्म प्रोसेसिंग—आपको बार‑बार आने वाले ऐसे अक्षर मिल सकते हैं जो अर्थपूर्ण डेटा का हिस्सा नहीं होते (उदाहरण के लिए, विभाजक के रूप में उपयोग किए जाने वाले हाइफ़न, पृष्ठ संख्या, या सजावटी प्रतीक)। OCR इंजन को इनको छोड़ने के लिए कहने से आप पोस्ट‑प्रोसेसिंग प्रयास को कम कर सकते हैं और डाउनस्ट्रीम विश्लेषण की समग्र विश्वसनीयता को बढ़ा सकते हैं।

## चरण‑दर‑चरण मार्गदर्शिका

### चरण 1: अपनी दस्तावेज़ निर्देशिका सेट करें

अपनी दस्तावेज़ों के संग्रहित होने वाले निर्देशिका को निर्दिष्ट करके शुरू करें। `"Your Document Directory"` को अपने दस्तावेज़ों के वास्तविक पथ से बदलें।

```csharp
// The path to the documents directory.
string dataDir = "Your Document Directory";
```

### चरण 2: Aspose.OCR को प्रारंभ करें

OCR इंजन का एक इंस्टेंस बनाएं। यह ऑब्जेक्ट सभी बाद की पहचान कॉल्स को संभालेगा।

```csharp
// Initialize an instance of AsposeOcr
AsposeOcr api = new AsposeOcr();
```

### चरण 3: अनदेखे अक्षरों के साथ छवि को पहचानें

छवि फ़ाइल को एक `RecognitionSettings` ऑब्जेक्ट के साथ पास करें जो उन अक्षरों की सूची देता है जिन्हें आप अनदेखा करना चाहते हैं। इस उदाहरण में हम अक्षर `a`, `b` और `1` को अनदेखा करते हैं।

```csharp
// Recognize image with specified ignored characters
RecognitionResult result = api.RecognizeImage(dataDir + "SpanishOCR.bmp", new RecognitionSettings
{
    IgnoredCharacters = "ab1"
});
```

### चरण 4: पहचाना गया टेक्स्ट प्रदर्शित करें

अंत में, साफ़ किया गया टेक्स्ट कंसोल या किसी अन्य आउटपुट में प्रदर्शित करें जो आप पसंद करते हैं।

```csharp
// Display the recognized text
Console.WriteLine(result.RecognitionText);
```

## सामान्य समस्याएँ और सुझाव

- **गलत पथ:** सुनिश्चित करें कि `dataDir` आपके OS के अनुसार एक पथ विभाजक (`\` या `/`) के साथ समाप्त हो।  
- **असमर्थित भाषा:** OCR इंजन के पास स्रोत छवि के लिए भाषा पैक होना चाहिए; अन्यथा, अनदेखे अक्षर सही ढंग से लागू नहीं होंगे।  
- **लाइसेंस त्रुटियाँ:** यदि आपको लाइसेंसिंग अपवाद दिखता है, तो जांचें कि अस्थायी लाइसेंस फ़ाइल आपके प्रोजेक्ट में सही ढंग से संदर्भित है।
