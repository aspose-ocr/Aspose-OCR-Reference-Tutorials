# ocr दस्तावेज़ मोड – OCR इमेज पहचान में डिटेक्ट एरिया मोड

## परिचय

आधुनिक .NET विकास में, **ocr दस्तावेज़ मोड** वह प्रमुख तरीका है **OCR सटीकता सुधारने** के लिए जब आपको छवियों के भीतर टेक्स्ट की पहचान पर सटीक नियंत्रण चाहिए। Aspose.OCR for .NET विभिन्न डिटेक्शन स्ट्रैटेजी के बीच स्विच करना आसान बनाता है, जिससे आप रसीदें, इनवॉइस या मल्टी‑कॉलम दस्तावेज़ जैसे जटिल लेआउट से **टेबल टेक्स्ट इमेज निकाल** सकते हैं। यह **aspose ocr tutorial c#** आपको Detect Areas Mode फीचर के माध्यम से ले जाएगा, बताएगा कि प्रत्येक मोड कब उपयोग करना है, और एक तैयार‑चलाने योग्य कोड नमूना दिखाएगा।

## त्वरित उत्तर
- **ocr दस्तावेज़ मोड क्या है?** Aspose.OCR को बताने वाले डिटेक्शन स्ट्रैटेजी (PHOTO, DOCUMENT, COMBINE) का एक सेट जो टेक्स्ट क्षेत्रों को खोजता है।
- **टेबल के लिए कौन सा मोड सबसे अच्छा है?** `PHOTO` मोड टेबल टेक्स्ट इमेज और छोटे टेक्स्ट ब्लॉक्स निकालने में उत्कृष्ट है।
- **क्या विकास के लिए लाइसेंस चाहिए?** परीक्षण के लिए एक मुफ्त ट्रायल लाइसेंस पर्याप्त है; उत्पादन के लिए एक वाणिज्यिक लाइसेंस आवश्यक है।
- **कौन से .NET संस्करण समर्थित हैं?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6 और बाद के संस्करण।
- **सेटअप में कितना समय लगता है?** आमतौर पर नमूना कोड को एकीकृत करने और चलाने में 10 मिनट से कम समय लगता है।

## Detect Areas Mode के साथ OCR सटीकता कैसे सुधारें?

सही **Detect Areas Mode** चुनना संरचित छवियों पर OCR सटीकता बढ़ाने का सबसे प्रभावी तरीका है। इंजन को यह बताकर कि छवि फोटो, प्रिंटेड दस्तावेज़ या दोनों का मिश्रण है, आप गलत detections को कम करते हैं, प्रोसेसिंग गति बढ़ाते हैं, और साफ़ टेक्स्ट आउटपुट प्राप्त करते हैं—विशेष रूप से टेबल, रसीदें और मल्टी‑कॉलम लेआउट के लिए।

## ocr दस्तावेज़ मोड क्या है?

`ocr दस्तावेज़ मोड` वह कॉन्फ़िगरेशन है जो Aspose.OCR को बताता है कि टेक्स्ट पहचान करने से पहले छवि को कैसे विभाजित किया जाए। तीन बिल्ट‑इन मोड हैं:

- **PHOTO** – फ़ोटोग्राफ़, रसीदें, इनवॉइस और छोटे टेक्स्ट क्षेत्रों के लिए अनुकूलित (टेबल टेक्स्ट इमेज निकालने के लिए आदर्श)।
- **DOCUMENT** – मल्टी‑कॉलम प्रिंटेड पेज और एम्बेडेड ग्राफ़िक्स वाले दस्तावेज़ों के लिए उपयुक्त।
- **COMBINE** – PHOTO और DOCUMENT के परिणामों को मिलाकर सबसे व्यापक कवरेज प्रदान करता है।

## Detect Areas Mode क्यों उपयोग करें?

उपयुक्त डिटेक्शन मोड चुनने से फॉल्स पॉज़िटिव कम होते हैं, प्रोसेसिंग तेज़ होती है, और सटीकता बढ़ती है—मुख्य कारक जब आप टेबल जैसे संरचित डेटा पर **OCR सटीकता सुधारने** का लक्ष्य रखते हैं। मोड को अपनी छवि प्रकार के अनुसार अनुकूलित करने से व्यापक पोस्ट‑प्रोसेसिंग की आवश्यकता समाप्त हो जाती है।

## सामान्य उपयोग केस

| परिदृश्य | सिफ़ारिश किया गया मोड | क्यों मदद करता है |
|----------|------------------|--------------|
| घनी टेबल वाली रसीदें या इनवॉइस | **PHOTO** | छोटे टेक्स्ट ब्लॉक्स पर केंद्रित और टेबल लेआउट को संरक्षित करता है |
| मल्टी‑कॉलम मैगज़ीन या रिपोर्ट | **DOCUMENT** | कॉलम विभाजन और एम्बेडेड ग्राफ़िक्स को संभालता है |
| स्कैन किए हुए दस्तावेज़ जिनमें फोटो और टेक्स्ट दोनों हैं | **COMBINE** | PHOTO और DOCUMENT दोनों की ताकतों का उपयोग करता है |

## आवश्यकताएँ

शुरू करने से पहले, सुनिश्चित करें कि आपके पास है:

- **Aspose.OCR for .NET** – लाइब्रेरी को [Aspose.OCR for .NET documentation](https://reference.aspose.com/ocr/net/) से डाउनलोड और इंस्टॉल करें।
- **Document Directory** – आपके मशीन पर एक फ़ोल्डर जिसमें वह इमेजेज़ हैं जिन्हें आप प्रोसेस करना चाहते हैं (उदा., `table.png`)।

## नेमस्पेस इम्पोर्ट करें

सबसे पहले, अपने C# प्रोजेक्ट में Aspose.OCR के साथ काम करने के लिए आवश्यक नेमस्पेस इम्पोर्ट करें।

```csharp
using System;
using System.IO;
using Aspose.OCR;
```

## चरण 1: Aspose.OCR को इनिशियलाइज़ करें

OCR इंजन का एक इंस्टेंस बनाएं और उसे अपने डेटा फ़ोल्डर की ओर इंगित करें।

```csharp
// The path to the documents directory.
string dataDir = "Your Document Directory";

// Initialize an instance of AsposeOcr
AsposeOcr api = new AsposeOcr();
```

## चरण 2: इमेज लोड करें और Detect Areas Mode चुनें

टार्गेट इमेज लोड करें और उस डिटेक्शन स्ट्रैटेजी को निर्दिष्ट करें जो आपके परिदृश्य से मेल खाती है।

```csharp
// Recognize image
RecognitionResult result = api.RecognizeImage(dataDir + "table.png", new RecognitionSettings
{
    // Choose the Detect Areas Mode
    DetectAreasMode = DetectAreasMode.PHOTO
    // Other options: NONE, DOCUMENT, COMBINE
});
```

## चरण 3: पहचाने गए टेक्स्ट को प्राप्त करें और प्रदर्शित करें

OCR पूर्ण होने के बाद, आप निकाले गए टेक्स्ट तक पहुंच सकते हैं—आगे की प्रोसेसिंग या डेटाबेस में स्टोर करने के लिए एकदम उपयुक्त।

```csharp
// Display the recognized text
Console.WriteLine(result.RecognitionText);

Console.WriteLine("OCRDetectAreasMode executed successfully");
```
