# OCR इमेज रिकग्निशन में आयतें तैयार करें

## परिचय

ऑप्टिकल कैरेक्टर रिकग्निशन (OCR) दृश्य सामग्री को खोज योग्य, संपादन योग्य टेक्स्ट में बदलने के लिए आवश्यक है। इस ट्यूटोरियल में आप **इमेज से टेक्स्ट निकालेंगे** कस्टम आयतें तैयार करके जो OCR इंजन को विशिष्ट क्षेत्रों पर केंद्रित करती हैं। Aspose.OCR for .NET का उपयोग करके, हम हर कदम से गुजरेंगे—अपने प्रोजेक्ट को सेटअप करने से लेकर पहचाने गए टेक्स्ट को प्राप्त करने तक—ताकि आप अपनी .NET एप्लिकेशन्स में शक्तिशाली इमेज‑टू‑टेक्स्ट कार्यक्षमता को एकीकृत कर सकें।

## त्वरित उत्तर
- **“इमेज से टेक्स्ट निकालना” का क्या मतलब है?** इसका अर्थ है चित्र में मौजूद दृश्य अक्षरों को मशीन‑रीडेबल स्ट्रिंग्स में बदलना।  
- **.NET में इसके लिए कौन सी लाइब्रेरी मदद करती है?** Aspose.OCR for .NET।  
- **क्या विकास के लिए लाइसेंस चाहिए?** परीक्षण के लिए मुफ्त ट्रायल काम करता है; उत्पादन के लिए लाइसेंस आवश्यक है।  
- **क्या मैं विशिष्ट क्षेत्रों को टार्गेट कर सकता हूँ?** हाँ, आयतें परिभाषित करके OCR का दायरा सीमित किया जा सकता है।  
- **कौन से .NET संस्करण समर्थित हैं?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7।

## आयतों के साथ “इमेज से टेक्स्ट निकालना” क्या है?
जब आप इमेज पर आयताकार ज़ोन परिभाषित करते हैं, तो OCR इंजन केवल उन ज़ोन को प्रोसेस करता है। इससे सटीकता बढ़ती है, प्रोसेसिंग समय घटता है, और आप शोरयुक्त बैकग्राउंड या अप्रासंगिक सेक्शन को अनदेखा कर सकते हैं।

## OCR से पहले आयतें क्यों तैयार करें?
- **संबंधित सामग्री पर फोकस:** हेडर, फुटर या सजावटी ग्राफिक्स को छोड़ दें।  
- **प्रदर्शन बढ़ाएँ:** छोटे क्षेत्र तेज़ पहचान का मतलब होते हैं।  
- **सटीकता सुधारें:** कम दृश्य शोर से साफ़ परिणाम मिलते हैं।

## पूर्वापेक्षाएँ

- C# और .NET विकास की परिचितता।  
- Aspose.OCR for .NET लाइब्रेरी स्थापित – आप इसे **[यहाँ](https://releases.aspose.com/ocr/net/)** डाउनलोड कर सकते हैं।  
- एक सैंपल इमेज (जैसे `sample.png`) जिसमें वह टेक्स्ट हो जिसे आप निकालना चाहते हैं।

## नेमस्पेसेस इम्पोर्ट करें

पहले, आवश्यक नेमस्पेसेस को स्कोप में लाएँ:

```csharp
using System;
using System.Collections.Generic;
using System.Drawing;
using System.IO;
using Aspose.OCR;
```

## चरण 1: अपने डॉक्यूमेंट डायरेक्टरी सेट करें

निर्दिष्ट करें कि आपकी इमेज फ़ाइलें कहाँ स्थित हैं और OCR इंजन का एक इंस्टेंस बनाएँ।

```csharp
// The path to the documents directory.
string dataDir = "Your Document Directory";

// Initialize an instance of AsposeOcr
AsposeOcr api = new AsposeOcr();
```

## कई आयतों का उपयोग करके इमेज से टेक्स्ट निकालने का तरीका

### चरण 2: कई आयतों के साथ इमेज को पहचानें

#### 2.1 आयतें परिभाषित करें

`Rectangle` ऑब्जेक्ट्स की एक लिस्ट बनाएँ जो उन क्षेत्रों की रूपरेखा तय करती है जिन्हें आप OCR इंजन को स्कैन कराना चाहते हैं।

```csharp
List<Rectangle> rects = new List<Rectangle>()
{
    new Rectangle(138, 352, 2033, 537),
    new Rectangle(147, 890, 2033, 1157),
    new Rectangle(923, 2045, 465, 102),
    new Rectangle(104, 2147, 2076, 819)
};
```

#### 2.2 OCR पहचान करें

इमेज पाथ और आयत लिस्ट को `RecognizeImage` में पास करें। यह मेथड स्ट्रिंग्स का एक कलेक्शन रिटर्न करता है—प्रत्येक एंट्री एक आयत से मेल खाती है।

```csharp
// first case
List<string> listResult = api.RecognizeImage(dataDir + "sample.png", rects);

// Display the recognized text
foreach (string s in listResult)
{
    Console.WriteLine(s);
}
```

### चरण 3: रिकग्निशन सेटिंग्स के साथ इमेज को पहचानें (वैकल्पिक तरीका)

यदि आप `RecognitionSettings` का उपयोग करना पसंद करते हैं, तो आप थोड़ा अलग API कॉल के साथ वही परिणाम प्राप्त कर सकते हैं।

#### 3.1 रिकग्निशन सेटिंग्स परिभाषित करें

```csharp
RecognitionResult result = api.RecognizeImage(dataDir + "sample.png", new RecognitionSettings
{
    RecognitionAreas = rects
});
```

#### 3.2 पहचाने गए टेक्स्ट को दिखाएँ

```csharp
// Display the recognized text
foreach (string s in result.RecognitionAreasText)
{
    Console.WriteLine(s);
}
```

## सामान्य समस्याएँ और सुझाव

- **गलत आयत निर्देशांक:** सुनिश्चित करें कि `X`, `Y`, `Width`, और `Height` मान सही ढंग से उस क्षेत्र को मैप करते हैं जिसे आप चाहते हैं।  
- **इमेज क्वालिटी:** लो‑रेज़ोल्यूशन इमेजेज़ खराब OCR परिणाम दे सकती हैं; प्री‑प्रोसेसिंग (जैसे बाइनराइज़ेशन) पर विचार करें।  
- **खाली परिणाम:** जाँचें कि आयतों में वास्तव में टेक्स्ट है; अन्यथा इंजन खाली स्ट्रिंग रिटर्न करेगा।
