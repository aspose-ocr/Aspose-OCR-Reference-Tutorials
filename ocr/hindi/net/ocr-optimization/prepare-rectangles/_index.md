# OCR इमेज रिकग्निशन में आयतें तैयार करें

## परिचय

ऑप्टिकल कैरेक्टर रिकग्निशन (OCR) दृश्य सामग्री को खोज योग्य, संपादन योग्य टेक्स्ट में बदलने के लिए आवश्यक है। इस ट्यूटोरियल में आप **छवि से टेक्स्ट निकालेंगे** कस्टम आयतें तैयार करके जो OCR इंजन को विशिष्ट क्षेत्रों पर केंद्रित करती हैं। Aspose.OCR for .NET का उपयोग करके, हम हर कदम से गुजरेंगे—अपने प्रोजेक्ट को सेट अप करने से लेकर पहचाने गए टेक्स्ट को प्राप्त करने तक—ताकि आप अपनी .NET एप्लिकेशन में शक्तिशाली इमेज‑टू‑टेक्स्ट कार्यक्षमता को एकीकृत कर सकें।

## त्वरित उत्तर
- **छवि से टेक्स्ट निकालना** क्या मतलब है? यह चित्र में दृश्य अक्षरों को मशीन‑पठनीय स्ट्रिंग्स में बदलना है।  
- **.NET में इसके लिए कौन सी लाइब्रेरी मदद करती है?** Aspose.OCR for .NET.  
- **क्या विकास के लिए लाइसेंस चाहिए?** परीक्षण के लिए एक मुफ्त ट्रायल काम करता है; उत्पादन के लिए लाइसेंस आवश्यक है।  
- **क्या मैं विशिष्ट क्षेत्रों को लक्षित कर सकता हूँ?** हाँ, OCR सीमा को सीमित करने के लिए आयतें परिभाषित करके।  
- **कौन से .NET संस्करण समर्थित हैं?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7.

## आयतों के साथ “छवि से टेक्स्ट निकालना” क्या है?
जब आप छवि पर आयताकार ज़ोन परिभाषित करते हैं, तो OCR इंजन केवल उन ज़ोन को प्रोसेस करता है। इससे सटीकता बढ़ती है, प्रोसेसिंग समय घटता है, और आप शोरयुक्त पृष्ठभूमि या अप्रासंगिक हिस्सों को अनदेखा कर सकते हैं।

## OCR से पहले आयतें क्यों तैयार करें?
- **संबंधित सामग्री पर ध्यान केंद्रित करें:** हेडर, फुटर या सजावटी ग्राफिक्स को छोड़ दें।  
- **प्रदर्शन बढ़ाएँ:** छोटे क्षेत्रों का मतलब तेज़ पहचान।  
- **सटीकता सुधारें:** कम दृश्य शोर से साफ़ परिणाम मिलते हैं।

## वास्तविक‑दुनिया के प्रोजेक्ट्स के लिए यह क्यों महत्वपूर्ण है
कई व्यावसायिक दस्तावेज़—रसीदें, इनवॉइस, आईडी कार्ड—में मिश्रित लेआउट होते हैं जहाँ केवल कुछ हिस्सों में मूल्यवान टेक्स्ट होता है। आयतों का उपयोग करके, आप केवल आवश्यक फ़ील्ड निकाल सकते हैं, जिससे पोस्ट‑प्रोसेसिंग कार्य में काफी कमी आती है और आपके ऑटोमेशन पाइपलाइन की समग्र विश्वसनीयता बढ़ती है।

## सामान्य उपयोग केस
- **डेटा एंट्री ऑटोमेशन:** स्कैन किए गए फ़ॉर्म से विशिष्ट फ़ील्ड निकालें।  
- **अनुपालन जांच:** कानूनी टेक्स्ट ब्लॉक्स को अलग करें और सत्यापित करें।  
- **कंटेंट इंडेक्सिंग:** खोज इंजनों के लिए छवि के केवल हेडलाइन या कैप्शन को इंडेक्स करें।  

## पूर्वापेक्षाएँ

- C# और .NET विकास की परिचितता।  
- Aspose.OCR for .NET लाइब्रेरी स्थापित – आप इसे **[here](https://releases.aspose.com/ocr/net/)** से डाउनलोड कर सकते हैं।  
- एक सैंपल इमेज (जैसे, `sample.png`) जिसमें वह टेक्स्ट हो जिसे आप निकालना चाहते हैं।

पहले, आवश्यक नेमस्पेस को स्कोप में लाएँ:

```csharp
using System;
using System.Collections.Generic;

```

## चरण 1: अपने दस्तावेज़ डायरेक्टरी सेट अप करें

निर्दिष्ट करें कि आपकी इमेज फ़ाइलें कहाँ स्थित हैं और OCR इंजन का एक इंस्टेंस बनाएं।

```csharp
// The path to the documents directory.
string dataDir = "Your Document Directory";

// Initialize an instance of AsposeOcr
AsposeOcr api = new AsposeOcr();
```

## कई आयतों का उपयोग करके छवि से टेक्स्ट निकालने का तरीका

### चरण 2: कई आयतों के साथ इमेज को पहचानें

#### 2.1 आयतें परिभाषित करें

`Rectangle` ऑब्जेक्ट्स की एक सूची बनाएं जो उन क्षेत्रों को रेखांकित करती है जिन्हें आप OCR इंजन को स्कैन करना चाहते हैं।

```csharp
List<Rectangle> rects = new List<Rectangle>()
{
    new Rectangle(138, 352, 2033, 537),
    new Rectangle(147, 890, 2033, 1157),
    new Rectangle(923, 2045, 465, 102),
    new Rectangle(104, 2147, 2076, 819)
};
```

#### 2.2 OCR पहचान निष्पादित करें

इमेज पाथ और आयत सूची को `RecognizeImage` को पास करें। यह मेथड स्ट्रिंग्स का संग्रह लौटाता है—प्रत्येक एंट्री एक आयत से संबंधित होती है।

```csharp
// first case
List<string> listResult = api.RecognizeImage(dataDir + "sample.png", rects);

// Display the recognized text
foreach (string s in listResult)
{
    Console.WriteLine(s);
}
```

### चरण 3: रिकग्निशन सेटिंग्स के साथ इमेज को पहचानें (वैकल्पिक तरीका)

यदि आप `RecognitionSettings` का उपयोग करना पसंद करते हैं, तो आप थोड़ा अलग API कॉल के साथ वही परिणाम प्राप्त कर सकते हैं।

#### 3.1 रिकग्निशन सेटिंग्स परिभाषित करें

```csharp
RecognitionResult result = api.RecognizeImage(dataDir + "sample.png", new RecognitionSettings
{
    RecognitionAreas = rects
});
```

#### 3.2 पहचाना गया टेक्स्ट दिखाएँ

```csharp
// Display the recognized text
foreach (string s in result.RecognitionAreasText)
{
    Console.WriteLine(s);
}
```

## सामान्य समस्याएँ और टिप्स

- **गलत आयत निर्देशांक:** सुनिश्चित करें कि `X`, `Y`, `Width`, और `Height` मान सही ढंग से उस क्षेत्र से मेल खाते हैं जिसे आप चाहते हैं।  
- **इमेज क्वालिटी:** कम‑रिज़ॉल्यूशन वाली इमेजेज़ खराब OCR परिणाम दे सकती हैं; प्री‑प्रोसेसिंग (जैसे, बाइनराइज़ेशन) पर विचार करें।  
- **खाली परिणाम:** पुष्टि करें कि आयतों में वास्तव में टेक्स्ट है; अन्यथा इंजन खाली स्ट्रिंग्स लौटाएगा।

## समस्या निवारण और सर्वोत्तम प्रथाएँ

| लक्षण | संभावित कारण | उपाय |
|---------|--------------|--------|
| कोई आउटपुट नहीं या खाली स्ट्रिंग्स | आयतें इमेज की सीमा के बाहर | इमेज के आयाम और आयत निर्देशांक को दोबारा जांचें |
| गड़बड़ अक्षर | कम कंट्रास्ट या शोर | OCR से पहले इमेज क्लीनिंग (ग्रेस्केल, थ्रेशहोल्ड) लागू करें |
| बड़ी फ़ाइलों पर धीमी प्रदर्शन | बहुत अधिक आयतें या बहुत बड़ी इमेज | इमेज को विभाजित करें या जहाँ संभव हो आयतों की संख्या कम करें |
