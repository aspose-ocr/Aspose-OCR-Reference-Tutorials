# लेआउट विश्लेषण OCR – छवियों से लाइन आयत प्राप्त करें

## परिचय

इस ट्यूटोरियल में आप Aspose.OCR for .NET के साथ **OCR लाइन आयतें कैसे प्राप्त करें** जानेंगे। गाइड के अंत तक आप **छवि में टेक्स्ट लाइनों को पहचानने** और प्रत्येक पहचानी गई लाइन के लिए **लाइन निर्देशांक निकालने** में सक्षम होंगे—जो **लेआउट विश्लेषण OCR**, डेटा निष्कर्षण, या कस्टम रेंडरिंग जैसी डाउनस्ट्रीम प्रोसेसिंग के लिए उपयुक्त है।

## त्वरित उत्तर
- **“OCR लाइन आयतें प्राप्त करें” का क्या अर्थ है?** यह छवि में पहचानी गई प्रत्येक टेक्स्ट लाइन के बाउंडिंग बॉक्स लौटाता है।  
- **कौन सा API मेथड उपयोग किया जाता है?** `AsposeOcr.GetRectangles(..., AreasType.LINES, ...)`।  
- **क्या मुझे लाइसेंस चाहिए?** विकास के लिए एक मुफ्त ट्रायल काम करता है; उत्पादन के लिए एक व्यावसायिक लाइसेंस आवश्यक है।  
- **समर्थित इमेज फ़ॉर्मेट?** PNG, JPEG, BMP, TIFF, और अधिक।  
- **क्या मैं इसे .NET Core पर चला सकता हूँ?** हाँ, Aspose.OCR पूरी तरह से .NET Core और .NET 5/6 को सपोर्ट करता है।

## पूर्वापेक्षाएँ

ट्यूटोरियल शुरू करने से पहले सुनिश्चित करें कि आपके पास निम्नलिखित पूर्वापेक्षाएँ मौजूद हैं:

- C# और .NET विकास का बुनियादी ज्ञान।  
- Visual Studio जैसे एकीकृत विकास वातावरण (IDE)।  
- Aspose.OCR for .NET लाइब्रेरी स्थापित। आप इसे [यहाँ](https://releases.aspose.com/ocr/net/) डाउनलोड कर सकते हैं।  
- OCR पहचान के लिए टेक्स्ट वाली एक नमूना छवि।

## नेमस्पेस आयात करें

सुनिश्चित करें कि आपके प्रोजेक्ट में आवश्यक नेमस्पेस आयात किए गए हैं। अपने C# फ़ाइल के शीर्ष पर निम्नलाइनों को जोड़ें:

```csharp
using System;
using System.Collections.Generic;
using System.Drawing;
using System.IO;
using Aspose.OCR;
```

अब, चलिए OCR इमेज पहचान में लाइनों के लिए आयतें प्राप्त करने की प्रक्रिया को आसान‑से‑अनुसरणीय चरणों में विभाजित करते हैं।

## लेआउट विश्लेषण OCR – चरण‑दर‑चरण गाइड

### चरण 1: अपने दस्तावेज़ डायरेक्टरी सेट करें

```csharp
// ExStart:3
string dataDir = "Your Document Directory";
// ExEnd:3
```

`"Your Document Directory"` को उस फ़ोल्डर के वास्तविक पथ से बदलें जिसमें आपकी नमूना छवि रखी है।

### चरण 2: Aspose.OCR को इनिशियलाइज़ करें

```csharp
// ExStart:4
AsposeOcr api = new AsposeOcr();
// ExEnd:4
```

`AsposeOcr` क्लास का एक इंस्टेंस बनाकर OCR कार्यक्षमता तक पहुँचें।

### चरण 3: इमेज पाथ निर्दिष्ट करें

```csharp
// ExStart:5
string fullPath = dataDir + "sample.png";
// ExEnd:5
```

उस इमेज का पूर्ण पाथ निर्धारित करें जिस पर आप OCR करना चाहते हैं।

### चरण 4: इमेज को पहचानें और आयतें प्राप्त करें

```csharp
// ExStart:6
List<Rectangle> lines = api.GetRectangles(fullPath, AreasType.LINES, false);
// ExEnd:6
```

`GetRectangles` मेथड `Rectangle` ऑब्जेक्ट्स की एक सूची लौटाता है, प्रत्येक एक पहचानी गई टेक्स्ट लाइन के निर्देशांक दर्शाता है। यह **OCR लाइन आयतें प्राप्त करने** का मूल है और **लेआउट विश्लेषण OCR** को सक्षम बनाता है।

### चरण 5: परिणाम प्रिंट करें

```csharp
// ExStart:7
Console.WriteLine("Areas coordinates:");
lines.ForEach(a => Console.WriteLine($"x:{a.X} y:{a.Y} width:{a.Width} height:{a.Height}"));
// ExEnd:7
```

पहचाने गए क्षेत्रों के निर्देशांक को कंसोल में प्रिंट करें। आप ऐसे मान देखेंगे जिन्हें आप बाद में कस्टम प्रोसेसिंग के लिए **लाइन निर्देशांक निकालने** में उपयोग कर सकते हैं।

## लाइन आयतों के लिए Aspose.OCR क्यों उपयोग करें?

- **उच्च सटीकता** – उन्नत एल्गोरिदम शोरयुक्त या तिरछी छवियों में भी लाइनों का पता लगाते हैं।  
- **क्रॉस‑प्लेटफ़ॉर्म** – .NET Framework, .NET Core, और .NET 5/6 पर काम करता है।  
- **कोई बाहरी निर्भरताएँ नहीं** – शुद्ध .NET लाइब्रेरी, शिप करने के लिए कोई नेटिव DLL नहीं।  
- **समृद्ध आउटपुट** – लाइन आयतों के अलावा आप शब्द, अक्षर, और कॉन्फिडेंस स्कोर भी प्राप्त कर सकते हैं।

## सामान्य समस्याएँ और समाधान

| समस्या | समाधान |
|-------|----------|
| **कोई आयतें वापस नहीं मिलीं** | सुनिश्चित करें कि छवि में स्पष्ट, क्षैतिज टेक्स्ट हो और `AreasType.LINES` निर्दिष्ट किया गया हो। |
| **गलत निर्देशांक** | छवि के DPI की जाँच करें; कम‑रिज़ॉल्यूशन वाली छवियों से असटीक बाउंड्स हो सकते हैं। |
| **बड़ी छवियों पर प्रदर्शन बाधा** | `GetRectangles` कॉल करने से पहले छवि को उचित रिज़ॉल्यूशन में री‑साइज़ करें। |
| **लाइसेंस अपवाद** | परीक्षण के लिए ट्रायल लाइसेंस उपयोग करें; उत्पादन में मूल्यांकन सीमाओं से बचने के लिए पूर्ण लाइसेंस लागू करें। |

## अक्सर पूछे जाने वाले प्रश्न

**प्रश्न: क्या मैं पूरी लाइनों के बजाय व्यक्तिगत शब्द निकाल सकता हूँ?**  
**उत्तर:** हाँ, शब्द‑स्तर के बाउंडिंग बॉक्स प्राप्त करने के लिए उसी `GetRectangles` मेथड के साथ `AreasType.WORDS` का उपयोग करें।

**प्रश्न: क्या API मल्टी‑पेज PDFs को सपोर्ट करता है?**  
**उत्तर:** पहले प्रत्येक PDF पेज को इमेज में बदलें, फिर प्रत्येक इमेज पर `GetRectangles` कॉल करें।

**प्रश्न: घुमा हुआ टेक्स्ट कैसे संभालें?**  
**उत्तर:** OCR सेटिंग्स में ऑटो‑रोटेट विकल्प सक्षम करें या प्रोसेसिंग से पहले इमेज को पूर्व‑रोटेट करें।

**प्रश्न: प्रत्येक लाइन के लिए कॉन्फिडेंस स्कोर प्राप्त करने का कोई तरीका है?**  
**उत्तर:** आयतें प्राप्त करने के बाद, `api.RecognizeImage(...).Lines` को कॉल करके लाइन ऑब्जेक्ट्स तक पहुँचें जिनमें कॉन्फिडेंस मान शामिल होते हैं।

**प्रश्न: कौन से .NET संस्करण संगत हैं?**  
**उत्तर:** लाइब्रेरी .NET Framework 4.5+, .NET Core 3.1+, और .NET 5/6 के साथ काम करती है।

## वास्तविक‑विश्व उपयोग केस

- **डॉक्यूमेंट लेआउट विश्लेषण OCR** – कॉलम संरचनाओं को पुनर्निर्मित करने के लिए लाइन आयतों को लेआउट इंजन में फीड करें।  
- **स्वचालित डेटा निष्कर्षण** – डाउनस्ट्रीम NLP पाइपलाइन के लिए व्यक्तिगत लाइनों को क्रॉप करने हेतु निर्देशांक उपयोग करें।  
- **कस्टम रेंडरिंग** – विज़ुअल वेरिफिकेशन या UI ओवरले के लिए मूल छवि पर बाउंडिंग बॉक्स ओवरले करें।
