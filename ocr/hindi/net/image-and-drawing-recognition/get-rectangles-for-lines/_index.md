# इमेज टेक्स्ट लाइनों के लिए OCR लाइन रेक्टेंगल प्राप्त करें

## परिचय

इस ट्यूटोरियल में आप Aspose.OCR for .NET के साथ **OCR लाइन रेक्टेंगल कैसे प्राप्त करें** सीखेंगे। गाइड के अंत तक आप **इमेज में टेक्स्ट लाइनों को पहचानने** और प्रत्येक पहचानी गई लाइन के लिए **लाइन कोऑर्डिनेट्स निकालने** में सक्षम होंगे—जो लेआउट विश्लेषण, डेटा एक्सट्रैक्शन या कस्टम रेंडरिंग जैसे डाउनस्ट्रीम प्रोसेसिंग के लिए उपयुक्त है।

## त्वरित उत्तर
- **“get OCR line rectangles” का क्या मतलब है?** यह इमेज में पहचानी गई प्रत्येक टेक्स्ट लाइन के बाउंडिंग बॉक्स को लौटाता है।  
- **कौन सा API मेथड उपयोग किया जाता है?** `AsposeOcr.GetRectangles(..., AreasType.LINES, ...)`.  
- **क्या मुझे लाइसेंस चाहिए?** विकास के लिए फ्री ट्रायल काम करता है; प्रोडक्शन के लिए कमर्शियल लाइसेंस आवश्यक है।  
- **समर्थित इमेज फॉर्मेट?** PNG, JPEG, BMP, TIFF, आदि।  
- **क्या इसे .NET Core पर चलाया जा सकता है?** हाँ, Aspose.OCR पूरी तरह से .NET Core और .NET 5/6 को सपोर्ट करता है।

## पूर्वापेक्षाएँ

ट्यूटोरियल शुरू करने से पहले सुनिश्चित करें कि आपके पास निम्नलिखित पूर्वापेक्षाएँ मौजूद हैं:

- C# और .NET विकास का बुनियादी ज्ञान।  
- Visual Studio जैसे एकीकृत विकास वातावरण (IDE)।  
- Aspose.OCR for .NET लाइब्रेरी स्थापित। आप इसे [यहाँ](https://releases.aspose.com/ocr/net/) से डाउनलोड कर सकते हैं।  
- OCR पहचान के लिए टेक्स्ट वाली एक नमूना इमेज।

## नेमस्पेस आयात करें

सुनिश्चित करें कि आपके प्रोजेक्ट में आवश्यक नेमस्पेस आयात किए गए हैं। अपनी C# फ़ाइल के शीर्ष पर निम्नलाइनों को जोड़ें:

```csharp
using System;
using System.Collections.Generic;
using System.Drawing;
using System.IO;
using Aspose.OCR;
```

अब, OCR इमेज पहचान में लाइनों के लिए रेक्टेंगल प्राप्त करने की प्रक्रिया को आसान‑से‑अनुसरण करने योग्य चरणों में विभाजित करते हैं।

## चरण 1: अपने डॉक्यूमेंट डायरेक्टरी को सेट अप करें

```csharp
// ExStart:3
string dataDir = "Your Document Directory";
// ExEnd:3
```

`"Your Document Directory"` को उस फ़ोल्डर के वास्तविक पाथ से बदलें जिसमें आपका नमूना इमेज स्थित है।

## चरण 2: Aspose.OCR को इनिशियलाइज़ करें

```csharp
// ExStart:4
AsposeOcr api = new AsposeOcr();
// ExEnd:4
```

OCR कार्यक्षमता तक पहुँचने के लिए `AsposeOcr` क्लास का एक इंस्टेंस बनाएं।

## चरण 3: इमेज पाथ निर्दिष्ट करें

```csharp
// ExStart:5
string fullPath = dataDir + "sample.png";
// ExEnd:5
```

उस इमेज का पूर्ण पाथ परिभाषित करें जिस पर आप OCR चलाना चाहते हैं।

## चरण 4: इमेज को पहचानें और रेक्टेंगल प्राप्त करें

```csharp
// ExStart:6
List<Rectangle> lines = api.GetRectangles(fullPath, AreasType.LINES, false);
// ExEnd:6
```

`GetRectangles` मेथड `Rectangle` ऑब्जेक्ट्स की एक सूची लौटाता है, प्रत्येक एक पहचानी गई टेक्स्ट लाइन के कोऑर्डिनेट्स को दर्शाता है। यह **OCR लाइन रेक्टेंगल प्राप्त करने** का मुख्य भाग है।

## चरण 5: परिणाम प्रिंट करें

```csharp
// ExStart:7
Console.WriteLine("Areas coordinates:");
lines.ForEach(a => Console.WriteLine($"x:{a.X} y:{a.Y} width:{a.Width} height:{a.Height}"));
// ExEnd:7
```

पता लगाए गए क्षेत्रों के कोऑर्डिनेट्स को कंसोल में प्रिंट करें। आप ऐसे मान देखेंगे जिन्हें आप बाद में कस्टम प्रोसेसिंग के लिए **लाइन कोऑर्डिनेट्स निकालने** हेतु उपयोग कर सकते हैं।

## Aspose.OCR को लाइन रेक्टेंगल के लिए क्यों उपयोग करें?

- **उच्च सटीकता** – उन्नत एल्गोरिदम शोरयुक्त या तिरछी इमेज में भी लाइनों का पता लगाते हैं।  
- **क्रॉस‑प्लेटफ़ॉर्म** – .NET Framework, .NET Core, और .NET 5/6 पर काम करता है।  
- **कोई बाहरी निर्भरताएँ नहीं** – शुद्ध .NET लाइब्रेरी, शिप करने के लिए कोई नेटिव DLL नहीं।  
- **समृद्ध आउटपुट** – लाइन रेक्टेंगल के अलावा आप शब्द, अक्षर और कॉन्फिडेंस स्कोर भी प्राप्त कर सकते हैं।

## सामान्य समस्याएँ और समाधान

| समस्या | समाधान |
|--------|----------|
| **कोई रेक्टेंगल नहीं मिला** | सुनिश्चित करें कि इमेज में स्पष्ट, क्षैतिज टेक्स्ट है और `AreasType.LINES` निर्दिष्ट किया गया है। |
| **कोऑर्डिनेट्स गलत हैं** | इमेज DPI जांचें; कम‑रिज़ॉल्यूशन इमेज से बाउंड्स में त्रुटि हो सकती है। |
| **बड़ी इमेज पर प्रदर्शन बाधा** | `GetRectangles` कॉल करने से पहले इमेज को उचित रिज़ॉल्यूशन पर रिसाइज़ करें। |
| **लाइसेंस अपवाद** | परीक्षण के लिए ट्रायल लाइसेंस उपयोग करें; प्रोडक्शन में पूर्ण लाइसेंस लागू करें ताकि मूल्यांकन सीमाएँ न आएँ। |

## FAQ's

### Q1: क्या मैं Aspose.OCR for .NET को किसी भी प्रकार की इमेज के साथ उपयोग कर सकता हूँ?

A1: Aspose.OCR विभिन्न इमेज फॉर्मेट्स को सपोर्ट करता है, जिससे आपके OCR एप्लिकेशन में लचीलापन मिलता है।

### Q2: OCR पहचान की सटीकता कितनी है?

A2: Aspose.OCR उन्नत एल्गोरिदम का उपयोग करके उच्च सटीकता प्रदान करता है, जो विभिन्न टेक्स्ट पहचान परिदृश्यों के लिए उपयुक्त है।

### Q3: क्या ट्रायल संस्करण उपलब्ध है?

A3: हाँ, आप [फ्री ट्रायल](https://releases.aspose.com/) के साथ Aspose.OCR for .NET की क्षमताओं का अन्वेषण कर सकते हैं।

### Q4: विस्तृत दस्तावेज़ीकरण कहाँ मिल सकता है?

A4: विस्तृत जानकारी और उपयोग दिशानिर्देशों के लिए [डॉक्यूमेंटेशन](https://reference.aspose.com/ocr/net/) देखें।

### Q5: सहायता चाहिए या विशेष प्रश्न हैं?

A5: समुदाय समर्थन और चर्चा के लिए [Aspose.OCR फ़ोरम](https://forum.aspose.com/c/ocr/16) पर जाएँ।

## अक्सर पूछे जाने वाले प्रश्न

**प्रश्न: क्या मैं पूरी लाइनों के बजाय व्यक्तिगत शब्द निकाल सकता हूँ?**  
उत्तर: हाँ, वही `GetRectangles` मेथड के साथ `AreasType.WORDS` उपयोग करके शब्द‑स्तर के बाउंडिंग बॉक्स प्राप्त कर सकते हैं।

**प्रश्न: क्या API मल्टी‑पेज PDFs को सपोर्ट करता है?**  
उत्तर: प्रत्येक PDF पेज को पहले इमेज में बदलें, फिर प्रत्येक इमेज पर `GetRectangles` कॉल करें।

**प्रश्न: घुमाए हुए टेक्स्ट को कैसे संभालें?**  
उत्तर: OCR सेटिंग्स में ऑटो‑रोटेट विकल्प सक्षम करें या प्रोसेसिंग से पहले इमेज को प्री‑रोटेट करें।

**प्रश्न: प्रत्येक लाइन के लिए कॉन्फिडेंस स्कोर प्राप्त करने का कोई तरीका है?**  
उत्तर: रेक्टेंगल प्राप्त करने के बाद `api.RecognizeImage(...).Lines` कॉल करें ताकि लाइन ऑब्जेक्ट्स मिलें जिनमें कॉन्फिडेंस वैल्यू शामिल हो।

**प्रश्न: कौन‑से .NET संस्करण संगत हैं?**  
उत्तर: लाइब्रेरी .NET Framework 4.5+, .NET Core 3.1+, और .NET 5/6 के साथ काम करती है।
