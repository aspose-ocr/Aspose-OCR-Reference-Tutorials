# पैराग्राफ के लिए आयतें निकालने का तरीका OCR इमेज रिकग्निशन में

## परिचय

Aspose.OCR for .NET के साथ OCR इमेज रिकग्निशन में पैराग्राफ के लिए **how to extract rectangles** के बारे में हमारा व्यापक गाइड में आपका स्वागत है। यदि आप अपने दस्तावेज़‑प्रोसेसिंग पाइपलाइन को बेहतर बनाना चाहते हैं, पैराग्राफ की सीमाएँ निकालना चाहते हैं, और डेटा एक्सट्रैक्शन को स्वचालित करना चाहते हैं, तो आप सही जगह पर हैं। हम हर कदम को समझाएंगे—पर्यावरण सेटअप से लेकर आयत के निर्देशांक प्रिंट करने तक—ताकि आप तुरंत OCR परिणामों का उपयोग शुरू कर सकें।

## त्वरित उत्तर
- **“extract rectangles” क्या मतलब है?** यह पहचान किए गए टेक्स्ट क्षेत्रों के बाउंडिंग बॉक्स (x, y, width, height) लौटाता है।  
- **कौन सा API मेथड आयतें प्रदान करता है?** `AsposeOcr.GetRectangles` with `AreasType.PARAGRAPHS`.  
- **क्या विकास के लिए लाइसेंस चाहिए?** परीक्षण के लिए एक फ्री ट्रायल काम करता है; उत्पादन के लिए एक वाणिज्यिक लाइसेंस आवश्यक है।  
- **क्या मैं एक साथ कई इमेज प्रोसेस कर सकता हूँ?** हाँ—अपनी इमेज सूची पर लूप करें और प्रत्येक फ़ाइल के लिए `GetRectangles` कॉल करें।  
- **कौन से फ़ॉर्मेट समर्थित हैं?** PNG, JPEG, TIFF, BMP, और कई अन्य।

## OCR में “how to extract rectangles” क्या है?

OCR शब्दावली में, आयतें निकालना का मतलब है छवि के भीतर प्रत्येक पैराग्राफ या टेक्स्ट लाइन को घेरने वाली ज्यामितीय सीमाओं की पहचान करना। ये निर्देशांक आपको स्कैन किए गए दस्तावेज़ के विशिष्ट भागों को हाइलाइट, क्रॉप, या आगे विश्लेषण करने की अनुमति देते हैं।

## पैराग्राफ निर्देशांक निकालने का क्यों?

- **सटीक पोस्ट‑प्रोसेसिंग** – आप प्रत्येक आयत को डाउनस्ट्रीम वर्कफ़्लो (जैसे, अनुवाद, रेडैक्शन) में फीड कर सकते हैं।  
- **बेहतर UI/UX** – मूल छवि पर बाउंडिंग बॉक्स ओवरले करें ताकि उपयोगकर्ताओं को दिखाया जा सके कि टेक्स्ट कहाँ मिला।  
- **बैच ऑटोमेशन** – बड़े दस्तावेज़ सेट में जल्दी से पैराग्राफ खोजें और अलग करें।

## पूर्वापेक्षाएँ

- C# और .NET विकास का बुनियादी ज्ञान।  
- Aspose.OCR for .NET स्थापित वाला विकास वातावरण – आप इसे [यहाँ](https://releases.aspose.com/ocr/net/) डाउनलोड कर सकते हैं।  
- इमेज प्रोसेसिंग अवधारणाओं की परिचितता और स्कैन किए गए फ़ाइलों से टेक्स्ट निकालने में OCR के महत्व को समझना।

## नेमस्पेस इम्पोर्ट करें

अपने C# फ़ाइल में, आवश्यक नेमस्पेस इम्पोर्ट करें ताकि OCR क्लास उपलब्ध हों:

```csharp
using System;
using System.Collections.Generic;
using System.Drawing;
using System.IO;
using Aspose.OCR;
```

## चरण 1: अपने दस्तावेज़ डायरेक्टरी सेट करें

उस फ़ोल्डर को परिभाषित करें जिसमें वे इमेज हैं जिन्हें आप विश्लेषण करना चाहते हैं:

```csharp
string dataDir = "Your Document Directory";
```

## चरण 2: AsposeOcr इंस्टेंस इनिशियलाइज़ करें

`AsposeOcr` ऑब्जेक्ट बनाएं – यह आपको सभी OCR फ़ंक्शन तक पहुंच देता है:

```csharp
AsposeOcr api = new AsposeOcr();
```

## चरण 3: इमेज पाथ निर्दिष्ट करें

उस इमेज फ़ाइल की ओर इशारा करें जिसे आप प्रोसेस करना चाहते हैं:

```csharp
string fullPath = dataDir + "sample.png";
```

## चरण 4: इमेज को पहचानें और पैराग्राफ आयतें प्राप्त करें

`GetRectangles` मेथड को कॉल करें। `detect_areas` को `true` सेट करने से इंजन को **पैराग्राफ** आयतें लौटाने को कहा जाता है:

```csharp
List<Rectangle> rectangles = api.GetRectangles(fullPath, AreasType.PARAGRAPHS, true);
```

## चरण 5: परिणाम प्रिंट करें

निर्देशांक आउटपुट करें ताकि आप देख सकें कि कौन से **extract paragraph coordinates** पहचाने गए हैं:

```csharp
Console.WriteLine("Areas coordinates:");
rectangles.ForEach(a => Console.WriteLine($"x:{a.X} y:{a.Y} width:{a.Width} height:{a.Height}"));
```
