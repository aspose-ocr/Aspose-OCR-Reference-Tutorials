# OCR के लिए इमेज को डेस्क्यू कैसे करें – पूर्ण जावा प्री‑प्रोसेसिंग गाइड

क्या आपने कभी **इमेज को डेस्क्यू** करने के बारे में सोचा है इससे पहले कि आप उन्हें OCR इंजन को दें? आप अकेले नहीं हैं। कई वास्तविक‑दुनिया प्रोजेक्ट्स में—जैसे स्कैन किए गए इनवॉइस, हाथ से लिखे फ़ॉर्म, या पुराने समाचारपत्र अभिलेख—एक तिरछी स्कैन पहचान की सटीकता को बिगाड़ सकती है। अच्छी खबर? सिर्फ कुछ जावा लाइनों और Aspose OCR लाइब्रेरी के साथ, आप अपनी तस्वीरों को सीधा, साफ़ और बाइनराइज़ कर सकते हैं ताकि OCR इंजन उन्हें प्रो की तरह पढ़े।

इस ट्यूटोरियल में हम पूरे पाइपलाइन को देखेंगे: स्कैन किए गए फ़ॉर्म को लोड करना, डेस्क्यू फ़िल्टर लागू करना, शोर हटाना, साफ़ काली‑और‑सफ़ेद इमेज में बदलना, और अंत में टेक्स्ट निकालना। अंत तक आप **OCR सुधारने** के तरीके, **OCR के साथ इमेज प्रोसेस** करने की विश्वसनीय विधि, और एक तैयार‑चलाने‑योग्य कोड सैंपल प्राप्त करेंगे जो **फ़ॉर्म से टेक्स्ट निकालता** है सेकंड में।

## आपको क्या चाहिए

- **Java Development Kit (JDK) 8 या नया** – कोड किसी भी नवीनतम JDK के साथ कम्पाइल होता है।
- **Aspose.OCR for Java** लाइब्रेरी (लेखन के समय का नवीनतम संस्करण, 23.12)। आप इसे Maven Central से प्राप्त कर सकते हैं या Aspose की साइट से JAR डाउनलोड कर सकते हैं।
- परीक्षण के लिए एक इमेज फ़ाइल (जैसे, `scanned_form.jpg`)। बेहतर होगा कि वह स्कैन किया हुआ दस्तावेज़ थोड़ा टिल्टेड हो।
- आपका पसंदीदा IDE (IntelliJ IDEA, Eclipse, VS Code…) – कोई भी जो आपको एक साधारण `main` मेथड चलाने दे।

> **Pro tip:** यदि आप Maven का उपयोग कर रहे हैं, तो नीचे दी गई डिपेंडेंसी को अपने `pom.xml` में जोड़ें। यह सभी आवश्यक ट्रांज़िटिव लाइब्रेरीज़ को स्वचालित रूप से शामिल कर लेगा।

```xml
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-ocr</artifactId>
    <version>23.12</version>
</dependency>
```

## चरण 1 – OCR इंजन इंस्टेंस बनाएं  

पहला काम है `OcrEngine` को स्पिन अप करना। इसे उस दिमाग़ की तरह सोचें जो बाद में आपकी इमेज के अक्षरों को पढ़ेगा।

```java
import com.aspose.ocr.*;

public class DeskewDemo {
    public static void main(String[] args) throws Exception {

        // Initialize the OCR engine – this object holds all settings.
        OcrEngine ocrEngine = new OcrEngine();
```

यह चरण क्यों महत्वपूर्ण है? बिना इंजन के, बाद में जोड़ने वाले प्री‑प्रोसेसिंग फ़िल्टरों को कहीं भी अटैच नहीं किया जा सकता। इंजन भाषा पैक्स, रिकग्निशन मॉडल और आउटपुट फ़ॉर्मेट भी मैनेज करता है।

## चरण 2 – वह इमेज लोड करें जिसे आप साफ़ करना चाहते हैं  

अब इंजन को उस फ़ाइल की ओर इंगित करें जिसे आप सीधा करना चाहते हैं। `ImageStream.fromFile` फ़ाइल को एक स्ट्रीम में पढ़ता है जिसे Aspose उपयोग कर सकता है।

```java
        // Load the image (replace the path with your own file location)
        ocrEngine.setImage(ImageStream.fromFile("YOUR_DIRECTORY/scanned_form.jpg"));
```

यदि इमेज JAR के अंदर किसी रिसोर्स फ़ोल्डर में रहती है, तो आप `ImageStream.fromResource` का उपयोग कर सकते हैं। मुख्य बात यह है कि इंजन को एक **bitmap** मिले जिसे वह मैनीपुलेट कर सके।

## चरण 3 – सही क्रम में प्री‑प्रोसेसिंग फ़िल्टर जोड़ें  

यहीं पर जादू होता है। हम तीन फ़िल्टरों को चेन करेंगे:

1. **DeskewFilter** – स्वचालित रूप से टिल्ट एंगल का पता लगाता है और इमेज को क्षैतिज में वापस घुमा देता है।
2. **NoiseRemovalFilter** – उन धब्बों और दाने को हटाता है जो आमतौर पर कम‑गुणवत्ता वाले स्कैन में दिखते हैं।
3. **BinarizationFilter** – तस्वीर को शुद्ध काली‑और‑सफ़ेद में बदलता है, जिसे अधिकांश OCR इंजन पसंद करते हैं।

```java
        // Attach preprocessing filters: deskew → denoise → binarize
        ocrEngine.getEngineOptions()
                 .addPreprocessingFilter(new DeskewFilter())
                 .addPreprocessingFilter(new NoiseRemovalFilter())
                 .addPreprocessingFilter(new BinarizationFilter());
```

> **Why this order?** डेस्क्यू पहले सुनिश्चित करता है कि रोटेशन मूल पिक्सेल पर लागू हो; रोटेशन के बाद क्लीनिंग नई शोर उत्पन्न होने से बचाती है। बाइनराइज़ेशन अंत में OCR को एक तेज़, हाई‑कॉन्ट्रास्ट इमेज देता है—बिल्कुल वही जो आपको **OCR के साथ इमेज प्रोसेस** करने के लिए चाहिए।

## चरण 4 – प्री‑प्रोसेस्ड इमेज पर OCR चलाएँ  

अब हम इंजन से टेक्स्ट पढ़ने को कहते हैं। `process()` कॉल एक `OcrResult` लौटाता है जिसमें पहचाना गया स्ट्रिंग और वैकल्पिक कॉन्फिडेंस स्कोर होते हैं।

```java
        // Perform OCR on the cleaned image
        OcrResult ocrResult = ocrEngine.process();

        // Print the extracted text to the console
        System.out.println("=== Recognized Text ===");
        System.out.println(ocrResult.getText());
    }
}
```

यदि सब कुछ ठीक चलता है, तो आप मूल फ़ॉर्म पर मौजूद कच्चे अक्षर देखेंगे। यह **फ़ॉर्म से टेक्स्ट निकालने** वर्कफ़्लो का मूल है—एक बार स्ट्रिंग मिल जाए, तो आप फ़ील्ड्स को पार्स कर सकते हैं, डेटाबेस में डाल सकते हैं, या PDFs जेनरेट कर सकते हैं।

## चरण 5 – आउटपुट सत्यापित करें और पैरामीटर ट्यून करें  

हल्के टिल्टेड इनवॉइस पर डेमो चलाने से पठनीय आउटपुट मिलना चाहिए। हालांकि, किनारे के केस मौजूद हैं:

- **अत्यधिक कोण (>15°)** – आपको `DeskewFilter` की सहनशीलता को `setAngleThreshold` के माध्यम से बढ़ाने की आवश्यकता हो सकती है।
- **भारी बैकग्राउंड पैटर्न** – बाइनराइज़ेशन से पहले `ContrastEnhancementFilter` जोड़ने पर विचार करें।
- **बहु‑पृष्ठ PDFs** – प्रत्येक पृष्ठ पर लूप करें, पहले उसे इमेज में बदलें, फिर वही इंजन इंस्टेंस पुनः उपयोग करें।

नीचे 10‑डिग्री घुमाए गए रसीद का एक नमूना कंसोल आउटपुट दिया गया है:

```
=== Recognized Text ===
Date: 02/13/2026
Item          Qty   Price
Coffee        2     $4.00
Bagel         1     $2.50
Total                $6.50
```

ध्यान दें कि मूल टिल्ट के बावजूद टेक्स्ट लाइन्स पूरी तरह से संरेखित हैं। यही है **इमेज को डेस्क्यू** करने की सही समझ की शक्ति।

        // Save the pre‑processed image for inspection

                 .getPreprocessedImage()

## पुनरावलोकन – हमने क्या कवर किया  

- `DeskewFilter` का उपयोग करके **इमेज को डेस्क्यू** कैसे करें।  
- एक पूर्ण **OCR के लिए इमेज प्री‑प्रोसेस** श्रृंखला (डेस्क्यू → डीनॉइज़ → बाइनराइज़)।  
- Aspose OCR के साथ **फ़ॉर्म फ़ाइलों से टेक्स्ट निकालने** के लिए सटीक कोड।  
- **OCR सुधारने** के टिप्स और जटिल किनारे के मामलों को संभालने के लिए सुझाव।  
- प्रोडक्शन‑रेडी जावा मेथड में **OCR के साथ इमेज प्रोसेस** करने का तेज़ तरीका।

## अगले कदम  

अब जब आप एक पेज को सीधा कर पढ़ सकते हैं, तो स्केल अप करने पर विचार करें:

1. **बैच प्रोसेसिंग** – स्कैन की फ़ोल्डर पर इटररेट करें, वही पाइपलाइन लागू करें।  
2. **फ़ील्ड एक्सट्रैक्शन** – रेगुलर एक्सप्रेशन या Apache PDFBox जैसी लाइब्रेरी का उपयोग करके कच्चे टेक्स्ट को संरचित डेटा में मैप करें।  
3. **क्लाउड सर्विसेज़ के साथ इंटीग्रेशन** – साफ़ की गई इमेज को Azure Form Recognizer या Google Document AI को उन्नत लेआउट विश्लेषण के लिए भेजें।  

इनमें से प्रत्येक विषय उस नींव पर बना है जो आपने अभी स्थापित की है, और सभी को एक ठोस **OCR के लिए इमेज प्री‑प्रोसेस** रूटीन से लाभ मिलता है।

### अंतिम विचार  

परफ़ेक्ट OCR परिणाम प्राप्त करना कभी एक ही ट्रिक से नहीं होता; यह एक अनुशासित वर्कफ़्लो पर निर्भर करता है। **इमेज को डेस्क्यू** करने में महारत हासिल करके, आपने सबसे बड़ी बाधा को हटा दिया है। अब आप अन्य फ़िल्टरों के साथ प्रयोग कर सकते हैं, थ्रेशोल्ड समायोजित कर सकते हैं, और अपनी रिकग्निशन रेट्स को बढ़ते देख सकते हैं।

यदि आपको कोई समस्या आती है या आगे सुधारों के लिए आपके पास विचार हैं, तो नीचे टिप्पणी छोड़ें। हैप्पी कोडिंग, और आपकी स्कैन हमेशा पूरी तरह सीधी रहें!
