# इमेज से टेक्स्ट पहचानें और टेक्स्ट एरिया रेक्टेंगल प्राप्त करें

## परिचय

यदि आपको एक Java एप्लिकेशन में **इमेज फ़ाइलों से टेक्स्ट पहचानने** की आवश्यकता है, तो Aspose.OCR for Java तेज़ और सटीक तरीका प्रदान करता है। इस ट्यूटोरियल में हम इमेज से पैराग्राफ निकालने, प्रत्येक टेक्स्ट एरिया के बाउंडिंग रेक्टेंगल प्राप्त करने, और उन कॉर्डिनेट्स को कंसोल में प्रिंट करने के सटीक चरणों को देखेंगे। अंत तक आप समझेंगे कि यह तरीका क्यों काम करता है, लाइब्रेरी को कैसे इंटीग्रेट करें, और अपने उपयोग‑केस के लिए इसे कहाँ विस्तारित किया जा सकता है।

## त्वरित उत्तर
- **“इमेज से टेक्स्ट पहचानना” का क्या अर्थ है?** यह तस्वीर में मौजूद दृश्य अक्षरों को संपादन योग्य स्ट्रिंग डेटा में बदलना है।  
- **Java में यह कौन सी लाइब्रेरी संभालती है?** Aspose.OCR for Java।  
- **क्या विकास के लिए लाइसेंस चाहिए?** परीक्षण के लिए एक टेम्पररी लाइसेंस उपलब्ध है; उत्पादन के लिए पूर्ण लाइसेंस आवश्यक है।  
- **क्या मैं एकल शब्दों के बजाय पैराग्राफ निकाल सकता हूँ?** हाँ – `AreasType.PARAGRAPHS` का उपयोग करके पैराग्राफ‑लेवल रेक्टेंगल प्राप्त करें।  
- **क्या कोड Java 11+ के साथ संगत है?** बिल्कुल, API Java 11 और बाद के संस्करणों के साथ काम करता है।

## Aspose.OCR में “इमेज से टेक्स्ट पहचानना” क्या है?
Aspose.OCR का `RecognizePage` मेथड बिटमैप का विश्लेषण करता है, OCR एल्गोरिदम लागू करता है, और पहचाना गया स्ट्रिंग लौटाता है। जब आप टेक्स्ट एरिया का अनुरोध करते हैं, तो लाइब्रेरी प्रत्येक टेक्स्ट ब्लॉक के सटीक `Rectangle` कॉर्डिनेट्स भी गणना करती है, जिससे बाद में विशिष्ट सेक्शन को हाइलाइट या प्रोसेस करना आसान हो जाता है।

## Java के लिए Aspose.OCR क्यों चुनें?
- **उच्च सटीकता** – कई भाषाओं और जटिल फ़ॉन्ट्स को सपोर्ट करता है।  
- **आसान इंटीग्रेशन** – एक ही JAR से पूरी OCR क्षमता मिलती है।  
- **लचीला आउटपुट** – आप रॉ टेक्स्ट, फॉर्मेटेड HTML, या सटीक टेक्स्ट‑एरिया रेक्टेंगल प्राप्त कर सकते हैं।  
- **थ्रेड‑सेफ़** – उच्च‑थ्रूपुट सर्वर वातावरण के लिए उपयुक्त।

## पूर्वापेक्षाएँ

- **Java Development Kit** (JDK 11 या नया) आपके मशीन पर स्थापित हो।  
- **Aspose.OCR for Java** लाइब्रेरी – आधिकारिक साइट से [यहाँ](https://releases.aspose.com/ocr/java/) डाउनलोड करें।  
- एक IDE या बिल्ड टूल (Maven/Gradle) ताकि JAR डिपेंडेंसी मैनेज की जा सके।

## पैकेज इम्पोर्ट करें

अपने Java प्रोजेक्ट में आवश्यक क्लासेस इम्पोर्ट करें:

```java
package com.aspose.ocr.examples.OcrFeatures;

import com.aspose.ocr.AreasType;
import com.aspose.ocr.AsposeOCR;
import com.aspose.ocr.examples.License.SetLicense;
import com.aspose.ocr.examples.Utils;

import java.awt.*;
import java.io.IOException;
import java.util.ArrayList;
```

## चरण‑दर‑चरण गाइड

### चरण 1: प्रोजेक्ट सेट अप करें
एक नया Java प्रोजेक्ट बनाएं (या मौजूदा में जोड़ें) और Aspose.OCR JAR को क्लासपाथ पर रखें। यदि आप Maven उपयोग करते हैं, तो डाउनलोड पैकेज में बताए अनुसार डिपेंडेंसी जोड़ें।

### चरण 2: डॉक्यूमेंट डायरेक्टरी और इमेज पाथ निर्धारित करें
अपने सैंपल इमेज का स्थान निर्दिष्ट करें:

```java
// The path to the documents directory.
String dataDir = "Your Document Directory";

// The image path
String imagePath = dataDir + "p3.png";
```

### चरण 3: Aspose.OCR इंस्टेंस बनाएं
OCR इंजन को इंस्टैंशिएट करें:

```java
// Create Aspose.OCR instance
AsposeOCR api = new AsposeOCR();
```

### चरण 4: इमेज में टेक्स्ट पहचानें
`RecognizePage` को कॉल करके चित्र को साधारण टेक्स्ट में बदलें। यह चरण **इमेज से टेक्स्ट पहचानना** की मुख्य क्षमता दर्शाता है:

```java
try {
    // Recognize page by full path to file
    String result = api.RecognizePage(imagePath);
    System.out.println("Result: " + result);
} catch (IOException e) {
    e.printStackTrace();
}
```

### चरण 5: टेक्स्ट एरिया के रेक्टेंगल प्राप्त करें
अब प्रत्येक पैराग्राफ (या अन्य एरिया टाइप) के बाउंडिंग रेक्टेंगल प्राप्त करें। यही वह है जहाँ आप **इमेज से पैराग्राफ निकालते** हैं और उनके कॉर्डिनेट्स प्राप्त करते हैं:

```java
// Get rectangles with text areas in the image.
ArrayList<Rectangle> rectResult = api.getTextAreas(imagePath, AreasType.PARAGRAPHS, true);

// Print each text area rectangle
for (Rectangle r : rectResult) {
    System.out.println("Text area:" + r);
}
```

## सामान्य समस्याएँ और ट्रबलशूटिंग

| लक्षण | संभावित कारण | समाधान |
|---------|--------------|-----|
| `IOException` on `RecognizePage` | गलत फ़ाइल पाथ या पढ़ने की अनुमति नहीं है | सुनिश्चित करें कि `imagePath` मौजूद PNG/JPG की ओर इशारा करता है और एप्लिकेशन को फ़ाइल सिस्टम एक्सेस है। |
| खाली परिणाम स्ट्रिंग | कम‑गुणवत्ता वाली इमेज या असमर्थित भाषा | इमेज को प्री‑प्रोसेस करें (कॉन्ट्रास्ट बढ़ाएँ, बाइनराइज़ करें) या `api.setLanguage("eng")` से सही भाषा निर्दिष्ट करें। |
| कोई रेक्टेंगल नहीं मिला | गलत `AreasType` उपयोग किया (जैसे, पैराग्राफ की अपेक्षा `WORDS`) | आवश्यकतानुसार `AreasType.PARAGRAPHS` या `AreasType.LINES` में स्विच करें। |
