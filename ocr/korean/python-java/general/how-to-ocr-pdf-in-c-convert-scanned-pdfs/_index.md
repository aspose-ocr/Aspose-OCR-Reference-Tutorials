# C#에서 PDF OCR하기 – 스캔된 PDF 변환

스캔된 파일 때문에 **PDF를 OCR하는 방법**을 찾고 계셨나요? 여러분만 그런 것이 아닙니다. 실제 프로젝트—예를 들어 청구서 디지털화나 역사적 보고서 보관—에서는 PDF 안에 이미지가 가득한 경우가 많으며, 이를 검색 가능하게 만들려면 OCR을 적용해야 합니다.  

좋은 소식은 몇 줄의 C# 코드만으로 **스캔된 PDF**를 **검색 가능한 PDF**로 변환하고, 은은한 워터마크를 추가하며, 원시 텍스트를 추출해 추가 처리까지 할 수 있다는 것입니다. 아래 예제는 바로 그 전체 흐름을 보여줍니다.

## 이 튜토리얼에서 다루는 내용

* `OcrEngine` 클래스를 사용한 **PDF를 OCR하는 방법**.  
* 스캔된 PDF를 **검색 가능한 PDF**로 변환하기.  
* 첫 페이지에 워터마크 추가 (**add watermark pdf**).  
* 결과에서 텍스트 문자열 추출 (**extract text from pdf**).  
* 다중 페이지 문서 처리나 저해상도 스캔과 같은 흔한 함정들.

외부 문서 링크나 미완성 스니펫은 없습니다. 필요한 모든 것이 여기 있습니다.

## 사전 준비

* .NET 6.0 이상 (.NET 5에서도 컴파일 가능).  
* `OcrEngine` 및 `ImageFormats`를 제공하는 OCR 라이브러리 참조(e.g., `Aspose.OCR` 또는 유사 패키지).  
* `input.pdf`라는 이름의 입력 PDF가 있는 폴더.  
* C# 콘솔 앱에 대한 기본 지식.

위 조건을 모두 갖췄다면, 바로 시작해 보세요.

## 1단계: 프로젝트 설정 및 의존성 가져오기

새 콘솔 프로젝트를 만들고 NuGet을 통해 OCR 패키지를 추가합니다:

```bash
dotnet new console -n OcrPdfDemo
cd OcrPdfDemo
dotnet add package Aspose.OCR
```

이제 `Program.cs`를 열고, 필요한 네임스페이스를 상단에 추가합니다:

```csharp
using System;
using Aspose.OCR;          // OcrEngine lives here
using Aspose.OCR.Image;   // ImageFormats enum
```

*왜 중요한가*: 올바른 네임스페이스를 가져와야 컴파일러가 `OcrEngine`을 찾을 수 있습니다. 이 단계를 건너뛰면 `CS0246` 오류가 발생해 빌드가 중단됩니다.

## 2단계: OCR 엔진 초기화

엔진은 전체 프로세스의 핵심입니다. 한 번 인스턴스를 만들고 모든 페이지에 재사용합니다.

```csharp
// Step 2: Initialize the OCR engine
OcrEngine engine = new OcrEngine();
```

*팁*: 여러 PDF를 연속으로 처리한다면 같은 `engine` 인스턴스를 재사용해 라이선스 검사를 반복하지 않도록 하세요.

## 3단계: 원본 PDF 로드

엔진에 작업할 파일을 알려줍니다. `SetImageFromFile` 메서드는 이미지든 PDF든 모두 받아들입니다.

```csharp
// Step 3: Load the scanned PDF you want to OCR
engine.SetImageFromFile(@"YOUR_DIRECTORY\input.pdf");
```

> **이 단계가 중요한 이유** – 엔진은 각 페이지를 비트맵 형태로 필요로 합니다. PDF를 전달하면 기본 DPI(보통 300)로 페이지를 래스터화합니다. 원본 PDF가 이미 텍스트 기반이라면 엔진은 그대로 통과시켜 시간을 절약합니다.

## 4단계: OCR 인식 실행

이제 본격적인 작업이 시작됩니다. `Recognize` 메서드는 모든 페이지를 스캔하고 글리프를 추출해 검색 가능한 레이어를 만듭니다.

```csharp
// Step 4: Perform OCR on the loaded document
engine.Recognize();
```

*자주 묻는 질문*: *PDF가 50페이지라면?*  
`Recognize`는 기본적으로 전체 문서를 처리합니다. 메모리가 제한된 환경에서는 `engine.PageIndex`와 `engine.PageCount`를 설정해 페이지별로 처리할 수 있습니다.

## 5단계: 검색 가능한 PDF 저장 (1페이지에 워터마크)

결과를 PDF로 저장합니다. OCR 라이브러리는 기본적으로 “Generated by Aspose.OCR” 오버레이를 첫 페이지에 추가합니다. 커스텀 워터마크가 필요하면 저장 전에 `engine.Watermark` 속성을 수정하면 됩니다.

```csharp
// Step 5: Save the OCR‑processed PDF with a watermark on page 1
engine.Save(@"YOUR_DIRECTORY\output.pdf", ImageFormats.Pdf);
```

이 호출이 끝나면 **검색 가능한 PDF**가 생성되어 텍스트 선택, 복사, 검색이 가능한 상태가 됩니다.

## 6단계: 순수 텍스트 추출 (선택 사항이지만 유용)

인덱싱이나 추가 분석을 위해 **PDF에서 텍스트 추출**이 필요하다면, 엔진은 원시 문자열에 접근할 수 있게 해줍니다.

```csharp
// Step 6: Pull the recognized text into a string
string extractedText = engine.Text;
Console.WriteLine("=== Extracted Text Start ===");
Console.WriteLine(extractedText);
Console.WriteLine("=== Extracted Text End ===");
```

`engine.Text` 속성은 모든 페이지의 텍스트를 줄 바꿈을 유지한 채 연결합니다. 페이지별 구분이 필요하면 `\f`(폼 피드) 기준으로 분할하면 됩니다.

## 전체 동작 예제

아래는 완전한 프로그램 코드입니다. `Program.cs`에 복사‑붙여넣기하고, `YOUR_DIRECTORY`를 절대 경로나 상대 경로로 바꾼 뒤 `dotnet run`을 실행하세요.

```csharp
using System;
using Aspose.OCR;
using Aspose.OCR.Image;

namespace OcrPdfDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // 1️⃣ Initialize the OCR engine
            OcrEngine engine = new OcrEngine();

            // 2️⃣ Load the source PDF to be processed
            // Make sure the file exists; otherwise you'll get a FileNotFoundException.
            string inputPath = @"YOUR_DIRECTORY\input.pdf";
            engine.SetImageFromFile(inputPath);

            // 3️⃣ Perform OCR recognition on the loaded document
            engine.Recognize();

            // 4️⃣ Save the recognized document as a PDF (watermark appears on page 1)
            string outputPath = @"YOUR_DIRECTORY\output.pdf";
            engine.Save(outputPath, ImageFormats.Pdf);

            // 5️⃣ (Optional) Extract plain text for further use
            string extractedText = engine.Text;
            Console.WriteLine("=== Extracted Text Start ===");
            Console.WriteLine(extractedText);
            Console.WriteLine("=== Extracted Text End ===");

            Console.WriteLine($"OCR complete! Searchable PDF saved to: {outputPath}");
        }
    }
}
```

### 예상 출력

프로그램을 실행하면 콘솔에 추출된 텍스트가 출력되고 `output.pdf`가 생성됩니다. PDF 뷰어(Adobe Acrobat, Edge 등)에서 원본 스캔에 있던 단어를 검색해 보세요—이제 검색이 가능합니다. 첫 페이지에 기본 워터마크가 표시되어 **add watermark pdf** 단계가 성공했음을 확인할 수 있습니다.

## 자주 묻는 질문 & 엣지 케이스

| Question | Answer |
|----------|--------|
| **스캔 해상도가 낮으면 어떻게 하나요?** | 인식 전에 DPI를 높입니다: `engine.ImageInfo.DpiX = engine.ImageInfo.DpiY = 600;` 이렇게 하면 OCR 엔진이 더 많은 디테일을 활용할 수 있습니다. |
| **커스텀 워터마크를 지정할 수 있나요?** | 예. `engine.Watermark = "Confidential – Processed on " + DateTime.Now.ToShortDateString();` 를 `Save` 전에 설정하면 됩니다. |
| **비밀번호가 걸린 PDF는 어떻게 처리하나요?** | `engine.LoadPassword = "yourPassword";` 를 `SetImageFromFile` 전에 지정합니다. |
| **특정 페이지만 OCR하고 싶다면?** | `engine.PageIndex = 2;` 와 `engine.PageCount = 5;` 를 설정하면 3‑7페이지만 처리합니다. |
| **원본 이미지를 그대로 유지하고 싶다면?** | OCR 후에 원본 PDF와 OCR 레이어를 PDF 라이브러리(e.g., `Aspose.PDF`)를 사용해 병합하면 이미지 품질을 보존할 수 있습니다. |

## 팁 & 모범 사례

* **팁:** UI 앱에 통합한다면 OCR을 백그라운드 스레드에서 실행해 UI가 멈추는 것을 방지하세요.  
* **주의:** 래스터와 벡터 콘텐츠가 혼합된 PDF. 엔진은 래스터 페이지만 OCR하고, 벡터 텍스트는 자동으로 선택 가능하게 남습니다.  
* **성능 최적화:** OCR 언어 데이터를 캐시합니다(`engine.Language = OcrLanguage.English;`) 이렇게 하면 문서마다 다시 로드할 필요가 없습니다.

## 결론

이제 C#에서 **PDF를 OCR하는 방법**에 대한 완전한 엔드‑투‑엔드 솔루션을 갖추었습니다. `OcrEngine`을 초기화하고, 스캔 파일을 로드하고, 텍스트를 인식하고, **검색 가능한 PDF**를 저장하고, 워터마크를 추가하며, 필요에 따라 **PDF에서 텍스트 추출**까지 수행하면 이미지 기반 PDF를 검색 가능하고 인덱싱 가능한 자산으로 바꿀 수 있습니다.

다음 단계는? 이 워크플로를 문서 관리 시스템과 연결하거나, 다른 언어(`engine.Language = OcrLanguage.French;`)를 실험해 보세요. 폴더 내 여러 파일을 배치 처리하고 싶다면, 매번 새로운 `engine` 인스턴스를 사용해 루프를 돌리면 됩니다.

행복한 코딩 되시고, 여러분의 PDF가 언제나 검색 가능하길 바랍니다!
