# Så gör du OCR på PDF i C# – Konvertera skannade PDF-filer

Har du någonsin behövt **how to OCR PDF** men fastnat på en skannad fil som vägrade samarbeta? Du är inte ensam. I många verkliga projekt—tänk fakturadigitalisering eller arkivering av historiska rapporter—slutar du med en hög av bilder gömda i en PDF, och det enda sättet att göra dem sökbara är att köra OCR på dem.  

Den goda nyheten? Med bara några rader C# kan du **convert scanned PDF** till en **create searchable PDF**, lägga till ett diskret vattenmärke och till och med hämta den råa texten för vidare bearbetning. Nedan ser du ett komplett, färdigt‑att‑köra exempel som gör exakt det.

## Vad den här handledningen täcker

* Hur man **how to OCR PDF** med `OcrEngine`-klassen.  
* Att omvandla en skannad PDF till en **create searchable PDF**.  
* Lägga till ett vattenmärke på första sidan (**add watermark pdf**).  
* Extrahera rena textsträngar från resultatet (**extract text from pdf**).  
* Vanliga fallgropar—som att hantera flersidiga dokument eller lågupplösta skanningar.

Inga externa dokumentationslänkar, inga halvfärdiga kodsnuttar. Allt du behöver finns här.

## Förutsättningar

* .NET 6.0 eller senare (koden kompileras även med .NET 5).  
* En referens till OCR‑biblioteket som tillhandahåller `OcrEngine` och `ImageFormats` (t.ex. `Aspose.OCR` eller ett liknande paket).  
* En inmatnings‑PDF med namnet `input.pdf` placerad i en mapp du kontrollerar.  
* Grundläggande kunskap om C#‑konsolappar.

Om du redan har dem, bra—låt oss hoppa in.

## Steg 1: Skapa projektet och importera beroenden

Skapa ett nytt konsolprojekt och lägg till OCR‑paketet via NuGet:

```bash
dotnet new console -n OcrPdfDemo
cd OcrPdfDemo
dotnet add package Aspose.OCR
```

Öppna nu `Program.cs`. Lägg till de namnrymder du behöver högst upp:

```csharp
using System;
using Aspose.OCR;          // OcrEngine lives here
using Aspose.OCR.Image;   // ImageFormats enum
```

*Varför detta är viktigt*: Att importera rätt namnrymder låter kompilatorn hitta `OcrEngine`. Att hoppa över detta steg ger ett `CS0246`‑fel och stoppar bygget.

## Steg 2: Initiera OCR‑motorn

Motorn är hjärtat i processen. Du instansierar den en gång och återanvänder den för varje sida.

```csharp
// Step 2: Initialize the OCR engine
OcrEngine engine = new OcrEngine();
```

*Proffstips*: Om du planerar att bearbeta många PDF‑filer i rad, överväg att återanvända samma `engine`‑instans för att undvika upprepade licenskontroller.

## Steg 3: Ladda käll‑PDF‑filen

Berätta för motorn vilken fil den ska arbeta med. Metoden `SetImageFromFile` accepterar vilken bild‑ eller PDF‑källa som helst.

```csharp
// Step 3: Load the scanned PDF you want to OCR
engine.SetImageFromFile(@"YOUR_DIRECTORY\input.pdf");
```

> **Varför detta steg är avgörande** – Motorn behöver en bitmap‑representation av varje sida. När du matar den med en PDF rasteriserar den varje sida internt baserat på standard‑DPI (vanligtvis 300). Om käll‑PDF‑filen redan är textbaserad kommer motorn helt enkelt att skicka den vidare oförändrad, vilket sparar tid.

## Steg 4: Kör OCR‑igenkänning

Nu sker det tunga arbetet. Metoden `Recognize` skannar varje sida, extraherar glyfer och bygger ett sökbart lager.

```csharp
// Step 4: Perform OCR on the loaded document
engine.Recognize();
```

*Vanlig fråga*: *Vad händer om PDF‑filen har 50 sidor?*  
`Recognize` bearbetar hela dokumentet som standard. För minnesbegränsade miljöer kan du sätta `engine.PageIndex` och `engine.PageCount` för att arbeta sida‑för‑sida.

## Steg 5: Spara den sökbara PDF‑filen (Vattenmärke på sida 1)

Vi sparar resultatet som en PDF. Första sidan får automatiskt ett vattenmärke eftersom OCR‑biblioteket som standard lägger till en “Generated by Aspose.OCR”-översikt. Om du behöver ett eget vattenmärke kan du ändra egenskapen `engine.Watermark` innan du sparar.

```csharp
// Step 5: Save the OCR‑processed PDF with a watermark on page 1
engine.Save(@"YOUR_DIRECTORY\output.pdf", ImageFormats.Pdf);
```

Efter detta anrop har du en **create searchable PDF** där du kan markera, kopiera och söka i texten precis som i vilken native PDF som helst.

## Steg 6: Extrahera ren text (Valfritt men praktiskt)

Om du också vill **extract text from pdf** för indexering eller vidare analys, ger motorn dig tillgång till den råa strängen.

```csharp
// Step 6: Pull the recognized text into a string
string extractedText = engine.Text;
Console.WriteLine("=== Extracted Text Start ===");
Console.WriteLine(extractedText);
Console.WriteLine("=== Extracted Text End ===");
```

`engine.Text`‑egenskapen sammanfogar texten från alla sidor och bevarar radbrytningar. Du kan dela den med `\f` (form feed) om du behöver sid‑specifik granularitet.

## Fullständigt fungerande exempel

Nedan är hela programmet. Kopiera‑klistra in det i `Program.cs`, ersätt `YOUR_DIRECTORY` med en absolut eller relativ sökväg, och kör `dotnet run`.

```csharp
using System;
using Aspose.OCR;
using Aspose.OCR.Image;

namespace OcrPdfDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // 1️⃣ Initialize the OCR engine
            OcrEngine engine = new OcrEngine();

            // 2️⃣ Load the source PDF to be processed
            // Make sure the file exists; otherwise you'll get a FileNotFoundException.
            string inputPath = @"YOUR_DIRECTORY\input.pdf";
            engine.SetImageFromFile(inputPath);

            // 3️⃣ Perform OCR recognition on the loaded document
            engine.Recognize();

            // 4️⃣ Save the recognized document as a PDF (watermark appears on page 1)
            string outputPath = @"YOUR_DIRECTORY\output.pdf";
            engine.Save(outputPath, ImageFormats.Pdf);

            // 5️⃣ (Optional) Extract plain text for further use
            string extractedText = engine.Text;
            Console.WriteLine("=== Extracted Text Start ===");
            Console.WriteLine(extractedText);
            Console.WriteLine("=== Extracted Text End ===");

            Console.WriteLine($"OCR complete! Searchable PDF saved to: {outputPath}");
        }
    }
}
```

### Förväntad output

När programmet körs skrivs den extraherade texten ut i konsolen och `output.pdf` skapas. Öppna PDF‑filen i någon visare (Adobe Acrobat, Edge osv.) och försök söka efter ett ord som fanns i den ursprungliga skanningen—du kommer att se att den nu är sökbar. Första sidan visar standardvattenmärket, vilket bekräftar att steget **add watermark pdf** lyckades.

## Vanliga frågor & kantfall

| Fråga | Svar |
|----------|--------|
| **What if the scan is low‑resolution?** | Öka DPI innan igenkänning: `engine.ImageInfo.DpiX = engine.ImageInfo.DpiY = 600;` Detta ger OCR‑motorn mer detaljer att arbeta med. |
| **Can I choose a custom watermark?** | Ja. Sätt `engine.Watermark = "Confidential – Processed on " + DateTime.Now.ToShortDateString();` innan `Save`. |
| **How do I handle password‑protected PDFs?** | Använd `engine.LoadPassword = "yourPassword";` före `SetImageFromFile`. |
| **Is there a way to limit OCR to specific pages?** | Sätt `engine.PageIndex = 2;` och `engine.PageCount = 5;` för att bearbeta endast sidorna 3‑7. |
| **What if I need to keep the original images untouched?** | Efter OCR kan du slå ihop den ursprungliga PDF‑filen med OCR‑lagret med ett PDF‑bibliotek (t.ex. `Aspose.PDF`) för att bevara bildkvaliteten. |

## Tips & bästa praxis

* **Proffstips:** Kör OCR på en bakgrundstråd om du integrerar detta i en UI‑app—annars fryser gränssnittet.  
* **Se upp för:** PDF‑filer med blandat raster‑ och vektor­innehåll. Motorn OCR‑ar bara raster‑sidor; vektortext förblir automatiskt markerbar.  
* **Prestandajustering:** Cacha OCR‑språkdatan (`engine.Language = OcrLanguage.English;`) för att undvika att ladda om den för varje dokument.

## Slutsats

Du har nu en komplett, end‑to‑end‑lösning för **how to OCR PDF** i C#. Genom att initiera `OcrEngine`, ladda en skannad fil, känna igen texten, spara en **create searchable pdf**, lägga till ett vattenmärke och eventuellt **extract text from pdf**, kan du förvandla vilken bild‑baserad PDF som helst till en sökbar, indexerbar resurs.

Nästa steg? Försök kedja detta arbetsflöde med ett dokumenthanteringssystem, eller experimentera med olika språk (`engine.Language = OcrLanguage.French;`). Du kan också utforska batch‑bearbetning av flera filer i en mapp—loopa bara över stegen med en ny `engine`‑instans varje gång.

Lycka till med kodningen, och må dina PDF‑filer alltid vara sökbara!
